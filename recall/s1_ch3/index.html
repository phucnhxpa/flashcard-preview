S1 Ch 3 — Representations of Data

The whole chapter in one mental model. Five graphical tools (histograms, box plots, stem-and-leaf, frequency polygons, back-to-back diagrams) help you see a distribution. Each one reveals shape (skewness), outliers and key quartiles. To compare two data sets you compare a measure of location with a measure of spread — pair them honestly: mean+SD or median+IQR, never crossed.

1. Jargon glossary

Term Meaning Where
Histogram Bar chart for grouped continuous data where area = $k\times$ frequency. Heights are frequency density, not frequency. Bars touch (no gaps) because the variable is continuous; their widths reflect class widths so unequal classes are handled correctly. 3.1
Frequency density Height of a histogram bar: $\text{f.d.} = \dfrac{\text{frequency}}{\text{class width}}$ (when $k=1$). Lets unequal classes be drawn without misleading the eye — the area you see equals the count of observations. 3.1
Frequency polygon Join the midpoint of the top of each histogram bar with straight lines. A skeleton view of the distribution useful for overlaying / comparing two histograms on one plot. 3.1
Outlier Any value far from the bulk of the data. Two common rules: (IQR rule) $xQ_3+k(Q_3-Q_1)$, with $k$ usually 1.5 or 1; (SD rule) $|x-\bar x|>k\sigma$, usually $k=2$. The exam tells you which rule to use. 3.2
Anomaly An outlier that is clearly an error (impossible value, recording mistake) — these can legitimately be deleted. Compare to a legitimate outlier (e.g. an unusually long snail), which must be kept. 3.2
Cleaning data The act of removing anomalies before analysis. Must be justified ("this value is impossible / clearly mistyped") — don't just delete inconvenient data points. 3.2
Box plot (box-and-whisker) Picture of $Q_1,Q_2,Q_3$ as a box, with whiskers reaching the smallest/largest non-outlier values; outliers are marked separately with $\times$. The whisker stops at the outlier fence, not the actual extreme value. 3.3
Stem-and-leaf diagram Each number split into a stem (leading digits) and leaf (last digit). Preserves all original values and shows the distribution shape. Always include a key, e.g. "1|4 = 1.4". 3.4
Back-to-back stem-and-leaf Two distributions on a shared stem with leaves growing left and right. Reading the left side, leaves go from the stem outwards (i.e. right-to-left). Useful for direct comparison of two groups. 3.4
Skewness A measure of asymmetry. Symmetric: mean ≈ median ≈ mode. Positive skew: long tail to the right (mode < median < mean). Negative skew: long tail to the left (mode > median > mean). Has both a quartile-based test and a numerical coefficient. 3.5
Pearson coefficient of skewness $\dfrac{3(\text{mean}-\text{median})}{\sigma}$. 0 ⇒ symmetric; positive ⇒ positive skew; negative ⇒ negative skew. The further from 0, the more skewed. Compares mean against median directly. 3.5
Quartile-based skewness test Compare $Q_3-Q_2$ with $Q_2-Q_1$. Equal ⇒ symmetric; $Q_3-Q_2 > Q_2-Q_1$ ⇒ positive skew (right tail longer); $Q_3-Q_2 < Q_2-Q_1$ ⇒ negative skew. 3.5
Comparing data sets State one measure of location and one measure of spread, in context. Use mean+SD when data are clean and symmetric; use median+IQR when there are outliers. Never mix (no median+SD, no mean+IQR). 3.6
Class width $U-L$ where $L,U$ are the lower and upper class boundaries. Determines histogram bar width. Shifted by $\pm 0.5$ when rounded data has gaps (e.g. "13–14, 15–17"). 3.1

2. Histogram mechanics

Concept Formula / rule Notes
Frequency density $\text{f.d.}=\dfrac{f}{\text{class width}}$ Use this as bar height when $k=1$
General relation Area of bar $= k\times f$ $k=1$ is easiest; some questions pick $k$ to keep numbers tidy
Recover frequency from histogram $f = \text{f.d.}\times\text{class width}$ Or count "small squares" if no scale given
Estimate count in a sub-range Integrate the area inside that sub-range (trapezium rule on each bar) E.g. count of "36 – 45 mins" = sum of partial-bar areas
Two bars compared Ratio of heights = ratio of (f / cw) values Useful when only widths/heights are given
Histogram width and height (cm scale) $\text{width}\propto\text{class width}$; $\text{height}\propto\text{f.d.}$ When given a sample bar (e.g. width 4 cm, height 6 cm), find scale factors first

3. Outlier rules — which formula to apply

Rule given Formula Threshold Typical $k$
1.5×IQR rule (Tukey) $x>Q_3+1.5(Q_3-Q_1)$ or $x "Fences" 1.5 (default)
General IQR Same with $k$ stated in question Wider/narrower fences 1, 2, 3
SD rule $|x-\bar x|>k\sigma$ Distance from mean $k=2$ usually
One-sided check Only one fence cited E.g. "above $\bar x+2\sigma$" —

Procedure:

  1. Compute $Q_1,Q_3,\text{IQR}$ (or $\bar x,\sigma$).
  2. Compute the upper and lower fences with the given $k$.
  3. Compare each suspect value against the fences.
  4. If asked to clean, remove justified anomalies and recompute $n,\sum x,\sum x^2,\bar x,\sigma$.

4. Box plot — anatomy

Feature Meaning
Left of box $Q_1$
Line in box $Q_2$ (median)
Right of box $Q_3$
Box width IQR $=Q_3-Q_1$
Whiskers Extend to smallest/largest non-outlier value (or to the fence if outliers exist)
Cross ($\times$) Marked outlier

Drawing rules:

  • Use a clear linear scale; label axis with units.
  • Two box plots on one diagram for comparison must share the same scale.
  • Each box plot needs a title.

Reading skewness from a box plot:

Box-plot shape Skewness
Whiskers symmetric, median centred in box Symmetric
Long right whisker, median in left half Positive skew
Long left whisker, median in right half Negative skew

5. Stem-and-leaf — construction & quartiles

Step Action
1 Sort data in ascending order
2 Decide stem (leading digits) and leaf (final digit) — usually one decimal place or one integer position
3 List each row with its stem and the leaves in order
4 Add a key (e.g. "2 | 1 = 2.1")
5 Often add a count column (number of leaves per row)
6 Quartiles: use discrete-data rules ($\tfrac{n}{4},\tfrac{n}{2},\tfrac{3n}{4}$ with rounding/halfway) on the ordered values

Back-to-back layout:

  • Shared central stem column.
  • Right group: leaves grow left-to-right normally.
  • Left group: leaves grow outward from the stem (i.e. right-to-left). Re-read the row, the smallest leaf sits next to the stem.

6. Skewness — three tests, decision table

Test Symmetric Positive skew Negative skew
Quartile distances $Q_2-Q_1=Q_3-Q_2$ $Q_2-Q_1 $Q_2-Q_1>Q_3-Q_2$
Order of central measures mean ≈ median ≈ mode mode < median < mean mode > median > mean
Pearson coefficient $\dfrac{3(\bar x-\text{median})}{\sigma}$ 0 positive (>0) negative (<0)
Box plot whisker length equal whiskers longer right whisker longer left whisker
Histogram tail symmetric tail extends to the right (large values rare but extreme) tail extends to the left

Memory trick: the skew is named for where the long tail goes, not where the bulk sits.

7. Comparing two data sets

Situation Use these statistics Why
Clean data, no outliers, fairly symmetric Mean and SD Mean uses every value; SD captures full spread
Outliers present / skewed Median and IQR Both robust to extreme values
Already given two box plots Compare medians, IQRs, and ranges Read directly off the diagram
Already given two summary stats tables Compute means/SDs from $\sum x,\sum x^2$ first Then compare

Comparison sentence template:

"The [median/mean] of A is [higher/lower] than B, indicating [...]. The [IQR/SD] of A is [larger/smaller], so A's data are more/less spread out."

Always interpret in context — refer to the actual variable (temperature, age, etc.).

8. Worked archetypes (memorise the shape)

# Setup Method
1 Histogram with class widths $5,5,5,10,30$, frequencies $55,39,68,32,6$. Find f.d. Divide each $f$ by its class width: $11, 7.8, 13.6, 3.2, 0.2$
2 "5 people in 25 small squares". Find count for 20 small squares. $\frac{5}{25}\cdot 20 = 4$ people
3 $Q_1=3.2, Q_3=4.0$. IQR rule with $k=1.5$. Fences: $3.2-1.2=2.0$ and $4.0+1.2=5.2$
4 $\bar x=6.1, \sigma^2=4.2$. SD rule with $k=2$. Fences: $6.1\pm2\sqrt{4.2}=6.1\pm 4.10$ ⇒ $2.00$ to $10.20$
5 Pearson skewness with $\bar x=57.46, \text{med}=60, \sigma=15.7$. $\dfrac{3(57.46-60)}{15.7}=-0.486$ ⇒ negative skew
6 Compare via box plots (medians, IQRs, ranges, outliers, in context). Always two sentences: location + spread

9. Common exam pitfalls

Pitfall Fix
Using frequency as histogram height (instead of f.d.) Always divide by class width
Forgetting class boundaries shift by 0.5 for rounded data with gaps "13–14, 15–17" → boundaries 12.5/14.5/17.5
Whisker drawn to the actual outlier instead of the fence Whisker ends at the next non-outlier (or at the fence if convention says so), outlier marked separately
Skewness named after the bulk of data Named after the tail — long right tail = positive skew
Comparing median with standard deviation Pair: median ↔ IQR; mean ↔ SD. Never cross
Not stating skewness "in context" Always refer back to the data ("times taken to run", "ages of foxes", …)
Cleaning data without justifying Only clean anomalies (errors / impossible values), not legitimate outliers
Stem-and-leaf with no key Always include a key like "2|3 = 23 marks"
Using $\tfrac{n+1}{2}$ for grouped median That's discrete only. Stem-and-leaf with $n=30$? Use the discrete rule $\tfrac{n}{2}=15$th + $\tfrac{n}{2}+1=16$th, mean of those

10. Big picture (30-second mental model)

Take any dataset, do four passes:

  1. Visualise — histogram (continuous, possibly unequal classes), stem-and-leaf (small data, preserves values), or box plot (summary).
  2. Hunt outliers — apply the rule named in the question (IQR-based or SD-based); decide whether each is a legitimate outlier or an anomaly to clean.
  3. Describe shape — symmetric / positive skew / negative skew, supported by quartile distances, order of measures, or Pearson's coefficient.
  4. Compare — when two sets of data are given, choose mean+SD or median+IQR (never mix), state both, then interpret in context.

Three diagrams, two outlier rules, three skewness tests — that's the chapter.
