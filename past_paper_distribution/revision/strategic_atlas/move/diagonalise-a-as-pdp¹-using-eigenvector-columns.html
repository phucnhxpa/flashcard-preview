Matrices

Diagonalise A as PDP⁻¹ using eigenvector columns

Family: Diagonalisation Frequency: 1 questions

Trigger — When to use

Fires once a 2×2 or 3×3 matrix has been shown to possess a full set of linearly independent eigenvectors, and the question asks to diagonalise A, find a matrix P and diagonal D such that P⁻¹AP = D, or compute Aⁿ for large/general n. Common phrasings: 'find a matrix P such that P⁻¹AP is diagonal', 'hence find A⁵', 'express A in the form PDP⁻¹'. The presence of distinct eigenvalues is a strong cue that diagonalisation is available.

Motivation — Why this works

If Av = λv for each eigenvector–eigenvalue pair, stacking the eigenvectors as columns of P gives AP = PD, where D is the diagonal matrix of eigenvalues in matching order. Because the columns of P are independent, P is invertible, so A = PDP⁻¹. This decouples the action of A into independent scalings along eigenvector directions, making powers Aⁿ = PDⁿP⁻¹ computable by raising only diagonal entries to the n-th power.

Thought Process — The step-by-step reasoning

  1. Notice the question asks for a power, exponential, or explicit factorisation — a signal that diagonal form will trivialise the computation.
  2. Rule out direct multiplication of A by itself (infeasible for general n) and rule out Cayley–Hamilton shortcuts unless specifically prompted.
  3. Commit to PDP⁻¹ once three (or two) independent eigenvectors are confirmed, since this guarantees P is invertible.
  4. Bridge: assemble P column-by-column from eigenvectors and write D with eigenvalues in the same order.
  5. Decide the column ordering early (e.g. ascending eigenvalues) and stick with it to avoid mismatches when computing Dⁿ or P⁻¹.

Mechanic — Concrete operations

  1. List each eigenvalue λᵢ with a chosen eigenvector eᵢ.
  2. Form P = (e₁ | e₂ | e₃) with eigenvectors as columns.
  3. Form D = diag(λ₁, λ₂, λ₃) in the matching order.
  4. Compute P⁻¹ using the standard inverse method (cofactors/row reduction).
  5. Optional check: verify AP = PD by computing one column.
  6. State A = PDP⁻¹, and for powers write Aⁿ = PDⁿP⁻¹.

Worked Micro-Example

Let $A=\begin{pmatrix}2&1\\0&3\end{pmatrix}$ with eigenvalues $\lambda_1=2,\lambda_2=3$ and eigenvectors $e_1=\binom{1}{0}, e_2=\binom{1}{1}$. Then $P=\begin{pmatrix}1&1\\0&1\end{pmatrix}$, $D=\begin{pmatrix}2&0\\0&3\end{pmatrix}$, $P^{-1}=\begin{pmatrix}1&-1\\0&1\end{pmatrix}$, so $A=PDP^{-1}$ and $A^n=P\begin{pmatrix}2^n&0\\0&3^n\end{pmatrix}P^{-1}$.

Variants — Same move, different surface

Symmetric matrix (real)
Eigenvectors for distinct eigenvalues are orthogonal; normalise them so P is orthogonal and P⁻¹ = Pᵀ, removing the inverse computation.
2×2 matrix with two distinct eigenvalues
Same procedure with smaller P and D; P⁻¹ obtained from the 2×2 inverse formula.
Computing Aⁿ or solving a linear recurrence/system
Use Aⁿ = PDⁿP⁻¹, raising only the diagonal entries to the n-th power.
Repeated eigenvalue with two independent eigenvectors
Diagonalisation still works; just place the repeated eigenvalue twice on D's diagonal with its two eigenvectors as columns of P.

Common Misreadings

  • Writing A = P⁻¹DP instead of A = PDP⁻¹ (swapping the roles of P and P⁻¹).
  • Mismatching the column order of P with the diagonal order of D.
  • Assuming diagonalisation is always possible — overlooking the need for a full set of independent eigenvectors.

Pitfalls

  • Arithmetic slips when computing P⁻¹ propagate through every later step.
  • Forgetting to raise each diagonal entry of D to the n-th power, or raising P or P⁻¹ to the n-th power by mistake.
  • Using un-normalised eigenvectors but treating P as orthogonal in the symmetric case.

A-level Scope Notes

Sits squarely in Edexcel IAL Further Pure (FP3) eigenvalue/eigenvector work for 2×2 and 3×3 matrices; Jordan form and defective cases are out of scope.
