Substitute known x-value into polynomial to form linear equation in unknown coefficient

Family: Polynomial Evaluation Frequency: 1 questions

Trigger — When to use

A polynomial f(x) contains a single unknown coefficient (e.g. a) and the question supplies one numerical input-output pair such as f(2) = 3, or equivalently states 'the remainder when f(x) is divided by (x − 2) is 3'. The pairing of one unknown with one functional condition is the cue. Phrasings like 'find the value of a given that...' or 'when f(x) is divided by (x − c) the remainder is r' all signal this move.

Motivation — Why this works

Substituting x = c into f(x) collapses every term into a number except the one carrying the unknown coefficient, so the polynomial identity reduces to a linear equation in that coefficient. This works because the Remainder Theorem guarantees f(c) equals the remainder on division by (x − c), making direct evaluation equivalent to (and far quicker than) polynomial long division. Linearity in the unknown is preserved because each coefficient appears only once and multiplied by a constant power of c.

Thought Process — The step-by-step reasoning

  1. Notices that f(x) has exactly one unknown parameter and exactly one numerical condition f(c) = k — so one equation in one unknown is solvable.
  2. Rules out factorising, dividing, or root-finding: the question asks for a coefficient, not roots, and f(c) ≠ 0 so (x − c) is not a factor anyway.
  3. Commits to substitution because the Remainder Theorem reframes f(c) = k as a direct evaluation, and c is small enough that powers c², c³ are trivial to compute.
  4. Bridges to the first operation by writing f(c) term-by-term, keeping the unknown coefficient symbolic while evaluating every numerical power and product.
  5. Tactically combines all pure constants on one side, isolates the term in a, then solves the resulting linear equation, double-checking signs of negative coefficients and odd powers.

Mechanic — Concrete operations

  1. Write f(c) by substituting x = c into every term, keeping a symbolic.
  2. Evaluate the numerical powers c², c³, etc.
  3. Multiply each evaluated power by its coefficient, retaining signs carefully.
  4. Combine all pure constants into a single number, leaving the form (coefficient)·a + constant.
  5. Set the expression equal to the given value k and solve the resulting linear equation for a.

Worked Micro-Example

Given $f(x) = ax^3 + 3x^2 - 8x + 2$ with $f(2) = 3$. Substitute: $f(2) = 8a + 12 - 16 + 2 = 8a - 2$. Set $8a - 2 = 3$, so $8a = 5$ and $a = \tfrac{5}{8}$.

Variants — Same move, different surface

Negative substitution value, e.g. f(−1) = k
Track sign changes carefully on odd powers: (−1)³ = −1, (−1)² = 1.
Remainder Theorem phrasing: 'remainder on division by (x − c) is r'
Translate directly to f(c) = r and substitute as normal.
Division by (ax − b) with remainder r
Use f(b/a) = r — substitute the rational root of the divisor.
Two unknown coefficients a and b
Need two conditions, e.g. f(2) = 3 and f(−1) = 7; substitute each to get two linear equations and solve simultaneously.
(x − c) is a factor of f(x)
Use f(c) = 0 instead of f(c) = k; same substitution mechanic but right-hand side is zero.

Common Misreadings

  • Mistaking f(2) = 3 for the statement that (x − 2) is a factor (which would require f(2) = 0).
  • Reading 'remainder is 3' as 'f(3) = 0' rather than f(2) = 3.
  • Choosing polynomial long division instead of direct substitution — valid but slower and more error-prone.

Pitfalls

  • Sign slips when evaluating negative-coefficient terms, e.g. writing +16 instead of −16 for −8(2).
  • Forgetting to apply the power: writing a(2) instead of a(2)³ = 8a.
  • Arithmetic error combining constants (e.g. 12 − 16 + 2 should give −2).
  • With negative c, mishandling parity: (−1)³ = −1, not +1.

A-level Scope Notes
