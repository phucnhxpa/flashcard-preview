Sequences & Series

Reject algebraic solution by checking physical validity of nth term

Family: Contextual Validation Frequency: 1 questions

Trigger — When to use

Fires when a quadratic-in-n equation (typically from S_n = target) yields two positive integer roots and the question asks which is 'not sensible', 'must be rejected', or 'in this context'. Common surface: a decreasing arithmetic sequence (negative common difference) modelling a non-negative real-world quantity such as monthly savings, daily production, or distance walked. The phrase 'explain why' or 'state, with a reason' is a strong cue that contextual — not algebraic — justification is required.

Motivation — Why this works

An arithmetic sum S_n is quadratic in n, so two values of n can give the same total. With d < 0, the terms eventually pass through zero and become negative; beyond that point the model is no longer faithful to the real situation (you cannot 'save' a negative amount). The larger root corresponds to a regime where negative terms cancel earlier positive ones to recreate the same total — mathematically valid, contextually nonsense. Validating u_n ≥ 0 selects the physically meaningful root.

Thought Process — The step-by-step reasoning

  1. Notice that two algebraic roots have appeared and the question explicitly demands a reason — so the rejection is contextual, not algebraic.
  2. Rule out 'just pick the smaller n' as a justification; examiners want the mechanism, not the magnitude.
  3. Commit to the test 'is u_n ≥ 0 for all terms in the sum?' because d < 0 means terms must eventually go negative.
  4. Bridge: locate the threshold where u_n = 0, i.e. solve a + (n−1)d = 0, then compare each candidate root to that threshold.
  5. Tactically, either compute u at the suspect root directly or find the zero-crossing index — whichever is faster — and phrase the conclusion in terms of the context (savings becoming negative).

Mechanic — Concrete operations

  1. Identify the candidate roots n_1 < n_2 from the quadratic in n.
  2. Find the index where u_n = 0: solve a + (n−1)d = 0 to get n*.
  3. Evaluate u at the larger candidate, e.g. u_{n_2} = a + (n_2 − 1)d, and check its sign.
  4. If u_{n_2} < 0 (equivalently n_2 > n*), reject n_2 because the model requires non-negative terms.
  5. State the conclusion in context: e.g. 'monthly savings would be negative, which is not sensible', and report the remaining root.

Worked Micro-Example

Lina saves with $a = 600$, $d = -10$, and $S_n = 18200$ gives $n = 56$ or $n = 65$. Zero-crossing: $600 + (n-1)(-10) = 0 \Rightarrow n = 61$. Then $u_{65} = 600 + 64(-10) = -40 < 0$, so from month 62 onward she would be 'saving' a negative amount. Reject $n = 65$; the sensible answer is $n = 56$ (where $u_{56} = 50 > 0$).

Variants — Same move, different surface

Decreasing AP modelling daily output of a worker/machine until production stops
Reject the root for which u_n < 0, since output cannot be negative; threshold via a + (n−1)d = 0.
Walker covering decreasing distances each day
Same logic: distance per day must be ≥ 0; the larger root corresponds to 'walking backwards'.
Arithmetic series where the question asks for the largest n with S_n still increasing
Use u_n ≥ 0 as the cutoff; n* = 1 − a/d gives the last positive term.
Geometric series context with |r| < 1 and a target sum
Algebraic rejection differs (uniqueness from log), but validity check still uses 'terms must remain meaningful in context'.

Common Misreadings

  • Saying 'n = 65 is too large' without referencing negative terms — examiners require the specific mechanism.
  • Comparing n to an unrelated number from another part of the question (e.g. another person's timeframe).
  • Treating the rejection as if it followed from the algebra (e.g. 'the discriminant rules it out') when both roots are genuinely valid solutions of the equation.

Pitfalls

  • Computing u at the wrong candidate (e.g. checking u_{56} instead of u_{65}).
  • Sign slip in u_n = a + (n−1)d when d is negative.
  • Stating only 'u_{65} < 0' without translating into context ('savings would be negative, not sensible').
  • Using S_n instead of u_n for the validity test — the sum can still hit the target while individual terms are negative.

A-level Scope Notes

Fully within Edexcel IAL P1/P2 arithmetic series: uses only u_n = a + (n−1)d, S_n formulae, and contextual reasoning expected in 'explain why' marks.
