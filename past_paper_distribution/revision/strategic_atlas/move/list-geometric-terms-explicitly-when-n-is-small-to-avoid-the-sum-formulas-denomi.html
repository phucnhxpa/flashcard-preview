Sequences & Series

List geometric terms explicitly when n is small to avoid the sum formula's denominator

Family: Formula Selection Frequency: 1 questions

Trigger — When to use

A geometric series problem involves only 2, 3, or occasionally 4 terms, and you need to set the sum equal to a target value or substitute it into another equation. Applying S_n = a(r^n − 1)/(r − 1) would introduce a denominator (r − 1) that must be cleared later. Cues include phrases like 'the sum of the first three terms is...' or a system combining a GP sum with a term equation. The unknown is typically r (or a), so the resulting equation will be polynomial in r.

Motivation — Why this works

For small n, the identity a + ar + ar² + ... + ar^(n−1) is already fully expanded and factors cleanly as a(1 + r + r² + ...). Keeping the sum in this form preserves a polynomial structure, which is essential when the next step is substitution into another equation or solving a cubic/quadratic in r. The closed-form fraction S_n = a(r^n − 1)/(r − 1) is algebraically equivalent but forces multiplication by (r − 1), risks introducing/losing the spurious root r = 1, and obscures factorisation.

Thought Process — The step-by-step reasoning

  1. Notice that the GP sum involves only a small number of terms (here, 3), so direct enumeration is feasible.
  2. Rule out the closed-form S_n = a(r^n − 1)/(r − 1) because its denominator will need clearing and carries the r ≠ 1 caveat.
  3. Commit to writing the sum as a + ar + ar² and factor out a to get a(1 + r + r²), keeping everything polynomial.
  4. Bridge: equate this product directly to the given sum value, e.g. a(1 + r + r²) = 39.
  5. Tactically, pair this with the other given equation (often a term value like ar^k = c) to eliminate a and reduce to a single polynomial in r.

Mechanic — Concrete operations

  1. Write S_n explicitly: e.g. S_3 = a + ar + ar²
  2. Factor out a: a(1 + r + r²)
  3. Set equal to the given sum value
  4. Use the second condition to express a in terms of r (or vice versa)
  5. Substitute to obtain a single polynomial equation in r and solve

Worked Micro-Example

Given a GP with $a + ar + ar^2 = 39$ and $ar^2 = 27$. From the second equation, $a = 27/r^2$. Substituting: $\frac{27}{r^2}(1 + r + r^2) = 39$, so $27(1 + r + r^2) = 39r^2$, giving $12r^2 - 27r - 27 = 0$, i.e. $4r^2 - 9r - 9 = 0$, hence $r = 3$ or $r = -3/4$.

Variants — Same move, different surface

GP with only 2 terms summed (a + ar = k)
Factor as a(1 + r) = k; trivially polynomial, no formula needed.
GP with 4 terms summed
Listing a + ar + ar² + ar³ = a(1 + r)(1 + r²) is still cleaner than the fraction formula and exposes useful factorisation.
GP with 5+ terms or symbolic n
Switch to the closed-form S_n = a(r^n − 1)/(r − 1); listing becomes impractical.
Arithmetic series with small n
The AP formula S_n = n/2(2a + (n−1)d) is already polynomial, so no analogous issue — use it directly.
Sum to infinity of a GP (|r| < 1)
Use S_∞ = a/(1 − r) directly; explicit listing is impossible.

Common Misreadings

  • Treating the closed-form formula as universally preferable and reaching for it reflexively even when n = 2 or 3.
  • Thinking the listed form is only valid for specific values of r, when in fact a(1 + r + r²) holds for all r including r = 1.

Pitfalls

  • Forgetting to exclude r = 1 when using the fraction formula, or multiplying both sides by (r − 1) and inadvertently introducing r = 1 as a spurious solution.
  • Mis-factoring 1 + r + r² as (1 + r)² — it does not factor over the reals.
  • Sign errors when expanding ar² + ar + a if r is later taken negative.

A-level Scope Notes
