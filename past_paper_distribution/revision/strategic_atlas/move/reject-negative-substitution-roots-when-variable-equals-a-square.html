Algebra & Functions

Reject negative substitution roots when variable equals a square

Family: Domain Filtering Frequency: 1 questions

Trigger — When to use

Fires whenever a substitution u = x² (or u = x⁴, u = |x|, u = e^x, u = √x) reduces a higher-degree equation to a quadratic/linear in u, and solving for u yields one or more negative values. Cue phrases include 'let u = x²', 'this is a quadratic in x²', or any disguised quadratic such as ax⁴ + bx² + c = 0. Also fires when the question explicitly asks for real solutions or for the number of real solution pairs.

Motivation — Why this works

The substitution u = x² imposes the hidden constraint u ≥ 0 because squares of real numbers cannot be negative. Any negative u-root corresponds to x² = (negative), which has no real solutions, so it cannot contribute to the answer set. Filtering these roots is a logical necessity, not cosmetic tidying — it directly determines how many real x-values exist. Forgetting this step leads to phantom solutions or attempts to take square roots of negatives.

Thought Process — The step-by-step reasoning

  1. Notice the equation has been reduced to a quadratic in u via a substitution u = x² (or similar non-negative expression).
  2. Rule out any u-value that violates the implicit domain of the substitution — here, any u < 0.
  3. Commit to keeping only the non-negative u-roots because the question asks for real x.
  4. Bridge: for each surviving u ≥ 0, recover x by taking x = ±√u, remembering both signs.
  5. Tactically count solution pairs: u = 0 gives one x-value, u > 0 gives two, u < 0 gives none.

Mechanic — Concrete operations

  1. Solve the quadratic in u to obtain u-roots.
  2. Test each u-root against the domain constraint u ≥ 0 implied by u = x².
  3. Discard any u-root with u < 0, stating explicitly 'no real x since x² ≥ 0'.
  4. For each retained u ≥ 0, solve x² = u to give x = ±√u (or x = 0 if u = 0).
  5. Count and list all real x-values; substitute back if solution pairs (x, y) are required.

Worked Micro-Example

Solve $x^4 - 15x^2 - 25 = 0$ for real $x$. Let $u = x^2$: $u^2 - 15u - 25 = 0$ gives $u = \tfrac{15 \pm \sqrt{325}}{2}$. One root is negative, so reject it; the positive root $u \approx 16.51$ gives $x = \pm\sqrt{16.51}$, yielding exactly two real solutions.

Variants — Same move, different surface

Exponential substitution u = e^x in an equation like e^{2x} - 3e^x + 2 = 0
Domain constraint becomes u > 0; reject any u ≤ 0 before taking ln u.
Trig substitution u = sin θ or u = cos θ
Domain constraint becomes -1 ≤ u ≤ 1; reject u-roots outside this interval.
Surd substitution u = √x
Domain constraint is u ≥ 0; reject negative u, then square the surviving u to recover x = u².
Modulus substitution u = |x|
Domain is u ≥ 0; for each retained u > 0, recover x = ±u.

Common Misreadings

  • Confusing 'x² cannot be negative' with 'x cannot be negative' and discarding x = -√u as well as the negative u-root.
  • Forgetting the ± when recovering x from a valid positive u, giving only one real solution instead of two.
  • Treating u = 0 as if it gave two x-values rather than the single value x = 0.

Pitfalls

  • In Further Pure with complex numbers permitted, a negative u still yields valid complex x = ±i√|u| — read the question to see whether real or all solutions are required.
  • Failing to state the rejection reason explicitly can cost a justification mark in 'how many real solutions' style questions.
  • Mis-solving the quadratic in u so that a sign error makes a genuinely positive root appear negative (or vice versa).

A-level Scope Notes

Standard across Edexcel IAL Pure (P1–P4) for disguised quadratics in x², e^x, sin θ, etc.; the complex-root variant is only relevant in Further Pure when the question explicitly allows non-real solutions.
