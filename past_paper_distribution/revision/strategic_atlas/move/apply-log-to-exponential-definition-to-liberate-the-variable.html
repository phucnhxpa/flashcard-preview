Algebra & Functions

Apply log-to-exponential definition to liberate the variable

Family: Log Exponential Bridge Frequency: 2 questions

Trigger — When to use

Fires whenever the equation has been reduced (or already given) in the form log_b(A) = C, where A contains the unknown and C is either a constant or an expression free of logarithms. Visual cues: a single isolated logarithm on one side, no log on the other side, and a target form that is either a numerical value of x or a clean exponential expression like x = pq^(-t). Equivalent phrasings of the trigger: 'one log term equals something non-log', 'the unknown is trapped inside log_b', or 'I need to anti-log both sides'. This is typically the final exit step after using log laws to condense multiple log terms into one.

Motivation — Why this works

The defining identity log_b(A) = C ⟺ A = b^C is the inverse relationship between logarithm and exponentiation; applying it removes the logarithm entirely and frees the unknown. This converts a transcendental-looking equation into an algebraic (often rational or linear) one, or transforms a log-form model into the exponential form required by the question. Without this step, no further algebraic manipulation of A is meaningful because the unknown remains locked inside the logarithm. It is the canonical bridge between log-world and exponential-world.

Thought Process — The step-by-step reasoning

  1. Notice the structural form: exactly one logarithm log_b(A) sits on one side, and the other side is either a constant or a log-free expression — so the equation is ready for direct conversion.
  2. Rule out further log-law manipulation (product/quotient/power rules): there is nothing left to combine, so condensing is finished.
  3. Rule out trying to solve for an inner variable first or taking logs again — both add complexity without freeing the unknown.
  4. Commit to the definition log_b(A) = C ⟺ A = b^C because it is the unique step that eliminates the log in one stroke.
  5. Tactical decision: identify the base b carefully (10, e, 3, etc.) — the resulting exponential must use the SAME base. Then write A = b^C explicitly, computing b^C numerically if C is a constant, or leaving it symbolic if C depends on another variable.
  6. Plan the next step: if A is rational/polynomial in x, cross-multiply and solve; if the goal is an exponential model, apply index laws to split b^C into the required product form.

Mechanic — Concrete operations

  1. Identify the base b and isolate the equation in the form log_b(A) = C.
  2. Apply the definition to rewrite as A = b^C.
  3. If C is numerical, evaluate b^C explicitly (e.g. 3^4 = 81, 10^{2.74} ≈ 549.5).
  4. If A is a rational expression, cross-multiply to obtain a polynomial equation; if A is just x, the conversion is the final answer (possibly followed by index-law splitting).
  5. Solve the resulting algebraic equation for x.
  6. Verify the solution lies in the domain of every original log argument (each argument must be > 0).

Worked Micro-Example

Solve $\log_3\!\left(\dfrac{x+5}{2x-1}\right) = 4$. Apply the definition: $\dfrac{x+5}{2x-1} = 3^4 = 81$. Cross-multiply: $x+5 = 81(2x-1) = 162x - 81$, giving $161x = 86$, so $x = \tfrac{86}{161}$. Check: $x>0$ and $2x-1 = \tfrac{11}{161} > 0$, so arguments are positive. ✓

Variants — Same move, different surface

Equation of the form log_b(A) = log_b(B) (logs on both sides, same base)
Skip exponentiation; equate arguments directly: A = B. Same bridge principle, applied to the injectivity of log_b.
Natural log: ln(A) = C
Use base e: A = e^C.
Log model needing exponential form, e.g. log_{10} x = a − bt with target x = pq^{−t}
Anti-log to get x = 10^{a-bt}, then split via index laws as x = 10^a · 10^{-bt} = 10^a · (10^b)^{-t}, identifying p = 10^a and q = 10^b.
Different base, e.g. log_2 x = f(t)
x = 2^{f(t)} — identical logic, just match the base.

Common Misreadings

  • Treating log_b(A) = C as if it implied A = C·b or A = C/b (confusing log with multiplication).
  • Trying to 'solve' for the parameter (e.g. t) instead of liberating the intended variable (e.g. x), losing sight of the target form.
  • Mistaking log_b(A) = log_b(B) for needing exponentiation when one can simply equate arguments.

Pitfalls

  • Arithmetic slips when computing b^C (e.g. writing 3^4 = 12 or 64 instead of 81).
  • Using the wrong base in the resulting exponential (e.g. using e when the original log was base 10).
  • Forgetting to check that the candidate solution keeps every original log argument strictly positive — extraneous roots are common when A is a rational function.
  • Applying the move before the equation is genuinely in single-log form (log laws not yet used to condense), leading to an invalid step.

A-level Scope Notes

Fully within Edexcel IAL Pure Maths (P2/P3) — the identity log_b(A) = C ⟺ A = b^C is foundational and assumed, not in the formula booklet. Stay with bases 10, e, and small integer bases that appear in problems; do not invoke change-of-base unless the bases on the two sides differ.
