Trigonometry

Translate the domain when substituting a phase-shifted argument

Family: Domain Shifting Frequency: 1 questions

Trigger — When to use

Fires whenever an equation has been reduced to a single trig function of a linear argument like sin(θ − α), cos(θ + α), or sin(2θ − α), and the original variable θ has a stated domain such as 0° ≤ θ < 360°. Cues include the appearance of an R sin(θ ± α) form, a substitution φ = θ ± α, or any compound argument inside one trig function. The student must convert the θ-bounds into bounds on the new argument before solving. Phrasings: 'find all θ in [0°,360°)', 'hence solve...', 'in the interval 0 ≤ θ < 2π'.

Motivation — Why this works

Standard solution techniques (principal value + symmetry, CAST, sine/cosine graph) deliver every solution of sin(φ)=k in a chosen φ-interval, so the φ-interval must exactly correspond to the allowed θ-interval. If you solve in the default [0°,360°) you may miss a solution whose θ-image lies in the domain but whose φ-image is negative, or include a spurious one whose θ-image exceeds the domain. Shifting bounds by −α (or scaling and shifting for kθ ± α) is the unique transformation that preserves the bijection between valid θ and valid φ.

Thought Process — The step-by-step reasoning

  1. I notice the equation now has a compound argument θ − α, not θ alone, so my standard solving template needs a new variable.
  2. I rule out solving sin(φ)=k on [0°,360°) and just adding α back — that silently assumes the φ-domain is the same as the θ-domain, which is false.
  3. I commit to letting φ = θ − α and explicitly translating the inequality 0° ≤ θ < 360° into bounds on φ.
  4. Concretely I subtract α throughout the θ-inequality to get −α ≤ φ < 360° − α, then solve sin(φ)=k inside this shifted window.
  5. Inside the move I list the principal value, use symmetry/periodicity to generate every φ in the shifted window (possibly including negative φ or φ slightly less than 360°), then add α back to recover θ.

Mechanic — Concrete operations

  1. Introduce φ = θ − α (or whatever linear argument appears).
  2. Apply the same linear transformation to every part of the θ-inequality to get the φ-domain.
  3. Find the principal value φ₀ = arcsin(k) (or arccos/arctan).
  4. Use symmetry (180° − φ₀ for sine, −φ₀ for cosine) and add/subtract 360° as needed to list every φ in the shifted window.
  5. Convert each valid φ back via θ = φ + α and verify θ lies in the original domain.

Worked Micro-Example

Solve $5\sin(\theta - 36.9°) = 2$ for $0° \le \theta < 360°$. Set $\varphi = \theta - 36.9°$, so $-36.9° \le \varphi < 323.1°$ and $\sin\varphi = 0.4$. Principal value $\varphi_1 = 23.6°$; symmetry gives $\varphi_2 = 180° - 23.6° = 156.4°$; both lie in the shifted window. Then $\theta = 23.6° + 36.9° = 60.5°$ or $\theta = 156.4° + 36.9° = 193.3°$.

Variants — Same move, different surface

Cosine with phase shift, e.g. $\cos(\theta + 40°) = k$ on $0° \le \theta < 360°$
Shifted window becomes $40° \le \varphi < 400°$; use cosine symmetry $\varphi = \pm\varphi_0 + 360°n$.
Double-angle plus shift, e.g. $\cos(2\theta + 30°) = k$ on $0° \le \theta < 360°$
Window becomes $30° \le 2\theta + 30° < 750°$, an interval of length 720°, so expect up to four solutions.
Radian problem, e.g. $\sin(\theta - \pi/6) = k$ on $0 \le \theta < 2\pi$
Shifted window is $-\pi/6 \le \varphi < 11\pi/6$; work in radians throughout.
R-formula form $R\sin(\theta - \alpha) = c$ from $a\sin\theta + b\cos\theta$
Same shift by $-\alpha$; the only extra step is computing $\alpha$ from $\tan\alpha = b/a$ first.

Common Misreadings

  • Solving $\sin\varphi = k$ on $[0°,360°)$ by default and then adding $\alpha$ — this can push a valid solution above 360° or omit a valid solution lying in $[-\alpha, 0°)$.
  • Forgetting that for arguments like $2\theta - \alpha$ the domain length doubles, so more than two solutions are expected.
  • Confusing the direction of the shift: writing $\alpha \le \varphi < 360° + \alpha$ instead of $-\alpha \le \varphi < 360° - \alpha$ when $\varphi = \theta - \alpha$.

Pitfalls

  • Dropping a solution that has negative $\varphi$ (e.g. $\varphi = -10°$) because it 'looks outside [0°,360°)', even though it lies in the shifted window.
  • Mixing degrees and radians when α is given as a decimal degree but the calculator is in radian mode.
  • Failing to test boundary cases: a φ exactly equal to $360° - \alpha$ corresponds to $\theta = 360°$, which is excluded by a strict inequality.

A-level Scope Notes

Fully within Edexcel IAL P2/P3 trigonometry: solving trig equations with compound or R-form arguments on a given interval, in degrees or radians.
