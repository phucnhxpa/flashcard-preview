Coordinate Geometry

Translate above/below boundary positions into inequality signs

Family: Region Inequality Assignment Frequency: 1 questions

Trigger — When to use

A figure shows a shaded planar region bounded by named curves and/or lines (often a parabola, a straight line, and an axis), and the task asks for a system of inequalities that defines the region. Phrasings include 'write down three inequalities that define R', 'find the inequalities satisfied by points in the shaded region', or 'describe R by inequalities'. Boundary equations are typically already given or easily readable from the figure. The move fires once each boundary curve has been identified.

Motivation — Why this works

Every boundary curve splits the plane into two half-regions, and the shaded region lies on exactly one side of each boundary. Reading 'shaded is below y = f(x)' yields y ≤ f(x); 'shaded is above' yields y ≥ f(x); for vertical boundaries x = a, 'shaded is to the left' yields x ≤ a. Whether the inequality is strict or non-strict is dictated by whether the boundary is drawn solid (included) or dashed (excluded). This turns a picture into a precise algebraic specification of R.

Thought Process — The step-by-step reasoning

  1. Notice the region R is enclosed by a finite list of named boundaries — typically a curve C, a line l, and a coordinate axis.
  2. Rule out trying to parametrise R or integrate; the question wants a static algebraic description, not a calculation.
  3. Commit to processing one boundary at a time, asking for each: 'on which side of this boundary does the shaded region sit?'
  4. Bridge: rewrite each boundary as y = f(x) (or x = g(y) if vertical) so that 'above/below' maps cleanly to ≥ / ≤.
  5. Tactically, keep each inequality in the form most natural to the given equation (e.g. preserve x + y ≤ 6 rather than re-expressing as y ≤ 6 − x), and check inclusion of the boundary by whether the figure draws it solid.

Mechanic — Concrete operations

  1. List all boundary equations of R from the figure.
  2. For each boundary, determine the side on which R lies (test a point inside R if unsure).
  3. Write the corresponding inequality: y ≤ f(x) for 'below', y ≥ f(x) for 'above', x ≤ g(y) or x ≥ g(y) for vertical-style boundaries.
  4. Choose ≤ / ≥ for solid boundaries (included) or < / > for dashed (excluded).
  5. Collect the inequalities as a system; verify by substituting one interior test point into all of them.

Worked Micro-Example

Region R is bounded by the curve $y = 6x - 2x^2 + 1$, the line $x + y = 6$, and the x-axis, with R lying below both the curve and the line and above the x-axis. Reading off: below curve gives $y \le 6x - 2x^2 + 1$; below line gives $x + y \le 6$; above x-axis gives $y \ge 0$. Test point $(1,1)$: $1 \le 5$ ✓, $2 \le 6$ ✓, $1 \ge 0$ ✓.

Variants — Same move, different surface

Region bounded on the side by a vertical line x = a
Use x ≤ a or x ≥ a depending on which side of the line R sits.
Interior of a circle (x − a)² + (y − b)² = r²
Inside is (x − a)² + (y − b)² ≤ r²; outside is ≥.
Region bounded by a curve given as x = g(y)
Use x ≤ g(y) for 'left of' and x ≥ g(y) for 'right of'.
Boundary drawn dashed (excluded)
Use strict inequality < or > instead of ≤ or ≥.
Region bounded by two curves with R sandwiched between them
Write a chain such as f(x) ≤ y ≤ g(x), one inequality per boundary.

Common Misreadings

  • Reversing direction — writing y ≥ f(x) when R lies below the curve.
  • Using strict < / > when the boundary is solid and therefore included.
  • Treating a curve given as x = g(y) as if it were y = … and getting the orientation wrong.
  • Assuming the region must be bounded by inequalities in y only, missing an x-based constraint.

Pitfalls

  • Forgetting a boundary entirely — e.g. omitting y ≥ 0 because the x-axis feels implicit.
  • Rearranging a line equation and introducing a sign error (e.g. writing y ≤ x − 6 instead of y ≤ 6 − x).
  • Mixing inequality directions when multiplying through by a negative to tidy a boundary equation.
  • Failing to verify with an interior test point, allowing a flipped inequality to slip through.

A-level Scope Notes

Standard Edexcel IAL Pure/Core content: linear programming-style region descriptions and curve-bounded regions appear in P1–P3 coordinate geometry and integration questions, using only polynomial, linear, and basic conic boundaries.
