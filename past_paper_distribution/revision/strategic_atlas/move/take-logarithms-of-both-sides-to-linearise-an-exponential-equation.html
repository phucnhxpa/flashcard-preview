Algebra & Functions

Take logarithms of both sides to linearise an exponential equation

Family: Log Linearisation Frequency: 4 questions

Trigger — When to use

Fires whenever the unknown is trapped in an exponent, e.g. equations of the form a^x = c, k·b^t = N, or models like N = ab^t and y = ae^(kt). Visual cues: a variable sitting as a power; a question asking to 'solve for t' in an exponential decay/growth model; a question stating that plotting log P (or ln P) against t yields a straight line; a target answer expressed as a ratio of logs. Equivalent phrasings: 'use algebra to find t', 'show that x = log A / log B', 'find a and b given the linear graph of log N vs t'.

Motivation — Why this works

Logarithms are the only algebraic tool that brings an exponent down to a coefficient, via the power rule log(a^b) = b·log(a). Combined with the product rule log(ab) = log a + log b, applying log to both sides of an exponential equation converts a multiplicative/exponential structure into an additive/linear one. This either isolates the unknown exponent directly (giving x = log c / log a) or transforms a model y = ab^t into Y = c + mt with Y = log y, enabling gradient–intercept extraction. The base of the logarithm is free to choose: match it to the form requested in the question or to the axis on the graph to avoid an unnecessary change-of-base step.

Thought Process — The step-by-step reasoning

  1. I notice the unknown is in the exponent (e.g. t in 0.72 = 0.999878^t, or constants a,b inside ab^t) — no purely algebraic rearrangement can extract it.
  2. I rule out trial-and-improvement, taking t-th roots (t is unknown so this fails), and inverse functions other than log; the question often explicitly says 'use algebra' or shows a log-axis plot, confirming logs are intended.
  3. I commit to taking logs of both sides because the power rule will pull the exponent down as a coefficient, producing a linear equation.
  4. Before logging, I isolate the exponential term if it has a coefficient (e.g. divide N = ab^t by nothing if I want to keep both constants, or divide 25·0.999878^t = 18 by 25 first if I want a single exponential).
  5. I choose the log base to match the target form: log₁₀ if the answer or graph uses log₁₀, ln if the model uses e^(kt). This avoids a redundant change-of-base at the end.
  6. I apply the product rule first (to split ab^t into log a + log b^t), then the power rule (to bring t down), then either divide to isolate x or compare term-by-term with y = mx + c.

Mechanic — Concrete operations

  1. If necessary, isolate the exponential term: e.g. k·b^t = N ⟹ b^t = N/k.
  2. Take log (chosen base) of both sides: log(b^t) = log(N/k), or log(ab^t) = log N for model fitting.
  3. Apply the product rule where applicable: log(ab^t) = log a + log(b^t).
  4. Apply the power rule: log(b^t) = t·log b.
  5. For solving: divide to isolate the unknown, e.g. t = log(N/k) / log b.
  6. For model-fitting: identify gradient m = log b and intercept c = log a from the linear equation log N = log a + t·log b, then recover a = 10^c (or e^c) and b = 10^m.

Worked Micro-Example

Solve $2 = 24^x$. Take $\log_{10}$ of both sides: $\log_{10} 2 = \log_{10}(24^x)$. Apply the power rule: $\log_{10} 2 = x\,\log_{10} 24$. Divide: $x = \dfrac{\log_{10} 2}{\log_{10} 24}$.

Variants — Same move, different surface

Single exponential equation a^x = c with answer required as log_a c
Take log in any base; result x = log c / log a is equivalent to log_a c by change of base.
Equation involving e, e.g. e^(kt) = c
Take ln (natural log) of both sides so ln(e^(kt)) = kt collapses cleanly, giving t = ln c / k.
Model N = ab^t with linear graph of log₁₀ N vs t
Take log₁₀ to get log₁₀ N = log₁₀ a + t·log₁₀ b; compare with y = mx + c to read off a = 10^(intercept), b = 10^(gradient).
Power-law model y = ax^n
Take log of both sides to get log y = log a + n·log x; both variables are logged, plot log y vs log x.
Coefficient in front of exponential, e.g. N = k·b^t with k known
Divide by k first to isolate b^t before logging, OR keep as log N = log k + t·log b for graph comparison.

Common Misreadings

  • Trying to take the t-th root of both sides when t is unknown — this does not extract t.
  • Using ln when the graph axis or target form uses log₁₀ (or vice versa), causing a base mismatch in coefficients.
  • Attempting to guess values of a and b numerically from data points instead of using the linearised comparison.

Pitfalls

  • Misapplying the power rule: writing log(b^t) = b·log t instead of t·log b.
  • Misapplying the product rule: writing log(ab) = log a · log b instead of log a + log b.
  • Inverting the final ratio: x = log 24 / log 2 instead of x = log 2 / log 24.
  • Sign errors when both arguments are between 0 and 1 (both logs negative, ratio positive) — students often drop a minus sign.
  • Forgetting to isolate the exponential before logging when there is a leading coefficient.
  • Using a rounded value of the base (e.g. b ≈ 0.999878) too early, amplifying rounding error in t.

A-level Scope Notes

Fully within Edexcel IAL P2/P3 (Exponentials and Logarithms). Uses the laws of logarithms (product, quotient, power) and the relationship log_a b = log b / log a (change of base, P3 formula booklet). Avoid generalising to complex-valued logs or to bases ≤ 0 — outside scope.
