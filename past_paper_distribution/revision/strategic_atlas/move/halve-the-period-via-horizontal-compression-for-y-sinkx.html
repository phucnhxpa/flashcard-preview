Trigonometry

Halve the period via horizontal compression for y = sin(kx)

Family: Graph Transformation Frequency: 1 questions

Trigger — When to use

The question asks you to sketch y = sin(kx), y = cos(kx), or y = tan(kx) — typically with k a positive integer such as 2 or 3 — often on the same axes as the parent curve y = sin x. The cue is that the argument has been multiplied by a constant inside the trig function, with no other change. Phrases like 'on the same diagram, sketch y = sin 2x for 0° ≤ x ≤ 360°' are the standard signal. This is a textbook f(kx) horizontal-stretch surface, not a translation or vertical scaling.

Motivation — Why this works

Replacing x by kx inside a trig function applies a horizontal stretch by scale factor 1/k, leaving amplitude unchanged. Because sin and cos have period 360°, the new period becomes 360°/k, so k complete cycles fit into [0°, 360°]. Treating it as a transformation (rather than plotting points) instantly relocates every zero, maximum, and minimum to k× as frequent x-positions. This is far quicker and less error-prone than tabulating values.

Thought Process — The step-by-step reasoning

  1. Notice the argument is kx (here 2x), not x + a or x − a, so this is an inside-the-function change affecting horizontal scale.
  2. Rule out a vertical stretch: there is no coefficient outside the sine, so amplitude stays at 1.
  3. Commit to the f(kx) rule: horizontal stretch by 1/k about the y-axis, hence period divides by k.
  4. Bridge: compute the new period 360°/k and list the five key x-values of one cycle (0, ¼T, ½T, ¾T, T).
  5. Tactical decision: extend the pattern across the stated domain so all k cycles are drawn, ensuring endpoints and intermediate zeros are placed accurately.

Mechanic — Concrete operations

  1. Identify k from the argument kx; here k = 2.
  2. New period T = 360°/k = 180°.
  3. List one cycle's key points: (0°, 0), (45°, 1), (90°, 0), (135°, −1), (180°, 0).
  4. Translate the pattern by T to fill the domain: (180°, 0), (225°, 1), (270°, 0), (315°, −1), (360°, 0).
  5. Draw a smooth sinusoidal curve through these points with amplitude 1, on the same axes as y = sin x for comparison.

Worked Micro-Example

Sketch $y=\sin 2x$ for $0°\le x\le 360°$. Here $k=2$, so period $=360°/2=180°$. Key points of cycle 1: $(0°,0),(45°,1),(90°,0),(135°,-1),(180°,0)$. Repeat for cycle 2: $(225°,1),(270°,0),(315°,-1),(360°,0)$. Join smoothly; amplitude remains $1$.

Variants — Same move, different surface

y = sin(3x) on [0°, 360°]
Period = 120°; three full cycles in the domain, key points at multiples of 30°.
y = cos(2x) on [0°, 360°]
Same period 180°, but start at maximum (0°, 1) rather than at 0.
y = sin(x/2) on [0°, 360°]
k = 1/2 gives period 720°, so only half a cycle appears in [0°, 360°].
y = tan(2x)
Parent period is 180°, so new period = 90°; vertical asymptotes at x = 45°, 135°, 225°, 315°.
y = 2 sin(2x)
Combine with vertical stretch: period 180° and amplitude 2; key y-values become ±2.

Common Misreadings

  • Treating the 2 as a vertical stretch and doubling the amplitude to 2.
  • Multiplying the period by k instead of dividing (drawing a slower curve with period 720°).
  • Confusing sin(2x) with sin(x) + sin(x) or with sin(x + 2).

Pitfalls

  • Placing the first maximum at 90° instead of 45° — halving the period but forgetting to halve the turning-point x-values.
  • Drawing only one cycle and stopping at 180° when the domain runs to 360°.
  • Miscounting zeros: there should be 5 zeros in [0°, 360°] for y = sin 2x including both endpoints.
  • Letting the curve change amplitude as it compresses — the peaks must still reach exactly ±1.

A-level Scope Notes

This is core Edexcel IAL P2 graph-transformation material covering y = f(kx) applied to sin, cos, and tan, with k a positive rational. No calculus or composite transformations beyond standard stretch/translate are required.
