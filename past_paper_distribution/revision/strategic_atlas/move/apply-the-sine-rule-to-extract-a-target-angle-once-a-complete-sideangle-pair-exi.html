Trigonometry

Apply the Sine Rule to extract a target angle once a complete side–angle pair exists

Family: Sine Rule Angle Frequency: 1 questions

Trigger — When to use

Fires in a triangle problem once you possess a complete opposite side–angle pair (a, A) and you want a second angle B whose opposite side b is known. Typical cues: 'find angle …' after the Cosine Rule has just delivered the missing third side, or after an isosceles/SAS setup leaves one side and its opposite angle paired. Equivalent phrasings: 'use the Sine Rule for the angle', 'solve sin B = b sin A / a'. Prefer this over the Cosine-Rule-for-an-angle when one side–angle pair is already clean, since it is a single-step solve.

Motivation — Why this works

The Sine Rule sin A / a = sin B / b is a direct proportion between each angle's sine and its opposite side, valid in any triangle. Once one ratio sin A / a is fixed, every other angle in the triangle is pinned down by its opposite side. Rearranging gives sin B = b sin A / a, a one-line solve requiring only a single arcsin. The Cosine Rule for the angle would re-use information you already have and introduce extra squaring, so the Sine Rule is the minimal-arithmetic route here.

Thought Process — The step-by-step reasoning

  1. Notice that the target angle ∠AOB has its opposite side AB = 1.4 already known, and a complete pair (∠OAB = 2 rad, OB ≈ 1.7376) is available.
  2. Rule out the Cosine Rule for the angle: it would require all three sides squared and more arithmetic for an identical result.
  3. Commit to the Sine Rule because exactly one unknown (∠AOB) sits in a single equation sin(∠AOB)/1.4 = sin 2 / 1.7376.
  4. Bridge to computation: isolate sin(∠AOB) = 1.4 sin 2 / 1.7376, then arcsin.
  5. Tactically check the ambiguous case: geometrically ∠AOB must be acute (it is half of θ < π), so arcsin's principal value is the correct branch; also keep OB unrounded to avoid drift in the final 3-s.f. answer.

Mechanic — Concrete operations

  1. Identify the complete pair (A, a) and the target pair (B, b): here A = ∠OAB = 2, a = OB ≈ 1.7376, b = AB = 1.4.
  2. Write the Sine Rule: sin(∠AOB)/1.4 = sin 2 / 1.7376.
  3. Rearrange: sin(∠AOB) = 1.4 sin 2 / 1.7376 ≈ 0.73264.
  4. Take arcsin (principal value, since the angle is known acute): ∠AOB ≈ 0.8222 rad.
  5. Feed back into the wider problem (e.g. θ = 2∠AOB ≈ 1.64 rad).

Worked Micro-Example

In $\triangle OAB$, $\angle OAB = 2$ rad, $OB \approx 1.7376$, $AB = 1.4$. By the Sine Rule, $\dfrac{\sin(\angle AOB)}{1.4} = \dfrac{\sin 2}{1.7376}$, so $\sin(\angle AOB) \approx 0.73264$. Since $\angle AOB$ is acute, $\angle AOB \approx \arcsin(0.73264) \approx 0.8222$ rad.

Variants — Same move, different surface

After SAS Cosine Rule gives the third side, asked for one of the remaining angles
Use the newly-found side as 'a' opposite the original given angle 'A', then Sine Rule onto the angle whose opposite side is one of the original two.
Isosceles triangle with apex angle and equal sides known, base required
Pair the apex angle with the base, then Sine Rule recovers a base angle (often redundant since base angles = (π − apex)/2).
Target angle could be obtuse (e.g. opposite the longest side)
After arcsin, replace by π − arcsin(...) when geometric reasoning (largest side ↔ largest angle, or diagram) indicates obtuse.
Bearings / triangle of forces problem with one side–angle pair given
Same algebra; just translate compass bearings into interior triangle angles before applying the rule.

Common Misreadings

  • Pairing an angle with a non-opposite side (e.g. using ∠OAB with AB instead of OB), which breaks the Sine Rule.
  • Assuming the Cosine Rule for the angle is required whenever three sides are known, missing the cheaper Sine Rule route.
  • Treating arcsin's output as automatically the correct angle without checking acute vs obtuse.

Pitfalls

  • Sine Rule ambiguity: arcsin returns a value in [0, π/2]; if the angle is actually obtuse you need π − arcsin(...). Here geometric reasoning confirms acute.
  • Premature rounding of the intermediate side (OB) propagating into the final angle and breaking the required 3-s.f. accuracy.
  • Mode error: working in radians for sin 2 but reading the calculator in degrees, or vice versa.
  • Algebraic slip when rearranging sin B / b = sin A / a — inverting the wrong ratio.

A-level Scope Notes
