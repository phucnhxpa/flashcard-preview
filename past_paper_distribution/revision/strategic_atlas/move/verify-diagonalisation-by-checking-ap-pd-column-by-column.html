Matrices

Verify diagonalisation by checking AP = PD column-by-column

Family: Verification Strategy Frequency: 2 questions

Trigger — When to use

Fires after eigenvalues and eigenvectors have been computed and assembled into P and D, and the student wants a fast confidence check before using the diagonalisation (e.g. for A^n or in a 'show that' / 'verify' sub-part). Also fires whenever the question carries enough marks to suggest verification is expected, or when arithmetic-heavy row reduction makes errors likely. Surface cues: a multi-mark diagonalisation question, a follow-on part that depends on P and D being correct, or explicit instructions to 'verify' an eigenpair.

Motivation — Why this works

The diagonalisation identity A = PDP⁻¹ is equivalent to AP = PD, which avoids inverting P and the two matrix multiplications needed to check PDP⁻¹ = A. PD is essentially free to write down (its j-th column is just λ_j times the j-th column of P), so the only real work is one matrix multiplication AP. Comparing column-by-column simultaneously verifies every eigenvalue–eigenvector pair, since column j of AP = PD is exactly the eigenpair equation Ae_j = λ_j e_j. Catching errors here prevents them propagating into A^n or any later computation.

Thought Process — The step-by-step reasoning

  1. Notice that the eigenvalues and eigenvectors came from multi-step algebra (characteristic polynomial, then row reduction), so arithmetic slips are likely and a check is worth the time.
  2. Rule out the brute-force verification PDP⁻¹ = A: it requires computing P⁻¹ plus two matrix multiplications — expensive and error-prone.
  3. Rule out merely checking det(P) ≠ 0 or det(A − λI) = 0: those confirm invertibility / eigenvalue only, not the eigenvectors.
  4. Commit to AP = PD because right-multiplying A = PDP⁻¹ by P eliminates P⁻¹ entirely, leaving a single matrix multiplication and a trivial scaling.
  5. Bridge to action: write down PD by scaling each column of P by its eigenvalue (no multiplication needed), then compute AP and compare.
  6. Tactical decision: if any column disagrees, the error is localised to that single eigenpair, so go back to the row reduction for that specific λ rather than redoing everything.

Mechanic — Concrete operations

  1. Form PD by multiplying column j of P by λ_j for each j (no full matrix multiplication required).
  2. Compute AP by ordinary matrix multiplication of A with P (be careful: AP, not PA).
  3. Compare AP and PD entry-by-entry; equivalently, check column j of AP equals λ_j times column j of P.
  4. If a column matches, that eigenpair (λ_j, e_j) is confirmed; if not, return to the linear system (A − λ_j I)x = 0 to locate the slip.

Worked Micro-Example

Suppose $A=\begin{pmatrix}2&1\\0&3\end{pmatrix}$, $P=\begin{pmatrix}1&1\\0&1\end{pmatrix}$, $D=\begin{pmatrix}2&0\\0&3\end{pmatrix}$. Then $PD=\begin{pmatrix}2&3\\0&3\end{pmatrix}$ (columns of $P$ scaled by $2$ and $3$), and $AP=\begin{pmatrix}2&3\\0&3\end{pmatrix}$. Columns agree, so both eigenpairs are verified in one multiplication.

Variants — Same move, different surface

Verifying a single eigenpair (e.g. a 'show that v is an eigenvector' sub-part)
Compute Av directly and check it equals λv — this is the column-by-column version of AP = PD applied to one column only.
Only the eigenvalue needs checking (not the eigenvector)
Substitute λ into det(A − λI) and confirm it gives 0; note this does NOT validate the eigenvector.
3×3 case with repeated eigenvalues
Same AP = PD check works; the column comparison still localises errors to a specific eigenvector even when λ is repeated.

Common Misreadings

  • Treating det(P) ≠ 0 as a verification — it only confirms P is invertible, not that the eigenpairs are correct.
  • Believing that verifying det(A − λI) = 0 is enough — it checks the eigenvalue but says nothing about the eigenvector.
  • Skipping verification entirely to save time, then losing follow-through marks on later parts (e.g. A^n) because of an early slip.

Pitfalls

  • Computing PA instead of AP — order matters; PA generally does not equal PD.
  • Scaling the wrong column of P by the wrong eigenvalue when forming PD (the j-th column of D must match the j-th column of P).
  • A sign error in the matrix-vector product can falsely flag a correct eigenvector as wrong — recheck the multiplication before redoing the eigenvector.

A-level Scope Notes

Within Edexcel IAL Further Pure scope for 2×2 and 3×3 diagonalisation; uses only matrix multiplication and the definition Av = λv (no formula booklet entry). Avoid generalisations to non-diagonalisable matrices or Jordan form, which are out of scope.
