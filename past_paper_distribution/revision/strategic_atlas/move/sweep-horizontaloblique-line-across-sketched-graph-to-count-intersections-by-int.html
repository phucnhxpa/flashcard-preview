Algebra & Functions

Sweep horizontal/oblique line across sketched graph to count intersections by interval

Family: Graphical Intersection Analysis Frequency: 4 questions

Trigger — When to use

The question asks for values of a parameter k such that an equation like f(x) = k (or f(x) = g_k(x)) has exactly a specified number of roots, where f has already been sketched (or is easy to sketch) — typically a V-shape from a modulus, a cubic, or a reciprocal. Visual cues: the parameter appears only as a height (y = k) or as a slope/intercept of a line, the equation would be algebraically intractable, and earlier parts of the question asked for a sketch. Equivalent phrasings: 'find the set of k for which the line y = k meets the graph in exactly two points', or 'state the number of solutions of f(x) = k'.

Motivation — Why this works

Each real root of f(x) = k corresponds to an x-coordinate where the curve y = f(x) meets the line y = k, so 'number of roots' becomes 'number of intersections'. As k increases, the horizontal (or oblique) line sweeps across the graph and the intersection count only changes at critical k-values: turning points (vertex of a V, local max/min) and domain-boundary y-values (open or closed endpoints). Identifying these critical k-values partitions the real line into intervals on which the count is constant, converting a hard 'for all k' algebraic problem into a simple read-off from a sketch. This is the standard examiner technique whenever a sketch in part (a) is followed by a 'how many roots' part.

Thought Process — The step-by-step reasoning

  1. Notice that the equation involves a parameter k that appears only as a horizontal level (or as a simple line parameter), and that earlier parts of the question already required a sketch — strong structural hint that a graphical approach is intended.
  2. Rule out direct algebra: clearing the modulus or cross-multiplying typically yields a high-degree polynomial (e.g. quintic) or a case-split that is more work than reading off a graph.
  3. Commit to graphical reasoning: the count of roots equals the number of intersections between y = f(x) and y = k, so identify all critical k-values where the count changes.
  4. Find these critical levels by computing (i) y-coordinates of vertices/turning points and (ii) y-values at any restricted-domain endpoints, being careful whether each endpoint is open or closed.
  5. Decide which arm/branch each intersection belongs to: e.g. for a V-shape with a truncated left arm, check whether the level k still reaches the left arm.
  6. Assemble the answer as a union of intervals, with strict inequalities where the line meets a vertex (giving a tangency / single root) or an open endpoint (root at excluded x-value), and weak inequalities where the endpoint is included.

Mechanic — Concrete operations

  1. Sketch (or recall the sketch of) y = f(x), marking vertex/turning points and any domain endpoints.
  2. Compute the y-coordinate of each vertex/turning point: e.g. for f(x) = 21 − 2|x − 2|, vertex at (2, 21).
  3. Compute f at each domain boundary: e.g. f(0) = 17 if the domain is x ≥ 0; note whether the endpoint is included (closed dot) or excluded (open dot).
  4. Determine the y-range covered by each arm/branch separately: left arm spans [17, 21], right arm spans (−∞, 21].
  5. For the required count (e.g. exactly two roots), find the k-values for which the horizontal line lies on the correct number of arms simultaneously.
  6. State the answer as an interval, using < at vertex levels (which give only one intersection) and using ≤ or < at boundary levels according to whether the endpoint is closed or open.

Worked Micro-Example

Let $f(x) = 21 - 2|x-2|$ for $x \ge 0$. Vertex: $(2, 21)$. Boundary: $f(0) = 21 - 4 = 17$ (closed). Left arm covers $y \in [17, 21]$; right arm covers $y \in (-\infty, 21]$. The line $y = k$ meets both arms iff $17 \le k < 21$ (excluding $k=21$ where the line is tangent at the vertex, giving only one root). Hence $f(x) = k$ has exactly two roots for $17 \le k < 21$.

Variants — Same move, different surface

Modulus on an open domain, e.g. f(x) = 2|3 − x| + 5 for x > 0
Boundary value f(0⁺) = 11 is not attained, so the upper bound is strict: 5 < k < 11 (strict at vertex k = 5 and strict at open endpoint k = 11).
Oblique line y = kx + c instead of horizontal y = k
Decompose the V-shape into two linear arms; solve y = kx + c against each arm separately. The critical k-values are the gradients of the two arms (parallel cases) and any value making the line pass through the vertex.
Equation f(x) = g(x) where both sides are sketched curves (e.g. cubic = reciprocal)
Count intersections of the two sketched curves directly, rather than intersecting one curve with a horizontal line — same principle, two non-trivial curves.
Unrestricted domain V-shape on all of ℝ
Endpoint considerations vanish; two roots occur for all k strictly below (or above, if inverted) the vertex y-value.

Common Misreadings

  • Trying to solve the equation algebraically by squaring or cross-multiplying, producing a high-degree polynomial that is intractable at A-level.
  • Ignoring the domain restriction and treating the graph as the full V-shape on ℝ, which gives a wrong (often unbounded) interval for k.
  • Confusing 'exactly two roots' with 'at least two roots' and accidentally including the vertex level.

Pitfalls

  • Including the vertex y-value k = (vertex height): the horizontal line is tangent there and gives only one root, so this k must be excluded.
  • Using ≤ versus < incorrectly at a domain boundary: closed endpoint (x = 0 included) → k = f(0) gives a genuine root, so use ≤; open endpoint (x = 0 excluded) → use strict <.
  • Miscounting which arm a horizontal line still reaches once one arm is truncated by the domain.
  • Producing an inaccurate sketch (e.g. wrong vertex location or wrong gradient sign) and reading off the wrong intervals.

A-level Scope Notes

Entirely within Edexcel IAL Pure Maths (P2/P3 modulus functions, transformations of graphs) and Further Pure (curve sketching of rational functions). No formula booklet entry is invoked; the technique is purely graphical. Avoid generalising to discontinuous or implicitly-defined curves where intersection counting requires calculus tools beyond A-level.
