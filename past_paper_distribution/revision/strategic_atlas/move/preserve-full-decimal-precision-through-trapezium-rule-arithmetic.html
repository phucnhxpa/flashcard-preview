Integration

Preserve full decimal precision through trapezium-rule arithmetic

Family: Precision Management Frequency: 1 questions

Trigger — When to use

Fires in numerical-integration questions (trapezium rule, Simpson's rule in FM) where the table of y-values is given or computed to a stated precision (typically 3 or 4 d.p.) and the final answer is requested to a specific accuracy. Cues include phrases like 'giving your answer to 4 decimal places', 'using the values in the table', or a mark scheme that lists a single exact terminating decimal as the target. Also fires whenever a multi-stage calculation chains sums and products of decimals.

Motivation — Why this works

Each rounding step injects an error of up to half a unit in the last place; these errors accumulate additively through sums and multiplicatively through the h/2 or h/3 factor, often pushing the final value outside the mark scheme's tolerance band. The mark scheme is computed by applying the formula to the tabulated values exactly, so the only way to land on the expected figure is to mirror that exact arithmetic. Holding full precision until the final reporting step is therefore the only reliable strategy.

Thought Process — The step-by-step reasoning

  1. I notice the y-values are quoted to 3 d.p. and the question asks for the final integral to 4 d.p. — the precision target is tighter than the inputs, so every digit matters.
  2. I rule out the temptation to 'tidy up' intermediate sums to 2 or 3 d.p. — that would discard information the mark scheme uses.
  3. I commit to carrying every digit (or storing values in calculator memory) until the very last step, because trapezium rule is a single closed-form expression and there is no benefit to early rounding.
  4. First concrete operation: form the bracket [y₀ + yₙ + 2(y₁ + ⋯ + yₙ₋₁)] using the full given decimals.
  5. Tactically, I keep the interior sum as one number, multiply by 2, add the endpoints, then multiply by h/2 in one go — and only round (or not) at the reporting stage.

Mechanic — Concrete operations

  1. Read the requested final precision and compare with the precision of the y-values.
  2. Write all y-values to their full given precision; do not pre-round.
  3. Compute the interior sum y₁ + y₂ + ⋯ + yₙ₋₁ keeping all decimal places.
  4. Apply the trapezium rule (h/2)[y₀ + yₙ + 2·(interior sum)] in a single chained calculation.
  5. Report the result to the precision demanded; if the result terminates within that precision, quote it exactly without further rounding.

Worked Micro-Example

With $h=0.25$ and $y_0=1.000,\ y_1=1.251,\ y_2=1.494,\ y_3=1.741,\ y_4=2.000$: interior sum $=1.251+1.494+1.741=4.486$. Bracket $=1.000+2.000+2(4.486)=11.972$. Integral $\approx \tfrac{0.25}{2}(11.972)=0.125\times 11.972=1.4965$. Quote $1.4965$ exactly — not $1.497$.

Variants — Same move, different surface

Simpson's rule (Further Pure)
Same principle: keep y-values full precision, apply (h/3)[y₀+yₙ+4(odd)+2(even)] in one chained step.
Question explicitly asks for answer to 3 s.f. or 2 d.p.
Still carry full precision internally; round only at the final reporting line.
y-values must be computed from a given function before the rule is applied
Store function values to at least one more decimal place than the requested final answer, then round each y to the table-precision specified in the question for display, but compute with the stored values.
Solution-of-equations or iteration questions in P-level
Carry extra digits through iterations and round only the reported root, to avoid spurious convergence behaviour.

Common Misreadings

  • Believing that because inputs are to 3 d.p., the answer must also be reported to 3 d.p.
  • Rounding each intermediate product or sum to match the input precision before continuing.
  • Treating a terminating decimal like 1.4965 as if it 'should' be rounded to 1.497 for neatness.

Pitfalls

  • Transcribing a rounded calculator display mid-calculation instead of using ANS or memory.
  • Truncating rather than rounding when copying digits, losing further accuracy.
  • Applying h/2 first to each ordinate separately and rounding each, instead of factoring h/2 outside the bracket.

A-level Scope Notes

Sits squarely within Edexcel IAL C2/P2 trapezium rule and FP Simpson's rule conventions; the precision discipline matches standard mark-scheme expectations on numerical-integration questions.
