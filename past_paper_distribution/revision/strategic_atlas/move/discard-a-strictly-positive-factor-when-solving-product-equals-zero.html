Discard a strictly positive factor when solving product equals zero

Family: Zero Product Property Frequency: 1 questions

Trigger — When to use

Fires when an equation of the form A(x)·B(x) = 0 arises (typically from setting dy/dx = 0) and one factor A(x) is provably non-zero on the given domain. Common cues: factors like x^{f(x)}, e^{f(x)}, or a variable y stated to be positive. Phrasings such as 'for x > 0', 'y > 0', or 'where y = x^{sin x}' signal the positivity. The move applies whenever a product is set to zero and at least one factor is structurally positive.

Motivation — Why this works

By the zero-product property, AB = 0 forces A = 0 or B = 0. If A is strictly positive on the relevant domain, the case A = 0 is impossible, so the equation reduces to B = 0. This collapses a two-case analysis into a single equation. For x^{sin x} = e^{sin x · ln x}, the exponential is always positive, so the factor cannot vanish.

Thought Process — The step-by-step reasoning

  1. Notice the derivative (or expression) has factored cleanly into a product equal to zero.
  2. Rule out one factor by checking the domain: rewrite x^{sin x} as e^{sin x · ln x} and observe the exponential is always positive.
  3. Commit to discarding that factor because the domain restriction (x > 0 or y > 0) guarantees non-vanishing.
  4. Bridge: divide both sides by the positive factor (with brief justification) so only the remaining bracket equation survives.
  5. Tactically, write a one-line justification ('since x^{sin x} > 0') to secure the reasoning mark before solving the reduced equation.

Mechanic — Concrete operations

  1. Identify the product structure A(x)·B(x) = 0.
  2. Rewrite the suspect factor in exponential form if helpful, e.g. x^{sin x} = e^{sin x · ln x}.
  3. State that this factor is strictly positive on the given domain, hence non-zero.
  4. Conclude the other factor must equal zero and write down the reduced equation.
  5. Proceed to solve B(x) = 0.

Worked Micro-Example

Given y = x^{sin x} with x > 0, differentiation gives dy/dx = x^{sin x}(cos x · ln x + sin x / x). Setting dy/dx = 0: since x^{sin x} = e^{sin x · ln x} > 0, we discard it. Hence cos x · ln x + sin x / x = 0, the equation to solve for stationary points.

Variants — Same move, different surface

dy/dx = e^{f(x)} · g(x) = 0
Discard e^{f(x)} since exponentials are always positive; solve g(x) = 0.
Implicit differentiation yielding y · h(x) = 0 with y > 0 stated
Discard y by the positivity hypothesis; solve h(x) = 0.
Product like (1 + x^2) · p(x) = 0 over reals
Discard 1 + x^2 since it is always ≥ 1 > 0; solve p(x) = 0.
Logarithmic differentiation gives y · (…) = 0 where y = a^{f(x)}, a > 0
Since a^{f(x)} > 0, discard the y factor and solve the bracket.

Common Misreadings

  • Forgetting to justify why the discarded factor is non-zero, losing the reasoning mark.
  • Treating x^{sin x} = 0 as a possible case and chasing a spurious solution branch.
  • Confusing 'strictly positive' with 'non-negative' and missing that a factor like 1 - cos x can vanish.

Pitfalls

  • Discarding a factor without checking the domain (e.g. x^{sin x} requires x > 0 to be well-defined).
  • Dividing by the factor without first stating it is non-zero, leading to mark deductions.
  • Overlooking that the exponential rewrite x^{sin x} = e^{sin x · ln x} relies on x > 0.

A-level Scope Notes

Standard for Edexcel IAL P3/P4 calculus where stationary points of products like x^{sin x}, e^{f(x)}g(x), or implicit forms appear. The justification 'positive base, so factor > 0' is expected reasoning.
