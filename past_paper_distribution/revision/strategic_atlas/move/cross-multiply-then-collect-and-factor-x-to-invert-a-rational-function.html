Algebra & Functions

Cross-multiply then collect and factor x to invert a rational function

Family: Inverse Function Application Frequency: 1 questions

Trigger — When to use

Fires when finding the inverse of a function f(x) = (ax+b)/(cx+d) where x appears in both numerator and denominator. The cue is that you cannot 'undo' f as a simple chain of operations because x is entangled on both sides of the fraction bar. Any rational function with x in both parts of the quotient — including disguised forms like f(x) = (3-x)/(x+2) or f(x) = x/(2x-1) — triggers this move.

Motivation — Why this works

Setting y = f(x) and clearing the denominator turns the equation into a linear equation in x whose coefficients are expressions in y. Once linear in x, the standard tactic of collecting all x-terms on one side, factoring x, and dividing isolates x cleanly. This works precisely because Möbius-type (linear fractional) functions have inverses that are themselves linear fractional, guaranteed by the algebraic structure (ad - bc ≠ 0).

Thought Process — The step-by-step reasoning

  1. Notice x appears in both numerator and denominator, so f is not a simple operation chain on x.
  2. Rule out 'reverse the operations' — there is no last operation to undo because x is in two places.
  3. Commit to the cross-multiply / collect / factor route as the only A-level technique that handles entangled x.
  4. Bridge: write y = f(x), then multiply both sides by the denominator to clear the fraction, producing a linear equation in x.
  5. Tactically decide which side to gather x-terms on (usually the side with the larger coefficient, to avoid sign slips), factor x, divide, and finally swap y and x to express f⁻¹(x).

Mechanic — Concrete operations

  1. Write y = (ax+b)/(cx+d)
  2. Multiply both sides by (cx+d): y(cx+d) = ax+b
  3. Expand: cxy + dy = ax + b
  4. Move all x-terms to one side: cxy - ax = b - dy
  5. Factor out x: x(cy - a) = b - dy
  6. Divide: x = (b - dy)/(cy - a)
  7. Swap variables to obtain f⁻¹(x), and state the domain restriction excluding the value that makes the denominator zero

Worked Micro-Example

Let $f(x) = \dfrac{2x+3}{x-1}$. Set $y = \dfrac{2x+3}{x-1}$, so $y(x-1) = 2x+3$, giving $yx - y = 2x + 3$. Collect: $yx - 2x = y + 3$, factor: $x(y-2) = y+3$, divide: $x = \dfrac{y+3}{y-2}$. Hence $f^{-1}(x) = \dfrac{x+3}{x-2}$, $x \neq 2$.

Variants — Same move, different surface

General Möbius form f(x) = (ax+b)/(cx+d)
Same procedure; result is f⁻¹(x) = (-dx+b)/(cx-a), valid when ad - bc ≠ 0.
Rational with x only in numerator, e.g. f(x) = (2x+3)/5
Cross-multiplying gives 5y = 2x+3 — a single x-term, so just rearrange directly without factoring.
Rational with x only in denominator, e.g. f(x) = 3/(x+1)
Cross-multiply to y(x+1) = 3, then solve linearly for x; no factoring step needed.
Quadratic-over-linear or higher, e.g. f(x) = (x²+1)/(x-2)
Cross-multiplying yields a quadratic in x; use the quadratic formula in y, and select the branch consistent with the stated domain.
Composite function inversion where one component is rational
Apply this move to invert the rational layer, then compose with the inverses of the other layers in reverse order.

Common Misreadings

  • Treating f as a chain of operations on x and trying to 'undo' them step-by-step.
  • Cross-multiplying but only moving one x-term, forgetting to gather both before factoring.
  • Leaving the final answer in terms of y instead of swapping to x.
  • Omitting the domain restriction on f⁻¹ that excludes the zero of the new denominator.

Pitfalls

  • Sign errors when subtracting across: writing yx + 2x instead of yx - 2x.
  • Distribution errors when expanding y(x-1) — e.g. writing yx - 1 instead of yx - y.
  • Forgetting to factor x before dividing, leading to an expression that isn't isolated.
  • Failing to state x ≠ (value making cy - a = 0) as the excluded value of f⁻¹'s domain.

A-level Scope Notes

Standard Edexcel IAL Pure Mathematics technique for inverting linear fractional (Möbius-type) functions; appears routinely in P2/P3 inverse function questions and in domain/range analysis.
