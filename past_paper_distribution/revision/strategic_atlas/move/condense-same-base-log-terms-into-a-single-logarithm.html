Algebra & Functions

Condense same-base log terms into a single logarithm

Family: Log Condensation Frequency: 1 questions

Trigger — When to use

An equation contains two or more logarithms with the same base on one side, often together with a constant or another isolated log. Cues include expressions like $\log_b A \pm \log_b B = c$ or $\log_b A = \log_b B + c$. The move also fires when coefficients sit in front of logs (e.g. $2\log_3 x$) that can be absorbed via the power rule before combining.

Motivation — Why this works

The log laws $\log_b A + \log_b B = \log_b(AB)$ and $\log_b A - \log_b B = \log_b(A/B)$ collapse multiple log terms into one, producing the canonical form $\log_b(\text{expression}) = \text{constant}$. From this form a single exponential conversion removes the logarithm and leaves a polynomial or rational equation in $x$. Without condensing first, one would need to exponentiate a sum/difference of logs, which is algebraically awkward and error-prone.

Thought Process — The step-by-step reasoning

  1. I notice multiple $\log_3$ terms plus a constant — this is a log equation, not an identity to verify.
  2. I rule out exponentiating term-by-term (you cannot apply $3^{(\cdot)}$ across a sum of logs cleanly) and rule out equating arguments (no matching $\log = \log$ form yet).
  3. I commit to condensing because the same base appears on every log term, so the product/quotient laws apply directly.
  4. First concrete step: move all log terms to one side and the constant to the other, so the laws have a clean target.
  5. Tactically, I check for coefficients (apply power rule first) and for sign (subtraction → quotient, addition → product) before combining.

Mechanic — Concrete operations

  1. Collect all log terms on one side and constants on the other.
  2. If any log has a coefficient $k$, rewrite $k\log_b A = \log_b A^k$ using the power rule.
  3. Apply $\log_b A + \log_b B = \log_b(AB)$ or $\log_b A - \log_b B = \log_b(A/B)$ to obtain a single log.
  4. Convert $\log_b(\text{expr}) = c$ to exponential form $\text{expr} = b^c$.
  5. Solve the resulting algebraic equation and check each candidate against the domain (arguments $>0$).

Worked Micro-Example

Solve $\log_3(x+5) - \log_3(2x-1) = 4$. Condense: $\log_3\!\left(\dfrac{x+5}{2x-1}\right) = 4$. Exponential form: $\dfrac{x+5}{2x-1} = 3^4 = 81$. Then $x+5 = 81(2x-1)$, giving $161x = 86$, so $x = \tfrac{86}{161}$. Check: $x+5>0$ and $2x-1>0$ must both hold (here the second fails, so reject and report no solution).

Variants — Same move, different surface

Sum of logs equal to a constant, e.g. $\log_2 x + \log_2(x-3) = 2$
Use the product rule: $\log_2(x(x-3)) = 2$, then $x(x-3)=4$.
Coefficient in front of a log, e.g. $2\log_5 x - \log_5(x+6) = 0$
Apply power rule first: $\log_5(x^2/(x+6)) = 0$, so $x^2/(x+6) = 1$.
Log equals log, e.g. $\log_b A = \log_b B$ after condensing both sides
Skip exponential conversion; equate arguments $A = B$ directly.
Natural logs in calculus contexts, e.g. $\ln(2x) - \ln(x+1) = 1$
Same condensation; convert via $e$ instead of $b$: argument $= e^1$.

Common Misreadings

  • Treating $\log_b A - \log_b B$ as $\log_b(A-B)$ instead of $\log_b(A/B)$.
  • Trying to exponentiate each log term separately (e.g. $b^{\log_b A} - b^{\log_b B}$) rather than condensing first.
  • Ignoring a coefficient and combining $2\log_b A + \log_b B$ as $\log_b(2AB)$ instead of $\log_b(A^2 B)$.
  • Mistaking different-base logs as combinable (the rule requires the same base).

Pitfalls

  • Forgetting to check that all original log arguments are positive at the candidate solution; extraneous roots are common.
  • Sign errors when moving log terms across the equals sign (subtraction vs addition mismatch).
  • Misapplying the power rule when the coefficient multiplies only part of an expression (e.g. $2\log_b(x+1) \ne \log_b(2x+2)$; it equals $\log_b((x+1)^2)$).

A-level Scope Notes

Fully within Edexcel IAL P2/P3 logarithm work: log laws, change to exponential form, and domain checks for log arguments. No use of complex logs or change-of-base beyond the standard A-level identities.
