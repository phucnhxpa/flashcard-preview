Algebra & Functions

Exponentiate both sides to invert a logarithm and free the unknown

Family: Inverse Function Application Frequency: 3 questions

Trigger — When to use

Fires whenever the unknown x is trapped inside a single logarithm and the equation/inequality has been rearranged to the form log_a(f(x)) = c, ln(f(x)) = c, or log_a(f(x)) ⋚ c, or when an expression y = log_a(f(x)) must be inverted to give x in terms of y. Visual cues: a lone log or ln term equal to (or compared with) a constant or another variable, with no other log terms left to combine. Equivalent phrasings: 'undo the log', 'remove the logarithm', 'switch to exponential form'. Also fires when an inequality of the form ln(f(x)) < c needs solving, since exp is monotonic.

Motivation — Why this works

Exponentiation with base a is the inverse of log_a, so applying a^(·) to both sides exploits the identity a^(log_a A) = A, stripping the logarithm and leaving a purely algebraic equation/inequality in x. This is essentially the only efficient exit route once the equation has been reduced to a single log equal to a constant. Because the exponential function is strictly increasing on ℝ, the operation also preserves the direction of any inequality — no flipping required, in contrast to multiplying by a negative. The same inverse-function principle works in reverse: ln undoes e^(·).

Thought Process — The step-by-step reasoning

  1. Notice that x is buried inside log_a(f(x)) and ordinary algebra cannot reach it while the log is still wrapped around the expression.
  2. Rule out log laws (combining/splitting logs) — there is only one log term left, so there is nothing to combine; rule out change of base, since converting to ln still leaves x inside a logarithm.
  3. Commit to applying the inverse function: raise the base of the log to the power of each side, using a^(log_a A) = A.
  4. Bridge to operation: rewrite c as log_a(a^c) if it helps conceptually, or simply apply a^(·) to both sides and simplify the LHS via the inverse identity.
  5. Tactical check: if dealing with an inequality, confirm exp is increasing so the inequality direction is preserved; if inverting y = log_a(f(x)), keep the constant a^y exact rather than approximating; verify the original domain f(x) > 0 to avoid spurious solutions.

Mechanic — Concrete operations

  1. Isolate the log term so the equation reads log_a(f(x)) = c (or ln(f(x)) = c).
  2. Apply a^(·) to both sides: a^(log_a(f(x))) = a^c.
  3. Simplify the LHS using the inverse identity: f(x) = a^c.
  4. Solve the resulting algebraic equation for x by standard rearrangement.
  5. For inequalities, repeat the same operation keeping the inequality sign unchanged (since a^(·) is strictly increasing for a > 1).

Worked Micro-Example

Solve $\ln(x+1) = \tfrac{1}{2}$. Apply $e^{(\cdot)}$ to both sides: $e^{\ln(x+1)} = e^{1/2}$, giving $x+1 = e^{1/2}$. Hence $x = e^{1/2} - 1$ (exact form retained).

Variants — Same move, different surface

Natural log equation ln(f(x)) = c
Exponentiate with base e: f(x) = e^c.
General log equation log_a(f(x)) = c
Raise both sides as powers of a: f(x) = a^c.
Inverting y = log_a(f(x)) to express x in terms of y
Apply a^(·): a^y = f(x), then solve linearly/algebraically for x.
Logarithmic inequality ln(f(x)) < c
Exponentiate; since exp is increasing, keep the inequality direction: f(x) < e^c.
Exponential equation e^(f(x)) = k
Same inverse principle in reverse: take ln of both sides to give f(x) = ln k.

Common Misreadings

  • Trying to 'distribute' the log across a sum, e.g. writing log(2x+1) as log(2x) + log(1) — log does not distribute over addition.
  • Treating ln(f(x)) = c as ready to solve without exponentiating, by incorrectly stripping the ln symbol.
  • Reaching for change-of-base when the log is already alone equal to a constant — this just relabels the problem without progressing.

Pitfalls

  • Prematurely converting e^c or a^c to a decimal approximation and losing the exact form required by the markscheme.
  • Arithmetic slips with negative exponents: e^(-1/4) is NOT -e^(1/4); it equals 1/e^(1/4).
  • When exponentiating an inequality, mistakenly flipping the sign — exp is increasing, so the direction is preserved.
  • Forgetting the domain condition f(x) > 0 needed for the original log to be defined; check candidate solutions against this.

A-level Scope Notes

Fully within Edexcel IAL P2/P3 scope (logarithms and exponentials). Uses the inverse-function identities a^(log_a A) = A and ln/exp inverse pair from the formula booklet's logarithms section. Avoid generalising to complex-valued logs or to bases ≤ 0, which are out of scope.
