Trigger — When to use

The question presents a complex number in Cartesian form a + bi (or an equation z^n = w with w in Cartesian form) and asks for a power, an nth root, a product/quotient, or a geometric locus. Visual cues include phrases like 'find all roots of z^n = ...', 'express in the form r(cos θ + i sin θ)', 'use de Moivre's theorem', or modulus/argument inequalities defining a region on the Argand diagram. Equivalently: any time repeated Cartesian multiplication would be unwieldy, or the geometry (rotation, scaling) is the natural language of the problem. Also fires when |z − z₁| = r or arg(z − z₁) = θ boundaries need explicit Cartesian curves to be drawn and shaded.

Motivation — Why this works

Modulus-argument form z = r(cos θ + i sin θ) = re^{iθ} separates magnitude from direction, so multiplication becomes 'multiply moduli, add arguments' and division becomes 'divide moduli, subtract arguments'. This is the prerequisite for de Moivre's theorem, which gives z^n = r^n(cos nθ + i sin nθ) and the n distinct nth roots r^{1/n}(cos((θ+2kπ)/n) + i sin((θ+2kπ)/n)). For loci, the polar description identifies circles (constant modulus) and half-lines (constant argument) directly, making boundary curves trivial to draw. Without this conversion, powers, roots, and geometric structure are hidden behind algebraic clutter.

Thought Process — Step-by-step reasoning

  1. Notice that the complex number is in a + bi form but the question demands a power, an nth root, a product/quotient, or a region on the Argand diagram — operations whose natural home is polar form.
  2. Rule out brute-force Cartesian expansion: (a + bi)^n by binomial expansion is error-prone for n ≥ 3, and conjugate-rationalisation for quotients is heavier than polar division.
  3. Commit to polar form because de Moivre's theorem (the standard route for powers and roots) and locus interpretations both require r and θ as inputs.
  4. Compute the modulus r = √(a² + b²), then the reference angle α = arctan(|b|/|a|).
  5. Pin down the correct quadrant from the signs of a and b, adjusting α to give the principal argument θ ∈ (−π, π] (Edexcel convention): Q1: θ = α; Q2: θ = π − α; Q3: θ = −(π − α); Q4: θ = −α.
  6. Check whether θ matches a standard exact angle (π/6, π/4, π/3, π/2, 2π/3, 3π/4, 5π/6, π) so the answer can be given exactly.

Mechanic — Concrete operations

  1. Write z = a + bi and identify a, b.
  2. Compute modulus: r = |z| = √(a² + b²).
  3. Compute reference angle: α = arctan(|b|/|a|) (handle a = 0 separately: θ = ±π/2).
  4. Determine quadrant from signs of a, b and assign θ accordingly, with −π < θ ≤ π.
  5. Write z = r(cos θ + i sin θ) or z = re^{iθ}.
  6. (For loci) Convert |z − z₁| = r to (x − a)² + (y − b)² = r², and arg(z − z₁) = θ to a half-line through (a, b) at angle θ; test a point to decide which side to shade.

Worked Micro-Example

Convert z = -1 + i√3 to polar form. Modulus: r = √(1 + 3) = 2. Reference angle: α = arctan(√3/1) = π/3. Since a < 0, b > 0, z is in Q2, so θ = π − π/3 = 2π/3. Therefore z = 2(cos(2π/3) + i sin(2π/3)) = 2e^{i·2π/3}.

Variants — Same move, different surface

Quotient of two Cartesian complex numbers, e.g. (1 + i)/(√3 − i)
Convert numerator and denominator separately to polar, then divide moduli and subtract arguments — avoids conjugate-rationalisation entirely.
Equation z^n = w with w in Cartesian form (e.g. z^4 = −4)
Convert w to polar (w = 4(cos π + i sin π)), then apply de Moivre to get the n roots r^{1/n}(cos((θ + 2kπ)/n) + i sin((θ + 2kπ)/n)) for k = 0, 1, …, n−1.
Locus inequality |z − z₁| ≤ r or α ≤ arg(z − z₁) ≤ β
Convert each boundary to Cartesian: circle (x − a)² + (y − b)² = r² or half-line (y − b)/(x − a) = tan θ with the correct ray direction; test a point not on the boundary to choose the shaded side.
Apollonius locus |z − a| = k|z − b| with k ≠ 1
Square both sides and rearrange to (x − p)² + (y − q)² = R² — same Cartesian-conversion logic produces a circle.
Purely imaginary z = bi (a = 0)
Skip arctan; assign θ = π/2 if b > 0, θ = −π/2 if b < 0.

Common Misreadings

  • Using arctan(b/a) directly without adjusting for quadrant, giving wrong arguments for numbers in Q2, Q3, Q4.
  • Confusing arg(−4) with 0 or −π instead of π for negative real numbers.
  • Shading the exterior of a circle for |z − z₁| ≤ r instead of the interior.
  • Treating arg(z − z₁) = θ as a full line rather than a half-line emanating from z₁.

Pitfalls

  • arctan returns values only in (−π/2, π/2), so manual quadrant correction is essential.
  • Sign errors when b < 0 or a < 0, especially distinguishing −2π/3 from 4π/3.
  • Forgetting Edexcel's principal-argument convention −π < θ ≤ π and giving θ outside this range.
  • Using degrees instead of radians when the mark scheme expects radians.
  • Including the boundary when the inequality is strict (< vs ≤).
  • Choosing a test point that lies on the boundary, which gives no information about the inequality direction.
