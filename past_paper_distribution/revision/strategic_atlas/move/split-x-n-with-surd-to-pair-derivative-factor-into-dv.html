Trigger — When to use

Fires when deriving a reduction formula for I_n = ∫ x^n √(f(x)) dx where f(x) is a polynomial whose derivative f'(x) is a power of x (e.g. f(x) = x³+1 gives f'(x) = 3x²). The surd alone has no elementary antiderivative, so a naive u = x^n, dv = √(f(x)) dx split is blocked. Recognise that peeling off the factor matching f'(x) from x^n into dv makes v computable by a direct substitution t = f(x). Cue phrases: 'establish a reduction formula for ∫ x^n √(x³+1) dx'.

Motivation — Why this works

The substitution t = f(x), dt = f'(x) dx, turns f'(x)√(f(x)) dx into t^{1/2} dt, which integrates to (2/3)t^{3/2}. So pairing f'(x) with the surd in dv yields a clean v = (2/(3·k))(f(x))^{3/2} where k is the constant from f'. Any split that leaves a non-derivative factor next to the surd in dv fails to integrate. The matching split also forces u = x^{n - deg(f')}, producing exactly the index drop needed for the reduction.

Thought Process — Step-by-step reasoning

  1. I see I need IBP for a reduction, and the surd √(x³+1) cannot be integrated on its own.
  2. I rule out u = x^n, dv = √(x³+1) dx (no closed-form v) and u = √(x³+1), dv = x^n dx (gives a harder integral, not a reduction).
  3. I notice d/dx(x³+1) = 3x², so x²√(x³+1) dx is a perfect substitution candidate — this commits me to packaging x² with the surd inside dv.
  4. I rewrite x^n = x^{n-2} · x², take dv = x²√(x³+1) dx and u = x^{n-2}, so du brings down a factor (n-2)x^{n-3} that drops the index by 3.
  5. Tactically, I compute v by t = x³+1, dt = 3x² dx before assembling the IBP formula, keeping the 1/3 factor visible to avoid arithmetic slips.

Mechanic — Concrete operations

  1. Rewrite I_n = ∫ x^{n-2} · x²√(x³+1) dx.
  2. Choose u = x^{n-2} and dv = x²(x³+1)^{1/2} dx.
  3. Compute v: substitute t = x³+1, dt = 3x² dx, so v = (1/3)∫ t^{1/2} dt = (2/9)(x³+1)^{3/2}.
  4. Compute du = (n-2)x^{n-3} dx.
  5. Apply IBP: I_n = (2/9) x^{n-2}(x³+1)^{3/2} − (2(n-2)/9) ∫ x^{n-3}(x³+1)^{3/2} dx.
  6. Express the remainder via I_{n-3} after expanding (x³+1)^{3/2} = (x³+1)·(x³+1)^{1/2} if a closed reduction in I_n is required.

Worked Micro-Example

For $I_n = \int x^n \sqrt{x^3+1}\,dx$ take $u = x^{n-2}$, $dv = x^2(x^3+1)^{1/2}dx$. Then $v = \tfrac{2}{9}(x^3+1)^{3/2}$ and $du = (n-2)x^{n-3}dx$, giving $I_n = \tfrac{2}{9}x^{n-2}(x^3+1)^{3/2} - \tfrac{2(n-2)}{9}\int x^{n-3}(x^3+1)^{3/2}dx$.

Variants — Same move, different surface

I_n = ∫ x^n √(x²+1) dx
Pair x with the surd: u = x^{n-1}, dv = x(x²+1)^{1/2} dx, giving v = (1/3)(x²+1)^{3/2} via t = x²+1; index drops by 2.
I_n = ∫ x^n (x²+a²)^{p} dx with p = 1/2 or other fractional power
Same principle: peel off one x to pair with dv so that t = x²+a² makes v = (x²+a²)^{p+1}/(2(p+1)).
I_n = ∫ x^n e^{x²} dx (related IBP-split idea, non-surd)
Pair x with e^{x²} in dv since d(x²) = 2x dx; take u = x^{n-1}, dv = x e^{x²} dx so v = (1/2)e^{x²}.
I_n = ∫ x^n √(1−x²) dx on [0,1]
Take u = x^{n-1}, dv = x(1−x²)^{1/2} dx; then v = −(1/3)(1−x²)^{3/2} via t = 1−x².

Common Misreadings

  • Choosing u = √(x³+1), dv = x^n dx because the surd 'looks like the messy bit' — this differentiates the surd but produces a harder integral, not a reduction.
  • Choosing u = x^n, dv = √(x³+1) dx because dv is the obvious leftover — but √(x³+1) has no elementary antiderivative.
  • Splitting as u = x^{n-1}, dv = x√(x³+1) dx by analogy with the √(x²+1) case, ignoring that here the derivative-factor is x², not x.

Pitfalls

  • Dropping the factor of 1/3 from dt = 3x² dx when computing v.
  • Mismatching the index drop: the split must leave x^{n-3} (not x^{n-2} or x^{n-1}) in the remainder to match the target I_{n-3}.
  • Forgetting to expand (x³+1)^{3/2} = (x³+1)(x³+1)^{1/2} if the question demands a reduction expressed purely in terms of I_k with the same surd power.

A-level Scope Notes

Standard FP-level reduction-formula technique in Edexcel IAL Further Pure: integration by parts with substitution to compute v, all polynomial/surd manipulation within syllabus.
