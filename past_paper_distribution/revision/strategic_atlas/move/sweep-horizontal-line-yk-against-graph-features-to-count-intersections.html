Algebra & Functions

Sweep horizontal line y=k against graph features to count intersections

Family: Graphical Reasoning Frequency: 1 questions

Trigger — When to use

The question gives a sketched curve (or one you have just sketched) and asks for the value(s) of $k$ for which $y = k$ meets the curve a specified number of times — most often exactly once. Phrasings include 'state the values of $k$ such that the equation $f(x) = k$ has exactly one solution' or 'find $k$ so that the line $y = k$ is a tangent to the curve'. The presence of a labelled turning point and/or a horizontal asymptote on the diagram is a strong cue.

Motivation — Why this works

The number of solutions of $f(x) = k$ equals the number of intersections of $y = f(x)$ with the horizontal line $y = k$. As $k$ varies continuously, the intersection count only changes at critical $y$-values: the $y$-coordinates of turning points (where a tangent line gives a double contact) and the $y$-values of horizontal asymptotes (where a branch transitions). Cataloguing these critical heights partitions the $k$-axis into bands of constant intersection count, so the boundary cases can be read off directly without solving any equation.

Thought Process — The step-by-step reasoning

  1. I notice the question is about how many times a horizontal line meets the curve, so I think of $y = k$ as a moving ruler I slide up and down the diagram.
  2. I rule out algebraic methods (solving $f(x) = k$) because the curve is given graphically and the critical behaviour is visible from the sketch.
  3. I commit to listing every horizontal feature: each turning point's $y$-coordinate and each horizontal asymptote, since these are exactly where the intersection count can change.
  4. I sweep $k$ from $-\infty$ upward, recording the intersection count in each band, and flag the boundary $k$-values where the count equals the target (here, one).
  5. Tactically, I check whether the curve actually crosses any asymptote at a finite point — many exam graphs do — because that crossing itself contributes a 'one-intersection' value of $k$.

Mechanic — Concrete operations

  1. Read off all turning point $y$-coordinates from the labelled diagram.
  2. Read off all horizontal asymptotes $y = a$ and decide from the sketch whether the curve crosses $y = a$ at a finite point.
  3. Order these critical $y$-values and sweep $y = k$ through each band, counting intersections in each band.
  4. At each critical $y$-value, decide the intersection count exactly (tangency at a max/min gives one local contact; an asymptote crossing gives one finite intersection).
  5. Select the $k$-values whose intersection count matches the requirement (here, exactly one).

Worked Micro-Example

Curve has maximum at $(4,9)$ and horizontal asymptote $y = 4$, with the curve crossing $y = 4$ once before approaching it from above. Sweep: for $k > 9$, $0$ intersections; at $k = 9$, $1$ (tangent at max); for $4 < k < 9$, $2$; at $k = 4$, $1$ (single finite crossing); for $k < 4$, $0$. Hence exactly one intersection when $k = 4$ or $k = 9$.

Variants — Same move, different surface

Curve with both a local maximum and a local minimum (e.g. cubic)
Both turning-point $y$-values are critical; intersection counts go $1 \to 2 \to 3 \to 2 \to 1$ as $k$ sweeps, so 'exactly one' typically gives $k$ above the max or below the min, plus the two tangent values.
Modulus graph $y = |f(x)|$ with a cusp on the $x$-axis
Add $k = 0$ as a critical value because the cusp/contact with the $x$-axis changes the count; reflected portions create extra intersections for small positive $k$.
Reciprocal-type curve with a vertical asymptote
The sweep is unchanged for horizontal lines, but each branch must be counted separately; horizontal asymptote $y = a$ may give $0$ extra intersections if the curve never crosses it.
Question asks for 'no solutions' or 'exactly two solutions' instead of one
Same critical $y$-values; just pick the bands or boundaries matching the new count.

Common Misreadings

  • Believing the curve never meets its horizontal asymptote, so omitting $k = a$ as a one-intersection value when the graph clearly shows a finite crossing.
  • Treating 'tangent to the curve' as requiring calculus, instead of just reading the turning-point height off the diagram.
  • Confusing the $x$-coordinate of a turning point with the critical $k$-value (which is the $y$-coordinate).

Pitfalls

  • Giving only $k = 9$ (the maximum) and forgetting the asymptote-crossing case.
  • Using strict vs non-strict inequalities incorrectly when stating ranges of $k$ for a given intersection count.
  • Miscounting intersections in a band by ignoring a hidden branch of the curve off the visible part of the sketch.

A-level Scope Notes

Standard Edexcel IAL P2/P3 graph-transformation and curve-sketching territory; the analysis is purely visual and requires no calculus beyond identifying labelled turning points and asymptotes.
