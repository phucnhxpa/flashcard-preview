Impose non-negativity of RHS when solving modulus equations

Family: Domain Validity Check Frequency: 1 questions

Trigger — When to use

Fires whenever an equation has the form |E(x)| = F(x) where F(x) is not a constant or is not manifestly non-negative. Typical cues: a linear or rational expression on the right of a modulus equation, or a parameter-dependent equation like |kx − 9| = 5 − 2x where validity must be tested. Also fires when squaring both sides produces algebraic candidates that must be filtered. Any solution must satisfy F(x) ≥ 0, otherwise it is extraneous.

Motivation — Why this works

Since |E| ≥ 0 for all real E, the equation |E| = F can only hold where F ≥ 0. Squaring or splitting cases removes the modulus but may introduce phantom roots that satisfy |E|² = F² yet fail |E| = F because F < 0 there. Imposing F ≥ 0 on each candidate filters these out. In parameter problems this becomes an inequality in the parameter, restricting the admissible parameter range.

Thought Process — The step-by-step reasoning

  1. Notice the modulus equation has a non-constant RHS, so RHS sign is not automatic.
  2. Rule out blindly accepting algebraic candidates from squaring or case-splitting; recall |E| ≥ 0 forces F ≥ 0.
  3. Commit to checking F(x) ≥ 0 at each candidate, since this is the only constraint that distinguishes genuine roots from extraneous ones.
  4. Substitute each candidate x (possibly in terms of a parameter) into F(x) ≥ 0 to obtain the validity condition.
  5. Simplify the resulting inequality, performing careful sign analysis if it is rational, and intersect with any pre-existing parameter constraints.

Mechanic — Concrete operations

  1. Identify F(x), the RHS of |E(x)| = F(x), and write the constraint F(x) ≥ 0.
  2. Solve |E(x)| = F(x) by case-splitting (E = F and E = −F) or by squaring, obtaining candidate solutions.
  3. Substitute each candidate into F(x) ≥ 0; if candidates depend on a parameter, this yields an inequality in the parameter.
  4. For rational inequalities like (ak + b)/(ck + d) ≥ 0, do sign analysis using critical points and any given parameter restrictions (e.g. k > 0).
  5. Intersect the validity conditions from all retained cases to obtain the final admissible set.

Worked Micro-Example

Solve $|2x-1| = x-3$. Squaring gives $(2x-1)^2 = (x-3)^2 \Rightarrow 3x^2 + 2x - 8 = 0$, so $x = 4/3$ or $x = -2$. Validity needs $x - 3 \geq 0$, i.e. $x \geq 3$. Both candidates fail, so there is no solution.

Variants — Same move, different surface

|f(x)| = c with c a numerical constant
Just check c ≥ 0 once; if c < 0 there are no solutions, otherwise both branches are kept.
|f(x)| = |g(x)|
No RHS sign check needed; square both sides directly since both sides are non-negative.
Parameter problem: find values of parameter for which |E(x;k)| = F(x;k) has solutions
Substitute candidate x(k) into F ≥ 0 to derive an inequality in k; combine with any given range of k.
Modulus inequality |E(x)| ≤ F(x)
Still requires F(x) ≥ 0 (otherwise no solutions), then equivalent to −F ≤ E ≤ F.
Modulus inequality |E(x)| ≥ F(x)
If F(x) < 0 the inequality is automatic; only enforce E ≥ F or E ≤ −F where F ≥ 0.

Common Misreadings

  • Treating both algebraic candidates from squaring as automatically valid without checking RHS sign.
  • Believing the constraint is on E(x) rather than on F(x), the RHS.
  • Confusing this with the |f| = |g| case, where no sign check is needed.
  • In parameter problems, ignoring the given parameter range (e.g. k > 0) when doing sign analysis on the rational inequality.

Pitfalls

  • Sign errors in rational inequalities like (5k − 18)/(k − 2) ≥ 0; failing to consider the sign of the denominator.
  • Forgetting to intersect validity conditions across both cases when both must hold simultaneously.
  • Accepting strict vs non-strict inequality incorrectly at boundary points (≥ 0 includes equality, giving F = 0 and hence E = 0).

A-level Scope Notes

Standard Edexcel IAL Pure content: modulus equations and inequalities are covered in P2/P3, including parameter-dependent versions where validity conditions yield inequalities in the parameter.
