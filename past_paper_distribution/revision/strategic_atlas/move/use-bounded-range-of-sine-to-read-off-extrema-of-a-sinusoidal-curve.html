Trigonometry

Use bounded range of sine to read off extrema of a sinusoidal curve

Family: Range Exploitation Frequency: 1 questions

Trigger — When to use

The question asks for the coordinates of a maximum or minimum point on a curve of the form y = a·sin(x) + k (or a·cos(x) + k), often within a stated interval such as 0° ≤ x ≤ 360°. Cues include 'state the coordinates of P', 'write down the maximum value', or a sketch with marked turning points. It also fires when a follow-up part shifts the curve vertically and re-asks for an extremum. No calculus is hinted at, and the function is a single sine/cosine term, not a sum of two.

Motivation — Why this works

Because −1 ≤ sin(x) ≤ 1 for all real x, the expression a·sin(x) + k (with a > 0) attains its maximum a + k exactly when sin(x) = 1 and its minimum −a + k exactly when sin(x) = −1. The x-coordinates at which these occur are standard angles (90° and 270° in the principal interval), so the extrema can be written down directly. This converts a 'turning point' question into substitution plus recall of the unit circle, avoiding differentiation entirely.

Thought Process — The step-by-step reasoning

  1. Notice the curve is a single sine (or cosine) term plus a constant — a pure sinusoid, not a sum requiring R-form.
  2. Rule out differentiation: setting dy/dx = 0 just recovers cos(x) = 0, i.e. x = 90°, 270° — information already known from the graph shape.
  3. Commit to using the bounded range of sin(x): the extrema of a·sin(x) + k are forced by sin(x) = ±1.
  4. Bridge: read off the amplitude a and vertical shift k, then compute y_max = a + k and y_min = −a + k.
  5. Tactically pair each y-value with the correct x: sin(x) = 1 at x = 90°, sin(x) = −1 at x = 270° (within 0° ≤ x ≤ 360°), and check sign of a in case the curve is reflected.

Mechanic — Concrete operations

  1. Identify the amplitude a and vertical shift k from y = a·sin(x) + k.
  2. Recall max(sin x) = 1 at x = 90° + 360°n and min(sin x) = −1 at x = 270° + 360°n.
  3. For the maximum: substitute sin(x) = 1 to get y = a + k, paired with x = 90° (in the given interval).
  4. For the minimum: substitute sin(x) = −1 to get y = −a + k, paired with x = 270°.
  5. State coordinates as (x, y) pairs, ensuring x lies in the stated domain.

Worked Micro-Example

Find the max and min of $y = 3\sin x + 7$ for $0° \le x \le 360°$. Since $-1 \le \sin x \le 1$, we have $4 \le y \le 10$. Max: $\sin x = 1$ at $x = 90°$, giving $(90°, 10)$. Min: $\sin x = -1$ at $x = 270°$, giving $(270°, 4)$.

Variants — Same move, different surface

y = a·cos(x) + k
Max occurs at x = 0° (and 360°) where cos(x) = 1; min at x = 180° where cos(x) = −1.
y = a·sin(bx + c) + k with b > 0
Range is still [k − a, k + a], but solve bx + c = 90° for the max-x and bx + c = 270° for the min-x, then list all solutions in the given interval.
Negative amplitude, e.g. y = −2·sin(x) + 1
The roles flip: max y = 1 + 2 = 3 occurs when sin(x) = −1 (x = 270°); min y = −1 occurs when sin(x) = 1 (x = 90°).
R-form expression R·sin(x + α) + k from a·sin x + b·cos x + k
First convert to single-sinusoid form, then apply the same bounded-range argument with amplitude R.

Common Misreadings

  • Confusing amplitude with range: stating max y = a and min y = 0 instead of min y = −a + k.
  • Forgetting to add the vertical shift k after substituting sin(x) = ±1.
  • Treating a negative coefficient a as if it were positive, so swapping which extremum sits at x = 90° vs 270°.

Pitfalls

  • Placing the minimum of sin(x) at x = 180° instead of x = 270°.
  • Giving only y-values when the question asks for full coordinates.
  • Missing extra extrema when the interval extends beyond 360° or when b ≠ 1 inside sin(bx + c).

A-level Scope Notes

Fully within Edexcel IAL Pure Maths (P2) trigonometric graphs and transformations; no calculus required, and the unit-circle facts sin 90° = 1, sin 270° = −1 are standard.
