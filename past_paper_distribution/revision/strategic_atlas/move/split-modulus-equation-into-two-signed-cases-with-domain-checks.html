Split modulus equation into two signed cases with domain checks

Family: Modulus Case Analysis Frequency: 1 questions

Trigger — When to use

Fires when an equation contains an isolated modulus expression |f(x)| equal to a non-modulus expression g(x), and the equation cannot be solved by inspection. Typical cues: |ax+b| = (linear or quadratic in x), or 'solve algebraically' instructions on a curve-intersection problem involving a V-shaped graph. Also fires when the modulus has already been isolated on one side after rearrangement. Squaring is available but rejected when it would raise the degree or introduce extraneous roots that are awkward to vet.

Motivation — Why this works

By definition, |f(x)| = f(x) when f(x) \ge 0 and |f(x)| = -f(x) when f(x) < 0. Replacing the modulus by each signed branch converts one modulus equation into two ordinary equations, each valid only on the half-line where its sign assumption holds. This is exhaustive (every real x falls into one branch) and, with the domain check, sound (no extraneous roots). It is preferred over squaring whenever squaring would inflate the polynomial degree.

Thought Process — The step-by-step reasoning

  1. Notice the equation has an isolated |f(x)| equal to some other expression — the standard modulus template.
  2. Rule out squaring because here f(x) is linear and the RHS is quadratic, so squaring would give a quartic with potential extraneous roots.
  3. Commit to the two-case definition since it is guaranteed complete and keeps each sub-equation at most quadratic.
  4. Identify the critical value where f(x) changes sign (here 3x - 22 = 0 \Rightarrow x = 22/3) to label the two domain regions.
  5. Inside each case, simplify, solve the resulting polynomial, then filter candidates against that case's domain inequality before reporting.

Mechanic — Concrete operations

  1. Find the critical value x_0 where the argument changes sign: solve f(x) = 0.
  2. Case 1 (f(x) \ge 0, i.e. x \ge x_0): replace |f(x)| by f(x) and solve the resulting equation.
  3. Case 2 (f(x) < 0, i.e. x < x_0): replace |f(x)| by -f(x) and solve the resulting equation.
  4. For each candidate root, check the domain inequality of its case; discard those that fail.
  5. If the question asks for points of intersection, substitute each retained x into the original non-modulus side to get y.

Worked Micro-Example

Solve |3x - 22| = 14 - \tfrac{1}{9}x^2. Critical value: x = 22/3.\n Case 1 (x \ge 22/3): 3x - 22 = 14 - \tfrac{1}{9}x^2 \Rightarrow x^2 + 27x - 324 = 0 \Rightarrow (x+36)(x-9) = 0; keep x = 9, reject x = -36.\n Case 2 (x < 22/3): 22 - 3x = 14 - \tfrac{1}{9}x^2 \Rightarrow x^2 - 27x + 72 = 0 \Rightarrow (x-3)(x-24) = 0; keep x = 3, reject x = 24.\n Solutions: x = 3, 9.

Variants — Same move, different surface

|f(x)| = |g(x)| (modulus on both sides)
Case-split into f(x) = g(x) and f(x) = -g(x); no domain restriction is needed because both sides are non-negative, but check both equations exhaust all solutions.
|f(x)| < g(x) or |f(x)| \ge g(x) (modulus inequality)
Use -g(x) < f(x) < g(x) (for <) or split into f(x) \ge g(x) or f(x) \le -g(x) (for \ge); intersect with any implicit domain such as g(x) > 0.
Equation containing two separate moduli, e.g. |x-1| + |x+2| = 5
Use the critical points to split the real line into three (or more) regions and rewrite each modulus with the correct sign in each region; solve and check membership.
Curve intersection y = |f(x)| with y = g(x) asking for coordinates
Apply the case-split to find x, then substitute into g(x) (the non-modulus side) for the y-coordinates of the intersection points.

Common Misreadings

  • Setting up cases based on the sign of the RHS rather than the sign of the modulus argument.
  • Treating |f(x)| = g(x) as automatically equivalent to f(x) = \pm g(x) without recording the domain conditions, then forgetting to filter.
  • Reading off intersections from a sketch when the question explicitly says 'solve algebraically'.
  • Confusing |3x - 22| with |3x| - 22 or with 3|x - 22| when expanding.

Pitfalls

  • Forgetting to test each candidate root against its case's domain inequality, leading to spurious solutions.
  • Sign slip in Case 2: writing 3x - 22 instead of -(3x - 22) = 22 - 3x.
  • Arithmetic errors when clearing the fraction by multiplying through by 9 — failing to multiply every term.
  • Reporting only x-values when the question asks for coordinates of intersection points.

A-level Scope Notes

Standard Edexcel IAL Pure (P3) modulus-equation technique; case-splitting with domain checks is the expected algebraic method, and curve-intersection follow-ups appear routinely in the same papers.
