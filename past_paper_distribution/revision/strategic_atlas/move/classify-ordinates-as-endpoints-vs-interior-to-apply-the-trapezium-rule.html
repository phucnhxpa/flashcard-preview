Integration

Classify ordinates as endpoints vs interior to apply the trapezium rule

Family: Trapezium Rule Application Frequency: 2 questions

Trigger — When to use

The question explicitly instructs 'use the trapezium rule' (often with the phrase 'with all the y-values from the table') to estimate a definite integral. A table of equally-spaced x-values with corresponding y-values (or a function whose antiderivative is awkward, e.g. mixing exponentials, surds, and polynomials) is supplied. Alternative phrasings include 'estimate the area', 'find an approximate value of ∫...dx', or 'using n strips of width h'. The structural cue is: tabulated ordinates + an instruction to approximate, rather than integrate exactly.

Motivation — Why this works

The trapezium rule ∫_a^b y dx ≈ (h/2)[y₀ + yₙ + 2(y₁ + y₂ + … + y_{n−1})] approximates the area under a curve as a sum of trapezium areas, weighting the two boundary ordinates with coefficient 1 and every interior ordinate with coefficient 2. Correctly partitioning the table entries into 'endpoints' versus 'interior' is the single decision that determines whether each y-value is multiplied by 1 or by 2 — misclassifying one ordinate halves or doubles its contribution and corrupts the whole estimate. This is the most common source of lost marks on this question type.

Thought Process — The step-by-step reasoning

  1. Notice the keyword 'trapezium rule' (or a table with equally-spaced x-values and an instruction to estimate) — this immediately rules out analytic integration even when it might be possible.
  2. Rule out Simpson's rule (different weighting 1-4-2-…-4-1) and rule out integrating term-by-term: the question explicitly wants a numerical estimate from the tabulated ordinates.
  3. Commit to the formula (h/2)[y₀ + yₙ + 2(y₁ + … + y_{n−1})]. Count the x-values in the table: k values give n = k − 1 strips (a frequent miscount).
  4. Determine h either from consecutive x-values or from h = (b − a)/n; verify by checking the spacing is constant.
  5. Tag the first and last y-values as endpoints (coefficient 1) and every other y-value as interior (coefficient 2). Write them down explicitly before substituting to avoid mis-assignment.
  6. Carry full table precision (≥3 d.p.) through the arithmetic; round only at the very end to the precision the question asks for.

Mechanic — Concrete operations

  1. Identify n = (number of ordinates) − 1 and compute h = (b − a)/n.
  2. Write the formula: Area ≈ (h/2)[y₀ + yₙ + 2(y₁ + y₂ + … + y_{n−1})].
  3. Sum the endpoints: S_end = y₀ + yₙ.
  4. Sum the interior ordinates: S_int = y₁ + y₂ + … + y_{n−1}, then double: 2·S_int.
  5. Add: bracket = S_end + 2·S_int.
  6. Multiply by h/2 to obtain the estimate, then round to the requested accuracy.

Worked Micro-Example

Table: x = 0, 0.25, 0.5, 0.75, 1; y = 1.000, 1.284, 1.649, 2.117, 2.718 (so n=4, h=0.25). Then ∫_0^1 y dx ≈ (0.25/2)[1.000 + 2.718 + 2(1.284 + 1.649 + 2.117)] = 0.125[3.718 + 2(5.050)] = 0.125 × 13.818 = 1.727 (3 d.p.).

Variants — Same move, different surface

Only 3 ordinates given (2 strips)
Formula collapses to (h/2)[y₀ + 2y₁ + y₂] — same endpoint/interior logic with a single interior term.
Question states 'use n strips' but gives no table
First derive h = (b − a)/n, generate x-values x_i = a + ih, evaluate y_i = f(x_i), then proceed as normal.
Simpson's rule (Further Maths)
Same endpoint vs interior classification, but interior weights alternate 4, 2, 4, 2, … 4 and the multiplier is h/3.
Table values were computed in an earlier part
Use the values to full given precision; do not re-round, and check earlier values are correct since errors propagate.

Common Misreadings

  • Counting k ordinates as k strips (so taking n = k instead of n = k − 1), giving the wrong h.
  • Treating an endpoint as interior (doubling y₀ or yₙ), or treating an interior ordinate as an endpoint (failing to double it).
  • Misreading 'use all y-values from the table' as permission to ignore some ordinates.

Pitfalls

  • Forgetting the factor of 2 on the interior sum, or doubling every ordinate including the endpoints.
  • Using h instead of h/2 as the outer multiplier (answer comes out exactly twice too large).
  • Halving h twice — once when forming h/2 and again somewhere in the arithmetic.
  • Arithmetic slip when summing the interior ordinates; this sum dominates the answer, so re-check it.
  • Premature rounding of table values (e.g. to 2 d.p.) before substitution, losing accuracy in the final figure.

A-level Scope Notes

Within Edexcel IAL P2/P3 scope; the trapezium rule formula is provided in the formula booklet (Numerical Methods). Stay with uniform spacing h — non-uniform spacing and error-bound analysis (e.g. via the second derivative) are out of scope for IAL Maths.
