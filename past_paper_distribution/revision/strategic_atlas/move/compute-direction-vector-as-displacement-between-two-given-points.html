Vectors

Compute direction vector as displacement between two given points

Family: Direction From Points Frequency: 1 questions

Trigger — When to use

The question specifies a line passing through two known points (e.g. 'line l₁ passes through A and B' or 'find the vector equation of the line through P and Q') and asks for its vector/parametric equation. It also fires whenever you need *any* direction along such a line — for angle calculations, intersections, perpendicularity checks, or distance-from-point problems. Cues include being handed coordinates of two points but no explicit direction. The move applies in both 2D and 3D.

Motivation — Why this works

A straight line is fully determined by one point on it and any non-zero vector parallel to it. The displacement \(\vec{AB} = \mathbf{b} - \mathbf{a}\) is by construction parallel to the line through A and B, so it is a legitimate direction vector. Combined with either endpoint as position vector, this yields the vector equation \(\mathbf{r} = \mathbf{a} + \lambda(\mathbf{b} - \mathbf{a})\). Any non-zero scalar multiple of this displacement is equally valid, so no normalisation is required unless explicitly asked.

Thought Process — The step-by-step reasoning

  1. I notice the line is specified by two points, so I have position information but no direction yet.
  2. I rule out needing extra geometric input (no plane normal, no parallel line, no perpendicularity condition is given).
  3. I commit to the identity 'direction = second point − first point' because it is the standard and shortest route.
  4. I bridge to computation by subtracting coordinate-wise to get \(\mathbf{b}-\mathbf{a}\).
  5. I decide which point to use as the base (either works) and whether to simplify the direction by dividing out a common factor for neatness.

Mechanic — Concrete operations

  1. Identify position vectors \(\mathbf{a}\) and \(\mathbf{b}\) of the two given points.
  2. Compute the direction vector \(\mathbf{d} = \mathbf{b} - \mathbf{a}\) component-wise.
  3. (Optional) Divide \(\mathbf{d}\) by any common integer factor to get a tidier parallel vector.
  4. Choose a base point (commonly A) and write \(\mathbf{r} = \mathbf{a} + \lambda \mathbf{d}\).
  5. State the parameter \(\lambda \in \mathbb{R}\) if the mark scheme expects it.

Worked Micro-Example

Given \(A(6,6,2)\) and \(B(6,0,7)\): \(\vec{AB} = \mathbf{b}-\mathbf{a} = (0,-6,5)\). Taking A as base point gives \(\mathbf{r} = \begin{pmatrix}6\\6\\2\end{pmatrix} + \lambda\begin{pmatrix}0\\-6\\5\end{pmatrix},\ \lambda \in \mathbb{R}\).

Variants — Same move, different surface

Line given by a point and an explicit direction vector (e.g. l₂ stated as 'through P with direction \(\mathbf{d}\)')
No subtraction needed — read off \(\mathbf{d}\) directly and write \(\mathbf{r} = \mathbf{p} + \mu \mathbf{d}\).
Midpoint or section-point of AB given instead of an endpoint
Direction is still \(\mathbf{b}-\mathbf{a}\); use the given midpoint/section point as the base position vector.
2D line in \(\mathbb{R}^2\) through two points
Same subtraction in two components; can be converted to Cartesian form \(y - y_1 = m(x - x_1)\) using the gradient implied by \(\mathbf{b}-\mathbf{a}\).
Line of intersection between two planes where two points on it have been found
Use those two points to form \(\mathbf{b}-\mathbf{a}\) as the direction; alternatively use \(\mathbf{n}_1 \times \mathbf{n}_2\).
Need a unit direction vector (e.g. for foot-of-perpendicular shortcuts)
Compute \(\mathbf{b}-\mathbf{a}\) then divide by \(|\mathbf{b}-\mathbf{a}|\).

Common Misreadings

  • Reading \(\mathbf{a}-\mathbf{b}\) instead of \(\mathbf{b}-\mathbf{a}\) — gives the opposite direction but the same line, so still valid.
  • Treating the position vector of one point as the direction vector of the line.
  • Thinking the direction must be a unit vector before writing the equation.

Pitfalls

  • Sign errors during component-wise subtraction, especially with negative coordinates.
  • Forgetting to attach the parameter \(\lambda\) (writing \(\mathbf{r} = \mathbf{a} + (\mathbf{b}-\mathbf{a})\), which is just the point B).
  • Mixing up which point is the base, then using inconsistent coordinates later when substituting \(\lambda\) values.
  • Over-simplifying by cancelling a factor that is not actually common to all components.

A-level Scope Notes
