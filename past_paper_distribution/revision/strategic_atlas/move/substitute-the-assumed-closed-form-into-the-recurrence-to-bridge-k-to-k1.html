Strategic Atlas / Proof / Recurrence Induction Bridge
Proof

Substitute the assumed closed form into the recurrence to bridge k to k+1

Family: Recurrence Induction Bridge Frequency: 2 questions

Trigger — When to use

The question is a proof by induction where a sequence is defined by a recurrence relation (e.g. u_{k+1} = 2u_k − 2k + 1 or u_{k+1} = 3u_k + 1) and you are asked to prove a stated closed-form formula for u_n. You have completed the base case and stated the inductive hypothesis 'assume true for n = k', and now need to deduce the result for n = k+1. Visual cue: the question hands you BOTH a recurrence (linking consecutive terms) AND a target closed form (explicit in n) — the inductive step must connect them. Equivalent phrasings: 'prove by induction that u_n = ...', 'hence show that the formula holds for all positive integers n'.

Motivation — Why this works

The recurrence relation is the ONLY given link between u_k and u_{k+1}, so it is the natural — and essentially unique — bridge for the inductive step. By substituting the assumed closed form for u_k into the right-hand side of u_{k+1} = f(u_k, k), the expression for u_{k+1} becomes purely algebraic in k, which can then be simplified and matched against the target closed form evaluated at n = k+1. This avoids reverse-engineering the closed form from scratch and respects the induction framework. The underlying principle is that induction transfers truth along the recurrence's own arrow.

Thought Process — The step-by-step reasoning

  1. Notice the structure: a recurrence u_{k+1} = f(u_k, k) is given, and a closed form u_n = g(n) must be proved — the inductive step requires a bridge from k to k+1.
  2. Rule out alternatives: trying to manipulate g(k+1) backwards into g(k) is awkward with no clear starting point; deriving the closed form from scratch (e.g. via geometric-series summation) bypasses the required induction structure.
  3. Commit to using the recurrence as the bridge — it is the natural forward direction and the only legitimate link between consecutive terms.
  4. Plan the algebra: write down u_{k+1} from the recurrence, substitute u_k = g(k) from the inductive hypothesis, and aim to simplify to g(k+1).
  5. Before simplifying, write out g(k+1) explicitly as the target — this gives a concrete goal and prevents aimless algebra.
  6. Execute carefully: distribute, collect like terms (especially powers of the same base such as 2^k or 3^k), and combine fractions over a common denominator if needed.

Mechanic — Concrete operations

  1. State the inductive hypothesis: assume u_k = g(k) for some particular k ∈ ℤ⁺.
  2. Write u_{k+1} using the recurrence: u_{k+1} = f(u_k, k).
  3. Substitute the hypothesis: replace u_k by g(k) on the right-hand side.
  4. Simplify algebraically — distribute, combine like powers, get common denominators.
  5. Rearrange to recognise the result as g(k+1), the target closed form at n = k+1.
  6. Conclude: since true for n=1 and (true for k) ⇒ (true for k+1), it is true for all n ∈ ℤ⁺ by induction.

Worked Micro-Example

Recurrence: $u_{k+1} = 3u_k + 1$. Hypothesis: $u_k = \tfrac{3^k - 1}{2}$. Then $u_{k+1} = 3 \cdot \tfrac{3^k-1}{2} + 1 = \tfrac{3^{k+1} - 3}{2} + \tfrac{2}{2} = \tfrac{3^{k+1} - 1}{2}$, which is the closed form with $n = k+1$. ✓

Variants — Same move, different surface

Matrix induction, e.g. prove M^n = (closed form)
Use M^{k+1} = M · M^k as the bridge; substitute the assumed form for M^k and multiply out.
Summation induction, e.g. prove Σ_{r=1}^{n} f(r) = S(n)
Use Σ_{r=1}^{k+1} f(r) = Σ_{r=1}^{k} f(r) + f(k+1) as the bridge; substitute S(k) for the first sum.
Divisibility induction, e.g. prove 7 | (8^n − 1)
Write the (k+1)-expression in terms of the k-expression (e.g. 8^{k+1} − 1 = 8(8^k − 1) + 7), using the hypothesis as the bridge.

Common Misreadings

  • Trying to prove the recurrence itself rather than using it — the recurrence is given as a tool.
  • Deriving the closed form from scratch via geometric series instead of using the required induction framework.
  • Stating the hypothesis as 'true for all k' rather than 'true for some particular k'.

Pitfalls

  • Misreading the recurrence (e.g. confusing 2^n with 2n, or dropping a sign) which derails the entire algebraic step.
  • Forgetting to convert added integers to fractions with the correct common denominator before combining (e.g. '+1' must become '+2/2').
  • Failing to write the target g(k+1) explicitly before simplifying, leading to algebra with no clear destination.
  • Not collecting powers of the same base correctly, e.g. mishandling 2 · 2^{k-1} = 2^k or 3 · 3^k = 3^{k+1}.

A-level Scope Notes

Fully within Edexcel IAL Further Pure scope (proof by induction for sequences/recurrences, summations, matrix powers, divisibility). No formula booklet entry is required for the induction structure itself. Avoid invoking generating functions or characteristic-equation methods for solving the recurrence — these are out of scope and unnecessary.
