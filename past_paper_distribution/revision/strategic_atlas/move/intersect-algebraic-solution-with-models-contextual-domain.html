Algebra & Functions

Intersect algebraic solution with model's contextual domain

Family: Domain Restriction Frequency: 1 questions

Trigger — When to use

Fires whenever an inequality or equation has been solved algebraically inside a modelling question that specifies a restricted domain (e.g. a t-year monitoring period, a percentage between 0 and 100, or a physical region). Cues include phrases like 'over the period', 'for 0 ≤ t ≤ T', 'during the monitoring period', or a stated validity window for the model. It also fires when one of the algebraic critical values lies outside this stated domain. The student must reconcile the pure-algebra answer with the model's domain before stating the final contextual answer.

Motivation — Why this works

A model only describes reality within the domain on which it is defined; values of the variable outside that domain carry no meaning, even if the algebra produces them. Taking the intersection of the algebraic solution set with the contextual domain therefore produces the unique answer that is simultaneously mathematically valid and contextually meaningful. This reflects the A-level modelling principle that 'the algebra serves the context, not the other way round'. Examiners explicitly award marks for recognising and applying this restriction.

Thought Process — The step-by-step reasoning

  1. Notice that the algebraic solution interval extends beyond (or outside) the domain stated in the question's modelling preamble.
  2. Rule out simply quoting the raw algebraic interval, since values outside the model's domain are not admissible answers.
  3. Commit to taking the set intersection of the algebraic solution with the contextual domain, because this is the only set on which both the inequality and the model are valid.
  4. Bridge to operation: write the algebraic interval and the domain interval side by side, then compare endpoints.
  5. Tactically decide whether each boundary becomes strict or inclusive: an algebraic strict inequality stays strict, but a domain endpoint at which the inequality holds is included (≤), as the model is defined there.

Mechanic — Concrete operations

  1. Solve the inequality/equation algebraically and write the solution set in interval form.
  2. Write the contextual domain explicitly, e.g. 0 ≤ t ≤ 15.
  3. Compare endpoints: keep any algebraic endpoint that lies inside the domain; replace any endpoint outside the domain by the corresponding domain endpoint.
  4. Decide endpoint type: strict if it came from a strict algebraic inequality at a non-domain root, inclusive if it is a domain boundary at which the original inequality holds.
  5. State the final answer in the form a < t ≤ b (or appropriate variant) and interpret it in the context of the model.

Worked Micro-Example

Solve $t^2 - 20t + 42 < 0$ for the model valid on $0 \le t \le 15$. Roots: $t = 10 \pm \sqrt{58}$, so algebraically $10-\sqrt{58} < t < 10+\sqrt{58}$, i.e. $2.384 < t < 17.616$. Intersecting with $[0,15]$: the lower bound $2.384$ lies inside, the upper $17.616$ exceeds $15$, so the answer is $10-\sqrt{58} < t \le 15$.

Variants — Same move, different surface

Both algebraic roots lie inside the contextual domain
The intersection equals the algebraic interval; no replacement of endpoints is needed, but the domain check should still be stated.
Physical non-negativity constraint (e.g. time t ≥ 0, mass m ≥ 0)
Discard any negative portion of the algebraic solution; the lower bound becomes 0 (inclusive if the inequality holds at 0).
Percentage/probability model with 0 ≤ p ≤ 100 or 0 ≤ p ≤ 1
Cap algebraic values exceeding the upper limit at the domain ceiling, with inclusive endpoint where the model is defined.
Trigonometric equation on a restricted interval such as 0 ≤ x < 360°
Discard general-solution values outside the stated interval rather than intersecting an inequality interval.
Exponential/log decay model with an explicit time window
Same intersection logic applies; ensure the upper time bound is inclusive if the model is defined there.

Common Misreadings

  • Quoting the full algebraic interval and ignoring the modelling domain entirely, losing the context/modelling mark.
  • Capping the upper bound with a strict inequality (< 15) instead of inclusive (≤ 15) when the model is valid at the endpoint.
  • Treating the domain restriction as an additional algebraic constraint and re-solving, rather than simply intersecting sets.
  • Discarding the algebraic answer entirely when one root lies outside the domain, instead of keeping the in-domain portion.

Pitfalls

  • Forgetting that the model is valid at t = 15 (inclusive), so the upper bound should be ≤ 15 not < 15.
  • Mishandling endpoint inclusivity when the original inequality is strict but the boundary now comes from the domain.
  • Failing to verify that the lower algebraic root is actually inside the domain before retaining it.
  • Stating the final answer purely in interval notation without interpreting it in the modelling context, when the question demands an interpretation.

A-level Scope Notes

Standard Edexcel IAL Pure/Statistics modelling skill: solving an inequality and then restricting to a stated domain such as a time window or physical range. No measure-theoretic ideas needed — only basic interval intersection.
