Trigonometry

Flip the extremum when a negative coefficient scales a bounded trig function

Family: Extremum Of Scaled Trig Frequency: 1 questions

Trigger — When to use

Fires when asked for the maximum or minimum of y = A·cos(...) + c or y = A·sin(...) + c with A < 0, or any expression of the form y = c − k·trig(...) with k > 0. Cue phrases: 'find the maximum/minimum point', 'state the greatest value', 'coordinates of the highest/lowest point' on a transformed sine/cosine curve. The negative multiplier means the usual instinct (set cos = 1 for max) is wrong. Also fires for tan-free expressions where the trig factor is bounded in [−1, 1].

Motivation — Why this works

Since cos θ and sin θ are bounded in [−1, 1], any linear function y = A·trig(θ) + c is itself bounded, with extrema occurring at trig(θ) = ±1. When A < 0 the function is monotonically decreasing in trig(θ), so the maximum of y corresponds to the minimum of the trig factor (−1), and vice versa. This converts an optimisation question into a standard equation cos(...) = −1 or sin(...) = −1, whose solutions are easy to write down.

Thought Process — The step-by-step reasoning

  1. Notice the form y = A·cos(θ) + c (or sin) and read off the sign of A.
  2. Rule out the reflex 'maximum of cos is 1, so plug in 1' — that gives the minimum of y when A is negative.
  3. Commit to the rule: max of y ↔ min of trig factor when A < 0; min of y ↔ max of trig factor.
  4. Write down y_max = c − A (using trig = −1 with A negative) immediately to bag the y-coordinate.
  5. Then solve the reduced equation cos(θ) = −1 (or sin(θ) = −1) within the stated domain to obtain the x-coordinate.

Mechanic — Concrete operations

  1. Inspect the coefficient A in y = A·trig(argument) + c; note its sign.
  2. If A < 0, set trig(argument) = −1 to maximise y (or = +1 to minimise y).
  3. Compute y_extreme by substitution.
  4. Solve trig(argument) = ±1 for the variable, restricted to the given domain, to find the x-coordinate.
  5. State the extremum as a coordinate pair (x, y).

Worked Micro-Example

Find the maximum point of $y = -3\cos(x - 30)^\circ$ for $0 \le x \le 360$. Since the coefficient $-3$ is negative, $y$ is maximised when $\cos(x-30)^\circ = -1$, giving $y_{\max} = -3(-1) = 3$. Solving $\cos(x-30)^\circ = -1$ gives $x - 30 = 180$, so $x = 210$. Hence $Q = (210, 3)$.

Variants — Same move, different surface

y = A·sin(Bx + C) + D with A < 0
Maximum occurs where sin(Bx + C) = −1; y_max = D − A. Solve sin = −1 within domain.
y = c − k·cos(...) with k > 0
Rewrite as y = c + (−k)cos(...); same rule applies — max when cos = −1.
Minimum of y = A·cos(...) + c with A > 0
Standard case: set cos = −1 directly; no flipping needed.
Bounded combination R·sin(θ + α) after harmonic form
After expressing as R·sin(θ + α) + c with R > 0, the sign rule still governs whether max occurs at sin = +1 or −1 depending on any leading sign.
y = A·cos²(θ) + c using cos²θ ∈ [0,1]
Bounds are [0, 1] not [−1, 1]; with A < 0 the max occurs at cos²θ = 0, not 1.

Common Misreadings

  • Treating 'maximum of y' as automatically 'maximum of the trig part', ignoring the negative coefficient.
  • Confusing reflection in the x-axis (negative coefficient) with horizontal translation.
  • Assuming the maximum value is |A| + c without checking the constant term and sign carefully.

Pitfalls

  • Setting cos(x − 30)° = 1 instead of −1, producing the minimum point by mistake.
  • Forgetting to add the constant c, e.g. quoting y_max = −A instead of c − A.
  • Solving cos(...) = −1 outside the stated x-domain or missing solutions when the argument has a coefficient (e.g. 2x).
  • Mishandling the phase shift when reporting x: solving for the argument but forgetting to undo the shift.

A-level Scope Notes

Fully within Edexcel IAL P1/P2 trigonometry: uses only the bounds of sin and cos and standard transformations y = A·f(Bx + C) + D, with no calculus required.
