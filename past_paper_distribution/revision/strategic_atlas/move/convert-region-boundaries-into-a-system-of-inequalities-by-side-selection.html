Trigger — When to use

A diagram shows a shaded region R bounded by some combination of curves (parabolas, lines), axes, and/or vertical/horizontal lines, and the question asks you to write down the set of inequalities defining R. Visual cues include explicit boundary equations labelled on the sketch, dashed vs solid boundary lines (signalling strict vs non-strict inequalities), and a clearly enclosed shaded area. Equivalent phrasings: 'find inequalities that define R', 'describe the region by inequalities', 'state the inequalities satisfied by points in R'.

Motivation — Why this works

Each boundary equation y = f(x) or x = a divides the plane into two half-regions, and an inequality (y > f(x), y < f(x), x > a, x < a) selects exactly one of them. The shaded region R is the intersection of these chosen half-regions, so the conjunction of one correctly-oriented inequality per boundary defines R. The principle is purely set-theoretic: R = ⋂ᵢ Hᵢ where each Hᵢ is the half-region on the correct side of boundary i. Getting the inequality direction right is the entire mathematical content of the move.

Thought Process — Step-by-step reasoning

  1. I notice the diagram has a shaded region R with several labelled boundaries (a curve, one or more lines, possibly an axis); the question asks me to describe R algebraically rather than find an area, so I rule out integration and stick to inequality-writing.
  2. I rule out implicit/parametric descriptions because every boundary is given as an explicit equation y = f(x) or x = constant, which is exactly the format that converts cleanly to half-plane inequalities.
  3. I commit to a 'one inequality per boundary, then intersect' strategy: list every boundary equation, decide its inequality sign, and conjoin them.
  4. For each boundary I ask 'is R above/below this curve?' or 'is R left/right of this vertical line?'. I answer either by geometric reasoning (e.g. parabola opens upward, vertex below R, so R is above the curve) or by picking a clearly interior test point (x₀, y₀) ∈ R and checking which inequality it satisfies.
  5. I decide strict vs non-strict: dashed boundaries → strict (<, >); solid boundaries / 'including the boundary' → non-strict (≤, ≥). If the question is silent, I follow the diagram convention.
  6. Finally, I check my system is non-empty by re-testing one interior point against ALL the inequalities simultaneously, and I present the answer compactly (e.g. as a double inequality a < x < b together with f(x) < y < g(x) when the geometry permits).

Mechanic — Concrete operations

  1. List every boundary equation explicitly: e.g. y = f(x), y = mx + c, x = a, x = b.
  2. Pick a test point (x₀, y₀) clearly inside R (or use vertex/intercept reasoning).
  3. For each boundary y = g(x): substitute x₀ and compare y₀ with g(x₀); if y₀ > g(x₀) write y > g(x), else y < g(x).
  4. For each vertical boundary x = a: compare x₀ with a to choose x > a or x < a (similarly for horizontal lines).
  5. Choose strict (<, >) or non-strict (≤, ≥) based on whether the boundary is dashed or solid in the diagram.
  6. Combine into a simultaneous system, e.g. f(x) < y < g(x) with a < x < b, and verify the test point satisfies all parts.

Worked Micro-Example

Region R is bounded above by $y = 2(x-4)(x-10)$, below by $y = 3x - 12$, on the left by $x = 0$, on the right by $x = 4$. Test point $(2, 0)$: line gives $3(2)-12 = -6$, so $y > 3x-12$ ✓; curve gives $2(-2)(-8) = 32$, so $y < 2(x-4)(x-10)$ ✓. Hence $R: \; 3x - 12 < y < 2(x-4)(x-10)$ with $0 < x < 4$.

Variants — Same move, different surface

Region lies above an upward-opening parabola, below a line, and to the right of a vertical line
Write y ≥ ½x² − 10x + 22, y ≤ −⅛x + 25/4, x ≥ 10 — same side-selection logic, just different curves and directions.
Boundary is included in R (solid line in diagram)
Use ≤ or ≥ in place of < or >; the side-selection step is unchanged.
Region is the interior of a circle of radius r centred at (a, b)
Replace the linear/quadratic boundary with $(x-a)^2 + (y-b)^2 < r^2$ for the interior (or > for the exterior).
Region enclosed by curves on all sides with no vertical/horizontal cut-off
Find the x-coordinates of intersection points and use them as the implicit x-bounds inside the compound inequality.

Common Misreadings

  • Defaulting to y ≤ f(x) for every curve without checking direction, when the region is actually above the curve.
  • Confusing which labelled equation belongs to which boundary, leading to inequalities applied to the wrong curve.
  • Treating the question as an area-finding problem and starting integration instead of writing inequalities.

Pitfalls

  • Reversing the inequality for a vertical line (writing x ≤ a when R lies to the right of x = a).
  • Using ≤/≥ when the diagram shows a dashed boundary (or vice versa), losing the strict/non-strict mark.
  • Stating the inequalities as an unlinked list without making clear they must hold simultaneously — examiners require an 'and' or a compound double-inequality form.
  • Forgetting the x-bounds entirely, which leaves the described region unbounded and incorrect.
  • Not verifying with a test point, producing an inconsistent system that defines the empty set.

A-level Scope Notes

This is core Edexcel IAL Pure (P1/P2) coordinate-geometry content on linear and quadratic inequalities and their graphical interpretation; no formula booklet entry is invoked. Avoid generalising to systems of nonlinear inequalities involving more than one curved boundary unless the curves and their intersections have been computed explicitly — multi-region decomposition is not required at this level.
