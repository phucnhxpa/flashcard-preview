Coordinate Geometry

Set x = 0 and pick the non-origin root for y-axis intersection

Family: Axis Intersection Frequency: 1 questions

Trigger — When to use

Fires whenever a question names a point P where a curve meets the y-axis and either explicitly excludes the origin or implies P is distinct from another listed crossing. Cues include phrasing like 'the curve crosses the y-axis at P', 'P is not the origin', or a diagram showing two y-axis crossings. Common on implicit-curve questions where substituting x = 0 yields a quadratic (or higher) in y with multiple roots. Also fires when the curve passes through (0,0) by inspection and a second crossing must be located.

Motivation — Why this works

By definition, points on the y-axis have x = 0, so substitution collapses the curve equation to a single-variable equation in y. For implicit curves involving e^{kx} or similar, x = 0 typically reduces exponentials to 1, leaving a polynomial in y that factors. The presence of a y common factor (signalling y = 0 as a root) reflects the curve passing through the origin; the other factor pinpoints P. Selecting the non-trivial root is required because the question's constraint 'P is not the origin' singles out the second solution.

Thought Process — The step-by-step reasoning

  1. Notices that 'crosses the y-axis at P' is a coordinate condition: x_P = 0.
  2. Rules out trying to parametrise or use derivatives — the location of P is purely an algebraic substitution problem.
  3. Commits to substituting x = 0 because the equation simplifies dramatically (exponentials become 1, x-terms vanish).
  4. Bridges to the concrete operation: write the resulting equation in y, then factor rather than divide so no roots are lost.
  5. Tactical decision: identify which root is the origin (y = 0) and which is P; keep the non-origin root and record P = (0, y_P) for use in later parts (e.g., dy/dx, normal line).

Mechanic — Concrete operations

  1. Substitute x = 0 into the curve equation, simplifying any e^{kx} factor to 1.
  2. Rearrange to bring all terms to one side: f(y) = 0.
  3. Factor out y (or the obvious common factor) rather than dividing, to preserve y = 0 as a candidate root.
  4. Solve each factor: typically y = 0 and y = a/b.
  5. Apply the question's exclusion ('P is not the origin') to select the non-zero root.
  6. State P = (0, y_P) explicitly for downstream use.

Worked Micro-Example

Curve: $4y^2 + x^2 = 6y e^{x}$. Set $x = 0$: $4y^2 = 6y$, so $4y^2 - 6y = 0 \Rightarrow 2y(2y-3) = 0$. Roots: $y = 0$ or $y = \tfrac{3}{2}$. Since $P \neq O$, $P = \left(0, \tfrac{3}{2}\right)$.

Variants — Same move, different surface

Curve crosses the x-axis at a named non-origin point
Set y = 0 instead; factor out x and discard x = 0.
Explicit curve y = f(x) crossing y-axis
Just evaluate f(0) — no root selection needed since y is uniquely determined.
Implicit curve with no exponential, e.g. polynomial in x and y
Same procedure; substitution gives a polynomial in y that may need quadratic formula rather than simple factoring.
Parametric curve crossing the y-axis
Solve x(t) = 0 for t, reject the parameter value giving the origin, then evaluate y(t) at the surviving t.
Curve crossing a vertical line x = a (not the axis)
Substitute x = a instead of 0; same root-selection logic applies.

Common Misreadings

  • Reading 'crosses the y-axis' as 'has y-intercept' and stopping at y = 0 without checking for a second root.
  • Assuming the curve only crosses the y-axis once and not noticing the question names a specific non-origin point.
  • Confusing the y-axis condition (x = 0) with the x-axis condition (y = 0).

Pitfalls

  • Dividing both sides by y at the outset, which silently discards the y = 0 root and can mask whether the curve passes through the origin.
  • Forgetting that e^{0} = 1, leading to an incorrect simplified equation.
  • Sign or arithmetic slip when rearranging to factor form.
  • Reporting only the y-coordinate of P and forgetting to write the full pair (0, y_P).

A-level Scope Notes

Fully within Edexcel IAL Pure scope (P3/P4 implicit differentiation context); requires only substitution, factoring of a quadratic, and recognition that e^{0} = 1.
