Complex Numbers

Identify quadrant before applying arctan for principal argument

Family: Quadrant Analysis Frequency: 1 questions

Trigger — When to use

Fires whenever a question asks for arg(z) or the principal argument of a complex number z = x + iy. Cues include phrases like 'find arg(z)', 'give your answer in radians', or modulus-argument form r(cos θ + i sin θ). The move is essential because arctan(y/x) alone only returns values in (−π/2, π/2), so it gives the correct principal argument only when Re(z) > 0 (quadrants I and IV).

Motivation — Why this works

The principal argument is defined to lie in (−π, π], but arctan has range (−π/2, π/2), so it cannot distinguish z from −z. For example, 1 + i and −1 − i both give arctan(1) = π/4, yet their true arguments differ by π. Sketching z on an Argand diagram and identifying its quadrant tells the student whether to take arctan(y/x) directly, add π, or subtract π. Special axis cases (purely real or purely imaginary z) must also be handled separately.

Thought Process — The step-by-step reasoning

  1. Notices the question asks for arg(z) and recalls the principal argument lives in (−π, π].
  2. Rules out blindly typing arctan(y/x) into the calculator, since this only works when x > 0.
  3. Commits to a quick Argand sketch (or sign check on x and y) to locate the quadrant before any computation.
  4. Bridges to the concrete formula: Q1 or Q4 → arctan(y/x); Q2 → arctan(y/x) + π; Q3 → arctan(y/x) − π.
  5. Inside the move, decides on radian mode, rounding, and whether the answer should be left exact (e.g. π/3) or as a decimal.

Mechanic — Concrete operations

  1. Read off x = Re(z) and y = Im(z); note their signs.
  2. Identify the quadrant (or axis case) of z on the Argand diagram.
  3. Apply the correct formula: Q1/Q4 use arctan(y/x); Q2 add π; Q3 subtract π; axis cases use 0, π, π/2, or −π/2 directly.
  4. Set the calculator to radian mode (unless degrees are requested) and evaluate.
  5. Check the result lies in (−π, π] and round as instructed.

Worked Micro-Example

Find arg(z) for $z = 3 + 2i$. Here $x = 3 > 0$, $y = 2 > 0$, so $z$ lies in Q1. Therefore $\arg(z) = \arctan(2/3) = 0.5880\ldots \approx 0.59$ rad (2 d.p.). No π-adjustment is needed because Re(z) > 0.

Variants — Same move, different surface

z in Q2 (x < 0, y > 0), e.g. z = −1 + i
arg(z) = arctan(y/x) + π, giving 3π/4 here.
z in Q3 (x < 0, y < 0), e.g. z = −1 − i
arg(z) = arctan(y/x) − π, giving −3π/4 here.
z in Q4 (x > 0, y < 0)
arctan(y/x) gives a negative value directly — no adjustment needed.
Purely imaginary z (x = 0)
arg(z) = π/2 if y > 0, −π/2 if y < 0; arctan is undefined.
Negative real axis (y = 0, x < 0)
arg(z) = π exactly; arctan(0/x) = 0 would be wrong.
Positive real axis (y = 0, x > 0)
arg(z) = 0.

Common Misreadings

  • Assuming arctan(y/x) always returns the principal argument, regardless of quadrant.
  • Treating arg(z) and arg(−z) as equal because |y/x| is the same.
  • Confusing the principal argument range (−π, π] with [0, 2π).

Pitfalls

  • Calculator set to degree mode instead of radian mode (e.g. 33.69° instead of 0.59 rad).
  • Rounding arctan(2/3) = 0.588… up to 0.60 instead of 0.59.
  • Forgetting axis cases: writing arctan(0) = 0 for z = −2 instead of arg = π.
  • Adding π when the quadrant is actually Q4, pushing the answer outside (−π, π].

A-level Scope Notes

Fully within Edexcel IAL Further Pure (FP1/FP2) treatment of complex numbers in modulus-argument form; uses only arctan, sketching on the Argand diagram, and the principal argument convention (−π, π].
