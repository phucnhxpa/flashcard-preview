Sequences & Series

Back-substitute recovered parameter to evaluate a specific term

Family: Parameter Completion Frequency: 1 questions

Trigger — When to use

Fires in multi-part AP/GP questions where an earlier part has solved for one parameter (typically d or r) from simultaneous equations, and a later part asks for a specific term u_n or partial sum. The first term a is implicitly available but has not been explicitly stated. Cues include phrases like 'hence find u_15', 'find the 20th term', or 'find the value of the kth term' immediately after part (a) yielded d or r.

Motivation — Why this works

The nth-term formula u_n = a + (n−1)d (or ar^{n−1}) requires both parameters. Once d (or r) is fixed, any one of the original equations is the cheapest route to recover a, and a single substitution into u_n delivers the answer. This route earns all method marks transparently because each formula and substitution is explicit, whereas shortcut routes risk losing marks if working is not shown.

Thought Process — The step-by-step reasoning

  1. I notice the target u_15 needs both a and d, and d is already known from part (a).
  2. I rule out re-solving the simultaneous system from scratch — wasteful — and I rule out the index-shift shortcut u_15 = u_4 + 11d as risky for method marks.
  3. I commit to back-substitution into the simplest of the two original equations, since it gives a with one line of arithmetic.
  4. Bridge: pick the equation with the smallest coefficient on d (here a + 3d = 18) to minimise arithmetic error.
  5. Tactically, I write a explicitly before applying u_n = a + (n−1)d, double-checking that the index is (n−1) not n.

Mechanic — Concrete operations

  1. Take the simpler of the two original equations from part (a), e.g. a + 3d = 18.
  2. Substitute the known value of d to isolate a: a = 18 − 3(2.5) = 10.5.
  3. Write the nth-term formula: u_n = a + (n−1)d.
  4. Substitute a, d, and n: u_15 = 10.5 + 14(2.5).
  5. Evaluate: u_15 = 10.5 + 35 = 45.5.

Worked Micro-Example

Given an AP with $a + 3d = 18$ and $d = 2.5$ already established, find $u_{15}$. \newline Back-substitute: $a = 18 - 3(2.5) = 10.5$. \newline Apply formula: $u_{15} = a + 14d = 10.5 + 14(2.5) = 10.5 + 35 = 45.5$.

Variants — Same move, different surface

Geometric progression where r is known from part (a) and a term u_n is requested
Substitute r into ar^{k} = (known term) to recover a, then apply u_n = ar^{n−1}.
Question asks for S_n rather than u_n
After recovering a, apply S_n = (n/2)[2a + (n−1)d] (AP) or S_n = a(1−r^n)/(1−r) (GP).
Index-shift shortcut available because a known term u_k is given
Use u_n = u_k + (n−k)d (AP) or u_n = u_k · r^{n−k} (GP) to bypass finding a; show both methods if marks for stating a are at risk.
Sum to infinity of a GP with |r|<1 once r is found
Recover a from a known term, then apply S_∞ = a/(1−r).

Common Misreadings

  • Using exponent n instead of n−1 in u_n = a + (n−1)d, giving u_15 = a + 15d.
  • Treating the known term (e.g. u_4 = 18) as if it were a, leading to u_15 = 18 + 14d.
  • For a GP, writing u_n = ar^n instead of ar^{n−1}.

Pitfalls

  • Arithmetic slips with decimals (e.g. 18 − 7.5 miscomputed as 11.5).
  • Substituting d into the more complicated equation (e.g. a + 7d = 28) when a simpler one is available.
  • Forgetting to state a explicitly, costing a method/answer mark even if u_15 is correct.
  • Off-by-one error: using (n−1) = 15 because n = 15 was misread.

A-level Scope Notes

Standard Edexcel IAL P1/P2 arithmetic and geometric progression material; uses only the formulae u_n = a + (n−1)d, u_n = ar^{n−1}, and the corresponding sum formulae from the IAL formula booklet.
