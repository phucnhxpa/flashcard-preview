Binomial Expansion

Absorb the minus sign into b when matching (a+b)^n

Family: Parameter Identification Frequency: 1 questions

Trigger — When to use

Fires whenever a binomial expansion target has the form (A - B)^n rather than (A + B)^n. Typical surfaces include (2 - x/4)^{10}, (1 - 2x)^n, (3 - x)^6, or any 'find the coefficient of x^k in (... - ...)^n' question. The cue is a literal minus sign between the two terms inside the bracket, before any expansion or coefficient extraction begins.

Motivation — Why this works

The binomial theorem C(n,r) a^{n-r} b^r is stated for a sum, so the minus sign must live somewhere — either inside b or as a separate (-1)^r factor. Bundling it into b (so b = -B) lets the formula handle alternating signs automatically: even r gives +, odd r gives −, with no extra bookkeeping. This exploits (-B)^r = (-1)^r B^r as a single algebraic object, which is far less error-prone than tracking signs term by term. It also keeps the general term in one clean closed form, useful for 'find the coefficient of x^k' questions.

Thought Process — The step-by-step reasoning

  1. I notice the bracket is (2 - x/4)^{10}, a difference rather than a sum, so the standard (a+b)^n template doesn't match literally.
  2. I rule out keeping b = +x/4 and prepending an external minus sign per term, because that invites sign slips on every odd-power term.
  3. I commit to b = -x/4 because (-1)^r is then baked into b^r and signs alternate automatically.
  4. Concretely I set a = 2, b = -x/4, n = 10, and write the general term C(10,r) · 2^{10-r} · (-x/4)^r.
  5. Inside the move I keep the fraction intact in b (don't split -x and /4) so that powers of 4 in the denominator are tracked cleanly alongside the sign.

Mechanic — Concrete operations

  1. Identify a, n directly from the bracket; for the second term, set b equal to the WHOLE signed expression (including the minus).
  2. Write the general term T_{r+1} = C(n,r) · a^{n-r} · b^r.
  3. When evaluating b^r = (-B)^r, separate as (-1)^r · B^r so the sign is explicit.
  4. Compute terms for the required r-values; even r → positive, odd r → negative.
  5. If asked for a specific coefficient of x^k, solve the power equation in r first, then substitute.

Worked Micro-Example

Expand $(2 - x/4)^{10}$ up to $x^3$. Set $a=2$, $b=-x/4$, $n=10$. General term: $\binom{10}{r}2^{10-r}(-x/4)^r = \binom{10}{r}2^{10-r}(-1)^r x^r/4^r$. Then $r=0:\,1024$; $r=1:\,10\cdot 512\cdot(-1/4)x=-1280x$; $r=2:\,45\cdot 256\cdot(1/16)x^2=720x^2$; $r=3:\,120\cdot 128\cdot(-1/64)x^3=-240x^3$.

Variants — Same move, different surface

(1 - 2x)^n style brackets
Set a=1, b=-2x; general term C(n,r)(-2)^r x^r, so the sign and the coefficient 2^r combine inside (-2)^r.
Fractional second term such as (3 - x/2)^8
Keep the whole signed fraction as b = -x/2; do not split off the minus or the denominator prematurely.
Both terms with coefficients, e.g. (2x - 3)^7
Set a=2x, b=-3 (or swap); commitment to which is 'a' and which is 'b' depends on which power of x is asked for.
Series expansion (1 - x)^{-n} or (1 - x)^{1/2}
Same bundling: use b = -x in the generalised binomial $(1+b)^\alpha$, so signs come from b^r automatically; valid for |x|<1.

Common Misreadings

  • Reading (2 - x/4)^{10} as (2 + x/4)^{10} and producing all-positive terms.
  • Setting b = x/4 and trying to attach a separate minus sign 'when needed', then forgetting it on the r=3 term.
  • Treating the minus as belonging to a (i.e. a = -2) rather than to the second term.

Pitfalls

  • Writing (-x/4)^2 as -x^2/16 instead of +x^2/16 — the square kills the sign.
  • Writing (-x/4)^3 as +x^3/64 instead of -x^3/64.
  • Forgetting to raise the denominator: (-x/4)^r = (-1)^r x^r / 4^r, not (-1)^r x^r / 4.
  • When b = -2x, computing (-2x)^r as -2^r x^r instead of (-1)^r 2^r x^r.

A-level Scope Notes
