Sequences & Series

Reject extraneous quadratic root by checking physical validity of model term

Family: Contextual Root Rejection Frequency: 1 questions

Trigger — When to use

Fires when solving a sequence/series problem produces a quadratic in n (or N) with two positive integer roots, but the question asks for 'the value' (singular). Common in arithmetic sequence contexts where u_n = a + (n-1)d models a quantity (extraction, production, savings, attendance) that must remain non-negative. Cue phrases: 'hence state the value of N', 'find the year in which', or any decreasing AP where d < 0 so later terms eventually go negative. Also fires when both roots are mathematically valid but one violates the modelling assumption.

Motivation — Why this works

A quadratic has two roots by the fundamental theorem of algebra, but a contextual model imposes constraints (positivity, integer range, monotonicity) that may invalidate one. In a declining AP with d < 0, terms eventually become negative, so any root N giving u_N < 0 corresponds to a year past which the model has already broken down. Verifying u_N ≥ 0 for each candidate root is the standard discriminator. The singular phrasing 'state the value' is itself a strong signal that exactly one root survives.

Thought Process — The step-by-step reasoning

  1. Notice the question asks for a single value of N, but the quadratic will give two roots — so a rejection step is coming.
  2. Rule out the temptation to pick the larger root just because 'more years means more total'; in a declining AP later terms can be negative and meaningless.
  3. Commit to checking u_N for each root against the modelling constraint u_N ≥ 0.
  4. Bridge: factorise the quadratic (or use the formula), then substitute each root into u_N = a + (N-1)d.
  5. Tactically, find the cutoff year where u_n first turns negative (solve a + (n-1)d ≥ 0); any root beyond this is automatically rejected, and state the rejection reason explicitly for the method mark.

Mechanic — Concrete operations

  1. Solve the quadratic in N (factorise or use the formula) to obtain two candidate roots N₁, N₂.
  2. For each candidate, compute u_N = a + (N-1)d.
  3. Identify which u_N values satisfy the contextual constraint (typically u_N ≥ 0).
  4. Reject the root violating the constraint, stating the numerical reason (e.g. 'u₃₇ = -60 < 0, so impossible').
  5. State the surviving value as the answer.

Worked Micro-Example

An arithmetic sequence has $a = 480$, $d = -15$. Solving $S_N = 12978$ gives $N^2 - 65N + 1036 = 0$, so $(N-28)(N-37)=0$. Check: $u_{28} = 480 + 27(-15) = 75 \geq 0$ ✓; $u_{37} = 480 + 36(-15) = -60 < 0$ ✗. Hence $N = 28$.

Variants — Same move, different surface

Geometric series with sum condition giving a quadratic in n
Reject the root for which u_n or the sum violates positivity, or where n is non-integer.
Quadratic in the common ratio r from sum-to-infinity or two-term conditions
Reject |r| ≥ 1 if S_∞ is required to exist, or reject r < 0 if context demands positive terms.
Quadratic in first term or common difference from two simultaneous AP conditions
Reject the value producing a negative or non-integer term where context forbids it.
Mechanics-style modelling sequence (e.g. monthly savings, attendance) with a cap
Reject the root that exceeds the year at which the model term hits zero or the stated cap.

Common Misreadings

  • Assuming the larger root is correct because 'more years = more total', ignoring that terms become negative.
  • Treating both roots as valid answers and listing both, when the question demands a single value.
  • Rejecting on the wrong criterion (e.g. 'N must be small') without computing u_N.

Pitfalls

  • Failing to justify rejection: writing 'N = 37 is too large' without computing u₃₇ < 0 loses the reasoning mark.
  • Arithmetic slip when factorising large constants like 1036; the quadratic formula is a safe fallback.
  • Computing u_N with (N) instead of (N-1) in the AP formula, giving a wrong sign and misclassifying which root to reject.
  • Forgetting to check whether u_N = 0 is permitted by the context (boundary case).

A-level Scope Notes

Fully within Edexcel IAL P1/P2 sequences and series: arithmetic and geometric progressions, sums S_n, and modelling questions where context constrains the answer.
