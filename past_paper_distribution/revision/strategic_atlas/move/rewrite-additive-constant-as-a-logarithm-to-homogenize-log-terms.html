Algebra & Functions

Rewrite additive constant as a logarithm to homogenize log terms

Family: Constant To Log Conversion Frequency: 1 questions

Trigger — When to use

Fires when an equation or model has logarithm terms added to (or subtracted from) a bare numerical constant on the same side, e.g. log₁₀M = 1.93 log₁₀r + 0.684, and you need to combine everything into a single logarithm. Typical cues: a log-linear regression result of the form log y = m log x + c, an instruction to express y in the form pr^q or kx^n, or a need to apply the product/power laws but a non-log term blocks it. Also fires when 'undoing' a straight-line fit on a log-log plot back to a power-law relationship.

Motivation — Why this works

The log laws log(ab) = log a + log b and log(a^n) = n log a only combine logarithms with logarithms; a numerical constant c is not yet in log form. Since log₁₀ and 10^(·) are mutual inverses, c = log₁₀(10^c) is an identity that rewrites the constant as a logarithm without changing its value. Once every term on the side is a log₁₀ of something, the laws apply and the side collapses to a single log₁₀, after which both sides can be de-logged in one step.

Thought Process — The step-by-step reasoning

  1. I notice the RHS mixes log₁₀(r^{1.93}) with the bare constant 0.684 — they are not yet of the same type.
  2. I rule out exponentiating both sides immediately, because 10^{a+b} would split awkwardly and not directly give the target form M = pr^q.
  3. I commit to converting the constant into a log, since the target form is a single power-law expression and that requires combining via log laws.
  4. The bridging identity is c = log₁₀(10^c); applying it to 0.684 turns the RHS into a sum of two log₁₀ terms.
  5. Then I apply log a + log b = log(ab) to merge them, and finally remove the outer log₁₀ from both sides to read off p = 10^c and q = the gradient.

Mechanic — Concrete operations

  1. Identify the bare additive constant c on the side containing logarithms (here c = 0.684).
  2. Rewrite c = log₁₀(10^c), matching the base of the existing logarithms.
  3. Substitute to get all terms on that side as log₁₀(...): log₁₀M = log₁₀(r^{1.93}) + log₁₀(10^{0.684}).
  4. Combine with the product law: log₁₀M = log₁₀(10^{0.684} · r^{1.93}).
  5. Equate arguments (since log₁₀ is one-to-one): M = 10^{0.684} · r^{1.93}, then evaluate 10^{0.684} ≈ 4.83 to give M ≈ 4.83 r^{1.93}.

Worked Micro-Example

Given $\log_{10}M = 1.93\log_{10}r + 0.684$, write $0.684 = \log_{10}(10^{0.684})$. Then $\log_{10}M = \log_{10}(r^{1.93}) + \log_{10}(10^{0.684}) = \log_{10}(10^{0.684}\,r^{1.93})$. Hence $M = 10^{0.684}\,r^{1.93} \approx 4.83\,r^{1.93}$, so $p \approx 4.83$ and $q = 1.93$.

Variants — Same move, different surface

Natural-log model: ln y = m ln x + c
Use c = ln(e^c) instead of log₁₀(10^c), giving y = e^c · x^m.
General base: log_b y = m log_b x + c
Write c = log_b(b^c) so y = b^c · x^m.
Reading a straight line of best fit on a log-log plot with intercept c and gradient m
Same identity converts the intercept c into log(b^c), recovering the power law y = b^c x^m.
Exponential model log y = m x + c (semi-log, not log-log)
After rewriting c, get y = 10^c · 10^{mx} = A·k^x with A = 10^c, k = 10^m.
Constant on the LHS, e.g. 1 + log₁₀x = log₁₀(...)
Rewrite 1 = log₁₀10 (special case of c = log₁₀(10^c)) and combine.

Common Misreadings

  • Confusing the number with its log: writing 0.684 = log₁₀(0.684) instead of log₁₀(10^{0.684}).
  • Treating the constant as already a coefficient of a log, e.g. reading 0.684 as 0.684·log₁₀(something).
  • Thinking the rewrite changes the value of the constant rather than its form.

Pitfalls

  • Mismatching bases: writing 0.684 = ln(e^{0.684}) when the equation is in log₁₀, so the terms still cannot be combined.
  • Forgetting the rewrite and incorrectly exponentiating term-by-term, e.g. concluding M = r^{1.93} + 10^{0.684}.
  • Combining logs before homogenizing, e.g. writing log₁₀(r^{1.93} + 0.684).
  • Losing the constant when collecting, so the final p is reported as 1 instead of 10^{0.684}.

A-level Scope Notes

Fully within Edexcel IAL Pure Mathematics: laws of logarithms (P2) and reduction of experimental data to linear form log y = m log x + c to find power-law models y = a x^n (P2/P3 modelling).
