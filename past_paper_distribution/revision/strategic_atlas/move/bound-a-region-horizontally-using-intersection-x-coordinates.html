Coordinate Geometry

Bound a region horizontally using intersection x-coordinates

Family: Domain Restriction Frequency: 1 questions

Trigger — When to use

Fires when defining a finite enclosed region R by a system of inequalities and the curve/line inequalities alone leave the solution set unbounded or include extraneous regions. Typical cues: a shaded region in a diagram bounded by two curves meeting at points P and Q, a question asking to 'write down inequalities that define R', or part (a) having already produced intersection x-values. Also fires when the curves cross more than once and only the segment between two specific intersections is shaded.

Motivation — Why this works

Intersection points are where the relative ordering of the bounding curves can switch, so they are the natural horizontal limits of the enclosed region. Inside [x_P, x_Q] one curve sits above the other consistently; outside, the inequalities y ≤ f(x), y ≤ g(x), y ≥ 0 may still be satisfied but describe a different (often unbounded) region. Adding x_P ≤ x ≤ x_Q closes the system so its solution set is exactly the shaded R. This mirrors the role intersection abscissae play as integration limits when computing the same region's area.

Thought Process — The step-by-step reasoning

  1. Notices that the three inequalities y ≤ curve, y ≤ line, y ≥ 0 still admit points far to the left/right of the shaded R.
  2. Rules out adding more curve-based inequalities — the boundary curves are already used; what is missing is a horizontal cut-off.
  3. Commits to using the intersection x-coordinates of the bounding curves as the cut-off, since these are exactly where the ordering of the boundaries changes.
  4. Bridges to the concrete operation by retrieving x_P and x_Q from part (a) (e.g. solving curve = line) and writing x_P ≤ x ≤ x_Q.
  5. Decides on weak (≤) inequalities so the boundary points P and Q are included in R, matching the closed shaded region in the figure.

Mechanic — Concrete operations

  1. Identify the intersection points P and Q of the bounding curves (typically from part (a)).
  2. Read off their x-coordinates, e.g. x = 1 and x = 5/2.
  3. Write the horizontal bound: 1 ≤ x ≤ 5/2 (use ≤ to include endpoints).
  4. Combine with the curve/line/axis inequalities to give the full system defining R.
  5. Sanity-check by testing an interior point of R and an exterior point with the same y-conditions.

Worked Micro-Example

Curve $y = 6 - 2x$ and curve $y = \tfrac{2}{x}$ meet where $6 - 2x = 2/x \Rightarrow 2x^2 - 6x + 2 = 0$, giving $x = 1$ and $x = 5/2$. The shaded region $R$ above the x-axis between them is defined by $y \ge 0,\ y \le 6 - 2x,\ y \le 2/x,\ 1 \le x \le 5/2$. The final inequality is what closes the system.

Variants — Same move, different surface

Area between two curves by integration
The same intersection x-values become the limits of $\int_{x_P}^{x_Q}(\text{upper}-\text{lower})\,dx$.
Region bounded by a curve and a horizontal/vertical line
Use the line $x = a$ directly as the bound, or solve curve = line for x.
Region bounded by three curves meeting pairwise
Split R at the middle intersection and impose piecewise x-bounds, or add y-bounds as well.
Parametric or trig curves crossing multiple times
Restrict to the specific adjacent pair of intersection x-values that enclose the shaded lobe.

Common Misreadings

  • Believing the curve/line/axis inequalities alone already pin down a finite region.
  • Taking only one of P, Q as a bound (e.g. writing x ≥ 1) and leaving the other side open.
  • Using y-bounds (e.g. 0 ≤ y ≤ something) when the natural cut-off is horizontal.

Pitfalls

  • Propagating an incorrect intersection from part (a) — always re-check by substituting back.
  • Using strict inequalities $1 < x < 5/2$ when the shaded region is closed and includes P and Q.
  • Swapping the bounds (writing $5/2 \le x \le 1$) or mixing up which curve is upper on the interval.

A-level Scope Notes

Standard Edexcel IAL Pure content: defining regions by inequalities and locating intersections by simultaneous equations, directly linked to area-between-curves integration in P2/P3.
