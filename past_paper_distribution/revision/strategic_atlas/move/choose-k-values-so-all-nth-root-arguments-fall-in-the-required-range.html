Complex Numbers

Choose k-values so all nth-root arguments fall in the required range

Family: Range Selection Frequency: 1 questions

Trigger — When to use

The question gives an explicit argument range such as 0 < θ < 2π or −π < θ ≤ π for the n roots of a complex equation. After applying de Moivre's theorem to get θ = (θ₀ + 2kπ)/n, you must decide which n consecutive integer values of k to use. The cue is the explicit interval in the question stem, often combined with 'in the form r(cos θ + i sin θ)' with θ restricted.

Motivation — Why this works

Any n consecutive integers k generate the same set of n distinct roots, but the resulting arguments are shifted by multiples of 2π. Choosing the k-block that already lands inside the demanded interval avoids error-prone post-hoc additions or subtractions of 2π. For 0 < θ < 2π and a positive base argument, k = 0,1,…,n−1 typically works directly; for −π < θ ≤ π, a symmetric block around 0 is usually right. A quick check on the smallest and largest k confirms the choice.

Thought Process — The step-by-step reasoning

  1. I notice the question pins arguments to a specific interval, so listing roots with arbitrary k will not be accepted.
  2. I rule out blindly taking k = 0,…,n−1 without checking — it only works when the resulting θ-block lies inside the stated range.
  3. I commit to first computing θ for the smallest and largest candidate k, since these endpoints determine whether the whole block fits.
  4. I bridge to the first operation: write θ = (θ₀ + 2kπ)/n and substitute the boundary k-values.
  5. Tactically, if the test fails I shift the k-block by ±1 (or subtract 2π from offending arguments) rather than restart.

Mechanic — Concrete operations

  1. Write the general argument θ = (θ₀ + 2kπ)/n from de Moivre.
  2. Identify the required interval (e.g. 0 < θ < 2π or −π < θ ≤ π).
  3. Test the candidate smallest k: check θ exceeds the lower bound.
  4. Test the candidate largest k: check θ stays below the upper bound.
  5. If both endpoints fit, list all n arguments using that k-block; otherwise shift k-block by 1 or adjust by ±2π.
  6. Verify each of the n arguments lies strictly (or weakly, per question) inside the interval.

Worked Micro-Example

Solve $z^5 = 32i$ with $0 < \arg z < 2\pi$. Then $z = 2\,\text{cis}\!\left(\tfrac{\pi/2 + 2k\pi}{5}\right)$. Try $k=0$: $\theta = \pi/10 \in (0,2\pi)$ ✓. Try $k=4$: $\theta = 17\pi/10 < 2\pi$ ✓. So take $k=0,1,2,3,4$ giving $\theta = \tfrac{\pi}{10}, \tfrac{\pi}{2}, \tfrac{9\pi}{10}, \tfrac{13\pi}{10}, \tfrac{17\pi}{10}$.

Variants — Same move, different surface

Required range is −π < θ ≤ π (principal argument)
Use a symmetric k-block such as k = −2,−1,0,1,2 for n=5, or subtract 2π from any argument exceeding π.
Cube roots (n = 3) of a complex number
Same logic with k = 0,1,2 for [0,2π) or k = −1,0,1 for (−π,π].
Base argument θ₀ is negative (e.g. equation involves cis(−π/3))
Starting at k = 0 may give a negative θ; shift to k = 1,…,n for [0,2π).
Range stated as 0 ≤ θ < 2π or with weak inequalities
Endpoint inclusion may matter when an argument equals 0 or π; check boundary cases carefully.

Common Misreadings

  • Treating the range constraint as cosmetic and leaving arguments outside the stated interval.
  • Confusing the principal-argument convention (−π,π] with the (0,2π) convention used in the question.
  • Reading 'between 0 and 2π' as inclusive of endpoints when strict inequalities are written.

Pitfalls

  • Writing 13π/10 as −7π/10 (or vice versa) under the wrong convention.
  • Using n+1 k-values (e.g. k = 0,…,5 for fifth roots) and producing a duplicate root.
  • Forgetting to divide the 2kπ term by n before testing the interval.
  • Stopping after one endpoint check and missing that an interior argument falls outside the interval (rare but possible if θ₀ is awkwardly placed).
