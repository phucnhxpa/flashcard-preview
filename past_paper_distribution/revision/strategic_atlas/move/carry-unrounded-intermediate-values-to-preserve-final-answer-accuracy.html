Algebra & Functions

Carry unrounded intermediate values to preserve final-answer accuracy

Family: Precision Management Frequency: 2 questions

Trigger — When to use

Fires whenever a multi-part question asks for an intermediate quantity 'to 3 significant figures' (or similar) AND that quantity is reused in a later part. Visual cues: part (a)/(b) ends with 'give your answer to 3 s.f.', then part (b)/(c) begins 'Hence find...' or 'Using your answer to part (a)...'. Also fires when the reused value enters a sensitive operation — a subtraction near cancellation (e.g. π − 1.3 − θ), a trig function near a turning point, or a chained product. Equivalent phrasings: 'don't round too early', 'store full calculator value', 'avoid premature approximation'.

Motivation — Why this works

Each rounding step introduces an error of up to half a unit in the last retained digit. When a rounded value is fed forward, errors compound — and operations like subtraction of nearly-equal quantities, or sin/cos near extrema, amplify the relative error so the final answer can fall outside the mark scheme's accepted range. Edexcel mark schemes explicitly penalise 'premature approximation'. The principle: reporting precision (what you write down) is not the same as working precision (what you compute with) — keep full precision internally and round only the final reported answer.

Thought Process — The step-by-step reasoning

  1. I notice this is a multi-part question and the value I'm asked to report in this part is going to be reused in the next part (signalled by 'Hence' or by direct dependency).
  2. I rule out the naive approach of rounding to 3 s.f. and then re-typing that rounded number into part (c) — I recognise this would compound error.
  3. I rule out keeping the value symbolic only — sometimes that's neatest, but here a numerical value is needed and the calculator is the right tool.
  4. I commit to the strategy: compute and STORE the full-precision value (e.g. in calculator memory or as a long-decimal noted on paper), display the rounded version only as the written answer for the current part, and recall the stored value for every subsequent calculation.
  5. Tactical decision: I check whether the next-part operation is error-sensitive (subtraction, sin near 0 or π/2, division by a small number). If so, I am extra careful never to retype the rounded display value.
  6. I round only at the very last step, when producing the final reported answer of the chained calculation.

Mechanic — Concrete operations

  1. Compute the intermediate quantity to full calculator precision, e.g. q = 8 − 3.276 = 4.724... or θ = arccos(3.8/12.96) = 1.54131...
  2. Store this value in calculator memory (ANS, A, M+) — do not clear it between parts.
  3. Write down the rounded answer (e.g. 4.72 or 1.54) as the part (a)/(b) response only.
  4. In the next part, recall the stored value rather than retyping the rounded one; substitute it into all subsequent operations (sin θ, π − a − θ, area formulae, etc.).
  5. Apply rounding to 3 s.f. (or as instructed) only to the final composite answer.

Worked Micro-Example

Part (b): Find $q$ where $q = 8 - 3.276$. Calculator gives $q = 4.724$; report $q \approx 4.72$ (3 s.f.). Part (c): $T^2 = k q$ with $k = 70$. Using stored $q = 4.724$: $T^2 = 330.68$, so $T = 18.18... \approx 18.2$. Using prematurely-rounded $q = 4.72$ would give $T^2 = 330.40$, $T = 18.177...$ — same here, but in tighter problems the difference shifts the 3 s.f. answer.

Variants — Same move, different surface

Trigonometry / radians: an angle found in one part is reused inside sin, cos, or a subtraction from π.
Store the angle to all displayed digits (e.g. 1.54131…); never retype the rounded form. Especially critical when the next step is π − (small angle sum), where cancellation amplifies error.
Statistics: mean, variance or regression coefficient computed in part (a) feeds into a probability or further estimate in part (b).
Carry $\bar{x}$, $s^2$, or coefficients $a,b$ to full precision; round only the final probability/estimate.
Mechanics: a velocity or time obtained in stage 1 of a problem is used in stage 2 (e.g. SUVAT chained with another phase).
Carry the unrounded velocity/time into the next SUVAT equation; round only the requested final quantity.
Coordinate geometry: an intersection point or gradient is reused in a distance, area, or perpendicular calculation.
Keep the coordinates/gradient as exact surds or full-precision decimals through all subsequent steps.

Common Misreadings

  • Interpreting '3 significant figures' as an instruction about working precision rather than reporting precision — and so rounding internally too.
  • Believing that since each part is marked separately, the rounded answer from part (a) is the 'official' value to feed into part (b).
  • Thinking that storing more digits is unnecessary because '3 s.f. is plenty'.

Pitfalls

  • Retyping the displayed rounded answer from a previous part into the calculator instead of recalling the stored full-precision value.
  • Clearing calculator memory (or pressing AC) between parts and losing the stored value.
  • Over-truncating intermediate writes on paper (e.g. noting 0.36 instead of 0.3642…) and then using the truncated version.
  • Rounding within a sensitive subtraction such as π − 1.3 − θ, where small errors in θ shift the result by far more than 3 s.f. tolerance.

A-level Scope Notes

This is a universal exam-technique principle for Edexcel IAL Maths and Further Maths — not tied to a specific formula booklet entry. It applies wherever a question chains parts. No out-of-scope content is invoked; students should simply use their calculator's memory/ANS facility and round only when reporting.
