Vectors

Substitute parametric line into plane equation to find intersection point

Family: Parametric Substitution Frequency: 6 questions

Trigger — When to use

A line is given (or easily converted to) parametric/vector form r = a + td, and a plane is given in scalar form n·r = k or Cartesian form n₁x + n₂y + n₃z = k. The question asks for the point where the line meets the plane, or for a quantity that depends on this intersection (e.g. distance from a known point, reflection, foot of perpendicular). Equivalent phrasings: 'find the position vector of the point of intersection', 'find where the line crosses the plane', 'show that the line meets the plane at...'.

Motivation — Why this works

The line has exactly one degree of freedom (the parameter t), while the plane imposes exactly one scalar constraint. Substituting r = a + td into n·r = k therefore collapses the entire 3D geometric problem into a single linear equation in t — guaranteed to have a unique solution unless n·d = 0. This exploits the dot product's linearity (n·(a + td) = n·a + t(n·d)) and is dramatically more efficient than solving a 3×3 simultaneous system or eliminating t between Cartesian forms.

Thought Process — The step-by-step reasoning

  1. Notice the structural mismatch: the line is parametric (one unknown t generates all its points), while the plane gives one scalar equation in x, y, z.
  2. Rule out converting the line to Cartesian symmetric form and solving three simultaneous equations — that creates an over-determined-looking system and wastes algebra.
  3. Rule out trying to equate the vector form of the line with a vector form of the plane component-by-component — this confuses 'lying on' with 'equal to'.
  4. Commit to direct substitution because dotting r = a + td with n yields a single linear equation in t, exploiting the linearity of the dot product.
  5. Before substituting, sanity-check whether n·d = 0; if so, the line is parallel to the plane and one must compare n·a with k to decide between 'no intersection' and 'line lies in plane'.
  6. After solving for t, remember the goal is the point itself: substitute t back into r = a + td to read off the coordinates, and optionally verify by plugging into n·r = k.

Mechanic — Concrete operations

  1. Write the line explicitly as r = a + td, i.e. x = a₁ + td₁, y = a₂ + td₂, z = a₃ + td₃.
  2. Form the dot product n·r and set it equal to k: n·(a + td) = k, i.e. n·a + t(n·d) = k.
  3. Equivalently, in Cartesian form: n₁(a₁ + td₁) + n₂(a₂ + td₂) + n₃(a₃ + td₃) = k.
  4. Solve the resulting linear equation for t: t = (k − n·a) / (n·d), provided n·d ≠ 0.
  5. Substitute this value of t back into r = a + td to obtain the intersection point.
  6. (Optional) Verify by checking that the resulting point satisfies the original plane equation.

Worked Micro-Example

Line: $\mathbf{r} = (1,0,2) + t(2,1,-1)$. Plane: $x + 2y + z = 7$. Substitute: $(1+2t) + 2(t) + (2-t) = 7 \Rightarrow 3 + 3t = 7 \Rightarrow t = \tfrac{4}{3}$. Intersection: $\mathbf{r} = (1 + \tfrac{8}{3},\, \tfrac{4}{3},\, 2 - \tfrac{4}{3}) = (\tfrac{11}{3}, \tfrac{4}{3}, \tfrac{2}{3})$.

Variants — Same move, different surface

Line given in Cartesian symmetric form (x−a)/l = (y−b)/m = (z−c)/n
Set each ratio equal to t to recover parametric form, then substitute identically.
Plane given in vector form r·n = k versus Cartesian n₁x + n₂y + n₃z = k
These are the same equation; pick whichever makes the substitution cleaner — usually the scalar dot-product form.
Two planes are given instead of an explicit plane
First find the line of intersection (cross the normals for direction; solve for one point), then apply the substitution to whichever single plane equation is required.
Question asks for foot of perpendicular or reflection, not intersection
Build the perpendicular line through the given point using the plane's normal as direction, then apply this same substitution to find the foot.

Common Misreadings

  • Treating the line's vector equation and the plane's vector equation as componentwise-equal, producing a contradictory system.
  • Substituting only the direction vector d into the plane equation instead of the full position a + td.
  • Confusing the plane's scalar k with a position vector and trying to equate vectors.

Pitfalls

  • Failing to check n·d: if n·d = 0 the line is parallel to the plane (no intersection, or the entire line lies in the plane when also n·a = k).
  • Arithmetic errors when expanding three bracketed terms n_i(a_i + td_i) simultaneously — collect t-terms and constants separately.
  • Solving for t but forgetting to back-substitute to give the actual point requested.
  • Sign errors when the plane equation is written as n·r − k = 0 versus n·r = k.

A-level Scope Notes

Fully within Edexcel IAL Core/Further Pure (FP3 / P3-P4 vectors). Uses the standard line equation r = a + td and plane equations r·n = k or Cartesian form from the formula booklet. Do not invoke parametric plane forms or higher-dimensional generalisations; stick to 3D.
