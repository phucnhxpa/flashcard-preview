Trigger — When to use

Fires whenever taking a square root of a squared trig expression (e.g. √(sin²2y), √(cos²θ)) introduces a ± that must be resolved before differentiation, integration, or substitution can proceed. Cues: a domain restriction like 0 ≤ y ≤ π/2 has been given but not yet used; the working has just produced sin²(·) = … and the next natural step is √. Also fires when rewriting cos θ as ±√(1−sin²θ) inside a parametric or implicit-derivative chain. If a domain is supplied in a trig question, expect to need it for sign-fixing somewhere.

Motivation — Why this works

√(f²) = |f|, not f, so any square root of a squared quantity carries a ± unless the sign of the inside is pinned down. The given domain on the variable propagates to a domain on the trig argument (e.g. y ∈ [0,π/2] ⟹ 2y ∈ [0,π]), and on that interval the trig function has a definite sign (sin ≥ 0 on [0,π]). Committing to the correct sign keeps dy/dx single-valued and consistent with the geometry of the curve. Edexcel mark schemes explicitly reward stating this justification.

Thought Process — Step-by-step reasoning

  1. Notice: I have just produced sin²2y = (something in x), and the next step is to square-root — that creates a ± in principle.
  2. Rule out blindly writing +√: I need a reason. Scan the question — the domain 0 ≤ y ≤ π/2 has not yet been used, so it is almost certainly the justification.
  3. Commit: translate the domain on y to a domain on the trig argument 2y, then read off the sign of sin on that interval.
  4. Bridge to operation: 2y ∈ [0,π], and sin θ ≥ 0 there, so sin 2y ≥ 0 and the positive root is correct.
  5. Tactical: write the domain chain explicitly in the working ('0 ≤ y ≤ π/2 ⟹ 0 ≤ 2y ≤ π ⟹ sin 2y ≥ 0') so the examiner sees the justification, not just the conclusion.

Mechanic — Concrete operations

  1. Identify the ± introduced by √(sin²2y) = |sin 2y|.
  2. Translate the y-domain to the argument: 0 ≤ y ≤ π/2 ⟹ 0 ≤ 2y ≤ π.
  3. State the sign of sin on that interval: sin θ ≥ 0 for θ ∈ [0,π].
  4. Conclude |sin 2y| = sin 2y, so sin 2y = +√[(x+1)(3−x)]/2.
  5. Carry the positive root forward into dy/dx and check it is consistent with the monotonicity of the curve.

Worked Micro-Example

Given $\sin^2 2y = \tfrac{1}{4}(x+1)(3-x)$ with $0 \le y \le \tfrac{\pi}{2}$. Then $0 \le 2y \le \pi$, and $\sin\theta \ge 0$ on $[0,\pi]$, so $\sin 2y \ge 0$. Hence $\sin 2y = +\tfrac{1}{2}\sqrt{(x+1)(3-x)}$, and differentiating implicitly gives a well-defined positive $\dfrac{dy}{dx}$.

Variants — Same move, different surface

Rewriting cos θ = ±√(1−sin²θ) inside an implicit/parametric derivative
Use the given range of θ to decide the sign of cos: cos ≥ 0 on [−π/2, π/2], cos ≤ 0 on [π/2, 3π/2].
Inverse-trig simplification cos(arcsin x) or sin(arccos x)
The principal-value range of arcsin/arccos pins the sign automatically: cos(arcsin x) = +√(1−x²) since arcsin x ∈ [−π/2, π/2].
Polar/parametric curves with a stated parameter range
Translate the parameter range to the argument of the trig function and read off the sign on that subinterval.
Definite integrals using x = a sin θ or x = a tan θ substitution
The chosen θ-range (e.g. θ ∈ [−π/2, π/2]) is what justifies √(a²−x²) = a cos θ with the positive sign.

Common Misreadings

  • Treating √(sin²2y) as simply sin 2y by reflex, without noting the modulus issue.
  • Using the range of y instead of the range of 2y to decide the sign.
  • Assuming the positive root is 'obvious' and omitting any written justification.
  • Confusing the sign of sin 2y with the sign of cos 2y on the same interval.

Pitfalls

  • Writing ± in the final dy/dx and leaving the answer ambiguous.
  • Forgetting to scale the interval (using 0 ≤ y ≤ π/2 to claim sin y ≥ 0 when the expression involves sin 2y).
  • Picking the wrong sign when the trig argument crosses a zero of the function inside the stated interval — the move only works cleanly when the sign is constant on the whole interval.
  • Stating the conclusion without the intermediate step '2y ∈ [0,π]', which is what earns the justification mark.

A-level Scope Notes

Fully within Edexcel IAL Pure (P3/P4 implicit and parametric differentiation, trig substitution in integration) and Further Pure where inverse-trig identities arise; no analysis beyond standard sine/cosine sign behaviour on subintervals of [0, 2π] is needed.
