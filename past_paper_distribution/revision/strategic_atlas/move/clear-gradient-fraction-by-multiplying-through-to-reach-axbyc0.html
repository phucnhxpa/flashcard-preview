Coordinate Geometry

Clear gradient fraction by multiplying through to reach ax+by+c=0

Family: Integer Coefficient Presentation Frequency: 1 questions

Trigger — When to use

Fires whenever a straight-line equation has been written in point-slope form y − y₁ = m(x − x₁) with m a rational number p/q, and the question asks for the line 'in the form ax + by + c = 0' (or 'with integer coefficients', or 'where a, b, c are integers'). Common cues: a perpendicular/normal gradient that is the negative reciprocal of a fraction, a tangent gradient from differentiation that simplifies to p/q, or a chord/perpendicular bisector through given points. Also fires when the equation is already in y = mx + c form with fractional m or c and the final presentation must be integer.

Motivation — Why this works

Multiplying both sides by the denominator q at the point-slope stage eliminates the fraction in a single, sign-safe step before any expansion, so every subsequent term is automatically an integer. Distributing the fraction first risks fractional intermediate terms and arithmetic slips. The form ax + by + c = 0 is Edexcel's preferred presentation and the mark scheme typically reserves the final accuracy mark for it, often with a, b, c required to be integers (sometimes with a > 0 or no common factor).

Thought Process — The step-by-step reasoning

  1. Notice the gradient is a fraction p/q, so point-slope form will carry a denominator q.
  2. Rule out expanding first: distributing −(p/q) over (x − x₁) would introduce fractional terms unnecessarily.
  3. Commit to clearing the denominator immediately by multiplying both sides by q before any expansion.
  4. Bridge: write q·(y − y₁) = ±p·(x − x₁) as the next line, applying q to the whole left side as a bracket.
  5. Tactical: expand both sides, then move every term to one side to land on ax + by + c = 0; finally check the leading coefficient sign convention and that gcd(a,b,c) = 1.

Mechanic — Concrete operations

  1. Start from y − y₁ = (p/q)(x − x₁) with the gradient already simplified.
  2. Multiply both sides by q: q(y − y₁) = p(x − x₁).
  3. Expand each side: qy − qy₁ = px − px₁.
  4. Rearrange to collect all terms on one side: −px + qy + (px₁ − qy₁) = 0, or equivalently px − qy + (qy₁ − px₁) = 0.
  5. Tidy: ensure integer coefficients, optionally make the x-coefficient positive, and divide out any common factor.

Worked Micro-Example

Find the normal to y = f(x) at (2, 3) where the tangent gradient is 5/4, in the form ax + by + c = 0.\\ Normal gradient = −4/5, so y − 3 = −\tfrac{4}{5}(x − 2).\\ Multiply by 5: 5(y − 3) = −4(x − 2) ⇒ 5y − 15 = −4x + 8.\\ Rearrange: 4x + 5y − 23 = 0.

Variants — Same move, different surface

Gradient is −2/3 from a perpendicular calculation
Multiply both sides by 3 instead of 5; same procedure with denominator 3.
Equation written as y = mx + c with fractional m and c (different denominators)
Multiply through by the LCM of the denominators, then rearrange to ax + by + c = 0.
Perpendicular bisector of two points giving gradient p/q through midpoint (x_m, y_m)
Apply the same multiply-by-q step at the point-slope stage using the midpoint coordinates.
Question asks for y = mx + c form instead
Skip the integer rearrangement; expand and isolate y, leaving fractional coefficients if necessary.
Mark scheme requires a > 0 or gcd(a,b,c)=1
Multiply by −1 if the x-coefficient is negative, and divide through by any common factor at the end.

Common Misreadings

  • Treating 'integer coefficients' as optional and leaving the answer as y = −(4/5)x + 23/5.
  • Reading ax + by + c = 0 as requiring a specific sign for a even when the question does not specify.
  • Believing the fraction must be expanded before clearing, leading to fractional intermediate steps.

Pitfalls

  • Forgetting to multiply every term on one side by q (commonly the constant −y₁ is missed).
  • Sign errors when distributing −p across (x − x₁), especially when x₁ is negative.
  • Sign errors when transposing terms to achieve '= 0'.
  • Leaving a common factor (e.g., 8x + 10y − 46 = 0 instead of 4x + 5y − 23 = 0).
  • Using an unsimplified gradient like −8/10, which forces an unnecessarily large multiplier.

A-level Scope Notes

Entirely within Edexcel IAL Pure (P1/P2 straight-line coordinate geometry, and tangent/normal applications in P1–P4 differentiation). No content beyond standard A-level techniques is required.
