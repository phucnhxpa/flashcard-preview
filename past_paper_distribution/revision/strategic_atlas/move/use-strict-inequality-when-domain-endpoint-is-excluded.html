Algebra & Functions

Use strict inequality when domain endpoint is excluded

Family: Endpoint Exclusion Analysis Frequency: 1 questions

Trigger — When to use

Fires whenever a function's domain is given with strict inequality (e.g. x > a) rather than x ≥ a, and you are asked for the range, the number of solutions of f(x) = k, or the values of k for which a horizontal line meets the graph a given number of times. Also fires when sketching piecewise/modulus graphs where one arm 'starts' at an open dot. Cue phrases: 'for x > 0', 'on the interval (a, b]', 'find the values of k such that …'.

Motivation — Why this works

An excluded endpoint means the limiting value is a supremum/infimum but not attained, so the function never equals that boundary value. Treating the endpoint as included would falsely add a value to the range and shift a strict inequality on k into a non-strict one. Edexcel mark schemes consistently award separate marks for the correct strict-vs-non-strict distinction, so this is a real mark-earning detail rather than cosmetic.

Thought Process — The step-by-step reasoning

  1. Notices the domain is written x > 0 (open) rather than x ≥ 0 (closed) — flags this immediately as a strict-vs-non-strict issue.
  2. Rules out the lazy reading 'x > 0 effectively means x ≥ 0' and refuses to plug x = 0 into f as if it were a legitimate input.
  3. Commits to treating f(0) as a limit value only: a supremum the function approaches but never reaches.
  4. Bridges to operation by computing the limit lim_{x→0⁺} f(x) to identify the excluded boundary value of y.
  5. Decides which inequality on k is strict: any horizontal line at the excluded y-value loses an intersection from that arm, so the corresponding bound on k must be strict.

Mechanic — Concrete operations

  1. Read the domain carefully and mark which endpoints are open vs closed.
  2. Compute f at each open endpoint as a limit, e.g. f(0⁺) = 2|3 − 0| + 5 = 11.
  3. State explicitly: f(x) < 11 on the open side, with f(x) → 11 as x → 0⁺.
  4. Translate to the k-question: a horizontal line y = k meets the open arm only when k is strictly less than the boundary value.
  5. Combine with the closed-end behaviour and write the final answer with the correct mix of < and ≤.

Worked Micro-Example

Let f(x) = 2|3 − x| + 5 for 0 < x ≤ 3. Then f(0⁺) → 11 (not attained) and f(3) = 5. On the left arm f decreases from 11 (excluded) to 5; on the right arm f increases from 5. For f(x) = k to have two roots we need k to hit both arms, giving 5 < k < 11 (strict at 11 because x = 0 is excluded).

Variants — Same move, different surface

Half-open interval [a, b) in a quadratic range question
Include f(a) in the range but exclude f(b); the bound from the b-side becomes strict.
Rational function with x ≠ c (point excluded mid-domain)
Compute the limit as x → c; the corresponding y-value is removed from the range, often producing a 'hole' that forces strict inequality on k.
Piecewise function defined with open dot on one branch
The branch's endpoint y-value is a sup/inf only; intersections counted by horizontal-line test exclude that y.
Inverse function domain question
Range of f becomes domain of f⁻¹; open endpoints in range of f transfer to open endpoints in domain of f⁻¹.
Modulus equation |g(x)| = k on a restricted domain
Same logic: count intersections of y = k with the graph, respecting which endpoints are dots vs filled.

Common Misreadings

  • Reading x > 0 as x ≥ 0 and substituting x = 0 directly into f.
  • Believing 'supremum' and 'maximum' are interchangeable on an open interval.
  • Assuming the answer must be a single inequality (e.g. k < 11) and forgetting the lower bound from the closed end.
  • Thinking the open/closed distinction only matters for sketching, not for the algebraic answer.

Pitfalls

  • Writing k ≤ 11 instead of k < 11 — a standard one-mark loss in Edexcel mark schemes.
  • Mixing up which side is strict (e.g. writing 5 ≤ k ≤ 11 when only the upper bound should be strict).
  • Stating f(0) = 11 as if it were a value of the function rather than a limit.
  • Forgetting to check the closed endpoint as well, leading to an incomplete range.

A-level Scope Notes

Fully within Edexcel IAL Pure (functions, modulus, range/domain, graph transformations); uses only one-sided limit reasoning at an informal level, not formal limit definitions.
