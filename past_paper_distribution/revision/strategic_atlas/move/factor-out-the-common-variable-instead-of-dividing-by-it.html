Algebra & Functions

Factor out the common variable instead of dividing by it

Family: Zero Product Property Frequency: 1 questions

Trigger — When to use

An equation has been rearranged to the form f(x) = 0 where every term shares a common variable factor, e.g. 5b² − 81b = 0, sin(x) − sin(x)cos(x) = 0, or x³ − 4x = 0. There is a strong temptation to 'simplify' by dividing both sides by the common variable factor. Cue phrases: 'every term contains x', 'cancel the x', 'divide through by sin x'. The move fires the moment you spot a shared variable factor across all terms of an equation equal to zero.

Motivation — Why this works

Dividing both sides of an equation by a variable expression silently assumes that expression is non-zero, which discards the root where it equals zero. Factoring and applying the zero-product property (if AB = 0 then A = 0 or B = 0) preserves every solution. Even when the lost root will later be rejected on domain grounds (e.g. log domain, or a physical constraint), the disciplined procedure is to enumerate all algebraic solutions first and reject afterwards — examiners reward this and penalise silent loss of roots.

Thought Process — The step-by-step reasoning

  1. I notice every term on the LHS shares a variable factor (e.g. b appears in both 5b² and 81b).
  2. I rule out dividing by b, because that would assume b ≠ 0 and silently delete the root b = 0 — even if b = 0 is later inadmissible, I should not lose it tacitly.
  3. I commit to factoring the common variable out, because the equation is already set equal to zero and the zero-product property will then split it cleanly.
  4. I write the equation as (common factor) × (remaining bracket) = 0, ready to split into two simpler equations.
  5. After solving each branch, I return to the original problem (domain, log/trig/physical constraints) to reject any inadmissible roots, keeping the rejection visible in the working.

Mechanic — Concrete operations

  1. Rearrange the equation to the form f(x) = 0.
  2. Identify the common variable factor across all terms.
  3. Factor it out: write as x · g(x) = 0.
  4. Apply zero-product property: x = 0 or g(x) = 0.
  5. Solve each branch independently.
  6. Return to the original context and reject any roots violating domain or stated constraints, stating the reason.

Worked Micro-Example

Solve $5b^2 - 81b = 0$. Factor: $b(5b - 81) = 0$. Zero-product: $b = 0$ or $5b - 81 = 0$, giving $b = 0$ or $b = \tfrac{81}{5}$. If this arose from $\log b$, reject $b = 0$ (outside domain), leaving $b = \tfrac{81}{5}$.

Variants — Same move, different surface

Trigonometric equation such as $\sin x \cos x = \sin x$ on $[0, 2\pi]$
Rearrange to $\sin x(\cos x - 1) = 0$; never divide by $\sin x$. Solve $\sin x = 0$ and $\cos x = 1$ separately, listing all roots in the interval.
Cubic with no constant term, e.g. $x^3 - 4x = 0$
Factor $x(x^2 - 4) = 0$ then $x(x-2)(x+2) = 0$, giving three roots; dividing by $x$ would lose $x = 0$.
Exponential equation like $x e^{x} - 3 e^{x} = 0$
Factor $e^{x}(x - 3) = 0$. Here $e^x \neq 0$ for all real $x$, so it is safe to discard that branch — but state the reason rather than dividing silently.
Parametric/quadratic-in-disguise like $t^2 \sin\theta - t \sin\theta = 0$ for $t$
Factor $t \sin\theta (t - 1) = 0$; treat each factor as a separate case rather than cancelling $\sin\theta$.

Common Misreadings

  • Treating 'divide both sides by b' as algebraically equivalent to factoring — it is not, because division presumes the divisor is non-zero.
  • Believing that since b = 0 will be rejected anyway (e.g. log domain), it is acceptable to skip it — examiners want the rejection shown explicitly.
  • Confusing the zero-product property with the no-zero-divisor reasoning used when cancelling: AB = AC does not generally give B = C unless A ≠ 0.

Pitfalls

  • Dividing through by the variable and losing a root.
  • Factoring correctly but then forgetting to write 'or' between the two cases, solving only one branch.
  • Failing to justify rejection of an inadmissible root, costing a reasoning mark.
  • Mis-factoring sign, e.g. writing $b(5b + 81) = 0$ instead of $b(5b - 81) = 0$.

A-level Scope Notes

Fully within Edexcel IAL Pure Maths (P1–P4) and Further Pure: zero-product property, polynomial factoring, and trig/log/exponential equation solving. No tools beyond standard A-level algebra are required.
