Coordinate Geometry

Apply point-gradient form then clear the fractional gradient to obtain integer coefficients

Family: Line Equation Construction Frequency: 6 questions

Trigger — When to use

Fires when you have (i) a known point (x₁, y₁) on a line, (ii) a fractional gradient m = p/q with q > 1, and (iii) a final-form requirement such as 'ax + by + c = 0 with a, b, c integers' or simply a clean tidy equation. Equivalent phrasings: 'Find the equation of the tangent/normal/line through ... in the form ax + by + c = 0', 'Show that the line has equation ...', or any tangent/normal problem where differentiation has produced a rational gradient. The cue is the combination of a fractional m and a request for integer-coefficient output.

Motivation — Why this works

Point-gradient form y − y₁ = m(x − x₁) is the most direct route from (point, gradient) to a line equation — unlike y = mx + c, it requires no separate computation of the intercept. When m = p/q is fractional, multiplying both sides of the equation by q in one stroke clears the only denominator, so every subsequent expansion involves only integers. Rearranging then trivially produces the required ax + by + c = 0 form. This avoids the error-prone alternative of expanding fractions term-by-term and clearing them later.

Thought Process — The step-by-step reasoning

  1. Notice the inputs: a specific point on the line and a fractional gradient p/q (often coming from differentiation or a perpendicular-gradient calculation), together with a final-form instruction asking for integer coefficients.
  2. Rule out y = mx + c: it requires solving for c by substitution, which is an extra arithmetic step on the same fraction — no benefit, more risk.
  3. Rule out trying to simplify the fraction or expanding before clearing: handling p/q across two terms multiplies the chance of arithmetic slips.
  4. Commit to point-gradient form because it loads both pieces of given data into one equation immediately, with the fraction confined to a single coefficient on the right.
  5. Bridge to action: write y − y₁ = (p/q)(x − x₁), then immediately multiply both sides by q (the denominator of the gradient) before expanding anything — this is the key tactical decision.
  6. Expand both sides cleanly with integer arithmetic, then move all terms to one side to match the required ax + by + c = 0 form, optionally adjusting signs so a > 0 if the mark scheme prefers it.

Mechanic — Concrete operations

  1. Write point-gradient form: y − y₁ = (p/q)(x − x₁).
  2. Multiply both sides by q: q(y − y₁) = p(x − x₁).
  3. Expand each side: qy − qy₁ = px − px₁.
  4. Rearrange to one side: px − qy + (qy₁ − px₁) = 0.
  5. Tidy signs/order so the equation reads ax + by + c = 0 with a, b, c integers (and a > 0 if convention requires).

Worked Micro-Example

Given gradient $m = -\tfrac{8}{9}$ and point $(-1, 3)$: write $y - 3 = -\tfrac{8}{9}(x + 1)$. Multiply by 9: $9(y-3) = -8(x+1)$, i.e. $9y - 27 = -8x - 8$. Rearrange: $8x + 9y - 19 = 0$.

Variants — Same move, different surface

Gradient is an integer (e.g. m = 4)
No fraction-clearing needed; point-gradient form already gives integer coefficients after expansion. Skip the multiply-through step.
Negative fractional gradient, e.g. m = −3/5 at (1, 2)
Same procedure: y − 2 = (−3/5)(x − 1), multiply by 5 → 5y − 10 = −3x + 3, rearrange to 3x + 5y − 13 = 0. Watch the sign on distribution.
Final form requested is y = mx + c instead of ax + by + c = 0
Skip the multiply-through; instead expand and isolate y to read off c directly.
Normal line rather than tangent
Use m_normal = −1/m_tangent first, then apply the same point-gradient + clear-fraction routine.

Common Misreadings

  • Treating the fractional gradient as a hint to use y = mx + c, then struggling with a fractional c.
  • Leaving the answer in unsimplified point-gradient form y − y₁ = (p/q)(x − x₁) when the mark scheme demands ax + by + c = 0 with integer coefficients.

Pitfalls

  • Sign error when distributing through (x − x₁) or (x + x₁), e.g. writing −8(x + 1) = −8x + 8 instead of −8x − 8.
  • Forgetting to multiply EVERY term by the denominator, leaving stray fractions on one side.
  • Arithmetic slip when combining the two constant terms during the final rearrangement (e.g. −2391 − 32 should give −2423).
  • Submitting with a < 0 when the mark scheme prefers a positive leading coefficient — multiply through by −1 if needed.

A-level Scope Notes

Fully within Edexcel IAL Pure Maths P1/P2 (straight lines) and P3/P4 (tangents and normals from differentiation). No formula-booklet entry is required — point-gradient form y − y₁ = m(x − x₁) is assumed knowledge. Do not generalise to non-linear 'tangent equations' in implicit form without first computing dy/dx at the specific point.
