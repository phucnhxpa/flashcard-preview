Algebra & Functions

Map labelled key points through the transformation rule

Family: Graph Transformation Analysis Frequency: 1 questions

Trigger — When to use

Fires when a sketch of y = f(x) is supplied with specific labelled points (axis intercepts, turning points, or touching points) and follow-up parts ask to sketch y = f(x+a), y = f(-x), y = f(x)+k, y = -f(x), y = af(x) or y = f(ax). Cues include phrases like 'on separate diagrams sketch...', 'showing the coordinates of points where the curve meets the axes', or labelled features such as '(2,0) is a point where the curve touches the x-axis'. The task is transformation, not re-derivation.

Motivation — Why this works

Each standard transformation is a coordinate map applied uniformly to every point on the curve, so the entire sketch is determined by where the labelled key points go plus preservation of qualitative shape (crossing vs touching, increasing/decreasing, asymptote behaviour). Inside-the-function changes act on x-coordinates in the inverse sense (x → x − a for f(x+a); x → −x for f(−x); x → x/a for f(ax)); outside-the-function changes act directly on y-coordinates. Mapping the few marked points captures all the marks the examiner is testing.

Thought Process — The step-by-step reasoning

  1. Notice that the question supplies a finite set of labelled points and a qualitative shape — this is the signature of a transformation-of-graph item, not a curve-sketching-from-scratch item.
  2. Rule out re-finding roots from a formula or recomputing y-values: the function f is not given explicitly, so only the transformation rule matters.
  3. Commit to the move because each labelled point carries a mark and the new sketch is fully determined by relocating those points and copying the shape between them.
  4. Bridge to operations: identify whether each transformation is inside or outside the function, then write down the coordinate map (x,y) → (x',y') before touching the diagram.
  5. Tactically, also flag any point lying on the axis of symmetry of a reflection (it is fixed) and any 'touching' point so the new sketch preserves repeated-root behaviour at the image.

Mechanic — Concrete operations

  1. Classify each requested transformation as inside-argument (horizontal) or outside (vertical), and write the explicit coordinate map.
  2. For y = f(x+2): use (x,y) → (x−2, y); for y = f(−x): use (x,y) → (−x, y).
  3. Apply the map to every labelled point, e.g. (−5,0)→(−7,0), (0,−3)→(−2,−3), (2,0)→(0,0) under f(x+2).
  4. Redraw the curve through the image points preserving original shape, monotonicity, curvature, and crossing-vs-touching behaviour at the x-axis.
  5. Label every image point with its new coordinates on the sketch.

Worked Micro-Example

Given y = f(x) with labelled points $(-5,0)$, $(0,-3)$, $(2,0)$ where $(2,0)$ is a touching point. For $y = f(x+2)$ apply $(x,y)\mapsto(x-2,y)$: points become $(-7,0),(-2,-3),(0,0)$, with the curve still touching the x-axis at $(0,0)$. For $y = f(-x)$ apply $(x,y)\mapsto(-x,y)$: points become $(5,0),(0,-3),(-2,0)$, with $(0,-3)$ fixed since it lies on the y-axis.

Variants — Same move, different surface

y = f(x) + k (outside vertical translation)
Map (x,y) → (x, y+k); x-coordinates of intercepts/turning points are unchanged, but x-axis crossings generally move off the axis.
y = -f(x) (reflection in x-axis)
Map (x,y) → (x, −y); points on the x-axis stay fixed, touching/crossing behaviour at x-axis preserved but flipped above/below.
y = a f(x) with a > 0 (vertical stretch)
Map (x,y) → (x, ay); x-intercepts fixed, y-values (including minima/maxima) scale by a.
y = f(ax) with a > 0 (horizontal stretch)
Map (x,y) → (x/a, y); y-intercept fixed, x-values scale by 1/a.
Combined y = f(2x+4)
Factor as f(2(x+2)): either translate by −2 then horizontally compress by factor 1/2, or compress first then translate by −2 — order matters, so be explicit.
y = |f(x)| or y = f(|x|) (modulus graphs, FP/IAL)
For |f(x)|, reflect the portion below the x-axis upwards; for f(|x|), discard x<0 part and reflect the x≥0 part in the y-axis.

Common Misreadings

  • Reading f(x+2) as a shift RIGHT by 2 instead of LEFT by 2 (inverting the inside-rule).
  • Reading f(−x) as a reflection in the x-axis (confusing it with −f(x)) and negating y-coordinates instead of x-coordinates.
  • Treating f(2x) as a horizontal stretch by factor 2 rather than factor 1/2.
  • Applying combined transformations in the wrong order without factorising the argument first.

Pitfalls

  • Failing to relabel image points on the new sketch — examiners mark per labelled coordinate.
  • Losing the 'touching' vs 'crossing' distinction at the x-axis after transformation.
  • Forgetting that a point on the axis of symmetry of a reflection is fixed (e.g. y-intercept under f(−x)).
  • Drawing the new curve with the wrong overall shape — the qualitative features (turning points, asymptotes) must be carried over.

A-level Scope Notes

Fully within Edexcel IAL Pure (graph transformations of y=f(x), including translations, reflections and stretches) and extends naturally to modulus graphs in Further Pure; no calculus or unknown function form required.
