Integration

Map Cartesian region boundaries to parameter t-limits

Family: Parametric Limit Finding Frequency: 1 questions

Trigger — When to use

Fires when a region R is described geometrically in Cartesian terms (e.g. bounded by the curve, the x-axis, and the y-axis, or between two axis intercepts), but the curve is given parametrically as x = f(t), y = g(t) and an area/volume must be computed via ∫ y (dx/dt) dt. The parameter range stated in the question (e.g. 0 ≤ t ≤ π/2) is typically larger than what R requires. Cue phrases: 'the region bounded by C and the coordinate axes', 'finite region enclosed by C and the x-axis', 'shaded region in the first quadrant'.

Motivation — Why this works

A parametric area integral integrates over the t-values that trace the boundary arc of R, not the entire parameter domain of the curve. Each Cartesian boundary (x = 0, y = 0, or another line) corresponds to a specific t-value where the curve meets it; these are the true limits of integration. Solving x(t) = 0 and y(t) = 0 (or x(t) = a, y(t) = b for other boundaries) within the allowed t-range pins down those endpoints exactly.

Thought Process — The step-by-step reasoning

  1. Notice that R is described by Cartesian boundaries (axes / lines), so the integration limits in t are not handed over directly — they must be deduced.
  2. Rule out using the full given t-range as limits: that would include parts of the curve outside R.
  3. Commit to finding t-values where the curve meets each Cartesian boundary, since these mark where the traced arc enters and leaves R.
  4. Bridge: set y(t) = 0 to find where the curve hits the x-axis, and x(t) = 0 to find where it hits the y-axis (and check each solution lies in the stated t-range).
  5. Tactically, back-substitute each candidate t into the other coordinate to confirm which axis it sits on, then order the two t-values to give the lower and upper limits.

Mechanic — Concrete operations

  1. Identify each Cartesian boundary curve of R (typically x = 0 and/or y = 0).
  2. For each boundary, solve the corresponding equation in t (e.g. y(t) = 0, x(t) = 0) within the stated t-domain.
  3. Substitute each t-solution back into the other coordinate to confirm the intersection point.
  4. Discard t-values outside the given range or not on the boundary of R.
  5. Order the surviving t-values as the lower and upper limits for the integral.

Worked Micro-Example

Curve: $x = 2\cos 2t$, $y = 4\sin t$, $0 \le t \le \pi/2$; R is bounded by C and the axes. Set $y = 0$: $4\sin t = 0 \Rightarrow t = 0$, giving $(2,0)$. Set $x = 0$: $\cos 2t = 0 \Rightarrow t = \pi/4$, giving $(0, 2\sqrt{2})$. Hence area $= \int_{0}^{\pi/4} y\,\frac{dx}{dt}\,dt$.

Variants — Same move, different surface

Region bounded by curve and the x-axis only (loop or arch)
Solve y(t) = 0 for two t-values; these become the limits, no x = 0 condition is needed.
Region bounded by curve and a vertical line x = a
Replace x(t) = 0 by x(t) = a; the other endpoint still comes from y(t) = 0 or a stated point.
Volume of revolution about the x-axis from a parametric curve
Same boundary logic: find t-limits from where the curve meets the axis or stated x-values, then use V = ∫ π y² (dx/dt) dt.
Curve crosses an axis multiple times in the given t-range
Pick the pair of consecutive t-values that bound the specific region described; reject solutions on the wrong arc.
Region bounded by curve and y-axis with integration in y (dy/dt)
Use ∫ x (dy/dt) dt; limits come from where the curve meets the y-axis (x = 0) and the other stated boundary.

Common Misreadings

  • Treating the question's stated parameter range (e.g. 0 ≤ t ≤ π/2) as the integration limits automatically.
  • Assuming the curve enters R at t = 0 without checking that (x(0), y(0)) actually lies on a boundary of R.
  • Confusing which axis a given t corresponds to and pairing the limits in the wrong order.

Pitfalls

  • Picking a t-solution outside the stated range (e.g. t = π when only [0, π/2] is allowed).
  • Not back-substituting to check which Cartesian point the t-value produces, leading to a wrong endpoint.
  • Reversing the limits and obtaining a negative area without recognising the sign issue.
  • Forgetting that dx/dt may change sign between the limits, requiring a sign check on the area.

A-level Scope Notes

Standard Edexcel IAL parametric area technique (C34/FP-level): solving simple trig or polynomial equations in t and using ∫ y (dx/dt) dt with appropriately deduced limits.
