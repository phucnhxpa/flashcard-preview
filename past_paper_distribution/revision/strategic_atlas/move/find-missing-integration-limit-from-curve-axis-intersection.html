Integration

Find missing integration limit from curve-axis intersection

Family: Region Identification Frequency: 1 questions

Trigger — When to use

Fires when a definite integral (area or volume of revolution) is set up against the x-axis but only one bound is supplied explicitly. The other bound must come from solving y = 0 for the given curve. Cues include phrases like 'the region bounded by the curve and the x-axis between ... and x = a', or a diagram showing the curve meeting the axis at an unspecified point. Especially common when the curve involves ln, e^x, or trig factors whose zeros are not obvious from inspection.

Motivation — Why this works

A region bounded by a curve and the x-axis terminates exactly where the curve crosses (or touches) y = 0. Solving y(x) = 0 within the curve's natural domain pinpoints those crossings, giving valid integration limits. Respecting the domain (e.g. x > 0 for ln x) eliminates spurious roots and avoids creating an improper integral. Without the correct lower/upper limit, both the integrand's setup and the final numerical answer are wrong.

Thought Process — The step-by-step reasoning

  1. Notice that the region is bounded by the curve and the x-axis, so a missing limit must be a zero of y.
  2. Rule out roots that lie outside the curve's domain (e.g. x = 0 for y = x ln x, since ln x requires x > 0).
  3. Commit to solving y = 0 algebraically using factor structure (product = 0 ⇒ each factor = 0).
  4. Bridge: substitute the candidate root back to confirm it lies on the x-axis and matches the diagram.
  5. Tactical check: verify the sign of y on the interval to ensure the curve stays on one side of the axis, so the area/volume integral is unambiguous.

Mechanic — Concrete operations

  1. Set y = 0 for the given curve.
  2. Factorise or apply domain restrictions to discard roots outside the valid domain.
  3. Solve the remaining equation for x to get the missing limit.
  4. Pair with the supplied bound, ordering them as lower < upper.
  5. Confirm y has constant sign on [lower, upper] so the region is well-defined.

Worked Micro-Example

Curve $y = x \ln x$, region bounded by the curve, the x-axis, and $x = e$. Set $x \ln x = 0$; since $x > 0$, $\ln x = 0 \Rightarrow x = 1$. Check: at $x=1$, $y=0$; at $x=e$, $y=e>0$. Limits: $x=1$ to $x=e$, giving $V = \pi \int_{1}^{e} (x \ln x)^2 \, dx$.

Variants — Same move, different surface

Curve y = (x − a)(x − b) bounded by x-axis only
Both limits come from y = 0; take the two real roots a, b as lower and upper.
Parametric curve x = f(t), y = g(t)
Solve g(t) = 0 for t-values, then read off corresponding x = f(t) for the x-limits (or integrate in t directly).
Region between two curves y₁ and y₂
Limits come from y₁ = y₂ rather than from y = 0.
Curve involving e^x or trig factor (e.g. y = e^x sin x)
Use the factor that can vanish (sin x = 0) since e^x > 0; pick the root in the relevant interval.

Common Misreadings

  • Treating x = 0 as a valid limit when the curve's domain excludes it (e.g. for x ln x).
  • Reading the missing limit off the diagram by eye instead of solving y = 0 algebraically.
  • Assuming the curve crosses the axis when it only touches it (repeated root), missing that the region may be on one side only.

Pitfalls

  • Swapping lower and upper limits, producing a sign error in the integral.
  • Failing to check that y ≥ 0 (or y² well-defined) on the whole interval, so the integral does not represent the intended region.
  • Including extraneous roots from factorisation that lie outside the curve's domain.
  • Setting up an improper integral unnecessarily by using a domain-boundary point as a limit.

A-level Scope Notes

Standard Edexcel IAL P3/P4 technique for areas and volumes of revolution; only requires solving y = 0 with elementary algebra and respecting the natural domain of ln, √, or rational expressions.
