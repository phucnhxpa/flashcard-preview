Integration

Determine trapezium-rule strip width h from a table of x-values

Family: Parameter Extraction Frequency: 1 questions

Trigger — When to use

A trapezium-rule question supplies a table of (x, y) values with uniformly spaced x-values, and the formula requires h as the leading multiplier. The cue is any phrasing like 'use the trapezium rule with n strips', '5 ordinates', or 'the values in the table'. It also fires when h is not stated explicitly and must be inferred from the tabulated x-column. Reading h correctly is a prerequisite before any y-value substitution.

Motivation — Why this works

In the trapezium rule $T = \tfrac{h}{2}\big[y_0 + y_n + 2(y_1+\dots+y_{n-1})\big]$, the factor h converts a weighted sum of ordinates into an area estimate. Any error in h scales the entire answer multiplicatively, so it must be pinned down first. Because the table lists x-values explicitly, h equals the common gap $x_1 - x_0$, equivalently $(b-a)/n$. Anchoring h to the table — rather than to the count of rows — eliminates the classic ordinates-versus-strips slip.

Thought Process — The step-by-step reasoning

  1. I notice the table lists evenly spaced x-values from a starting value a to a final value b, so h must be the common gap.
  2. I rule out simply counting table entries as the number of strips: the number of ordinates is one more than the number of strips.
  3. I commit to computing h either as $(b-a)/n$ or as the difference of two consecutive x-values, since both must agree.
  4. I read off $x_0 = a$ and $x_n = b$ from the table and identify n as (number of ordinates) − 1.
  5. I cross-check by verifying $x_1 - x_0 = x_2 - x_1$ to confirm uniform spacing before locking in h.

Mechanic — Concrete operations

  1. Count the ordinates (y-values) in the table; call this $n+1$, so the number of strips is n.
  2. Identify $a = x_0$ (first x-value) and $b = x_n$ (last x-value).
  3. Compute $h = (b - a)/n$, or equivalently $h = x_1 - x_0$.
  4. Verify uniform spacing by checking $x_2 - x_1 = x_1 - x_0$.
  5. Record h to be substituted into $\tfrac{h}{2}[\dots]$ in the next stage.

Worked Micro-Example

Table gives x = 1, 1.25, 1.5, 1.75, 2 with corresponding y-values. There are 5 ordinates, so $n = 4$ strips. Then $h = (2-1)/4 = 0.25$, matching $x_1 - x_0 = 1.25 - 1 = 0.25$. So the trapezium rule will use $T = \tfrac{0.25}{2}\big[y_0 + y_4 + 2(y_1+y_2+y_3)\big]$.

Variants — Same move, different surface

Question states 'using 5 ordinates' rather than 'using 4 strips'
Convert: ordinates = strips + 1, so 5 ordinates means $n = 4$ and $h = (b-a)/4$.
h is given directly in the stem (e.g. 'with h = 0.5')
Skip the derivation but still verify it is consistent with the tabulated x-values.
Only a and b and n are given, no full table
Compute $h = (b-a)/n$ directly; uniform spacing is implicit.
Integral involves a substitution producing a new variable u with its own limits
Re-derive h from the new u-limits and the requested number of strips, not from the original x-limits.

Common Misreadings

  • Treating the number of table entries as the number of strips (5 ordinates ⇒ 5 strips), giving h too small by a factor.
  • Using $h = b - a$ instead of $(b-a)/n$ when $n > 1$.
  • Reading h from a non-adjacent pair of x-values (e.g. $x_2 - x_0$) and getting double the true h.

Pitfalls

  • Forgetting to check uniform spacing — a misprinted or misread x-value can go undetected if h is taken from only the first pair.
  • Carrying h through with rounding too early (e.g. writing $h \approx 0.33$ instead of $1/3$), which corrupts the final area.
  • Mixing up h with $h/2$ when substituting into the formula.

A-level Scope Notes

Fully within Edexcel IAL P-level material: the trapezium rule with uniformly spaced ordinates is a standard numerical-integration tool, and h is always either stated or derivable from the given x-values.
