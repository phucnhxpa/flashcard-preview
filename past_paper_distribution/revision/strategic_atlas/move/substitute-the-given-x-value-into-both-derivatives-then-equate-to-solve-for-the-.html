Differentiation

Substitute the given x-value into both derivatives, then equate to solve for the parameter

Family: Parameter Determination Frequency: 1 questions

Trigger — When to use

Fires when a problem states a condition linking two derivative expressions (or a function value and a derivative) at a specific x-value, with an unknown parameter k embedded in f. Typical phrasings include 'given that f'(a) = f''(a)', 'the gradient at x = a equals the rate of change of the gradient at x = a', or 'f(a) and f'(a) satisfy [relation]'. The key cue is that x is pinned to a numerical value while a constant inside f is unknown.

Motivation — Why this works

Substituting the fixed x first reduces each derivative expression to a linear (or low-degree) expression in the parameter k alone. Equating these collapses a condition that looks like an equation between two functions into one elementary equation in one unknown. This works because differentiation and evaluation commute with the parameter — k passes through as a constant — so the structural complexity of the original problem disappears once x is numerical.

Thought Process — The step-by-step reasoning

  1. Notice that the condition pins x to a specific number, leaving k as the only true unknown.
  2. Rule out solving f'(x) = f''(x) symbolically first, since that produces a polynomial in x with k inside the coefficients — needlessly messy when x is already given.
  3. Commit to 'substitute first, equate second' because each evaluation produces a clean linear expression in k.
  4. Differentiate f cleanly to get f'(x) and f''(x), keeping k symbolic, then plug in x = a.
  5. Set the two numerical-in-k expressions equal, collect k-terms on one side, constants on the other, and solve.

Mechanic — Concrete operations

  1. Differentiate f(x) to obtain f'(x) and f''(x), keeping k as a literal constant.
  2. Substitute the specified x-value into f'(x) to get a linear expression in k.
  3. Substitute the same x-value into f''(x) to get a second linear expression in k.
  4. Set the two expressions equal to form a linear equation in k.
  5. Rearrange and solve for k, then state the value clearly.

Worked Micro-Example

Let $f(x) = x^3 - kx^2 + 14x$ with $f'(5) = f''(5)$. Then $f'(x) = 3x^2 - 2kx + 14$ and $f''(x) = 6x - 2k$. Evaluating: $f'(5) = 75 - 10k + 14 = 89 - 10k$ and $f''(5) = 30 - 2k$. Equating: $89 - 10k = 30 - 2k \Rightarrow 59 = 8k \Rightarrow k = \tfrac{59}{8}$.

Variants — Same move, different surface

Condition f(a) = f'(a) at a given point with unknown parameter
Evaluate f(a) and f'(a) directly in terms of k, then equate and solve.
Stationary point at x = a (i.e. f'(a) = 0) with unknown coefficient
Only one substitution needed; set f'(a) = 0 to get a single equation in k.
Point of inflection at x = a (f''(a) = 0)
Substitute a into f'' alone and solve for k.
Tangent at x = a has a stated gradient m
Set f'(a) = m and solve for k.
Curve passes through a given point (a, b) with parameter k
Set f(a) = b and solve for k; can be combined with a derivative condition for two-parameter problems.

Common Misreadings

  • Trying to solve f'(x) = f''(x) as a general equation in x and only afterwards imposing x = a, producing an unnecessary quadratic in x.
  • Treating k as if it depended on x and attempting to differentiate it, instead of carrying it as a constant.
  • Confusing the role of the fixed x-value with the role of the parameter — solving for x instead of for k.

Pitfalls

  • Arithmetic slips when squaring or cubing the substituted x-value (e.g. computing $3(5^2) + 14$).
  • Sign errors when moving the $-10k$ and $-2k$ terms across the equation.
  • Forgetting to multiply through when k appears with a coefficient (e.g. writing $-2k(5)$ as $-2k$).
  • Leaving the answer as an unsimplified fraction when the question expects an exact simplified value.

A-level Scope Notes

Fully within Edexcel IAL Pure Maths (P1/P2) — uses only polynomial differentiation, substitution, and linear-equation solving.
