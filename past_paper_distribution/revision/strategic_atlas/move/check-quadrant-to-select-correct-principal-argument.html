Complex Numbers

Check quadrant to select correct principal argument

Family: Quadrant Validation Frequency: 1 questions

Trigger — When to use

The question asks for arg(z) of a complex number z = x + iy, typically in the principal range (−π, π]. The student is about to apply arctan(y/x) but must first determine which quadrant z lies in. This fires whenever Re(z) and Im(z) are both nonzero, and especially when Re(z) < 0 where arctan alone gives the wrong answer. Also fires for purely real or purely imaginary z, where arctan is undefined or misleading.

Motivation — Why this works

The function arctan returns values only in (−π/2, π/2), corresponding to quadrants I and IV. For z in quadrant II or III, the raw arctan(y/x) value differs from the true argument by ±π. Checking the signs of Re(z) and Im(z) identifies the quadrant and tells the student whether to accept arctan(y/x), add π, or subtract π. This guarantees the answer lies in the required principal range.

Thought Process — The step-by-step reasoning

  1. I notice the question asks for arg(z) and I have explicit real and imaginary parts.
  2. I rule out blindly typing arctan(y/x) into the calculator — that only works in quadrants I and IV.
  3. I commit to a two-step procedure: first locate z in the Argand diagram by signs of Re(z) and Im(z), then compute arctan|y/x| as a reference angle.
  4. First concrete operation: read off the signs of Re(z) and Im(z) to fix the quadrant.
  5. Inside the move, I decide the adjustment: Q1 → θ = arctan(y/x); Q2 → θ = π − arctan|y/x|; Q3 → θ = −π + arctan|y/x|; Q4 → θ = −arctan|y/x|. Finally I confirm θ ∈ (−π, π].

Mechanic — Concrete operations

  1. Identify signs of Re(z) and Im(z) to determine the quadrant
  2. Compute the reference angle α = arctan|Im(z)/Re(z)|
  3. Apply the quadrant rule: Q1 → α; Q2 → π − α; Q3 → −(π − α); Q4 → −α
  4. Handle special cases: pure positive real → 0; pure negative real → π; pure positive imaginary → π/2; pure negative imaginary → −π/2
  5. Confirm the result lies in (−π, π] (or [0, 2π) if the question specifies that convention)
  6. Round/express as required (e.g. exact multiple of π, or to 2 d.p. in radians)

Worked Micro-Example

Find arg(w) where w = 1/2 + (1/2)i.\\ Re(w) = 1/2 > 0, Im(w) = 1/2 > 0, so w is in Q1.\\ Reference angle: \arctan(1/2 \div 1/2) = \arctan(1) = \pi/4.\\ Since w is in Q1, arg(w) = \pi/4 \approx 0.79 radians, which lies in (-\pi, \pi]. ✓

Variants — Same move, different surface

z = −1 + i in Q2
Reference angle α = arctan(1) = π/4; arg(z) = π − π/4 = 3π/4.
z = −1 − i in Q3
Reference angle α = π/4; arg(z) = −(π − π/4) = −3π/4.
z = 1 − i in Q4
Reference angle α = π/4; arg(z) = −π/4.
Purely imaginary z = bi (b ≠ 0)
arg(z) = π/2 if b > 0; arg(z) = −π/2 if b < 0; arctan is not used.
Negative real z = a (a < 0)
arg(z) = π directly; do not use arctan(0/a) = 0, which would be wrong.
Question requires argument in [0, 2π)
Convert any negative principal value θ to θ + 2π.
Argument of a quotient z₁/z₂ or product z₁z₂
Use arg(z₁z₂) = arg(z₁) + arg(z₂) and arg(z₁/z₂) = arg(z₁) − arg(z₂), then re-adjust the result back into (−π, π] by adding/subtracting 2π if needed.

Common Misreadings

  • Assuming arctan(y/x) always returns the principal argument without checking the quadrant.
  • Treating arctan(y/x) for z = −1 − i as π/4 (correct quadrant ignored), instead of −3π/4.
  • Confusing the required range: giving an answer in [0, 2π) when (−π, π] is asked, or vice versa.
  • Reading 'modulus-argument form' as requiring degrees, when radians are expected in IAL.

Pitfalls

  • Giving the argument in degrees when the question specifies radians (or vice versa).
  • For purely real or purely imaginary z, mechanically applying arctan instead of assigning 0, π, ±π/2 directly.
  • Forgetting to round to the required number of decimal places, or losing exactness when an exact multiple of π was expected.
  • After combining arguments via products/quotients, leaving the result outside (−π, π] without the ±2π correction.
  • Sign error in the reference angle when |y/x| is used but the quadrant adjustment is then misapplied.

A-level Scope Notes

Fully within Edexcel IAL Further Pure (FP1/FP2) treatment of complex numbers: principal argument in (−π, π], modulus-argument form, and arguments of products and quotients. No use of atan2 or branch-cut theory beyond this level.
