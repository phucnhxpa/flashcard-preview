Proof

Close induction with the three-part conclusion statement

Family: Induction Closure Frequency: 1 questions

Trigger — When to use

Fires at the very end of any proof by induction, once the base case has been verified and the inductive step (P(k) ⇒ P(k+1)) has been established. Cues include having just simplified the P(k+1) expression into the required target form, or finishing a divisibility/summation/matrix induction step. Edexcel mark schemes reserve a dedicated final mark (often labelled 'A1 cso' or 'conclusion') for this statement. If the script ends with the algebra and no concluding sentence, this move has been omitted.

Motivation — Why this works

Mathematical induction is a logical principle, not a calculation: it asserts that if P(1) holds and P(k) ⇒ P(k+1), then P(n) holds for every positive integer n. The three intermediate results are independent facts until a closing statement invokes the principle and binds them together. Without the conclusion, the script has demonstrated two true sub-claims but has not deduced the universal statement the question asked for. Edexcel's mark scheme treats the conclusion as the act of 'doing induction', which is why it carries its own mark.

Thought Process — The step-by-step reasoning

  1. I notice my P(k+1) expression has been rearranged into the required form, so the inductive step is finished and the proof body is complete.
  2. I rule out stopping here or writing a vague 'so true for all n' — Edexcel requires explicit reference to all three components and the correct domain.
  3. I commit to a templated closing sentence because the conclusion mark is awarded for specific phrasing, not for cleverness.
  4. I bridge to writing by recalling the three required clauses: base case verified, implication established, therefore result holds for all n in the stated domain.
  5. Tactically, I match the domain to the question (positive integers, integers n ≥ 0, etc.) and restate the original proposition in full rather than abbreviating to 'P(n)'.

Mechanic — Concrete operations

  1. State that the result is true for n = 1 (or the stated base value).
  2. State that if the result is true for n = k, then it is true for n = k+1.
  3. Invoke the principle: 'Therefore, by mathematical induction, [restate the original proposition] for all positive integers n.'
  4. Check that the domain ('positive integers', 'integers n ≥ 0', etc.) matches the question exactly.

Worked Micro-Example

Having shown $4^1 + 6(1) - 1 = 9$ is divisible by 9, and that $4^{k+1} + 6(k+1) - 1 = 4(4^k + 6k - 1) - 18k + 9 \cdot 1$ is divisible by 9 whenever $4^k + 6k - 1$ is, conclude: 'The statement is true for $n = 1$. If it is true for $n = k$, then it is true for $n = k+1$. Therefore, by mathematical induction, $4^n + 6n - 1$ is divisible by 9 for all positive integers $n$.'

Variants — Same move, different surface

Summation formula induction (e.g. $\sum_{r=1}^{n} r^2$)
Restate the summation identity in full inside the conclusion, e.g. '...therefore $\sum_{r=1}^{n} r^2 = \tfrac{1}{6}n(n+1)(2n+1)$ for all positive integers $n$.'
Matrix induction (e.g. $\mathbf{M}^n = \ldots$) in Further Pure
Conclude with the explicit matrix formula: '...therefore $\mathbf{M}^n = \begin{pmatrix} \ldots \end{pmatrix}$ for all positive integers $n$.'
Induction starting at $n = 0$
Replace 'all positive integers $n$' with 'all integers $n \geq 0$' or 'all non-negative integers $n$'.
Derivative/$n$th-derivative induction (e.g. $\frac{d^n}{dx^n}$ formula)
Same three-part structure, with the derivative formula restated in full inside the closing sentence.

Common Misreadings

  • Treating the algebraic completion of the inductive step as the end of the proof, not realising the conclusion itself carries a mark.
  • Believing 'P(n) is true for all n' is sufficient, without specifying the integer domain.
  • Thinking that one of the three clauses (base, implication, principle) can be implied rather than written.

Pitfalls

  • Writing 'true for all n' without specifying 'positive integers' — loses the conclusion mark.
  • Omitting the conclusion entirely, assuming the marker will infer it — Edexcel treats this as an incomplete proof.
  • Writing 'true for n = k+1' without explicitly linking it to 'if true for n = k' — fails to record the implication.
  • Forgetting to restate the original proposition, leaving an abstract 'P(n) is true' that does not match the mark scheme wording.
  • Mismatching the domain (e.g. concluding 'for all integers' when the question specified positive integers).

A-level Scope Notes

This closure protocol is the standard Edexcel IAL Further Pure expectation for every induction proof — divisibility, summation, matrix powers, and $n$th-derivative formulae — and uses only the principle of mathematical induction as stated in the specification.
