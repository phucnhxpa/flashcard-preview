Proof

Write the explicit P(k+1) target before manipulating

Family: Goal Directed Algebra Frequency: 1 questions

Trigger — When to use

Fires at the start of the inductive step in a proof by induction, immediately after assuming P(k). Cue phrases: 'we must show P(k+1)', 'required to prove', or whenever the closed-form RHS contains shifted factors like (k+1), (k+2), 2^{k+1}, etc. Particularly useful when the formula involves products, powers, or factorials whose shape will dictate the factoring strategy.

Motivation — Why this works

Induction proofs that begin by manipulating the LHS without a fixed destination tend to drift into algebraic dead-ends or circular reasoning. Substituting n = k+1 into the claimed formula produces a concrete target expression, and the factors visible in that target tell the student exactly which groupings to engineer during simplification. This converts an open-ended algebra problem into a directed one: every step is justified by progress toward a known endpoint.

Thought Process — The step-by-step reasoning

  1. I notice the inductive step asks me to prove a specific equality at n = k+1, so the RHS has a definite form.
  2. I rule out diving straight into LHS expansion — without a target I won't know which factorisations to choose.
  3. I commit to writing the target first because it converts the problem into 'reach this expression', which is much easier to plan against.
  4. I substitute n = k+1 into the original RHS and simplify the index shifts to get a clean target.
  5. I scan the target for distinctive factors (e.g. (k+2), (k+3)) so I know what to keep visible when I later combine the inductive hypothesis with the new term.

Mechanic — Concrete operations

  1. Write 'Required to prove: P(k+1)' to mark the goal.
  2. Take the claimed formula's RHS and substitute n := k+1 throughout.
  3. Simplify index shifts (e.g. (k+1)+1 = k+2) so the target is in fully reduced factored form.
  4. Box or underline the target expression.
  5. Now begin manipulating the LHS (using the inductive hypothesis) with the target in clear view, choosing factorisations that match the target's factors.

Worked Micro-Example

Claim: $\sum_{r=1}^{n} r(r+1) = \tfrac{n(n+1)(n+2)}{3}$. Inductive step target: substitute $n = k+1$ to get $\tfrac{(k+1)(k+2)(k+3)}{3}$. Now starting from $\tfrac{k(k+1)(k+2)}{3} + (k+1)(k+2)$, factor out $(k+1)(k+2)$ — chosen precisely because the target shows it must survive — giving $(k+1)(k+2)\!\left[\tfrac{k}{3} + 1\right] = \tfrac{(k+1)(k+2)(k+3)}{3}$, matching the target.

Variants — Same move, different surface

Divisibility induction (e.g. show $3 \mid 4^n - 1$)
State the target as 'must show $f(k+1) = 3M$ for some integer $M$', and pre-write $f(k+1) - f(k)$ or $f(k+1)$ in a form ready to absorb the hypothesis $f(k) = 3m$.
Matrix-power induction $M^n = \begin{pmatrix}\cdots\end{pmatrix}$
Substitute $n = k+1$ in each entry of the claimed matrix to get the explicit target matrix before computing $M^k \cdot M$.
Recurrence-defined sequences proving a closed form
Write the target $u_{k+1} = $ (closed form with $n=k+1$) so that after applying the recurrence to $u_k$ you know which algebraic shape to engineer.
Inequality induction (e.g. $2^n > n^2$ for $n \ge 5$)
State the target inequality at $n=k+1$ explicitly, so the chain of estimates from the hypothesis aims at a known RHS rather than drifting.

Common Misreadings

  • Treating 'state P(k+1)' as a formality and writing it without simplifying the index shifts, so the target isn't actually in the factored form needed to guide later steps.
  • Confusing P(k+1) (what must be proved) with the inductive hypothesis P(k) (what is assumed), and writing the wrong one as the target.
  • Believing the move is optional decoration; in fact it determines which factorisations to choose later.

Pitfalls

  • Substitution errors when shifting indices, e.g. writing $(k+1)(k+2)(k+4)/3$ instead of $(k+1)(k+2)(k+3)/3$.
  • Forgetting to substitute n in every occurrence (e.g. updating the bound of a sum but not the formula's RHS, or vice versa).
  • Leaving the target in unsimplified form like $(k+1)((k+1)+1)((k+1)+2)/3$, which obscures the factors and defeats the guiding purpose.
  • In divisibility proofs, stating the target as a specific multiple (like $3 \times \text{something fixed}$) rather than 'a multiple of 3', which over-constrains the proof.

A-level Scope Notes

Fully within Edexcel IAL Further Pure (FP1) proof by induction: summation formulae, divisibility results, matrix powers, and recurrence closed forms. No content beyond the IAL syllabus is required.
