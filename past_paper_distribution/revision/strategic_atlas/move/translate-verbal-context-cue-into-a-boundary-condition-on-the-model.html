Algebra & Functions

Translate verbal context cue into a boundary condition on the model

Family: Model Interpretation Frequency: 2 questions

Trigger — When to use

Fires whenever a worded modelling question contains a contextual phrase such as 'at the start of the study', 'initially', 'when extraction began', 'when the population dies out', or 'when the temperature reaches room temperature'. The model is given as a formula (often exponential) in a single parameter t, and you are asked for either the dependent variable's value or the time corresponding to that verbal cue. The signal is a natural-language description of a boundary moment that must be converted into either t = 0 or f(t) = constant before any algebra can begin.

Motivation — Why this works

Modelling questions hinge on correctly mapping English to a mathematical condition: 'start of study' → t = 0, 'starts being extracted' → cumulative G = 0, 'reaches room temperature' → T = T_room. Choosing the wrong condition (e.g. t = 0 when the cue actually means G = 0) derails the entire question. Once the correct condition is identified, the model collapses to a one-line substitution or a single equation in one unknown, which is then solved by standard exponential/log techniques.

Thought Process — The step-by-step reasoning

  1. Notice the question contains a worded condition rather than a numerical one — flag it as a translation step before doing any algebra.
  2. Identify what each variable in the model represents (cumulative quantity? rate? population? temperature?) so you know what 'start' or 'reaches' means physically.
  3. Rule out tempting alternatives: it is not a rate question (no dG/dt), not a maximum/minimum (no derivative=0), and not a 'long-term' question (no limit as t→∞).
  4. Decide which side of the equation the cue pins down: if the cue is about WHEN something happens, set the dependent variable equal to its boundary value and solve for t; if the cue is about a value AT a known time (typically the start), substitute that t and evaluate.
  5. Commit to the substitution: write the equation explicitly (e.g. N(0) = ... or G = 0) before manipulating, so the boundary condition is documented.
  6. Proceed with routine exponential algebra — using e^0 = 1 for initial-value substitutions, or isolating the exponential and taking ln for time-finding equations.

Mechanic — Concrete operations

  1. Read the worded cue and write down the corresponding mathematical condition (t = 0, or dependent variable = boundary value).
  2. Substitute the condition into the given model.
  3. If t = 0 was substituted, simplify using e^0 = 1 and evaluate arithmetically.
  4. If a value of the dependent variable was set, rearrange to isolate the exponential term: e^{...} = constant.
  5. Take natural logarithms of both sides and solve the resulting linear equation in t.

Worked Micro-Example

Model: $N = \dfrac{900}{2 + e^{-0.12t}}$. 'At the start of the study' → set $t=0$: $N = \dfrac{900}{2 + e^{0}} = \dfrac{900}{3} = 300$. Contrast: model $G = 40 - 30e^{1-0.05t}$, 'extraction began at $t=k$' → set $G=0$: $30e^{1-0.05k}=40$, then $1-0.05k = \ln(4/3)$, giving $k = \tfrac{1-\ln(4/3)}{0.05}$.

Variants — Same move, different surface

Initial value in radioactive decay $A = A_0 e^{-kt}$
'Initially' → t = 0, giving A = A_0 directly (no algebra needed).
Logistic / bounded growth model
Same t = 0 substitution extracts the initial population; long-term behaviour uses t → ∞ instead.
Newton's law of cooling $T = T_R + (T_0 - T_R)e^{-kt}$
'Reaches room temperature' translates to T = T_R (asymptotic), while 'initially' translates to t = 0 giving T = T_0.
Cumulative-quantity model (mass extracted, charge accumulated)
'When the process started' → cumulative quantity = 0, NOT t = 0, since t is measured from a reference epoch that may pre-date the process.

Common Misreadings

  • Treating 'start' as t = 1 (first year) instead of t = 0 (initial instant).
  • Confusing 'started being extracted' (cumulative G = 0) with 'maximum extraction rate' (dG/dt = max) — entirely different conditions.
  • Setting t = 0 when the cue actually fixes the dependent variable, or vice versa — getting the direction of the translation backwards.
  • Interpreting 'eventually' or 'in the long run' as a finite t-value rather than the limit t → ∞.

Pitfalls

  • Forgetting that e^0 = 1 (not 0) when simplifying after substituting t = 0.
  • Dropping a sign when isolating the exponential, e.g. writing $e^{1-0.05t} = -40/30$ from $0 = 40 - 30e^{1-0.05t}$.
  • Taking ln of a negative number after a rearrangement error — a clear signal to recheck the sign work.
  • Failing to state the boundary condition explicitly, then losing track of which equation is being solved.

A-level Scope Notes

Fully within Edexcel IAL P2/P3 exponential modelling. Uses only the standard identities $e^0 = 1$ and $\ln(e^x) = x$ from the formula booklet's logarithm/exponential section. Avoid invoking limits or differential-equation arguments beyond simple substitution at this stage.
