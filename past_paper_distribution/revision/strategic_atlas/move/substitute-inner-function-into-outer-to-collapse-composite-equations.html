Algebra & Functions

Substitute inner function into outer to collapse composite equations

Family: Function Composition Frequency: 3 questions

Trigger — When to use

Fires whenever a question involves a composite expression fg(x), f(g(x)), or f(h(x)) — either to evaluate at a point, solve fg(x) = k, or find roots of f(g(x)) = 0. Visual cues: the notation 'fg' (which means f∘g, NOT multiplication), nested brackets like f(2x+1), or a question part building on roots already found in an earlier part. Also fires when g(x) shares a constant with the denominator/structure of f, hinting at dramatic cancellation. Alternative phrasing: 'evaluate the composite', 'solve for x given fg(x) = …', or 'hence solve f(linear in x) = 0'.

Motivation — Why this works

Composition is defined inside-out: fg(x) = f(g(x)), so the inner function's output becomes the outer function's input. Substituting g(x) into f's formula either (a) produces a single rational/polynomial expression that simplifies — often dramatically when constants cancel — or (b) lets you bypass expansion entirely by equating the inner expression to known roots of the outer. The underlying principle is that solving f(g(x)) = 0 is equivalent to solving g(x) = r for each root r of f, which converts a hard composed equation into several simple ones. This is the workhorse technique behind every IAL composite-function question.

Thought Process — The step-by-step reasoning

  1. I read 'fg(x)' and confirm it means f(g(x)) — apply g first, then f. I rule out interpreting it as f(x)·g(x).
  2. I check whether the question wants a value (evaluate inside-out), a solution set (solve fg(x) = k), or roots (use known roots of f).
  3. If roots of f are already known from an earlier part, I rule out full substitution-and-expansion: instead I set g(x) equal to each known root — this is much faster and avoids cubic/quartic algebra.
  4. Otherwise I commit to substituting g(x) into the formula for f, watching for structural cancellation (e.g. constants in the denominator of f matching constants in g(x)).
  5. I distribute carefully, simplify numerator and denominator separately, and only then set the result equal to the target value.
  6. If a domain conflict arises (e.g. g(x) lands outside f's stated domain), in an exam context I still apply f's algebraic formula — mark schemes expect a numerical answer.

Mechanic — Concrete operations

  1. Identify the order: fg(x) = f(g(x)). Compute or substitute g(x) first.
  2. Replace every x in f(x) with the expression g(x), keeping brackets intact.
  3. Simplify numerator and denominator (or expanded polynomial) separately, looking for cancellation of constants.
  4. If solving fg(x) = k, cross-multiply or rearrange to a standard polynomial equation.
  5. If roots r₁, r₂, … of f are already known, instead solve g(x) = r₁, g(x) = r₂, … as separate simple equations.
  6. List all solutions; check each against any stated domain restriction on x.

Worked Micro-Example

Given $f(x) = \frac{4x+6}{x-5}$ and $g(x) = 5 - 2x^2$, solve $fg(x) = 3$.\\Substitute: $fg(x) = \frac{4(5-2x^2)+6}{(5-2x^2)-5} = \frac{26 - 8x^2}{-2x^2} = \frac{4x^2 - 13}{x^2}$.\\Set equal to 3: $4x^2 - 13 = 3x^2 \Rightarrow x^2 = 13 \Rightarrow x = \pm\sqrt{13}$.

Variants — Same move, different surface

Roots of f already known from a previous part, equation is f(linear in x) = 0
Skip substitution entirely; set the linear inner expression equal to each known root and solve the resulting linear equations. E.g. f(x/4) = 0 with roots 3/4, 5 gives x = 3, 20.
Composite evaluation at a point where g(a) lies outside f's stated domain
Apply f's algebraic formula anyway — mark schemes expect the computation. Only declare 'undefined' if the question explicitly tests domain awareness.
Inner function is a square, f(x²) = 0
Solve x² = r for each root r; take ±√r and reject any negative r (no real solutions there).
Outer function is rational and inner shares a constant with its denominator
Expect dramatic simplification — the constant terms cancel, leaving a monomial denominator. Always check for this before brute-force expansion.

Common Misreadings

  • Treating fg(x) as the product f(x)·g(x) instead of the composition f(g(x)).
  • Computing gf(x) instead of fg(x) — reversing the order of composition.
  • Substituting the input value directly into f instead of into g first when evaluating fg(a).
  • Re-expanding the full composite when known roots of f from an earlier part would solve the problem in one line.

Pitfalls

  • Sign errors when distributing across (a − bx²) inside f's numerator.
  • Forgetting constant terms (like the +6 in (4x+6)) before simplifying.
  • Arithmetic slip in evaluating squared brackets, e.g. (−2+3)² = 1, not 25.
  • Forgetting a root when f has multiple roots — every root of f generates an equation g(x) = r.
  • Failing to check final solutions against the domain of x stated in the original question.

A-level Scope Notes

Fully within Edexcel IAL P2/P3 Functions chapter. No formula-booklet entry is needed — composition is definitional. Avoid invoking inverse-function machinery (f⁻¹) unless the question explicitly asks; here we only use the forward composition f∘g.
