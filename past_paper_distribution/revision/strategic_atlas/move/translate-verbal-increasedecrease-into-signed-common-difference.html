Sequences & Series

Translate verbal increase/decrease into signed common difference

Family: Parameter Identification Frequency: 1 questions

Trigger — When to use

Fires whenever a worded problem describes a quantity changing by a fixed amount per period — e.g. 'saves £10 less each month', 'produces 3 fewer items each day', 'depreciates by £200 per year'. The constant per-period change is the signal for an arithmetic sequence, and the directional word ('more', 'less', 'fewer', 'extra', 'reduced by') determines the sign of d. Also fires when two parties are compared and only one has a decreasing pattern.

Motivation — Why this works

An arithmetic sequence is fully specified by (a, d), so extracting these from prose is the gateway to every later formula (u_n, S_n). The common difference d carries the direction of change: 'more' or 'extra' gives d > 0, while 'less' or 'fewer' gives d < 0. Getting the sign right is essential because a sign error on d silently propagates through u_n = a + (n-1)d and S_n = (n/2)(2a + (n-1)d), often producing impossible quadratics or spurious roots.

Thought Process — The step-by-step reasoning

  1. Notice the phrase 'each month/year/day' attached to a fixed monetary or quantity change — this flags arithmetic, not geometric.
  2. Rule out d = 0 (constant saving) and rule out a geometric model (no percentage or ratio language).
  3. Commit to identifying (a, d) by reading the starting value and the per-period change separately.
  4. Read the directional keyword: 'more/extra/additional' → d positive; 'less/fewer/reduced' → d negative.
  5. Double-check that the change refers to the per-period quantity itself (e.g. monthly saving), not to the running total or to a percentage.

Mechanic — Concrete operations

  1. Locate the starting amount in the prose and set a equal to it.
  2. Locate the fixed per-period change and its directional keyword.
  3. Assign d = +k if the keyword indicates increase, d = -k if it indicates decrease.
  4. State (a, d) explicitly before substituting into u_n or S_n.
  5. Sanity-check: compute u_2 = a + d and verify it matches the verbal description of the second period.

Worked Micro-Example

Lina saves £600 in month 1, then £10 less each month. Then $a = 600$ and $d = -10$, so $u_n = 600 + (n-1)(-10) = 610 - 10n$. Check: $u_2 = 590$, which is indeed £10 less than £600. Sum over 12 months: $S_{12} = \tfrac{12}{2}(2(600) + 11(-10)) = 6(1200 - 110) = 6540$.

Variants — Same move, different surface

Depreciation by a fixed amount per year (e.g. value falls by £500 each year)
Set a = initial value, d = -500; use u_n for value after (n-1) years.
Production decline (e.g. factory makes 3 fewer items each day)
d = -3; first day's output is a.
Training increase (e.g. runs 200 m more each week)
d = +200; standard positive case.
Comparison of two savers/producers in one question
Assign (a_B, d_B) and (a_L, d_L) separately, taking care that only one may be negative.

Common Misreadings

  • Reading 'saves £10 less each month' as d = +10, ignoring the decrease.
  • Confusing 'saves £10 less' with 'saves £10', taking a = 10 and d = 0.
  • Treating the per-period decrease as a percentage and switching to a geometric model.
  • Mistaking the change in the cumulative total for the change in the monthly amount.

Pitfalls

  • Sign error on d propagates through S_n, often producing a quadratic with no real (or wrong) roots when solving for n.
  • Forgetting that u_1 = a (not a + d) when listing terms.
  • Using n instead of (n-1) in u_n = a + (n-1)d after a correct sign assignment, masking the original error.

A-level Scope Notes

Fully within Edexcel IAL Pure Mathematics P1/P2 arithmetic sequences and series; uses only u_n = a + (n-1)d and S_n = (n/2)(2a + (n-1)d) with signed d.
