Trigonometry

Transform domain bounds under linear substitution to enumerate trig solutions correctly

Family: Domain Tracking Frequency: 2 questions

Trigger — When to use

Fires whenever a trig equation has its argument as a linear function of x — e.g. tan(2x − 15°) = k, sin(3x + α) = k, cos(x + 20°) = k — and a domain restriction on x is given (such as −90° < x < 90° or 0° ≤ x ≤ 360°). The visual cue is: a non-bare argument inside the trig function plus an explicit interval for x. Equivalently phrased: 'find all solutions of f(ax + b) = k in the given x-range' or 'how many solutions does this shifted/scaled trig equation have'. The original x-interval does NOT carry over directly to the new variable, so a domain transformation is mandatory.

Motivation — Why this works

Substituting y = ax + b reduces the equation to a standard f(y) = k, but the solution interval shifts and/or scales accordingly. Failing to transform the bounds causes students to either miss valid solutions (a periodic copy that lies inside the true y-range) or invent invalid ones (a copy that lies outside it). The transformed range tells you exactly how many full periods of the trig function fit, which dictates how many solutions to expect (period 180° for tan, 360° for sin/cos). This is the only reliable way to guarantee a complete and correct solution set when the argument is a non-trivial linear expression.

Thought Process — The step-by-step reasoning

  1. I notice the trig argument is not just x but a linear expression like 2x − 15° or x + 20°, AND the question gives an x-interval — that's the trigger.
  2. I rule out solving directly in x (messy, error-prone) and rule out using the original x-interval as if it applied to the new variable (a common trap).
  3. I commit to the substitution y = ax + b because it converts the equation to the canonical form tan y = k (or sin/cos), which I can solve with arctan plus periodicity.
  4. Bridge: I apply the same linear transformation to the inequality bounds — multiply by a, then add b — to get the y-interval. I check the direction of inequalities carefully if a < 0.
  5. I compute the length of the y-interval and divide by the period (180° for tan, 360° for sin/cos) to predict how many solutions to expect; this acts as a sanity check.
  6. I find the principal value y₀ = arctan(k) (or appropriate inverse), then generate all candidates y₀ + n·period (and the symmetry partner for sin/cos) and keep only those inside the transformed y-interval.
  7. Finally, I back-substitute x = (y − b)/a for each accepted y, and double-check each x lies in the original interval.

Mechanic — Concrete operations

  1. Let y = ax + b. Apply the transformation to the bounds: if p ≤ x ≤ q, then ap + b ≤ y ≤ aq + b (reverse inequalities if a < 0).
  2. Find the principal value y₀ = arctan(k) (use sin/cos symmetry instead if applicable).
  3. Generate candidates: y₀ + 180°n for tan; y₀ + 360°n and (180° − y₀) + 360°n for sin; ±y₀ + 360°n for cos.
  4. Filter: keep only candidates lying in the transformed y-interval; reject the rest explicitly.
  5. Back-substitute: x = (y − b)/a for each accepted y.
  6. Verify each resulting x lies in the original x-interval as a final check.

Worked Micro-Example

Solve $\tan(2x - 15°) = \tfrac{1}{3}$ for $-90° < x < 90°$. Let $y = 2x - 15°$, so $-195° < y < 165°$ (length 360°, so up to 2 solutions). Principal: $y_0 = \arctan(1/3) \approx 18.43°$. Candidates: $18.43°$ ✓, $18.43° - 180° = -161.57°$ ✓, $18.43° + 180° = 198.43°$ ✗. Back-substitute: $x = (y+15°)/2$, giving $x \approx 16.72°$ and $x \approx -73.28°$.

Variants — Same move, different surface

Pure shift, e.g. tan(x + 20°) = k on 0° ≤ x ≤ 360°
y-interval is just shifted: 20° ≤ y ≤ 380°. Same length as original, so same expected solution count, but the window is non-standard.
Scaling by integer > 1, e.g. sin(3x + α) = k on 0° ≤ x ≤ 360°
y-interval becomes α ≤ y ≤ 1080° + α, spanning three full cycles — up to 3 solutions for tan, up to 6 for sin/cos.
Half-angle substitution, e.g. u = x/2 on 0° ≤ x ≤ 360°
u-interval halves: 0° ≤ u ≤ 180°, so fewer solutions than for the bare variable.
sin or cos instead of tan
Use the appropriate symmetry partner (180° − y for sin, −y for cos) in addition to 360°-period copies, all filtered against the transformed interval.

Common Misreadings

  • Treating the original x-interval as if it applied to the substituted variable y, leading to missed or spurious solutions.
  • Solving only for the principal arctan value and assuming that's the unique solution.
  • Confusing the shift direction (e.g. using y = x − 20° instead of y = x + 20°).

Pitfalls

  • Forgetting to convert back from y to x at the end, leaving the answer in the wrong variable.
  • Reversing inequality direction incorrectly, especially if multiplying by a negative coefficient.
  • Stopping too early — not testing the next periodic copy to confirm it's actually outside the range.
  • Premature rounding of the principal value, which can push a borderline solution incorrectly in or out of range.

A-level Scope Notes

Fully within Edexcel IAL Pure Mathematics (P2/P3) trig chapters. Uses the standard periods (tan: 180°, sin/cos: 360°) and inverse trig from the formula booklet. Avoid generalising to complex-valued solutions or to general-solution formulas with arbitrary integer n unless the question explicitly asks for them — IAL questions almost always ask for solutions in a specified finite interval.
