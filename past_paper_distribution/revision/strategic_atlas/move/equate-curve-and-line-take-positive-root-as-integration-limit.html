Algebra & Functions

Equate curve and line, take positive root as integration limit

Family: Intersection For Limits Frequency: 1 questions

Trigger — When to use

Fires when a definite integral is needed but one (or both) of its x-limits is not stated explicitly — instead the boundary is described geometrically as 'where the curve meets the line y = k' or 'where the curves intersect'. Cues include a diagram showing a shaded region whose right edge is an intersection point, or wording like 'the region bounded by C and the line ...'. The limit must be manufactured by solving an equation before integration can begin.

Motivation — Why this works

An intersection point satisfies both equations simultaneously, so equating the y-expressions converts a geometric meeting condition into a single-variable algebraic equation. Solving this equation yields candidate x-values; the geometric context (sign restrictions, which branch of the region is shaded, x ≥ 0 conventions) selects the relevant root. This pinned-down x-value then serves as the upper (or lower) limit of the definite integral, allowing the area/volume calculation to proceed.

Thought Process — The step-by-step reasoning

  1. I notice the integral's upper limit is missing — the boundary is described as 'where the curve meets y = 17' rather than given numerically.
  2. I rule out guessing or reading from the diagram: I need an exact value (likely a surd) to keep the final area exact.
  3. I commit to equating the two y-expressions because intersection ⇔ same x and same y, which collapses to one equation in x.
  4. First concrete operation: write 2x² + 7 = 17 and rearrange to a clean quadratic 2x² = 10.
  5. Tactical decision inside the move: after solving x² = 5, I check the region's location (first quadrant / x ≥ 0) and discard the negative root, keeping x = √5 as the limit.

Mechanic — Concrete operations

  1. Set the curve equation equal to the line equation: 2x² + 7 = 17.
  2. Rearrange to isolate x²: 2x² = 10, then x² = 5.
  3. Solve: x = ±√5.
  4. Apply the geometric restriction (x ≥ 0 from the region) to select x = √5.
  5. Use x = √5 as the upper limit of the definite integral.

Worked Micro-Example

Curve $y = 2x^2 + 7$ meets line $y = 17$. Equate: $2x^2 + 7 = 17 \Rightarrow x^2 = 5 \Rightarrow x = \pm\sqrt{5}$. Region is in $x \geq 0$, so upper limit is $x = \sqrt{5}$. Area $= \int_{0}^{\sqrt{5}} \bigl(17 - (2x^2+7)\bigr)\,dx$.

Variants — Same move, different surface

Region bounded by two curves with no line
Equate the two curve expressions; both roots typically become the lower and upper limits.
Intersection equation is a non-trivial quadratic
Use factorisation or the quadratic formula rather than simple square-rooting; still apply geometric restriction to select roots.
Curve meets the x-axis (y = 0)
Set the curve expression equal to 0; roots give x-intercepts that serve as limits.
Volume of revolution about the x-axis
Same intersection step supplies the limits, but the integrand becomes π·y² rather than (y_top − y_bottom).
Parametric curve meets a line
Substitute the parametric x(t), y(t) into the line equation, solve for t, then translate to x-limits.

Common Misreadings

  • Treating the unknown limit as if it were given, and integrating with a placeholder that never gets evaluated.
  • Forgetting to reject the negative root when the region is restricted to x ≥ 0 (or vice versa).
  • Confusing 'where the curve meets the line' with 'where the curve meets the x-axis' and setting y = 0 by mistake.

Pitfalls

  • Arithmetic slip when rearranging: dividing by 2 before subtracting 7.
  • Writing x = √5 as a decimal approximation, losing the exact surd needed for an exact area.
  • Keeping both ±√5 and using −√5 as the lower limit, producing a symmetric but incorrect region.

A-level Scope Notes

Fully within Edexcel IAL Pure Mathematics (P2/P3 integration for area under a curve and between curves). Quadratic intersections solved by rearrangement, factorisation, or the formula are standard A-level content.
