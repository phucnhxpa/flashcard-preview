Algebra & Functions

Carry exact value forward; round only at final answer

Family: Precision Management Frequency: 1 questions

Trigger — When to use

A 'show that' part establishes a value to a stated accuracy (e.g. angle = 1.742 to 4 s.f., length = 4.83 m to 3 s.f.), and later parts demand a final answer to a tighter or comparable precision (1 d.p., 2 d.p., 3 s.f.). The displayed rounded value is tempting to reuse verbatim, but doing so risks losing accuracy. The cue is any chain where a derived numerical result feeds into sector areas, cosine rule, integration limits, or further trigonometry.

Motivation — Why this works

Rounding errors compound through multiplication, squaring, and trigonometric evaluation, so a value rounded to 4 s.f. can degrade to 2–3 s.f. accuracy after two operations. Carrying the exact symbolic form (e.g. $\pi - 1.4$) or the full calculator-precision decimal preserves all significant figures until the final rounding step. Edexcel mark schemes explicitly penalise answers that fall outside tolerance bands due to premature rounding, even when the method is correct.

Thought Process — The step-by-step reasoning

  1. I notice part (a) gave me a 'show that' result, and parts (b)/(c) reuse this value in further calculations.
  2. I rule out simply typing 1.742 into the next formula — that discards information I already have exactly.
  3. I commit to using the exact form $\pi - 1.4$ (or storing the full decimal in calculator memory) throughout.
  4. First operation: store $\theta = \pi - 1.4$ as variable A on the calculator before touching any subsequent formula.
  5. Tactically, I'll only press the round button when writing the final answer of each subpart, never mid-chain.

Mechanic — Concrete operations

  1. Identify the exact symbolic form behind the displayed rounded value (e.g. $\pi - 1.4$, $\sqrt{13}$, $\ln 5$).
  2. Store this exact value in calculator memory (A, B, Ans, etc.) or keep it symbolic on paper.
  3. Substitute the stored/symbolic value into every downstream formula (sector area, arc length, cosine rule, etc.).
  4. Perform all chained operations at full precision.
  5. Round only at the final answer of each part to the precision the question demands.

Worked Micro-Example

Part (a): show $\angle BOD = \pi - 1.4 = 1.742$ (4 s.f.). Part (b): area of sector $= \tfrac{1}{2}r^{2}\theta$ with $r=2$. Using stored $\theta = \pi - 1.4$: area $= 2(\pi-1.4) = 3.48318\ldots \approx 3.5$ (1 d.p.). Using rounded $1.742$ instead gives $3.484$, which still rounds correctly here, but in part (c) involving $\cos\theta$ the rounded version drifts into the wrong 1 d.p. band.

Variants — Same move, different surface

'Show that' establishes a length to 3 s.f. then asks for an area or volume
Keep the exact surd or expression (e.g. $\sqrt{13}$ rather than 3.61) in the area formula.
Iteration question where $x_1$ is found, then $x_2, x_3$ requested to 4 d.p.
Use the calculator's Ans key to feed full-precision $x_n$ into $x_{n+1}$, rounding only when reporting.
Trapezium rule where ordinates are computed from $f(x)$
Compute each ordinate to full calculator precision; round only the final estimate.
Statistics: standardising with a mean/sd given to 2 d.p.
Use the unrounded computed values (not the displayed rounded ones) when chaining into probability lookups.
Mechanics: a velocity is shown to equal $4.2\,\text{m s}^{-1}$, then used in energy/momentum
Carry the exact form $\sqrt{\ldots}$ or stored decimal into KE = $\tfrac{1}{2}mv^{2}$.

Common Misreadings

  • Thinking the 'show that' value is the exact answer to use, rather than a checkpoint.
  • Believing that 4 s.f. input guarantees 4 s.f. output — it does not after multiplication or trig.
  • Assuming calculator memory introduces error; in fact it preserves more digits than transcription.

Pitfalls

  • Re-entering 1.742 into each subsequent formula and producing a final answer just outside the accepted tolerance.
  • Mixing exact and rounded forms (using $\pi - 1.4$ once, then 1.742 later) leading to inconsistent intermediate results.
  • Rounding intermediate results to 'tidy' values like 1.74 to save typing.
  • Forgetting to clear or overwrite calculator memory between parts, reusing a stale stored value.

A-level Scope Notes

Standard Edexcel IAL expectation across Pure (radians, trig, iteration, numerical methods), Statistics (standardising), and Mechanics (chained kinematics): final-step rounding only, with full precision carried via calculator memory or exact symbolic form.
