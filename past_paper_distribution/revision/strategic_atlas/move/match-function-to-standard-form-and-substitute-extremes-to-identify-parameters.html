Trigonometry

Match function to standard form and substitute extremes to identify parameters

Family: Parameter Identification Frequency: 2 questions

Trigger — When to use

The question gives a trigonometric function in a parameterised form such as y = A sin(Bx) + C, y = A + B cos(kx), or y = A + B sin²(kx), and asks for amplitude, vertical shift, midline, or the values of A and B. The cues are: explicit constants in front of and added to a trig term, a request for 'amplitude/period/shift', or known maximum and minimum y-values from a graph. Alternative phrasings include 'find the values of A and B', 'state the amplitude', 'write down the equation of the principal axis', or 'given the maximum is … and minimum is …'.

Motivation — Why this works

Standard forms like A sin(Bx) + C are canonical decompositions in which each parameter independently controls one graphical feature: A scales the oscillation (amplitude = |A|), B controls period (2π/|B|), and C sets the midline. When the form is given explicitly, parameters can be read off by direct comparison; when only the max/min y-values are known, substituting the extreme values of the bounded trig component (cos ∈ [−1,1], sin² ∈ [0,1], etc.) gives two linear equations in A and C (or A and B) that pin them down uniquely. The principle is that bounded oscillating components attain their extremes at specific input values, so y_max and y_min translate directly into linear constraints.

Thought Process — The step-by-step reasoning

  1. Notice that the function is built from a constant plus a constant multiple of a bounded trig expression — this is the signature of a 'parameter-identification' problem.
  2. Rule out heavier techniques (R sin(x+α) compound-angle conversion, calculus to find stationary points): the structure already exposes the parameters, so direct comparison or extreme-value substitution will be cleaner.
  3. Identify the range of the oscillating component precisely: cos(kx), sin(kx) ∈ [−1,1]; sin²(kx), cos²(kx) ∈ [0,1]; |sin(kx)| ∈ [0,1]. This is the key tactical decision because it determines what to substitute.
  4. Commit to the move: if A,B,C are visible in the expression, read them off; if only y_max and y_min are given, write y_max = A + B·(max of trig component) and y_min = A + B·(min of trig component).
  5. Solve the resulting 2×2 linear system (or take absolute values for amplitude). Watch the sign of B: if B is negative, the maximum of y occurs at the minimum of the trig component, not its maximum.
  6. Sanity-check by computing the midline (y_max + y_min)/2 and half-range (y_max − y_min)/2 and seeing whether they match A and |B| under the standard cosine model — a mismatch signals the oscillating component is not a plain cos/sin.

Mechanic — Concrete operations

  1. Write the given function alongside the template, e.g. f(x) = A sin(Bx) + C or y = A + B·g(x) where g is the bounded trig component.
  2. Determine the range [m, M] of g(x) (e.g. m = −1, M = 1 for cos; m = 0, M = 1 for sin²).
  3. If parameters are explicit: read A, B, C directly; then amplitude = |A|, vertical shift = C, period = 2π/|B|.
  4. If only y_max, y_min are given: form equation (1) y_max = A + B·M (assuming B > 0) and equation (2) y_min = A + B·m.
  5. Solve simultaneously: subtract to get B = (y_max − y_min)/(M − m), then back-substitute for A.
  6. State the parameter values and, if asked, the amplitude |B|, midline y = A, etc.

Worked Micro-Example

Given $y = A + B\sin^2(kx)$ with $y_{\max}=7$ and $y_{\min}=2$. Since $\sin^2(kx)\in[0,1]$: at the minimum, $\sin^2=0 \Rightarrow A = 2$; at the maximum, $\sin^2=1 \Rightarrow A+B = 7 \Rightarrow B = 5$. Hence $y = 2 + 5\sin^2(kx)$.

Variants — Same move, different surface

Function given as A sin(Bx) + C with all constants visible
Read off parameters by direct comparison; amplitude = |A|, vertical shift = C, no equation-solving needed.
Function of the form y = A + B cos(kx) with y_max, y_min from graph
Use cos ∈ [−1,1]: A = (y_max + y_min)/2, B = (y_max − y_min)/2 (assuming B > 0).
Function of the form y = A + B sin²(kx) or y = A + B cos²(kx)
Use range [0,1]: A = y_min, B = y_max − y_min (assuming B > 0).
Negative leading coefficient, e.g. y = A − B cos(kx) with B > 0
y_max occurs when cos = −1, y_min when cos = 1; reverse the equations accordingly.
Form A cos(Bx + φ) + C asking for phase shift
Same parameter reading, but additionally extract phase shift = −φ/B.

Common Misreadings

  • Confusing the vertical shift C with the maximum value of y (the maximum is C + |A|, not C).
  • Reporting the amplitude as the full peak-to-trough distance (2|A|) rather than |A|.
  • Assuming every 'A + B·trig(x)' problem can be solved by midline = (max+min)/2, amplitude = (max−min)/2 — this fails when the trig component is sin² or cos² (range [0,1]) rather than sin/cos (range [−1,1]).
  • Mismatching which y-extreme corresponds to which extreme of the trig component when B is negative.

Pitfalls

  • Forgetting to take the absolute value: amplitude is |A|, not A, especially when A is negative (the graph is also reflected).
  • If the function is given in expanded form (e.g. 3 sin(2x)cos(1) + 3 cos(2x)sin(1) + 5), it must first be combined into standard form via compound-angle identities before parameters can be read.
  • Sign errors when solving the simultaneous equations: subtract carefully, and always verify by plugging A, B back into both equations.
  • Using cos²/sin² range [−1,1] instead of [0,1] — this corrupts both equations and gives wrong A and B.

A-level Scope Notes

Sits squarely within Edexcel IAL P2/P3 trigonometry (graphs of sin, cos, tan, transformations) and the modelling sections that use sin²/cos² (which can be linearised via the double-angle identities cos²x = (1+cos 2x)/2, sin²x = (1−cos 2x)/2 from the formula booklet). Avoid invoking Fourier-style decompositions or general harmonic analysis — these are out of scope.
