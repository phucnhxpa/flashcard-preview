Sequences & Series

Translate 'last/specific row' phrase into nth term u_n = a + (n−1)d

Family: Modelling Translation Frequency: 1 questions

Trigger — When to use

Fires whenever a worded AP context (rows of seats, stacked logs, salary years, distances on successive days) asks for the value of one specific term — most commonly phrased as 'the last row', 'in the kth year', 'on day n', or 'the final term'. The total number of terms n is either given or easily counted, and a and d are extractable from the wording. Distinguish from sum-questions which use phrases like 'total', 'altogether', 'in all'.

Motivation — Why this works

Any single term of an AP is reached in one substitution via u_n = a + (n−1)d, because the AP is defined by adding d exactly (n−1) times to the first term a. Computing it as S_n − S_{n−1} is algebraically equivalent but doubles the work and the error surface. Mapping the English noun phrase ('last row') to the index of the term is the only translation step required.

Thought Process — The step-by-step reasoning

  1. Notice the question targets one specific term, not a total — keyword 'the … row/year/day', not 'total'.
  2. Rule out the sum formula S_n; rule out listing terms one by one for large n.
  3. Commit to u_n = a + (n−1)d once a, d, and the term index are all identifiable from the wording.
  4. Bridge: read off a (first row/year/day), d (constant increment), and n (which term is being asked for).
  5. Tactical check: is the index n or n−1? 'Last of 29 rows' → index 29, multiplier (29−1) = 28.

Mechanic — Concrete operations

  1. Identify the term index from the wording (e.g. last row of 29 rows → u_{29}).
  2. Read a (first term) and d (common difference) from the context.
  3. Substitute into u_n = a + (n−1)d.
  4. Simplify arithmetically and state the answer in context (with units if appropriate).

Worked Micro-Example

A theatre has 29 rows; the front row has 18 seats and each subsequent row has 4 more. Find the number of seats in the last row.\\ Here a = 18, d = 4, n = 29, so u_{29} = 18 + (29-1)(4) = 18 + 112 = 130 seats.

Variants — Same move, different surface

GP context (e.g. populations multiplying each year)
Use u_n = ar^{n-1} instead of a + (n−1)d.
'First row/term that exceeds a threshold'
Set u_n > K, solve the inequality for n, then take the smallest integer n.
Index given by description rather than a number (e.g. 'the middle row of 29')
Translate to the correct index first (here n = 15) before substituting.
a or d unknown but two terms given
Form simultaneous equations from u_p = a + (p−1)d and u_q = a + (q−1)d before applying the move.

Common Misreadings

  • Reading 'last row' as a sum cue and computing S_n instead of u_n.
  • Treating the increment 'each row has 4 more' as the first term rather than as d.
  • Counting rows from 0, so taking the last row as u_{28} rather than u_{29}.

Pitfalls

  • Off-by-one: using n in place of (n−1) as the multiplier of d.
  • Sign error when d is negative (decreasing AP).
  • Forgetting to state units / context in the final answer.

A-level Scope Notes

Standard Edexcel IAL P1/P2 arithmetic-sequence material; uses only u_n = a + (n−1)d and basic worded-context translation.
