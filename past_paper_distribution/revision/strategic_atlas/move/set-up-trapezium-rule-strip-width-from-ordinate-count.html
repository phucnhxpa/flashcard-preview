Integration

Set up trapezium rule strip width from ordinate count

Family: Numerical Integration Frequency: 1 questions

Trigger — When to use

Question instructs 'use the trapezium rule with all the values' from a table of ordinates over an interval $[a,b]$. Phrasings include 'use the values in the completed table', 'with the values given' or specifying $h$ and asking for ordinates. Fires whenever a numerical integration estimate is required and the strip count must be inferred from the table.

Motivation — Why this works

The trapezium rule formula $\tfrac{h}{2}[y_0+y_n+2(y_1+\cdots+y_{n-1})]$ requires the correct strip width $h=(b-a)/n$, where $n$ is the number of strips, not the number of ordinates. Because $n+1$ equally-spaced ordinates partition $[a,b]$ into $n$ equal strips, miscounting by one propagates a multiplicative error through the entire estimate. Establishing $h$ and $n$ before substitution is the foundational setup step.

Thought Process — The step-by-step reasoning

  1. I notice the table lists tabulated values at equally spaced abscissae across $[a,b]$, and the instruction 'use all the values' fixes the strip count.
  2. I rule out treating the number of ordinates as the number of strips, since that would misalign with the equal spacing visible in the table.
  3. I commit to $n = (\text{number of ordinates}) - 1$ because $k$ points create $k-1$ gaps.
  4. I compute $h=(b-a)/n$ and cross-check it against the visible step between consecutive $t$-values in the table.
  5. I confirm I will use every ordinate (no skipping), since 'all the values' rules out coarser subdivisions.

Mechanic — Concrete operations

  1. Count the ordinates listed in the table; call this $k$.
  2. Set the number of strips $n = k - 1$.
  3. Compute $h = (b-a)/n$.
  4. Verify $h$ equals the constant spacing between consecutive abscissae in the table.

Worked Micro-Example

Table gives $P$ at $t=10,10.5,11,11.5,12$ (so $k=5$ ordinates). Then $n = k-1 = 4$ strips and $h=(12-10)/4 = 0.5$. This matches the visible step of $0.5$ between consecutive $t$-values, confirming the setup before applying $\int_{10}^{12} P\,dt \approx \tfrac{h}{2}[y_0+y_4+2(y_1+y_2+y_3)]$.

Variants — Same move, different surface

Six ordinates over $[0,5]$
$n = 6-1 = 5$ and $h = 5/5 = 1$.
Question states $h$ and asks how many ordinates to tabulate
Invert: $n=(b-a)/h$, then ordinate count $= n+1$.
Interval given as $[a,b]$ with non-integer endpoints, e.g. $[1.2, 3.2]$ with 5 ordinates
Same logic: $n=4$, $h=(3.2-1.2)/4 = 0.5$.
Table of $\ln x$ or $\sqrt{1+x^3}$ values for an integral
Setup is identical regardless of the integrand; $h$ depends only on $a,b,n$.

Common Misreadings

  • Treating the 5 listed ordinates as 5 strips, giving $h=(12-10)/5=0.4$.
  • Reading 'using 5 strips' from the prompt when the prompt actually says 'using all the values' (5 ordinates).
  • Assuming the spacing must equal an integer or the smallest visible decimal rather than computing $(b-a)/n$ directly.

Pitfalls

  • Using $n=5$ instead of $n=4$, producing $h=0.4$ and a wholly incorrect area estimate.
  • Forgetting to check that the table spacing is uniform — the trapezium rule formula assumes equal strips.
  • Substituting $h$ before fixing $n$, leading to an inconsistent value carried into the bracketed sum.

A-level Scope Notes

Standard Edexcel IAL Pure/Core trapezium rule setup; only equally spaced ordinates and the formula $\tfrac{h}{2}[y_0+y_n+2(y_1+\cdots+y_{n-1})]$ are in scope.
