Integration

Anchor reduction formula at base case then iterate upward

Family: Recursive Evaluation Frequency: 1 questions

Trigger — When to use

Fires once a reduction formula I_n = f(n) + g(n)·I_{n-k} has been derived (or stated) and a specific numerical value such as I_4 or I_5 is requested. The cue is a target index combined with a recurrence whose step is k = 1 or k = 2. Phrasings include 'Hence find I_4', 'Use the reduction formula to evaluate...', or 'Deduce the value of...'. The move applies whenever the recurrence is correct but the question still needs a starting integral to ground it.

Motivation — Why this works

A reduction formula only relates one integral to another — without a base case the chain never terminates. Because the step is typically 2, even-indexed integrals descend to I_0 and odd-indexed to I_1, both of which are elementary standard integrals. Once the base value is computed by direct integration, every higher-index value is obtained by pure substitution and arithmetic, which is far cheaper than integrating each I_n from scratch.

Thought Process — The step-by-step reasoning

  1. I notice a target index (say I_4) and a recurrence stepping by 2, so I trace the chain I_4 → I_2 → I_0 to identify which base case I need.
  2. I rule out integrating I_4 directly by parts again — the whole point of the reduction formula is to avoid that, and re-deriving it would waste marks.
  3. I commit to computing the lowest-index integral on the chain (I_0 here) directly, because it is a one-line standard integral.
  4. I evaluate the base case explicitly with limits, getting a clean number to feed into the recurrence.
  5. I apply the formula step-by-step upward, simplifying each I_n fully before substituting it into the next, and keep π powers exact to avoid arithmetic slips.

Mechanic — Concrete operations

  1. Identify whether the target index is even or odd to determine the base case (I_0 or I_1).
  2. Compute the base case by direct integration over the given limits.
  3. Substitute n = base+2 into the reduction formula and simplify to a closed form.
  4. Substitute n = base+4, using the previous value, and simplify again.
  5. Repeat until the target index is reached; present the final answer in fully simplified exact form.

Worked Micro-Example

Given $I_n = n(\pi/2)^{n-1} - n(n-1)I_{n-2}$ with $I_n = \int_0^{\pi/2} x^{n-1}\sin x\,dx$, find $I_4$. Base: $I_0 = \int_0^{\pi/2}\sin x\,dx = 1$. Then $I_2 = 2(\pi/2) - 2\cdot 1\cdot 1 = \pi - 2$. Finally $I_4 = 4(\pi/2)^3 - 12(\pi-2) = \tfrac{\pi^3}{2} - 12\pi + 24$.

Variants — Same move, different surface

Odd target index, e.g. I_5 from the same family
Chain down to I_1 instead of I_0; compute I_1 by a single integration by parts rather than a direct standard integral.
Wallis-type recurrence I_n = ((n-1)/n) I_{n-2} for ∫_0^{π/2} sin^n x dx
Base cases are I_0 = π/2 and I_1 = 1; the iteration is purely multiplicative with no additive term.
Step-1 recurrence such as I_n = x^n e^x|_0^1 - n I_{n-1}
Only one base case I_0 is needed and every index is reached directly without parity considerations.
Reduction with non-trivial limits or a parameter
Evaluate the base integral with those exact limits/parameter before iterating; the recurrence machinery is unchanged.

Common Misreadings

  • Treating the reduction formula itself as the final answer and forgetting that a numerical evaluation is required.
  • Believing the base case must be re-derived from the reduction formula rather than computed independently.
  • Assuming I_0 = 0 because 'the integrand has x^0' without actually evaluating the integral.

Pitfalls

  • Arithmetic slips in powers of π/2, e.g. writing $(\pi/2)^3 = \pi^3/2$ instead of $\pi^3/8$.
  • Using the wrong recurrence step (n−1 vs n−2) and so jumping to the wrong parity chain.
  • Trying to reach I_3 from I_0, or I_4 from I_1, mixing even and odd chains.
  • Forgetting to multiply through by the n(n−1) coefficient, or dropping the sign in the −n(n−1)I_{n−2} term.
  • Leaving the answer in unsimplified nested form rather than collecting like terms.

A-level Scope Notes

Fully within Edexcel IAL Further Pure scope: reduction formulae for ∫sin^n x, ∫cos^n x, ∫x^n e^x and ∫x^n sin x types are standard, and base cases use only elementary integrals from Core Pure.
