Coordinate Geometry

Convert boundary lines into inequalities using an interior test point

Family: Inequality From Boundary Frequency: 1 questions

Trigger — When to use

Fires when a question asks for the system of inequalities defining a closed region R bounded by lines (often after a part (a) that found vertices or intersection points). Cues include phrases like 'find the inequalities that define R', 'write down three inequalities', or a shaded region in a diagram. Typically appears in linear programming-flavoured coordinate geometry questions where R is a triangle or polygon. The boundary equations are already known; the task is purely to assign the correct direction (≤ or ≥) to each.

Motivation — Why this works

A closed convex polygonal region is the intersection of half-planes, one per boundary line. For each line, the region lies entirely on one side, so testing any single interior point (or a known interior vertex/centroid) against the line equation determines the correct inequality direction unambiguously. This converts a picture into an algebraic specification that can be checked mechanically. The principle generalises any time a region is described by boundaries: orientation is fixed by one interior point.

Thought Process — The step-by-step reasoning

  1. Notice the region R is closed and bounded by a known list of lines, so the answer must be one inequality per boundary line.
  2. Rule out guessing signs from the form of the equation (e.g. assuming y ≤ for any line written as y = ...); the orientation depends on where R sits, not on how the line is written.
  3. Commit to the test-point method: find any point demonstrably inside R (centroid of vertices, or an obvious lattice point), then check each boundary.
  4. Pick a concrete interior point — e.g. the centroid of the three vertices, or a visibly-inside integer point — and substitute into each line equation in the form LHS vs RHS.
  5. For each line, compare LHS and RHS at the test point; whichever sign holds is the correct inequality direction. Write inequalities in the form the mark scheme expects (often y on the left, or cleared of fractions).

Mechanic — Concrete operations

  1. List the boundary line equations in the form y = mx+c (or ax+by=c).
  2. Determine an interior test point: average the three vertex coordinates to get the centroid, or read off any clearly-interior point.
  3. For each boundary, substitute the test point's coordinates into both sides and observe which inequality (≤ or ≥) holds.
  4. Record that inequality, using the same algebraic form as the boundary equation.
  5. Sanity check: confirm all three inequalities are simultaneously satisfied at the test point and that each vertex satisfies them with equality on exactly two boundaries.

Worked Micro-Example

Triangle with vertices $(9/4,2)$, $(4,2)$, $(45/13,40/13)$, bounded by $y=\tfrac{8}{9}x$, $y=10-2x$, $y=2$. Centroid $\approx(3.42,\,2.69)$. Test $y$ vs $\tfrac{8}{9}x$: $2.69$ vs $3.04$, so $y\le \tfrac{8}{9}x$ fails — actually $\tfrac{8}{9}x>y$, giving $y\le\tfrac{8}{9}x$. Re-checking with point $(3.5,2.5)$: $\tfrac{8}{9}(3.5)\approx3.11>2.5$, confirms $y\le\tfrac{8}{9}x$ on this side. Test $y$ vs $10-2x$: $2.69$ vs $3.16$, so $y\le 10-2x$. Test $y$ vs $2$: $2.69\ge 2$. Inequalities: $y\le\tfrac{8}{9}x$, $y\le 10-2x$, $y\ge 2$ (direction follows the actual interior point — always trust the test).

Variants — Same move, different surface

Region bounded by a curve and a line (e.g. parabola and chord)
Same test-point method; substitute interior point into the curve equation, e.g. decide between $y\le x^2$ and $y\ge x^2$.
Mark scheme wants inequalities cleared of fractions
Multiply through, e.g. $y\ge\tfrac{8}{9}x \Leftrightarrow 9y\ge 8x$; preserve direction since multiplier is positive.
Strict inequalities for an open region
Use $<$ and $>$ if the boundary is dashed/excluded; otherwise use $\le,\ge$.
Region is unbounded (e.g. feasible region in linear programming extending to infinity)
Same per-line test; just no closure condition needed, but check a point clearly in the unbounded region.

Common Misreadings

  • Writing the inequality the wrong way round because no interior point was tested — sign chosen by feel.
  • Omitting the 'trivial' horizontal or vertical boundary (e.g. $y\ge 2$) and giving only two inequalities for a triangle.
  • Confusing 'above the line' with $y\ge$ when the line equation is not solved for $y$ (e.g. for $2x+y=10$, 'above' means $y\ge 10-2x$, not $2x+y\ge 10$ — actually these agree, but care is needed).

Pitfalls

  • Choosing a test point that lies on a boundary, so the comparison gives equality and doesn't fix the direction.
  • Leaving a parameter (e.g. $k$) in the inequality instead of substituting its determined value.
  • Multiplying or dividing by a negative quantity while rearranging without flipping the inequality sign.
  • Mixing strict and non-strict signs inconsistently across the three boundaries.

A-level Scope Notes

Standard Edexcel IAL Pure/Core coordinate geometry: linear boundaries, triangular or simple polygonal regions, exact-fraction vertices. No optimisation theory beyond identifying the region itself.
