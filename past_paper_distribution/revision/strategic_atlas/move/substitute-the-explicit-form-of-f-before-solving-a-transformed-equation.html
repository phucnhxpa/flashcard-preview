Trigonometry

Substitute the explicit form of f before solving a transformed equation

Family: Substitution Bridge Frequency: 1 questions

Trigger — When to use

The question asks for a concrete feature (intercepts, stationary points, specific values) of a transformed function such as y = f(x + a), y = f(x) + k, or y = af(bx), where f was identified in an earlier part. The transformation is given symbolically but f has a known explicit form (e.g. f(x) = 3cos x). Without substituting, no algebraic equation can be set up. Common phrasings: 'find the coordinates where the curve y = f(x + π/4) crosses the axes', 'state the exact value of...', 'solve f(2x) = 0'.

Motivation — Why this works

A transformation written as y = f(x + a) is just notation; it cannot be solved until f is replaced by its rule. Substituting converts the abstract transformation into a concrete trig equation that responds to standard techniques (set y = 0, evaluate at x = 0, apply identities). This is safer than shifting graph features by inspection because non-axis features (especially the y-intercept of a horizontally shifted curve) require a fresh evaluation, not a translated coordinate.

Thought Process — The step-by-step reasoning

  1. Notice the target — intercepts/values of y = f(x + π/4) — cannot be computed while f is still a symbol.
  2. Rule out trying to translate intercept coordinates from the original graph: this fails for the y-intercept under a horizontal shift, and is fragile for exact values.
  3. Commit to substitution: the earlier part already gave f(x) = 3cos x, so the transformed function has a fully explicit rule.
  4. Bridge to operation: write y = 3cos(x + π/4), then apply the standard intercept procedure (y = 0 for x-intercepts, x = 0 for y-intercept).
  5. Tactically decide whether to leave answers in exact surd form (e.g. 3cos(π/4) = 3√2/2) and which general solution range to take from the question.

Mechanic — Concrete operations

  1. Recall the explicit form of f from the earlier part (e.g. f(x) = 3cos x).
  2. Replace the argument: write y = 3cos(x + π/4).
  3. For x-intercepts, set 3cos(x + π/4) = 0 and solve cos(x + π/4) = 0 within the stated domain.
  4. For the y-intercept, substitute x = 0 to get y = 3cos(π/4) = 3√2/2.
  5. State coordinates explicitly and check they lie in any specified interval.

Worked Micro-Example

Given f(x) = 3cos x, find the intercepts of y = f(x + π/4) for 0 ≤ x ≤ 2π. Substitute: y = 3cos(x + π/4). y-intercept: x = 0 gives y = 3cos(π/4) = 3√2/2, so (0, 3√2/2). x-intercepts: cos(x + π/4) = 0 ⇒ x + π/4 = π/2, 3π/2 ⇒ x = π/4, 5π/4.

Variants — Same move, different surface

Vertical translation y = f(x) + k with f(x) = a sin x
Substitute to get y = a sin x + k; x-intercepts solve sin x = -k/a, y-intercept is just k.
Horizontal stretch y = f(bx) with f(x) = tan x
Substitute to get y = tan(bx); solve tan(bx) = 0 or evaluate at x = 0, remembering to divide solutions by b.
Reflection y = f(-x) or y = -f(x)
Substitute the explicit rule, then exploit even/odd symmetry of sin/cos rather than shifting points by eye.
Composite request like solving f(x + a) = c
After substitution, treat as a standard trig equation in (x + a), solve for the bracket, then subtract a.

Common Misreadings

  • Treating the y-intercept of y = f(x + a) as the y-intercept of f shifted left by a — but the y-intercept is a new evaluation f(a), not a translated point.
  • Shifting x-intercepts in the wrong direction (a translation by +a inside the bracket moves the graph left, not right).
  • Forgetting that the earlier part already identified f, and trying to work entirely graphically.

Pitfalls

  • Dropping the amplitude factor (writing cos(x + π/4) instead of 3cos(x + π/4)) when computing the y-intercept.
  • Solving cos(x + π/4) = 0 only for the principal value and missing solutions in the given interval.
  • Giving decimal approximations when the mark scheme expects exact surd form such as 3√2/2.

A-level Scope Notes

Standard Edexcel IAL P2/P3 content on graph transformations of trig functions and solving trig equations on a stated interval; uses only basic identities and exact values at π/4, π/3, π/6.
