Trigger — When to use

Fires whenever an equation reduces to tan x = k on a stated interval such as 0 < x ≤ π, 0 ≤ x < 2π, or -π < x ≤ π. Cues include: a single tan equation (or factorised form yielding tan x = 0 and tan x = k), strict vs non-strict interval endpoints, and a domain whose length is comparable to the period π. Also fires when k = 0 forces care at boundary 0 or π, and when k ≠ 0 forces a check on whether arctan k + π still lies inside the interval.

Motivation — Why this works

tan x has period π (not 2π), so within an interval of length π there is generically exactly one solution per equation tan x = k, but boundary inclusions/exclusions can shift the count. The general solution x = arctan k + nπ must be filtered against the stated interval, treating < and ≤ correctly. Methodically listing candidates for each n and testing endpoints prevents both omission and inclusion of spurious roots, both of which lose marks at A-level.

Thought Process — Step-by-step reasoning

  1. Notice the equation has been reduced to one or more tan x = k statements on a stated interval, and recall tan has period π.
  2. Rule out using period 2π or quadrant CAST diagrams designed for sin/cos: tan repeats every π, so only x and x+π type candidates need checking.
  3. Commit to the template x = arctan k + nπ for each k, then filter by the interval, paying explicit attention to strict vs non-strict endpoints.
  4. For tan x = 0 use the exact form x = nπ; for tan x = k with k ≠ 0 compute the principal value arctan k and test n = 0, 1 (and n = -1 if the interval extends below 0).
  5. Decide on exact vs decimal form: present x = π exactly when it is a boundary solution, and give arctan(3/2) ≈ 0.983 to 3 s.f. as required by the question.

Mechanic — Concrete operations

  1. Identify each equation tan x = k after factorising/reducing.
  2. Write the general solution x = arctan k + nπ (use x = nπ when k = 0).
  3. Substitute n = 0, ±1 and list candidate x values numerically.
  4. Filter candidates against the interval, carefully applying < vs ≤ at each endpoint.
  5. State surviving solutions in the form requested (exact where possible, else 3 s.f.).

Worked Micro-Example

Solve tan x (tan x - 3/2) = 0 for 0 < x ≤ π.\\ tan x = 0 ⇒ x = nπ; in (0, π] only x = π.\\ tan x = 3/2 ⇒ x = arctan(3/2) + nπ ≈ 0.983, 4.12, …; only 0.983 lies in (0, π].\\ Solutions: x = 0.983 (3 s.f.), x = π.

Variants — Same move, different surface

Interval 0 ≤ x < 2π with tan x = k
Two solutions generically: arctan k and arctan k + π, both typically in range; check endpoint 0 if k = 0.
Interval -π < x ≤ π
Test n = -1, 0; principal value may already be negative if k < 0, and x = π is included while x = -π is not.
Equation tan(2x) = k on 0 ≤ x < π
Let u = 2x ∈ [0, 2π), enumerate u = arctan k + nπ for n = 0,1, then halve to recover x.
Equation tan(x + α) = k
Let u = x + α, find u-interval by shifting, enumerate u = arctan k + nπ, then subtract α.
tan x = k expressed via sin x = k cos x
Same enumeration, but discard any spurious roots where cos x = 0 introduced during rearrangement.

Common Misreadings

  • Treating tan as having period 2π and producing two candidates per period unnecessarily.
  • Confusing the interval (0, π] with [0, π] and including x = 0 for tan x = 0.
  • Reading 'arctan(3/2) + π' as automatically valid because sin/cos equations usually give a second solution.

Pitfalls

  • Including x = 0 when the domain is 0 < x.
  • Including arctan(3/2) + π ≈ 4.12 despite it exceeding π.
  • Writing x ≈ 3.14 instead of the exact x = π.
  • Rounding arctan(3/2) inconsistently (e.g. 0.98 vs 0.983) when 3 s.f. is required.

A-level Scope Notes

Fully within Edexcel IAL P2/P3 trig solution work: uses only the period of tan, the principal value arctan, and standard interval filtering.
