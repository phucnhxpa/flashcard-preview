Algebra & Functions

Bridge parts via 'Hence' by selecting the admissible root

Family: Substitution Bridge Frequency: 1 questions

Trigger — When to use

Part (b) opens with 'Hence' (not 'Hence or otherwise'), forcing reuse of part (a)'s output. Part (a) yielded multiple roots, but part (b) involves an expression with a domain restriction (e.g. x^(3/2), √x, ln x) or explicitly says 'positive solution'. The student must filter the roots before substituting. Cue phrases: 'Hence find...', 'Hence show that...', combined with a fractional/irrational power or log in the target expression.

Motivation — Why this works

'Hence' is a mark-scheme directive: method marks are tied to using the previous result. Filtering to the admissible root is forced by the domain of the target function — fractional powers x^(p/q) with even q require x ≥ 0, and logs require x > 0. Selecting the wrong root either gives a non-real answer or violates the question's explicit constraint, costing accuracy marks even if arithmetic is clean.

Thought Process — The step-by-step reasoning

  1. Notice 'Hence' — part (b) is chained to part (a), so the roots from (a) are the only inputs allowed.
  2. Rule out roots incompatible with the target expression's domain: x = −3 fails because (−3)^(3/2) is not real.
  3. Commit to the positive root x = 5/2, which the question also flags explicitly via 'positive solution'.
  4. Set up the substitution y = (5/2)^(3/2) as the bridge to the evaluation phase.
  5. Plan the simplification route: split (5/2)^(3/2) = (5/2)·√(5/2) and rationalise to reach exact form.

Mechanic — Concrete operations

  1. Read the directive word: confirm 'Hence' (mandatory reuse) vs 'Hence or otherwise' (optional reuse).
  2. List all roots from part (a).
  3. Identify the domain constraint imposed by the target expression in part (b).
  4. Reject roots that violate the constraint or the question's explicit sign condition.
  5. Substitute the surviving root into the target expression.
  6. Simplify using index laws and rationalisation to reach exact form.

Worked Micro-Example

Part (a) gave roots $x = \tfrac{5}{2}$ and $x = -3$. Part (b): 'Hence find $y$ when $y = x^{3/2}$ for the positive solution.' Reject $x=-3$ (since $(-3)^{3/2}$ is non-real); substitute $x = \tfrac{5}{2}$: $y = \left(\tfrac{5}{2}\right)^{3/2} = \tfrac{5}{2}\sqrt{\tfrac{5}{2}} = \tfrac{5\sqrt{10}}{4}.$

Variants — Same move, different surface

Part (b) involves $\ln x$ or $\log x$
Reject any non-positive root on domain grounds ($x>0$ strictly), even without an explicit 'positive' instruction.
Part (b) involves $\sqrt{x}$ or $x^{1/2}$
Reject negative roots; if $x=0$ is a root, check whether the question requires strictly positive.
'Hence or otherwise' phrasing
Reuse is optional, but using part (a) is usually faster and aligns with mark scheme.
Trigonometric substitution where part (a) yields $\sin\theta = k$ with $|k|>1$ as one root
Reject the inadmissible root and proceed with the valid one within the stated interval.
Quadratic in disguise (e.g. $u = e^x$ giving two $u$-values)
Reject negative $u$ since $e^x>0$, then take logs of the positive value.

Common Misreadings

  • Treating 'Hence' as 'Hence or otherwise' and starting part (b) from scratch, losing method marks.
  • Substituting the negative root and producing a non-real or undefined value without recognising why.
  • Assuming 'positive solution' is a redundant hint and substituting both roots.
  • Confusing the domain restriction (a property of the target expression) with the question's explicit sign instruction — both should be checked.

Pitfalls

  • Carrying forward an error from part (a): method marks can still be awarded if the substitution and simplification are executed correctly.
  • Forgetting to rationalise or leave the answer in the required exact form after substitution.
  • Misapplying $x^{3/2} = x\sqrt{x}$ when $x$ is a fraction — the $\sqrt{x}$ part still needs rationalisation.

A-level Scope Notes

Standard Edexcel IAL P1–P4 chaining structure: 'Hence' directives appear routinely in indices, logarithms, and trig equation questions, and root-filtering by domain is core A-level technique.
