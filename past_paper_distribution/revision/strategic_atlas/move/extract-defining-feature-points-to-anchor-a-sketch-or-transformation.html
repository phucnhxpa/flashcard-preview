Algebra & Functions

Extract defining feature points to anchor a sketch or transformation

Family: Feature Point Extraction Frequency: 2 questions

Trigger — When to use

Fires whenever a question presents a curve graphically (a sketch with labelled coordinates) or via a modulus/transformation expression and asks you to draw a related sketch with labelled key points. Visual cues: a given diagram with marked intercepts/turning points, instructions like 'sketch y = f(x+2), labelling any points where the curve meets the axes', or a modulus graph y = |f(x)| whose V-shape vertex must be located. Equivalent phrasings: 'state the coordinates of the image of...', 'mark the new positions of the maximum/minimum', 'find where the graph meets the axes'. The common signal is that the curve is fully characterised by a small number of distinguished points rather than by a closed-form formula you need to manipulate.

Motivation — Why this works

Graph transformations and modulus reflections act uniformly on coordinates, so the entire transformed or reflected curve is determined by mapping its defining features — intercepts, turning points, vertices, and asymptotes. Extracting these as an explicit, finite list of coordinate pairs converts a vague 'draw the whole curve' task into a small set of point-mapping calculations. This is faster and far less error-prone than reverse-engineering an algebraic formula for f(x), and it directly produces the labelled coordinates the marker is looking for.

Thought Process — The step-by-step reasoning

  1. Notice that the curve is specified by a sketch (with labelled points) or by a modulus expression — there is no need, and often no way, to recover a full algebraic form.
  2. Rule out trying to deduce f(x) (e.g. fitting a cubic to the marked points) — this introduces unnecessary algebra and risks errors on a question designed to test transformation understanding.
  3. Commit to the principle that an A-level sketch is fully determined by its key features: axis intercepts, turning points, vertices of V-shapes, and any asymptotes.
  4. Bridge to operations: read off the coordinates of every labelled feature on the original (or, for a modulus graph, set the interior to zero to locate the vertex and substitute x=0 for the y-intercept).
  5. Tactically decide which features are 'defining' for this curve type — e.g. for a cubic, the two turning points and intercepts; for y=|ax+b|, the vertex and y-intercept; for a reciprocal, the asymptotes; for a trig curve, max/min and axis crossings within one period.
  6. Record these anchor points as an explicit list ready to be transformed or plotted, so the next part of the question becomes mechanical.

Mechanic — Concrete operations

  1. Identify the curve type and list which feature classes define it (intercepts, turning points, vertices, asymptotes).
  2. From a given sketch: read each labelled coordinate directly, e.g. (0,0), max (1,2), min (3,0).
  3. From a modulus expression y = |g(x)|: solve g(x)=0 to find the vertex x-coordinate, with y=0 there.
  4. Compute the y-intercept by substituting x=0 into the original expression and applying the modulus if present.
  5. If x-intercepts are needed for an inverted modulus (e.g. y = k − |...|), solve |...| = k.
  6. Record the resulting coordinates as the anchor list: these are the points to be mapped under the requested transformation or plotted on the new sketch.

Worked Micro-Example

Sketch y = |2x − k/3|. Set interior to zero: $2x - k/3 = 0 \Rightarrow x = k/6$, so the vertex is $(k/6,\,0)$. At $x=0$: $y = |-k/3| = k/3$, giving y-intercept $(0,\,k/3)$. These two anchor points plus gradients $\pm 2$ fully determine the V-shape sketch.

Variants — Same move, different surface

Curve given graphically, asked to sketch y = f(x+a) + b or y = af(bx)
Apply the transformation rule to each anchor point individually: (x,y) → (x−a, y+b) for translations, (x/b, ay) for stretches.
Reciprocal or rational curves with asymptotes
Treat asymptote lines themselves as features: transform them alongside intercepts (e.g. vertical asymptote x=c becomes x=c−a under a horizontal translation).
Trigonometric curves over one period
Anchor points become period endpoints, max/min within a period, and axis crossings; transform each.
Inverted modulus y = k − |ax+b|
Vertex is the maximum at x = −b/a with y = k; x-intercepts come from solving |ax+b| = k.

Common Misreadings

  • Trying to find an explicit algebraic equation of f(x) from a sketch and then substituting — wastes time and risks errors when the question only tests transformation of points.
  • Treating a modulus graph as a single smooth curve and missing that the vertex is a non-differentiable corner that must be labelled.

Pitfalls

  • Misreading coordinates from the original sketch, e.g. swapping the maximum and minimum points.
  • Sign error when zeroing the modulus interior (e.g. writing x = −k/6 instead of k/6).
  • Forgetting the modulus when computing the y-intercept (writing y = −k/3 instead of k/3).
  • Omitting the y-intercept when it coincides with the origin, leaving the sketch under-labelled.

A-level Scope Notes

Stays within Edexcel IAL P2/P3 graph-transformation and modulus-function content; no formula booklet entry is required. Avoid generalising to piecewise reasoning beyond a single modulus or to non-elementary asymptote analysis.
