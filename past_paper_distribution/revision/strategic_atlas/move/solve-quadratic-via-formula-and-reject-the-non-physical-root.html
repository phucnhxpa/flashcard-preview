Algebra & Functions

Solve quadratic via formula and reject the non-physical root

Family: Root Selection Frequency: 2 questions

Trigger — When to use

A context-based problem (geometry, mechanics, counting, finance) reduces to a quadratic ax² + bx + c = 0 in which the unknown represents a physically constrained quantity — a length, a time, a count of objects, a positive radius, etc. The coefficients are awkward (large constant term, c < 0, no obvious factor pair), so factorisation by inspection is unreliable. Equivalent cues: 'find the number of rows/terms n such that…', 'find the length x such that…', 'the area/perimeter is …, find x'. The quadratic formula is the safe, fast tool, and the modelling context will force you to discard one root.

Motivation — Why this works

The quadratic formula always returns both roots, even when the algebra is messy or irrational. In an applied problem the variable carries physical meaning, so any root that violates that meaning (negative length, negative time, non-integer count, value outside a stated domain) must be discarded — this is a modelling decision, not algebraic taste. When c/a < 0 the product of roots is negative, guaranteeing one positive and one negative root, so the rejection step is built into the structure. Examiners reward the explicit justification 'reject because …', not just the surface choice.

Thought Process — The step-by-step reasoning

  1. Notice the equation has the form ax² + bx + c = 0 with a large or awkward constant term (e.g. −1073, −1500), so scanning for integer factor pairs of |c| that sum to b is slow and error-prone.
  2. Rule out factorisation: either no obvious pair exists, or the discriminant may not even be a perfect square, in which case factorisation is impossible anyway.
  3. Commit to the quadratic formula because it is guaranteed to deliver both roots and immediately exposes whether the discriminant is a perfect square (clean integer answer) or irrational (decimal answer to suitable precision).
  4. Read the variable's meaning from the question: is it a length (>0), a time (≥0), a count (positive integer), a parameter on a curve (within a stated interval)? This determines which root to keep.
  5. Compute Δ = b² − 4ac carefully, evaluate √Δ (test nearby perfect squares if you suspect an integer answer), then write both roots explicitly before selecting.
  6. State the rejection in words: 'Since x is a length, x > 0, so reject x = …' — this secures the modelling mark.

Mechanic — Concrete operations

  1. Identify a, b, c from ax² + bx + c = 0.
  2. Compute the discriminant Δ = b² − 4ac.
  3. Evaluate √Δ; if you expect an integer answer, test nearby squares (e.g. 66² = 4356).
  4. Write both roots: x = (−b + √Δ)/(2a) and x = (−b − √Δ)/(2a).
  5. Identify the physical constraint on x (positive, integer, within a domain).
  6. Reject the root that violates the constraint and state the reason explicitly.
  7. Carry sufficient decimal precision forward if the surviving root is irrational.

Worked Micro-Example

Solve $n^2 + 8n - 1073 = 0$ where $n$ is a number of rows. $\Delta = 64 + 4292 = 4356 = 66^2$. So $n = \frac{-8 \pm 66}{2} = 29 \text{ or } -37$. Since $n$ is a count of rows, $n > 0$, so reject $n = -37$. Hence $n = 29$.

Variants — Same move, different surface

Length/radius problem with irrational discriminant (e.g. 2x² + 5x − 1500 = 0)
Use the formula, keep √Δ to 4–5 s.f., reject the negative root because x is a length, then propagate the decimal value into the next calculation without premature rounding.
Projectile/kinematics problem solving for time t
Same logic — reject t < 0 because time after release is non-negative; sometimes also reject t = 0 if it corresponds to the launch instant.
Counting problem (rows, terms of a series, number of objects)
Reject negative AND non-integer roots; if Δ is not a perfect square the model is likely wrong — re-check the set-up.
Logarithmic equation reducing to a quadratic in y = log x or in x itself
Reject any root that makes a log argument ≤ 0; the rejection is on domain grounds rather than physical grounds.
Coordinate-geometry intersection parameter
Reject roots outside the stated parameter interval (e.g. t ∈ [0, 1] for a line segment).

Common Misreadings

  • Trying to factorise a quadratic with irrational roots, wasting exam time.
  • Assuming both roots are valid and giving a set answer when the context demands a single value.
  • Treating the rejection as optional algebraic tidiness rather than a required modelling justification.

Pitfalls

  • Arithmetic slip in the discriminant — e.g. mis-adding 64 + 4292, or sign error when c is negative so −4ac becomes positive.
  • Failing to recognise a perfect square (test 60² = 3600, 70² = 4900, narrow to 66² = 4356).
  • Rounding √Δ too early so the final perimeter/area answer is off in the last significant figure.
  • Forgetting to write 'reject because …' — examiners deduct the modelling mark for an unjustified choice.
  • Keeping the 'positive' root without checking a second contextual constraint (e.g. outer radius must exceed inner radius, or n must be an integer).

A-level Scope Notes

Fully within Edexcel IAL Pure Maths (P1 quadratic formula, formula booklet entry x = (−b ± √(b² − 4ac))/(2a)) and reused in P2/P3 modelling, M1 kinematics, and S/D contexts. Do not invoke complex roots — if Δ < 0 in an applied context the model is wrong, not the algebra.
