Vectors

Divide normal vector by its GCD before plane and distance work

Family: Scalar Simplification Frequency: 1 questions

Trigger — When to use

Fires immediately after computing a normal vector (typically via cross product) when all components share an obvious common integer factor. Strong cues: components like (-42, 14, -21), (8, -4, 12), or any triple where a small prime divides each entry. Especially valuable when the question has follow-up parts requiring the plane equation, point-to-plane distance, or angle calculations where |n| appears in a denominator.

Motivation — Why this works

Any non-zero scalar multiple kn defines the same plane and the same direction perpendicular to it, so replacing n by n/gcd preserves all geometric content. Smaller integers reduce arithmetic load and the chance of sign/multiplication slips, and often produce a clean |n| (e.g. √49 = 7) that signals the simplification was intended by the examiner. The distance formula |n·(P−A)|/|n| is invariant under this rescaling, so no geometric information is lost.

Thought Process — The step-by-step reasoning

  1. I notice every component of the normal shares a visible common factor (here, 7 divides 42, 14, 21).
  2. I rule out keeping the raw normal — large components will inflate every subsequent dot product and the |n| computation.
  3. I commit because n and (1/k)n define the same plane and the same unit normal direction; nothing is lost.
  4. I divide by ±gcd, choosing the sign that leaves a tidy leading positive component if convenient.
  5. I recompute d = n·a using a point on the plane with the new n, rather than rescaling the old d (cleaner and avoids errors).

Mechanic — Concrete operations

  1. Compute gcd of |components| of the normal n.
  2. Divide each component by ±gcd to obtain a reduced normal n'.
  3. Verify |n'| simplifies cleanly (e.g. √(6²+2²+3²) = 7).
  4. Recompute the plane constant d' = n'·a using a known point a on the plane.
  5. Use n' and d' in all subsequent distance, angle, and intersection calculations.

Worked Micro-Example

Cross product gives $\mathbf{n} = (-42, 14, -21)$. Factor out $-7$: $\mathbf{n}' = (6, -2, 3)$, with $|\mathbf{n}'| = \sqrt{36+4+9} = 7$. Using point $A=(1,2,1)$ on the plane: $d' = 6(1)-2(2)+3(1) = 5$, so the plane is $6x-2y+3z = 5$. Distance from $P=(4,0,2)$ is $|6(4)-2(0)+3(2)-5|/7 = 25/7$.

Variants — Same move, different surface

Direction vector of a line has a common factor
Divide before writing the parametric/vector form so parameter values stay small integers.
Two normals being compared for parallelism
Reduce both to lowest terms; planes are parallel iff the reduced normals are equal up to sign.
Cross product yields a vector with a √ that factors
Pull the scalar outside before normalising; e.g. (2,4,4) → 2(1,2,2), |.|=6.
Computing angle between planes/lines
Scalar factors cancel in cos θ = (n₁·n₂)/(|n₁||n₂|), but reduced vectors still make the dot product cleaner.

Common Misreadings

  • Thinking the sign of n matters for the plane — it does not; n and −n define the same plane.
  • Believing rescaling changes the distance from a point to the plane — it does not, because |n| in the denominator scales identically.
  • Assuming the GCD must be positive — factoring out a negative is fine and often tidier.

Pitfalls

  • Forgetting to recompute d after rescaling n; reusing the old d gives a wrong plane equation.
  • Dividing only some components (arithmetic slip) — verify each component is exactly divisible first.
  • Cancelling a factor that isn't actually common to all three components.

A-level Scope Notes
