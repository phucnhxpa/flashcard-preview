Trigonometry

Enumerate tan solutions using 180° periodicity across the range

Family: Solution Enumeration Frequency: 1 questions

Trigger — When to use

After reducing an equation to the form tan x = k (or tan x = ±k) and being asked for all solutions in an interval such as 0° ≤ x < 360° or 0 ≤ x < 2π. The cue is a single tan equation (not sin or cos) combined with a range wider than one period of tan. Each value of k then contributes exactly two solutions per 360° interval, spaced 180° apart.

Motivation — Why this works

The tangent function has period 180° (π radians), so once one solution x₀ = arctan(k) is found, every x₀ + 180°n is also a solution. Calculators return a principal value in (−90°, 90°), which may lie outside the required range and must be shifted by +180° (or +360°) to enter it. Using 180° steps — not 360° — is essential because tan repeats twice as often as sin or cos.

Thought Process — The step-by-step reasoning

  1. Notices the equation has collapsed to tan x = k with a stated range spanning more than 180°, so multiple solutions are expected.
  2. Rules out the sin/cos enumeration patterns (180° − x or ±x symmetry) because tan has its own simpler period-180° rule.
  3. Commits to: principal value from arctan, then add 180° repeatedly until leaving the range; repeat for each k value.
  4. Computes x₀ = arctan(k); if x₀ < 0 (or otherwise outside the range), shifts by +180° until inside.
  5. Lists all x₀ + 180°n in range, checks the count matches expectations (e.g. two per k), and verifies no solution hits a point where the original equation was undefined.

Mechanic — Concrete operations

  1. Compute principal value: x₀ = arctan(k).
  2. If x₀ is outside the required range, add 180° (or 360° in radians: π) until it enters the range.
  3. Generate further solutions by adding 180° repeatedly while staying inside the range.
  4. Repeat for each distinct value of k (e.g. both +√5 and −√5).
  5. Sort, round to required accuracy, and discard any solution where the original equation was undefined.

Worked Micro-Example

Solve tan x = √5 for 0° ≤ x < 360°. Principal value: x = arctan(√5) ≈ 65.9°. Add 180°: x ≈ 245.9°. Both lie in range, so solutions are x ≈ 65.9°, 245.9°. If instead tan x = −√5, principal value ≈ −65.9°; add 180° → 114.1°, then 294.1°.

Variants — Same move, different surface

sin x = k on [0°, 360°)
Use the pair x₀ and 180° − x₀ (then add 360° for further periods), since sin has period 360° with reflection symmetry about 90°.
cos x = k on [0°, 360°)
Use the pair x₀ and 360° − x₀ (i.e. ±x₀ mod 360°), since cos has period 360° and is even.
tan(ax + b) = k
Solve for the compound argument first across an extended range, generating period-180° solutions in (ax+b)-space, then unwind to x.
Range given in radians, e.g. 0 ≤ x < 2π
Replace the 180° step with π; principal value comes from arctan in (−π/2, π/2).

Common Misreadings

  • Treating tan as having period 360° and adding 360° instead of 180°, which loses half the solutions.
  • Stopping after the principal value plus one more, giving only two solutions when four are required.
  • Trying to apply the sin-style 180° − x reflection rule to tan, producing spurious solutions.
  • Failing to recognise that ± k yields two distinct sets of solutions, not just sign-flipped duplicates.

Pitfalls

  • Forgetting to shift a negative principal value into the required range before generating further solutions.
  • Failing to discard any candidate solution that makes the original equation undefined (e.g. where tan, sec, or cot blew up before simplification).
  • Rounding the principal value too early and propagating the error when adding 180°.
  • Mixing degree and radian modes on the calculator when the range is stated in one unit only.

A-level Scope Notes

Standard Edexcel IAL Pure content (P2/P3): solving trig equations on a stated interval using the periodicity and symmetry of sin, cos, tan; no inverse trig calculus or complex roots required.
