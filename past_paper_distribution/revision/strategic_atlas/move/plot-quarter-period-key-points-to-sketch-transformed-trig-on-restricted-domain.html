Trigonometry

Plot quarter-period key points to sketch transformed trig on restricted domain

Family: Curve Sketching Frequency: 1 questions

Trigger — When to use

Fires when asked to sketch y = a sin(bx + c) + d or y = a cos(bx + c) + d on a closed interval that is NOT an integer number of periods (e.g. [0°, 225°] for cos(2x°), period 180°). Also fires when a follow-up part asks 'how many solutions of f(x) = p' on that interval, since accurate counting requires the true endpoint behaviour. Cue phrases: 'sketch on the interval', 'state coordinates of maxima/minima/intersections with axes', 'hence find the number of solutions'.

Motivation — Why this works

A transformed sine/cosine is fully determined on any interval by its values at quarter-period points (max, zero, min, zero, max, …). Evaluating at these points and joining with the standard sinusoidal shape reproduces the curve exactly, including any partial hump or partial trough at the right-hand endpoint. This is essential when the domain ends mid-period because symmetry/period arguments alone misrepresent the tail and lead to miscounted intersections with horizontal lines y = p.

Thought Process — The step-by-step reasoning

  1. Notice the stretched period: for cos(bx°) the period is 360/b, and the domain length is not an integer multiple of it.
  2. Rule out 'just draw one period and copy it' — the final fragment is partial and must be handled explicitly.
  3. Commit to a key-point table: list x-values at every quarter-period from the left endpoint up to and including the right endpoint.
  4. Translate each x into the inner argument (e.g. 2x°) so the cosine/sine evaluation is a standard exact value (0, ±1).
  5. Mark each (x, y) on axes, then join with smooth sinusoidal arcs, paying attention to whether the endpoint is on a max, min, zero, or partway between.

Mechanic — Concrete operations

  1. Identify the period T = 360°/b (or 2π/b in radians) and quarter-period T/4.
  2. List x-values: x₀ = left endpoint, x₀ + T/4, x₀ + T/2, … up to and including the right endpoint.
  3. For each x compute the inner argument and evaluate sin/cos at that standard angle (0, ±1, or known exact value).
  4. Tabulate (x, y) pairs and label maxima, minima, and axis crossings with coordinates.
  5. Sketch axes scaled to the domain, plot the points, and join with smooth sinusoidal curves.
  6. Check the right-hand endpoint: confirm whether the curve is ascending or descending into it, since this affects intersection counts.

Worked Micro-Example

Sketch $y = \cos(2x^\circ)$ on $[0, 225]$. Period $= 180$, quarter-period $= 45$. Key points: $(0,1), (45,0), (90,-1), (135,0), (180,1), (225,0)$. Joining with cosine arcs gives one full wave on $[0,180]$ plus a descending quarter-wave from $(180,1)$ to $(225,0)$.

Variants — Same move, different surface

$y = a\sin(bx) + d$ on a non-period domain
Start key-point list at a zero of sin rather than a max; quarter-period spacing is identical.
Radian domain such as $[0, 5\pi/4]$ for $y = \sin(2x)$
Use $T = \pi$, quarter-period $\pi/4$; evaluate sin at multiples of $\pi/2$.
Phase-shifted curve $y = \cos(2x - 30^\circ)$
Solve $2x - 30 = 0, 90, 180, \ldots$ to locate quarter-period x-values, then evaluate.
Vertical translation $y = a\cos(bx) + d$
Same x-values; shift all y-values by $d$; max becomes $d+a$, min becomes $d-a$.
Counting solutions of $f(x) = p$ on the domain (follow-up)
Use the sketch with the partial hump/trough at the endpoint to count intersections with the horizontal line $y = p$ correctly.

Common Misreadings

  • Treating the domain as exactly one or two full periods and copying the standard graph blindly.
  • Computing only at integer multiples of the original (untransformed) period (e.g. every 360°) and missing internal extrema.
  • Confusing degrees and radians when evaluating the inner argument.

Pitfalls

  • Using $T = 360°$ instead of $360°/b$ for the transformed curve.
  • Forgetting to include the right-hand endpoint in the key-point list, so the partial final piece is omitted.
  • Mislabelling a zero as a maximum because the table was set up at half-period rather than quarter-period spacing.
  • Sketching straight-line segments between key points instead of smooth sinusoidal arcs, which distorts intersection counts.

A-level Scope Notes

Uses only standard transformations $y = a\sin(bx+c)+d$, $y = a\cos(bx+c)+d$ and exact values of sin/cos at multiples of $90°$ (or $\pi/2$), all within Edexcel IAL P2/P3 trig graph-sketching.
