Sequences & Series

Apply ceiling to a strict inequality to find minimum integer n

Family: Integer Constraint Frequency: 1 questions

Trigger — When to use

After log-manipulation of a sequence/series inequality (e.g. solving a^n > k or S_n > M), you obtain a non-integer bound such as n − 1 > 127.734 or n > 12.6. The question asks for the smallest/minimum integer n (term number, year number, number of payments) for which the condition holds. Cue words: 'least value of n', 'minimum n', 'after how many terms/years', 'first term to exceed'. The variable is constrained to be a positive integer because it counts discrete objects.

Motivation — Why this works

A strict inequality n > c with c non-integer is satisfied by the smallest integer strictly greater than c, i.e. ⌈c⌉. Rounding down or to the nearest integer breaks the inequality at the boundary. Because n indexes a discrete sequence, only integer values are admissible, so the algebraic bound must be promoted to its ceiling. If the algebraic variable is a shifted form like n − 1, the ceiling must be applied to that expression first, then unshifted to recover n.

Thought Process — The step-by-step reasoning

  1. Notice the solved inequality has a non-integer right-hand side, e.g. n − 1 > 127.734.
  2. Rule out simply rounding to the nearest integer or truncating: 127 fails the strict inequality.
  3. Commit to the ceiling because n must be a positive integer term-number and the inequality is strict.
  4. Bridge: take ⌈127.734⌉ = 128 as the smallest admissible value of n − 1.
  5. Unshift carefully — since the bound is on n − 1, add 1 to recover n = 129; sense-check by substituting n = 128 to confirm it fails.

Mechanic — Concrete operations

  1. Isolate the integer-valued expression on one side: (n − 1) > 127.734… (or n > c).
  2. Apply the ceiling: smallest integer strictly greater than 127.734 is 128.
  3. Reverse any shift: n − 1 = 128 ⟹ n = 129.
  4. Verify: substitute n = 129 (works) and n = 128 (fails) into the original condition.

Worked Micro-Example

Find the least n with $0.9^{\,n-1} < 10^{-6}$. Taking logs: $(n-1)\log 0.9 < -6$, so $n - 1 > \dfrac{-6}{\log 0.9} \approx 131.13$. Hence $n - 1 \ge 132$, giving minimum $n = 133$.

Variants — Same move, different surface

Compound interest / savings: 'after how many complete years will the value first exceed £M'
Solve $P(1+r)^n > M$, take logs, then ceiling on n directly (no shift).
Geometric series sum threshold: least n with S_n > L
After solving for n in the log inequality, apply ceiling; check whether the inequality is strict or non-strict.
Non-strict inequality $n \ge c$ with c an exact integer
Answer is exactly c, not c + 1; no ceiling promotion needed.
Decreasing sequence where log step flips inequality
Remember to reverse the inequality when dividing by a negative log (e.g. log 0.9 < 0), then apply ceiling.

Common Misreadings

  • Treating the bound on n − 1 as a bound on n and forgetting the +1 shift.
  • Reading 'minimum n' as 'round to nearest integer' rather than 'ceiling'.
  • Missing that the inequality is strict and using = at the decimal value.

Pitfalls

  • Truncating 127.734 down to 127 and reporting n = 128.
  • Forgetting to flip the inequality sign when dividing by log of a number less than 1 (negative).
  • Reporting the ceiling of n − 1 (i.e. 128) as the final answer instead of 129.

A-level Scope Notes

Standard Edexcel IAL P2/P3 sequences and series content: geometric sequences, sums to n terms, and logarithmic solving of inequalities involving an integer index n.
