Trigger — When to use

Fires when a logarithmic equation has been (or can be) reduced to the form log_b(A) = log_b(B), with the SAME base on both sides and a SINGLE logarithm on each side. Visual cues: (i) every term is a logarithm (no stray constants outside logs), (ii) bases match (all log₃, all ln, etc.), (iii) after applying product/quotient/power rules you can collapse each side to one log. Alternative phrasings of the trigger: 'two log terms separated by =', 'ln(...) = ln(...)', 'log_b(f(x)) = log_b(g(x))'. If a constant sits outside a log (e.g. log₃(x) = 2), this move does NOT directly apply — first rewrite the constant as a log (2 = log₃ 9) or exponentiate.

Motivation — Why this works

Because log_b is a strictly monotonic (one-to-one) function on (0, ∞), the implication log_b(A) = log_b(B) ⇒ A = B holds. This single inference strips ALL logarithms from the equation, converting a transcendental equation into a purely algebraic one (typically polynomial or rational). It is the terminal solving step of essentially every same-base log equation, and is the reason for all prior consolidation work using the product, quotient, and power laws. Equating arguments is preferred over exponentiating both sides (b^LHS = b^RHS) because it is a single step rather than two.

Thought Process — Step-by-step reasoning

  1. Notice the equation contains only logarithmic terms with the same base, possibly with coefficients (e.g. 2 ln(2x+1) = ln 9) or sums (e.g. log₃(A) = log₃(B) + log₃(C)).
  2. Rule out exponentiating both sides — it works but is slower and reintroduces base-b expressions you'd then have to simplify.
  3. Rule out 'dividing by ln' or 'cancelling log_b' — log is a function, not a multiplicative factor, so this is meaningless.
  4. Commit to consolidating each side into a SINGLE logarithm using the laws: a log_b(X) → log_b(X^a) (power rule), log_b(X) + log_b(Y) → log_b(XY) (product rule), log_b(X) − log_b(Y) → log_b(X/Y) (quotient rule).
  5. Once both sides read log_b(A) = log_b(B), invoke injectivity to write A = B — the logarithms vanish in one stroke.
  6. Before declaring solutions, check the domain of the ORIGINAL equation: every argument inside an original log must be > 0. Reject any candidate solution that violates this.

Mechanic — Concrete operations

  1. Apply the power rule in reverse to absorb coefficients: c·log_b(u) = log_b(u^c).
  2. Apply product/quotient rules to fuse sums/differences: log_b(u) ± log_b(v) = log_b(uv) or log_b(u/v).
  3. Confirm both sides now have the form log_b(·) with the same base b.
  4. Equate arguments: A = B.
  5. Solve the resulting polynomial / rational / surd equation by standard algebra.
  6. Substitute each candidate back into the ORIGINAL log expressions; reject any that make an argument ≤ 0.

Worked Micro-Example

Solve $2\ln(2x+1) = \ln 9$. Rewrite using the power rule: $\ln(2x+1)^2 = \ln 9$. Equate arguments: $(2x+1)^2 = 9$, so $2x+1 = \pm 3$. Domain requires $2x+1 > 0$, so $2x+1 = 3$, giving $x = 1$.

Variants — Same move, different surface

Sum of logs equals a single log, e.g. log₃((1−x)²) + log₃(27) = log₃(32 − 12x)
Use the product rule on the sum first: log₃(27(1−x)²) = log₃(32 − 12x), then equate arguments to get 27(1−x)² = 32 − 12x and expand.
A constant appears on one side, e.g. log_b(f(x)) = c
Rewrite c as log_b(b^c) so both sides are single logs, OR exponentiate: f(x) = b^c. Same underlying principle.
Coefficient on a log, e.g. 3 ln x = ln 27
Power rule first: ln(x³) = ln 27, hence x³ = 27, x = 3.
Sum of logs equals a log of a constant, e.g. ln(x+1) + ln(x−1) = ln 8
Product rule: ln((x+1)(x−1)) = ln 8, so x² − 1 = 8, x² = 9. Domain forces x = 3 (reject x = −3).

Common Misreadings

  • Treating ln/log as a multiplicative factor and 'dividing it out', e.g. writing 2x+1 = 9/2 from 2 ln(2x+1) = ln 9.
  • Equating arguments when bases differ (e.g. log₂(A) = log₃(B)) — the injectivity argument requires the SAME base.
  • Equating arguments before consolidating: e.g. from log(A) + log(B) = log(C), wrongly writing A + B = C instead of AB = C.

Pitfalls

  • Failing to enforce the domain: candidate solutions must keep every original log argument positive. Spurious roots like 2x+1 = −3 (giving x = −2) must be rejected.
  • Sign errors when expanding and rearranging the resulting polynomial — e.g. mishandling −12x and −54x when moving terms across the equals sign.
  • Forgetting to expand (1−x)² before multiplying by a constant, leading to algebraic slips.
  • Not verifying the final answer in the ORIGINAL equation, especially when squaring or taking roots introduces extraneous solutions.

A-level Scope Notes

Fully within Edexcel IAL P2/P3 scope. Uses the laws of logarithms (product, quotient, power) listed in the formula booklet under 'Logarithms and exponentials'. The injectivity of log_b is assumed implicitly — students need not prove it. Avoid generalisations involving complex logarithms or change-of-base manipulations beyond the standard formula log_b(x) = ln x / ln b.
