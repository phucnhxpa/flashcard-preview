Algebra & Functions

Multiply through by denominator to clear fractions into integer-coefficient form

Family: Format Conversion Frequency: 1 questions

Trigger — When to use

Fires when an equation in point-slope form (or any linear/affine form) contains a fractional coefficient — most often a gradient like -4/3, 5/2, or 1/n — and the question demands the answer as ax + by + c = 0 with integer coefficients (often with a or a, b, c specified to be integers). Also triggers in perpendicular bisector, normal-line, and chord-equation tasks where the gradient is a ratio. Look for phrasing such as 'where a, b, c are integers', 'in the form ax + by + c = 0', or 'with integer coefficients'.

Motivation — Why this works

Multiplying both sides of y - y₁ = (p/q)(x - x₁) by q immediately removes the only denominator before expansion, so every subsequent term is automatically an integer. This works because multiplication by a non-zero constant preserves the solution set of an equation. Doing it pre-expansion is strictly fewer operations than expanding with a fraction and then clearing term-by-term, and it eliminates a common source of arithmetic slips.

Thought Process — The step-by-step reasoning

  1. Notices the gradient is a fraction p/q, so the point-slope equation will carry a denominator q.
  2. Rules out distributing the fraction first (would force fractional intermediate terms) and rules out leaving the answer in y = mx + c form (target form is ax + by + c = 0 with integers).
  3. Commits to scaling the whole equation by q up front, since one global multiplication removes all fractions in a single move.
  4. Bridges to the first operation: write y - y₁ = (p/q)(x - x₁), then multiply both sides by q to obtain q(y - y₁) = p(x - x₁).
  5. Inside the move: expand both sides, collect all terms on one side, and, if needed, divide through by any common factor or multiply by -1 so the leading coefficient a is positive (a common house-style requirement).

Mechanic — Concrete operations

  1. Write the point-slope equation y - y₁ = (p/q)(x - x₁).
  2. Multiply both sides by q: q(y - y₁) = p(x - x₁).
  3. Expand both brackets fully.
  4. Move every term to one side to reach ax + by + c = 0.
  5. If a < 0, multiply through by -1; if a, b, c share a common factor, divide through to give the simplest integer form.

Worked Micro-Example

Tangent at (-3, 12) with gradient -4/3: $y - 12 = -\tfrac{4}{3}(x+3)$. Multiply by 3: $3(y-12) = -4(x+3)$, so $3y - 36 = -4x - 12$. Rearrange: $4x + 3y - 24 = 0$.

Variants — Same move, different surface

Perpendicular bisector with gradient 2/5 through midpoint (1, -2)
Multiply by 5 before expanding: 5(y+2) = 2(x-1), then rearrange to 2x - 5y - 12 = 0.
Normal to a curve where dy/dx = 3/4 at the point, so normal gradient is -4/3
Same denominator-clearing step on the normal line equation; identical mechanic.
Equation involving two fractional terms, e.g. y/2 - x/3 = 1
Multiply by lcm(2,3) = 6 to clear both denominators simultaneously: 3y - 2x = 6, i.e. 2x - 3y + 6 = 0.
Chord/line through two given points giving gradient as a ratio (Δy/Δx)
Use the Δx as the multiplier on the point-slope form before expanding.

Common Misreadings

  • Treating the target form ax + by + c = 0 as flexible and leaving fractional coefficients.
  • Reading the gradient -4/3 as -4 only, dropping the denominator entirely.
  • Assuming the answer must have a = 1, and dividing through inappropriately.

Pitfalls

  • Forgetting to multiply every term by q — especially the constant on the LHS like -y₁.
  • Sign errors when transposing -4x to the other side (should become +4x).
  • Leaving a negative leading coefficient when the question implicitly expects a > 0.
  • Not reducing by a common factor, e.g. leaving 8x + 6y - 48 = 0 instead of 4x + 3y - 24 = 0.

A-level Scope Notes

Standard Edexcel IAL Pure (P1/C1-style) coordinate geometry technique for straight-line, tangent, normal, and perpendicular bisector questions; no content beyond linear equation manipulation is required.
