Trigger — When to use

Fires whenever a question asks for the shortest (perpendicular) distance from a specific point P(x₀, y₀, z₀) to a plane Π, AND the plane is available (or readily obtainable) in Cartesian form ax + by + cz = d (or ax + by + cz + d = 0, or vector form r·n = k). Typical phrasings: 'find the shortest distance from the point... to the plane Π', 'find the perpendicular distance', 'show that the distance from Q to Π is...'. Often the point may contain a parameter (e.g. Q = (2, k, −7)), signalling that a later part will solve an equation/inequality in that parameter. Also fires when asked for the distance between two parallel planes (variant).

Motivation — Why this works

The formula D = |ax₀ + by₀ + cz₀ − d| / √(a² + b² + c²) is derived by projecting the displacement vector from any point on the plane to P onto the unit normal n̂ = (a, b, c)/|n|. It converts a geometric perpendicular-distance problem into pure substitution, bypassing the need to construct the foot of the perpendicular (i.e. parameterising a line through P in direction n and intersecting it with Π). When the Cartesian equation is already in hand, this is by far the most efficient and least error-prone route to a single numerical (or parametric) answer.

Thought Process — Step-by-step reasoning

  1. Notice the question asks specifically for the shortest/perpendicular distance from a given point to a plane — geometric distance from a 0-dim object to a 2-dim object.
  2. Check that the plane is (or can be put) in Cartesian form ax + by + cz = d; if it's in vector form r·n = k, read off coefficients directly from n and k.
  3. Rule out alternatives: I could write a line through P with direction n, intersect with Π to get the foot F, then compute |PF| — but that's ~3 extra steps and more arithmetic. I could also use a projection of AP onto n̂ for any A∈Π — equivalent to the formula but messier. Commit to the direct formula.
  4. Bridge to operation: line up the plane as ax + by + cz − d = 0 so the substitution yields a single signed scalar; then take |·| because distance is non-negative.
  5. Tactical care: if the point contains a parameter k, keep the numerator factorised (e.g. |−2k − 14| = 2|k + 7|) so the next part (often an equation/inequality in k) is clean.
  6. Sanity check: the denominator |n| = √(a²+b²+c²) should match any earlier simplification; rationalise if exact form is required.

Mechanic — Concrete operations

  1. Write the plane as ax + by + cz − d = 0 (or ax + by + cz + d = 0 — be careful which sign convention).
  2. Substitute the point P = (x₀, y₀, z₀) into the LHS to obtain the signed scalar S = ax₀ + by₀ + cz₀ − d.
  3. Compute |n| = √(a² + b² + c²).
  4. Form D = |S| / |n|.
  5. Simplify: factorise the numerator if a parameter is present; rationalise the denominator if the mark scheme expects exact simplified form.

Worked Micro-Example

Find the distance from Q(2, k, −7) to Π: 6x − 2y + 3z = 5. Substitute: $6(2) - 2(k) + 3(-7) - 5 = -2k - 14$. Normal magnitude: $\sqrt{36+4+9} = \sqrt{49} = 7$. Hence $D = \dfrac{|-2k-14|}{7} = \dfrac{2|k+7|}{7}$.

Variants — Same move, different surface

Plane given in vector form r·n = k
Read (a, b, c) from n and the constant from k; formula becomes |r₀·n − k| / |n|.
Plane written as ax + by + cz + d = 0
Numerator becomes |ax₀ + by₀ + cz₀ + d| (do NOT subtract d again).
Distance between two parallel planes ax + by + cz = d₁ and ax + by + cz = d₂
Same denominator √(a²+b²+c²); numerator becomes |d₁ − d₂|.
Point contains an unknown parameter and a follow-up part sets the distance equal to a value
Keep the numerator factorised as a multiple of |k − α|; solve |k − α| = (mark-scheme value) at the next stage.

Common Misreadings

  • Confusing this with the distance between two points (just |PQ|, no plane involved).
  • Confusing this with the distance from a point to a LINE in 3D, which requires |AP × d| / |d|, not the plane formula.
  • Treating a plane given in vector parametric form r = a + λb + μc as if it were Cartesian — must convert (via cross product b×c for the normal) first.

Pitfalls

  • Forgetting the absolute value, leading to a negative 'distance'.
  • Sign error from misreading the plane's constant: ax+by+cz=d versus ax+by+cz+d=0 use opposite signs in the numerator.
  • Computing |n| with a sign mistake (e.g. (−2)² written as −4) or forgetting to square-root.
  • Failing to rationalise the denominator when Edexcel expects exact simplified surd form.
  • When the point has a parameter, expanding |−2k − 14| as −2k − 14 without the modulus, dropping a needed case.

A-level Scope Notes

Within Edexcel IAL Further Pure (FP3) vectors. The formula d = |ax₀ + by₀ + cz₀ − d| / √(a²+b²+c²) is in the IAL Further Maths formula booklet under '3D Vectors'. Do not generalise to distances in higher dimensions or to non-flat surfaces — those are out of scope.
