Trigonometry

Select Sine Rule for SSA configuration to find an unknown angle

Family: Rule Selection Frequency: 2 questions

Trigger — When to use

Fires when a triangle problem gives two sides and a non-included angle (SSA), and the target is an angle. Concretely: you can pair one given side with the angle opposite to it, and a second given side sits opposite the unknown angle you want. The known angle is NOT sandwiched between the two known sides (that would be SAS, the Cosine Rule's territory). Equivalent phrasings: 'I have a side–angle matched pair plus another side', or 'I'm missing the third side but only need an angle'.

Motivation — Why this works

The Sine Rule a/sin A = b/sin B = c/sin C directly links each side to the sine of its opposite angle, so as soon as you have one matched (side, opposite angle) pair plus one more side, you can solve for the angle opposite that second side in a single equation. The Cosine Rule, by contrast, needs either SAS or SSS — here you'd first have to find the missing third side (often via a quadratic) before extracting the angle, which is wasted work. Picking the Sine Rule exploits the matched-pair structure already present in the data and gives the answer in one line.

Thought Process — The step-by-step reasoning

  1. Catalogue the data: list which sides and which angle are given, and identify the target angle.
  2. Test for SAS: is the known angle sandwiched between the two known sides? If yes, abandon Sine Rule and use the Cosine Rule. If no, the configuration is SSA.
  3. Test for the Sine Rule's prerequisite: can I form a matched (side, opposite angle) pair from the data, with the unknown opposite the other known side? If yes, the Sine Rule will isolate the unknown in one equation.
  4. Rule out doing the Cosine Rule first to find the third side — that route requires solving a quadratic and an extra step, so it's strictly more work for an angle target.
  5. Commit to the Sine Rule and write it in the form sin(unknown angle)/(side opposite unknown) = sin(known angle)/(side opposite known angle), so the unknown sine sits cleanly on the left.
  6. Flag the SSA ambiguous case: arcsin returns the acute value, but an obtuse solution (180° − that value) may also be valid. Use any contextual information in the question (e.g. 'obtuse angle', a diagram, or a triangle inequality check) to pick the correct one.

Mechanic — Concrete operations

  1. Identify the matched pair: known angle A opposite known side a, and unknown angle B opposite known side b.
  2. Write the Sine Rule: sin(B)/b = sin(A)/a.
  3. Rearrange: sin(B) = b · sin(A) / a.
  4. Evaluate the right-hand side numerically; check it lies in [−1, 1] (otherwise no triangle exists).
  5. Take arcsin to get the principal (acute) value of B.
  6. Decide acute vs obtuse using the question's context; if obtuse is required, replace B with 180° − B (or π − B in radians).

Worked Micro-Example

In $\triangle ABC$, $AB = 6.5$, $BC = 4.7$, $\angle BAC = 35^\circ$. Find $\angle ACB$.\\ Sine Rule: $\dfrac{\sin(\angle ACB)}{6.5} = \dfrac{\sin 35^\circ}{4.7}$.\\ So $\sin(\angle ACB) = \dfrac{6.5 \sin 35^\circ}{4.7} \approx 0.7932$.\\ Hence $\angle ACB \approx \arcsin(0.7932) \approx 52.5^\circ$ (acute solution).

Variants — Same move, different surface

Known angle is between the two known sides (SAS)
Switch to the Cosine Rule: $a^2 = b^2 + c^2 - 2bc\cos A$ to find the third side first, then if needed use Sine Rule for remaining angles.
Angle given in radians (e.g. ∠AOD = 0.7 rad)
Same setup, but keep the calculator in radian mode; the arcsin output is in radians too.
Question states the target angle is obtuse
Take the principal arcsin value, then output $\pi - \theta$ (or $180^\circ - \theta$) as the answer.
Target is a side, not an angle, but the data is still SSA
Apply Sine Rule first to find the unknown angle, then use angle sum to find the third angle, then Sine Rule again for the missing side.

Common Misreadings

  • Defaulting to the Cosine Rule whenever 'find the angle' appears, even though the third side isn't known.
  • Treating SSA as SAS by miscounting which angle lies between which sides, especially when vertex labels are non-standard (e.g. AOD instead of ABC).
  • Forgetting that the Sine Rule needs a matched (side, opposite angle) pair — trying to pair a side with a non-opposite angle.

Pitfalls

  • Misidentifying which side is opposite which angle when the triangle's vertices are labelled unusually — always re-draw or re-label before writing the equation.
  • Forgetting the SSA ambiguous case: arcsin only returns the acute value, but the question may require the obtuse partner $180^\circ - \theta$.
  • Mixing degree and radian modes on the calculator when the given angle is in radians.
  • Failing to sanity-check that $b\sin(A)/a \le 1$; a value outside $[-1,1]$ signals no such triangle exists.

A-level Scope Notes

Sine Rule and Cosine Rule are in the Edexcel IAL formula booklet (Pure Mathematics, Trigonometry section). The ambiguous case of SSA is examinable at P2/P3 level. Avoid invoking the spherical or hyperbolic Sine Rule — these are out of scope.
