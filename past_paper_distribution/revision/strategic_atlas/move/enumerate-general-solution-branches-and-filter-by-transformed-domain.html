Trigonometry

Enumerate general-solution branches and filter by transformed domain

Family: General Solution Selection Frequency: 2 questions

Trigger — When to use

Fires when an equation of the form cos(ax + α) = k or sin(ax + α) = k must be solved either for the smallest positive root or for all roots in a specified interval, especially after an R-formula or differentiation step has produced a shifted argument. Visual cues: the variable appears inside a trig function with a phase shift (x − α, 2x − α, t − α), the domain is given for the outer variable (x or t) rather than the shifted argument, and the domain may exceed one period (2π or 360°). Equivalent phrasings include 'find the smallest positive value of x', 'solve for t in [0, T]', or 'find all stationary points in the given range'.

Motivation — Why this works

The move works because every trig equation has an infinite family of solutions parametrised by an integer n, and only by listing several consecutive members of that family can we be sure not to miss a valid root. Concretely, cos θ = k gives θ = ±arccos(k) + 2nπ and sin θ = k gives θ = arcsin(k) + 2nπ or π − arcsin(k) + 2nπ. Substituting θ = ax + α (or similar) converts each branch into a linear equation in the original variable, and translating the domain on x into a domain on θ tells us exactly which integers n to test. This guarantees completeness without relying on a CAST-diagram intuition that can mislead when the argument is shifted or scaled.

Thought Process — The step-by-step reasoning

  1. I notice the equation has a shifted/scaled argument like cos(2x − α) = 0 or cos(t − α) = k, and the question asks for a specific x (smallest positive) or all solutions in an interval — not a general solution.
  2. I rule out direct CAST-diagram reading on x itself, because CAST applies to the trig argument θ = ax + α, not to x; I also rule out just taking arccos and adding α, because that yields only one branch.
  3. I commit to writing the full general solution in θ, then back-substituting to x, because this systematically captures every branch and every period.
  4. I translate the domain: if 0 ≤ x ≤ X, then the θ-domain is −α ≤ θ ≤ aX − α. I check the width of this interval against 2π to decide how many values of n to enumerate.
  5. I list candidates θ = +θ₀ + 2nπ and θ = −θ₀ + 2nπ (or the sin analogue) for n = −1, 0, 1, …, convert each to x = (θ + α)/a, and discard those outside the original domain.
  6. Finally I select the smallest positive x (or report all valid x), keeping enough decimal places throughout to satisfy the 3 s.f. tolerance.

Mechanic — Concrete operations

  1. Identify the principal value θ₀ = arccos(k) (or arcsin(k)), keeping full calculator precision.
  2. Write the general solution: for cos, θ = ±θ₀ + 2nπ; for sin, θ = θ₀ + 2nπ or θ = π − θ₀ + 2nπ.
  3. Substitute θ = ax + α and rearrange: x = (θ + α)/a (or (θ − α)/a depending on sign convention).
  4. Translate the x-domain into a θ-domain to determine which integers n to test, ensuring the range covers more than one period if the θ-interval exceeds 2π.
  5. Enumerate n = …, −1, 0, 1, … for each branch, computing each candidate x and rejecting those outside the stated x-domain.
  6. Report the smallest positive x, or list all surviving x-values, rounded to the required accuracy (typically 3 s.f.).

Worked Micro-Example

Solve $\cos(2x - 1.107) = 0$ for the smallest positive $x$. General solution: $2x - 1.107 = \tfrac{\pi}{2} + n\pi$, so $x = \tfrac{1.107 + \pi/2 + n\pi}{2}$. Testing $n = -1$: $x \approx -0.232$ (reject). $n = 0$: $x \approx 1.34$. Smallest positive value: $x \approx 1.34$.

Variants — Same move, different surface

For sin(ax + α) = k instead of cos
Use the two branches θ = arcsin(k) + 2nπ and θ = π − arcsin(k) + 2nπ; the filtering procedure is identical.
Domain given in degrees with a shifted argument
Replace 2π with 360° throughout; principal value comes from arccos/arcsin in degree mode.
θ-domain exceeds one full period (e.g. spans more than 2π)
Test both n = 0 and n = 1 (and possibly n = −1) on each branch — expect more than two solutions.
Equation reduces to cos θ = 0 or sin θ = 0 (special k)
Use the compact form θ = π/2 + nπ (for cos) or θ = nπ (for sin) — a single family rather than two.

Common Misreadings

  • Treating the trig argument as if it were x and applying CAST directly, leading to an incorrect domain for the principal value.
  • Taking only the arccos/arcsin principal value and stopping, missing the negative-branch or next-period solutions.
  • Assuming a domain like [0, 2π] always gives exactly two solutions, without checking how the shift α moves the effective θ-interval.

Pitfalls

  • Only testing n = 0 and missing a smaller positive root from n = −1 or from the second branch (e.g. −arccos(k) + 2π).
  • Rounding α or θ₀ too early, causing the final x to drift outside the 3 s.f. tolerance.
  • Forgetting that with argument 2x (or generally ax), solutions in x are spaced π/a apart, not π apart — so more solutions fit inside the same x-domain.
  • Mis-signing the back-substitution: if the argument is t − α, then t = θ + α (add α); a sign error here shifts every answer.

A-level Scope Notes

Fully within Edexcel IAL Pure Maths (P2/P3) and uses standard general solutions of sin and cos equations; the R-formula (a cos x + b sin x = R cos(x ∓ α)) appears in the formula booklet. Avoid invoking complex-exponential or multi-angle generalisations beyond the stated identities.
