Reuse direction vector to build parallel line through new point

Trigger — When to use

Fires when a question asks for the equation of a line $l_2$ that passes through a stated point $P$ and is parallel to a given line $l_1$. Phrasings include 'parallel to $l_1$', 'in the same direction as $l_1$', or 'has the same direction vector as'. The cue is that no second point on $l_2$ is provided, but a direction is already available from $l_1$. Recognise that parallelism alone fully determines the direction, so no fresh computation is needed.

Motivation — Why this works

Two lines are parallel iff their direction vectors are scalar multiples of each other, so the simplest valid choice is to copy $l_1$'s direction verbatim. A line in 3D is uniquely specified by one point and one non-zero direction vector, so pairing $P$ with the inherited direction $\mathbf{d}$ closes the problem immediately. This avoids needless work like finding a second point on $l_2$ or recomputing a direction from coordinates. It also keeps the answer in the cleanest form for any follow-up (intersections, angles, distances).

Thought Process — Step-by-step reasoning

  1. Notice the word 'parallel' — this is a direction constraint, not a position constraint.
  2. Rule out finding a second point on $l_2$ or computing $\overrightarrow{AP}$-type directions; those would only be needed without a parallel reference.
  3. Commit to the template $\mathbf{r} = \text{point} + \lambda(\text{direction})$ with the direction lifted directly from $l_1$.
  4. Read $\mathbf{d}$ off $l_1$ as the coefficient of its parameter (not its position vector), and read $P$ from the question.
  5. Use a fresh parameter letter (e.g. $t$ or $\mu$) distinct from $l_1$'s parameter, especially if a later part asks about intersections.

Mechanic — Concrete operations

  1. Read the direction vector $\mathbf{d}$ off $l_1$ as the coefficient of its parameter.
  2. Read the anchor point $P$ from the question and write its position vector $\mathbf{p}$.
  3. Write $l_2: \mathbf{r} = \mathbf{p} + t\,\mathbf{d}$ with a fresh parameter $t$.
  4. (Optional) Simplify $\mathbf{d}$ by removing common factors if a tidier form is preferred.

Worked Micro-Example

Given $l_1: \mathbf{r} = \begin{pmatrix}2\\0\\-1\end{pmatrix} + \mu\begin{pmatrix}-5\\4\\3\end{pmatrix}$ and $P(1,5,2)$, the direction of $l_1$ is $\mathbf{d}=(-5,4,3)$. Since $l_2 \parallel l_1$, reuse $\mathbf{d}$ and anchor at $P$: $$l_2:\ \mathbf{r} = \begin{pmatrix}1\\5\\2\end{pmatrix} + t\begin{pmatrix}-5\\4\\3\end{pmatrix}.$$

Variants — Same move, different surface

$l_2$ is perpendicular to $l_1$ in 3D and must lie in a given plane
Find a new direction perpendicular to $\mathbf{d}_{l_1}$ using a dot-product condition or a cross product with the plane's normal — direction cannot be reused.
$l_2$ is parallel to a plane $\Pi$ rather than to a line
Direction is not unique; you need an additional constraint (e.g. another point or perpendicularity) to pin it down. Parallelism to a plane only forces $\mathbf{d}\cdot\mathbf{n}=0$.
Direction of $l_1$ is given via two points $A$ and $B$
First compute $\mathbf{d}=\overrightarrow{AB}=\mathbf{b}-\mathbf{a}$, then reuse this $\mathbf{d}$ at $P$.
$l_1$ given in Cartesian/symmetric form $\frac{x-x_0}{a}=\frac{y-y_0}{b}=\frac{z-z_0}{c}$
Read $\mathbf{d}=(a,b,c)$ from the denominators, then write $l_2$ in vector form through $P$.
Question asks for $l_2$ in Cartesian form
After writing $\mathbf{r}=\mathbf{p}+t\mathbf{d}$, eliminate $t$ to get $\frac{x-p_1}{d_1}=\frac{y-p_2}{d_2}=\frac{z-p_3}{d_3}$.

Common Misreadings

  • Treating $l_1$'s position vector as its direction vector and copying that instead.
  • Reading 'parallel' as 'coincident' and trying to force $P$ to lie on $l_1$.
  • Assuming 'parallel to $l_1$' means parallel to a particular axis or to the plane $l_1$ lies in.
  • Thinking a new direction must be computed because $P$ is a new point.

Pitfalls

  • Picking up the position vector of $l_1$ instead of the parameter coefficient as $\mathbf{d}$.
  • Reusing the same parameter letter as $l_1$ (e.g. $\mu$ on both lines), causing confusion in later intersection work.
  • Scaling $\mathbf{d}$ inconsistently and then misquoting it later when computing angles or checking parallelism.
  • Writing $l_2$ as $\mathbf{r}=\mathbf{d}+t\mathbf{p}$ — swapping anchor and direction roles.

A-level Scope Notes

Fully within Edexcel IAL Core Pure / FP vectors: vector equation of a line in 3D, parallelism via shared direction vector, and conversion between vector and Cartesian forms.
