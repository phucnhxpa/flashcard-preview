Strategic Atlas / Vectors / Perpendicular Foot Via Dot Product
Vectors

Parametrise point on line and impose dot-product perpendicularity to find foot

Family: Perpendicular Foot Via Dot Product Frequency: 3 questions

Trigger — When to use

Fires whenever the question asks for the foot of the perpendicular from a known external point P (often the origin, or a point on another line) to a given line l with direction vector d. Equivalent phrasings: 'find the point on l closest to P', 'shortest distance from P to l', 'find the point B on l such that PB is perpendicular to l', or 'find the projection of P onto l'. The structural cue is a fixed external point plus a parametric line — you need ONE specific point on l characterised by orthogonality.

Motivation — Why this works

The shortest distance from a point to a line is realised along the perpendicular from that point to the line. Since d is the direction of l, the foot B satisfies PB · d = 0 — the algebraic translation of 'PB ⊥ d' via the orthogonality criterion for the dot product. Writing B as a general point on l in terms of one parameter μ turns PB · d = 0 into a single linear equation in μ, which pins down the unique foot directly. This bypasses calculus-based minimisation of |PB|² (which gives the same answer but more algebra) and is more transparent than projection formulas at A-level.

Thought Process — The step-by-step reasoning

  1. I notice the question gives me a fixed point P and a line l with direction d, and asks for the closest point / foot of perpendicular / projection — all geometric synonyms for the same construction.
  2. I consider alternatives: I could minimise |PB|² as a function of μ via calculus, or use a projection formula scalar = (AP · d)/|d|². Both are valid, but they generate more bookkeeping than the direct orthogonality equation.
  3. I commit to PB · d = 0 because it converts a geometric perpendicularity condition into a single linear equation in one unknown — minimal algebra, maximal clarity.
  4. Bridge to action: I parametrise B as a general point on l using its parametric form (e.g. B = a + μd), so every coordinate of B is linear in μ.
  5. I form PB = B − P (keep this direction; reversing it changes the sign of the equation but not its solution) and dot it with the line's DIRECTION vector d — not with the position vector of any point on l.
  6. I solve the resulting linear equation for μ, then substitute back into B's parametric form to obtain the explicit coordinates of the foot.

Mechanic — Concrete operations

  1. Identify the external point P and the line l: r = a + μd, with direction vector d.
  2. Write the general point on l as B = a + μd, so each component of B is a linear expression in μ.
  3. Form the vector PB = B − P, keeping μ as the unknown in each component.
  4. Impose PB · d = 0 — expand the dot product carefully, watching signs.
  5. Collect terms to get a single linear equation Aμ + C = 0, and solve for μ.
  6. Substitute the value of μ back into B = a + μd to obtain the coordinates of the foot.
  7. (Optional) Compute |PB| to obtain the shortest distance from P to l.

Worked Micro-Example

Find the foot of perpendicular from $P=(10,-7,0)$ to $l: \mathbf{r}=(2,0,9)+\mu(2,1,3)$. Write $Q=(2+2\mu,\mu,9+3\mu)$, so $\vec{PQ}=(-8+2\mu,\mu+7,9+3\mu)$. Impose $\vec{PQ}\cdot(2,1,3)=0$: $2(-8+2\mu)+(\mu+7)+3(9+3\mu)=14\mu+18=0$, giving $\mu=-9/7$. Substituting: $Q=(-4/7,-9/7,36/7)$.

Variants — Same move, different surface

External point is the origin O
Use OX · d = 0 directly with X = a + λd; the equation simplifies to (a · d) + λ(d · d) = 0.
Foot of perpendicular from a point on one line l₁ to another line l₂
First find the fixed point A by substituting the given parameter into l₁'s equation; then apply AB · d₂ = 0 with B parametrised on l₂.
Foot of perpendicular from a point P to a plane Π with normal n
Construct the line through P with direction n: r = P + t·n, substitute into the plane equation to find t, then compute the foot.
Shortest distance between two skew lines
Parametrise points on both lines, form the connecting vector, and impose perpendicularity to BOTH direction vectors — gives two linear equations in two unknowns.

Common Misreadings

  • Setting PB · OB = 0 or PB · (position vector of a point on l) = 0 instead of PB · d — the perpendicularity must be against the DIRECTION vector of the line.
  • Setting PB · PB = 0, which incorrectly forces PB = 0 (i.e. demands P lie on the line).
  • Applying a cross-product distance formula to find |PB| but forgetting the question asked for the actual point B.

Pitfalls

  • Sign errors when computing PB = B − P, especially with negative or fractional coordinates of P.
  • Arithmetic errors when expanding the dot product, particularly with double negatives like −2(−9 − 2μ).
  • Substituting a negative or fractional μ back into the parametric form and slipping a sign in one component.
  • Confusing which line's direction vector to use when two lines are involved (must be the line CONTAINING the foot).

A-level Scope Notes

Fully within Edexcel IAL Pure Maths (P3/P4 vectors) and Further Pure (FP3 for 3D extensions). Uses the orthogonality criterion a·b = 0 ⇔ a ⊥ b, which is in the formula booklet. Avoid invoking matrix projection operators or Gram–Schmidt — these are out of scope and unnecessary here.
