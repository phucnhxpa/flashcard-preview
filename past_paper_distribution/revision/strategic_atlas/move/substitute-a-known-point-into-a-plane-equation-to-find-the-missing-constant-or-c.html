Strategic Atlas / Vectors / Point On Plane Substitution
Vectors

Substitute a known point into a plane equation to find the missing constant or coordinate

Family: Point On Plane Substitution Frequency: 1 questions

Trigger — When to use

Fires whenever a plane is given (or just constructed) in scalar/Cartesian form n·r = d or ax + by + cz = d, and exactly one quantity is unknown — either the constant d, or a single coordinate of a point asserted to lie on the plane. Common cues: 'the plane contains the point...', 'the line lies in the plane', or 'find the value of k such that (...) lies in Π'. Also fires after you have just computed a normal via a cross product and need d to finish the Cartesian equation. The hallmark is: one linear equation, one unknown.

Motivation — Why this works

A point (x₀, y₀, z₀) lies on the plane n·r = d iff n₁x₀ + n₂y₀ + n₃z₀ = d — this is the defining membership condition of a plane. So substitution converts a geometric incidence statement into a single linear scalar equation. With every other quantity known, the unknown is isolated immediately, with no need for parametrisation, projection, or simultaneous equations. It is the cheapest possible use of the plane equation.

Thought Process — The step-by-step reasoning

  1. Notice that the plane's normal direction is already pinned down, so the only freedom left in n·r = d is the scalar d (or the missing coordinate).
  2. Rule out heavier machinery: no need to parametrise a line and intersect, no need for a second cross product — one incidence fact gives one scalar equation, which is exactly what is needed.
  3. Commit because the point named in the question (or any point on a line stated to lie in the plane) automatically satisfies n·r = d, so substitution is logically forced.
  4. Bridge: pick the simplest available point — typically the position vector of a line in the plane, or a stated point — to minimise arithmetic.
  5. Tactically, keep the unknown symbolic on one side and evaluate the numeric dot product on the other; then read off the answer in one line.

Mechanic — Concrete operations

  1. Identify the normal n = (n₁, n₂, n₃) and write the plane as n₁x + n₂y + n₃z = d.
  2. Choose a point (x₀, y₀, z₀) known to lie on the plane (a stated point, or a position vector of a contained line).
  3. Substitute to get n₁x₀ + n₂y₀ + n₃z₀ = d (or = right-hand side, if the unknown is a coordinate).
  4. Solve the resulting linear equation for the single unknown.
  5. State the completed plane equation or the found coordinate clearly.

Worked Micro-Example

Plane Π has normal n = (3, 1, −1) and contains the point A = (1, 1, 1). Then d = 3(1) + 1(1) + (−1)(1) = 3, so Π: 3x + y − z = 3. If a further point (2, s, 0) lies on Π, then 3(2) + s − 0 = 3 ⇒ s = −3.

Variants — Same move, different surface

Finding d after computing a normal via cross product of two direction vectors
Use any point on either line as the substitution point; n is the cross product result.
Determining an unknown coordinate of a point asserted to lie in a given plane
Plane equation is fully numeric; substitute and solve for the single missing coordinate.
Finding a parameter λ so that a point on a parametrised line lies in a plane
Substitute the parametric expressions for x, y, z into ax + by + cz = d and solve for λ.
Checking whether a given point lies on a plane (verification rather than solving)
Substitute and confirm both sides are equal; no unknown to solve for.
Plane given in vector form r = a + λb + μc, but a Cartesian test is needed
Convert to n·r = d first (using n = b × c and d = n·a), then substitute.

Common Misreadings

  • Treating a point that lies on a different line/plane in the question as if it lay on Π.
  • Confusing the plane's constant d with the magnitude of n or with the perpendicular distance from the origin.
  • Reading n·r = d as n·r = 0 and forcing the wrong condition (that would mean the plane passes through the origin).

Pitfalls

  • Sign errors when coordinates or normal components are negative — especially with terms like −1·z₀.
  • Mixing up which line a given point belongs to when the question features two lines.
  • Forgetting to write the final plane equation after computing d, losing the presentation mark.
  • Substituting a direction vector instead of a position vector — directions do not satisfy n·r = d.

A-level Scope Notes

Fully within Edexcel IAL Core Pure / FP3 vector geometry: scalar product form of a plane, Cartesian equation ax + by + cz = d, and incidence of points and lines with planes.
