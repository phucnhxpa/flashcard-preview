Strategic Atlas / Integration / Zero Analysis For Limits
Integration

Use positivity of one factor to reduce a product-equals-zero to a single equation

Family: Zero Analysis For Limits Frequency: 1 questions

Trigger — When to use

Fires when an area/integral question asks for the region between a curve and the x-axis but does not state the limits of integration, and the curve has the form y = f(x)g(x) where one factor is strictly positive (e.g. e^{kx}, e^{-kx}, 1+x^2, sech^2 x). Also fires when locating x-intercepts of a product of functions, or when checking the sign of an integrand on an interval. Cue phrases: "the region R bounded by the curve and the x-axis", "shaded region", "first positive root".

Motivation — Why this works

A product is zero iff at least one factor is zero. If one factor is provably non-zero on the relevant domain (most commonly an exponential, since e^{anything} > 0), then the zeros of the product coincide exactly with the zeros of the other factor. This collapses a potentially awkward equation into a standard trigonometric, polynomial, or logarithmic equation whose roots give the integration limits directly.

Thought Process — The step-by-step reasoning

  1. Notice that the limits of integration are not stated — they must come from x-intercepts of y.
  2. Rule out the exponential factor as a source of zeros: e^{2x} > 0 for all real x, so it can never vanish.
  3. Commit to solving the remaining equation sin x = 0, which carries all the zero information.
  4. Bridge to the concrete step: list non-negative roots x = 0, π, 2π, … and pick the two consecutive ones bounding the shaded region.
  5. Check the sign of the integrand on [0, π] (sin x ≥ 0, e^{2x} > 0, so y ≥ 0) to confirm no absolute value is needed.

Mechanic — Concrete operations

  1. Set y = 0, i.e. e^{2x} sin x = 0.
  2. State that e^{2x} > 0 for all real x, so e^{2x} ≠ 0.
  3. Reduce to sin x = 0, giving x = nπ, n ∈ ℤ.
  4. Select the consecutive roots that bound R; for the first positive lobe these are x = 0 and x = π.
  5. Verify y ≥ 0 on [0, π] so Area = ∫₀^π e^{2x} sin x dx (no modulus needed).

Worked Micro-Example

Find the area between y = e^{2x} sin x and the x-axis for the first lobe with x ≥ 0. Zeros: e^{2x} sin x = 0 ⟹ sin x = 0 (since e^{2x} > 0) ⟹ x = 0, π, 2π, …. On (0, π), sin x > 0 and e^{2x} > 0, so y > 0. Hence Area = ∫₀^{π} e^{2x} sin x \, dx, which is then evaluated by integration by parts twice.

Variants — Same move, different surface

y = x^2 e^{-x} between curve and x-axis
e^{-x} > 0, so zeros come from x^2 = 0; only zero is x = 0 (double root), so a second boundary must come from another condition (e.g. a stated upper limit or another curve).
y = e^{-x} cos x on [0, π/2]
Exponential positive, so zeros given by cos x = 0; first positive zero x = π/2 supplies the upper limit.
y = (1 + x^2) sin(2x)
1 + x^2 > 0 always, so zeros come solely from sin(2x) = 0, i.e. x = nπ/2.
Polar area with r = e^{θ} sin θ
Same logic: r = 0 ⟺ sin θ = 0, giving θ-limits for the loop.

Common Misreadings

  • Reading 'find the area' as a request to use the limits given elsewhere in the question, rather than deducing them from y = 0.
  • Treating e^{2x} sin x = 0 as two separate cases and attempting to solve e^{2x} = 0.
  • Assuming the region runs between the first two positive zeros (π and 2π) instead of the first lobe (0 to π).

Pitfalls

  • Failing to state e^{2x} > 0 explicitly — examiners often want this justification written down.
  • Choosing x = 2π as the upper limit, which encloses both a positive and a negative lobe and gives the wrong (and possibly cancelled) value.
  • Forgetting to check the sign of y on the chosen interval; if y < 0 on part of it, the integral gives signed area, not geometric area.
  • Using x = nπ but starting at n = 1, missing x = 0 as the lower limit.

A-level Scope Notes

Entirely within Edexcel IAL Pure scope: uses positivity of e^{kx}, standard zeros of sin/cos, and definite integration (typically followed by integration by parts). No results beyond the specification are needed.
