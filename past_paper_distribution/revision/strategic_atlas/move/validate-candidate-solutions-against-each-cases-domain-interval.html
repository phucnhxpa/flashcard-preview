Algebra & Functions

Validate candidate solutions against each case's domain interval

Family: Back Checking Frequency: 1 questions

Trigger — When to use

Fires when a modulus equation, modulus inequality, or piecewise-defined equation has been split into cases, each assuming x lies in a specific interval (e.g. x < 0, 0 ≤ x < k/6, x ≥ k/6). After algebraically solving inside a case, you must verify the candidate actually belongs to that case's interval. Cue phrases: 'in this case we assume...', 'remove the modulus by assuming...', or any case-split where the assumption restricts x.

Motivation — Why this works

Stripping a modulus by assuming a sign for its interior is only logically valid on the interval where that sign actually holds. A candidate produced under such an assumption must satisfy the assumption a posteriori; otherwise the algebra rests on a false premise and the candidate is spurious. This is structurally identical to discarding extraneous roots after squaring or after multiplying by a variable expression — the case interval is the algebraic analogue of the squared equation's hidden assumption.

Thought Process — The step-by-step reasoning

  1. Notice that each case was opened with an assumption like 'suppose x ≥ k/6, so |6x − k| = 6x − k'; the assumption is a constraint, not a free choice.
  2. Rule out treating any value of x satisfying the resulting linear equation as automatically a solution of the original equation.
  3. Commit to a uniform back-check: for every candidate, test membership in its parent case's interval before accepting it.
  4. Bridge to the first operation by listing each case's interval explicitly alongside its candidate, so the comparison is mechanical.
  5. Tactically, if the candidate is in terms of a parameter (k), check the interval inequalities symbolically using the stated sign of the parameter (e.g. k > 0).

Mechanic — Concrete operations

  1. After solving within Case i, write the candidate x = c_i and the case interval I_i side by side.
  2. Substitute c_i into the inequalities defining I_i and verify each holds (using parameter sign assumptions where needed).
  3. If c_i ∈ I_i, retain it; if not, discard it as extraneous.
  4. If a case yields a contradiction (e.g. k/3 = k with k > 0), record 'no solution in this case' and move on.
  5. Collect all surviving candidates as the final solution set.

Worked Micro-Example

Solve $|x| + |6x - k| = k/2$ with $k > 0$. Case 1 ($x < 0$): $-x + (k - 6x) = k/2 \Rightarrow x = -k/14$? Recompute: $-7x = -k/2$, so $x = k/14$ — fails $x<0$, reject. Case 2 ($0 \le x < k/6$): equation gives $k = k/2$, contradiction. Case 3 ($x \ge k/6$): $x + 6x - k = k/2 \Rightarrow x = 3k/14$; check $3k/14 \ge k/6$ ✓. Final: $x = 3k/14$.

Variants — Same move, different surface

Modulus inequality split into cases
Intersect each case's solution set with that case's interval, then take the union across cases.
Piecewise-defined function equation f(x) = c
Solve on each piece using its formula, then keep only roots lying in that piece's domain.
Squaring both sides of |f(x)| = g(x)
Replace interval check with substitution into the original equation and the constraint g(x) ≥ 0.
Parametric modulus equation in k
Verify case-interval inequalities symbolically using the given sign/range of the parameter.

Common Misreadings

  • Treating the case assumption as a labelling device rather than a logical constraint that must be re-verified.
  • Believing that because the algebra was correct the candidate must satisfy the original equation.
  • Thinking the global domain check (e.g. x ∈ ℝ) is enough, ignoring the case-specific interval.

Pitfalls

  • Accepting a candidate without checking it lies in the correct case interval, producing extraneous solutions.
  • Checking only one of the interval's bounding inequalities (e.g. x ≥ 0) instead of all defining inequalities (e.g. 0 ≤ x < k/6).
  • Mishandling the parameter's sign when verifying inequalities like −k/6 < 0, leading to wrongly rejecting valid candidates.
  • Forgetting to record 'no solution' for a case that produced a contradiction, and later double-counting from another case.

A-level Scope Notes

Standard Edexcel IAL P2/P3 modulus-equation technique; case-splitting plus interval validation is the expected method, with no appeal to graphical or numerical solvers required.
