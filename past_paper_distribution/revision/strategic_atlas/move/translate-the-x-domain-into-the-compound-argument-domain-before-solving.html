Trigonometry

Translate the x-domain into the compound-argument domain before solving

Family: Domain Adjustment Frequency: 2 questions

Trigger — When to use

After applying the R-formula or a substitution, the equation has the form $\sin(x \pm \alpha) = k$, $\cos(x \pm \alpha) = k$, or similar with a compound argument, while the original domain (e.g. $0\degree \le x < 360\degree$) is stated for $x$ alone. The visible cue is a mismatch: the variable inside the trig function is no longer plain $x$ but $x \pm \alpha$ (or $2x \pm \alpha$, etc.). Equivalently phrased: 'the angle inside sin/cos has been shifted, but the bounds haven't been'. Whenever you set $y = x \pm \alpha$, you must immediately ask 'what range does $y$ live in?'.

Motivation — Why this works

Trig equations have infinitely many solutions, and the domain is what selects the finite set the question wants. If you solve $\sin y = k$ over the wrong $y$-range you will either miss solutions (a Q1/Q4 angle that only appears once you extend past $360\degree$, or that lives at negative $y$) or include spurious ones that map outside the original $x$-range. The principle is simply that an inequality survives translation: $a \le x < b$ implies $a \pm \alpha \le x \pm \alpha < b \pm \alpha$. This is the single most common source of dropped marks on R-formula questions in IAL P3.

Thought Process — The step-by-step reasoning

  1. I notice the equation now reads $\sin(x - \alpha) = k$ or $\cos(x + \alpha) = k$ — the argument is no longer just $x$.
  2. I rule out blindly using $\arcsin/\arccos$ on $[0\degree, 360\degree)$ for $x$ directly, because that ignores the phase shift.
  3. I commit to substituting $y = x \pm \alpha$ and translating the inequality, because every other approach (e.g. expanding back out) loses the structural simplicity.
  4. I apply the same arithmetic operation to all three parts of the inequality $0\degree \le x < 360\degree$, producing the new bounds for $y$.
  5. I then enumerate ALL $y$-solutions in this shifted window — including any that lie below $0\degree$ or above $360\degree$ — by adding/subtracting $360\degree$ to the principal value as needed.
  6. Finally I subtract/add $\alpha$ back to recover each $x$, and confirm each lies in $[0\degree, 360\degree)$.

Mechanic — Concrete operations

  1. Let $y = x \pm \alpha$ (matching the sign in the compound argument).
  2. Apply the same shift to the original domain: from $0\degree \le x < 360\degree$ obtain $\pm\alpha \le y < 360\degree \pm \alpha$.
  3. Compute the principal value $y_0 = \arcsin k$ or $\arccos k$.
  4. List the secondary value in one cycle: $180\degree - y_0$ for sine, $360\degree - y_0$ for cosine.
  5. Generate all values $y_0 + 360\degree n$ and (secondary) $+ 360\degree n$ that fall inside the shifted window $[\pm\alpha, 360\degree \pm \alpha)$ — this often means including a negative $y$ or a $y > 360\degree$.
  6. Recover $x = y \mp \alpha$ for each surviving $y$, and check each $x \in [0\degree, 360\degree)$.

Worked Micro-Example

Solve $\cos(x + 22.62\degree) = \tfrac{7}{13}$ for $0\degree \le x < 360\degree$. Let $y = x + 22.62\degree$, so $22.62\degree \le y < 382.62\degree$. Principal value: $y = \arccos(7/13) \approx 57.40\degree$; secondary: $y \approx 302.60\degree$. Check $57.40\degree + 360\degree = 417.40\degree \notin [22.62\degree, 382.62\degree)$, so no third solution. Subtract: $x \approx 34.78\degree$ or $x \approx 279.98\degree$.

Variants — Same move, different surface

Subtractive shift, e.g. $\sin(x - \alpha) = k$
The bounds shift down: $-\alpha \le y < 360\degree - \alpha$. Watch for solutions at negative $y$ which still correspond to valid $x$.
Multiple-angle plus shift, e.g. $\cos(2x + \alpha) = k$ on $0\degree \le x < 360\degree$
Let $y = 2x + \alpha$; the window becomes $\alpha \le y < 720\degree + \alpha$ (width doubled), so expect roughly twice as many solutions.
Radian domain $0 \le x < 2\pi$
Same logic, just with radian bounds: $\pm\alpha \le y < 2\pi \pm \alpha$.

Common Misreadings

  • Treating the compound argument $y = x \pm \alpha$ as if it inherits $[0\degree, 360\degree)$ unchanged.
  • Solving in $x$ directly via $\arccos$ without acknowledging the phase shift, producing answers off by $\alpha$.

Pitfalls

  • Forgetting to subtract/add $\alpha$ back at the end to recover $x$ from $y$.
  • Not checking whether $y_0 + 360\degree$ (or $y_0 - 360\degree$) falls inside the extended window — this is exactly where a 'hidden' third solution lives.
  • For $2x \pm \alpha$ problems, doubling the window but only listing solutions in one $360\degree$ cycle of $y$.

A-level Scope Notes

This is core Edexcel IAL P3 technique used immediately after applying the R-formula (booklet entries: $R\sin(x \pm \alpha)$, $R\cos(x \pm \alpha)$ with $R = \sqrt{a^2+b^2}$, $\tan\alpha = b/a$). Stay with linear arguments $ax+b$; do not generalise to non-linear arguments, which fall outside scope.
