Trigonometry

Reject trig roots outside the function's range to filter valid solutions

Family: Domain Filtering Frequency: 2 questions

Trigger — When to use

After substituting u = sin x, cos x, sec x, cosec x, tan x (etc.) and solving a quadratic or polynomial in u, you obtain two or more candidate values. Before proceeding to find angles, inspect each root against the natural range of the trig function: |sin x| ≤ 1, |cos x| ≤ 1, |sec x| ≥ 1, |cosec x| ≥ 1 (tan x is unbounded). The cue is a numerical root that violates one of these bounds — e.g. cos x = 2, sin 2θ = −3/2, or cosec 2θ = −2/3 (which forces sin 2θ = −3/2). Equivalent phrasings: 'discard the impossible root', 'check feasibility of each value', 'apply range constraint before inverting'.

Motivation — Why this works

A quadratic-in-trig is an algebraic equation in a substituted variable u; its roots are constrained only by the algebra, not by the trig function's actual range. So extraneous roots routinely appear. Filtering them via the range — sin/cos bounded in [−1,1], and reciprocally |sec|,|cosec| ≥ 1 — eliminates spurious branches before any inverse trig work, halving the case-analysis. This is also a marked step on Edexcel mark schemes: examiners reward an explicit statement of the rejection, not silent omission.

Thought Process — The step-by-step reasoning

  1. Notice that the equation has been reduced to a quadratic (or polynomial) in a single trig expression u, and you have two candidate values for u.
  2. Before reaching for the calculator and inverse trig, pause and recall the range of the function: sin and cos live in [−1,1]; sec and cosec live in (−∞,−1]∪[1,∞); tan is all reals.
  3. Rule out the impulse to compute cos⁻¹(2) or sin⁻¹(−3/2) on the calculator — these will throw a math error and waste time; the right move is a range argument.
  4. Commit to a feasibility check: test each root against the bound. If a root lies outside the allowed range, discard it and write a one-line justification (e.g. 'no solutions since |cos x| ≤ 1').
  5. For reciprocal functions, translate first: cosec 2θ = −2/3 ⟹ sin 2θ = −3/2, then apply |sin| ≤ 1 to reject. This conversion makes the bound transparent.
  6. Proceed to solve only with the surviving root(s), confident that no valid solutions are being lost.

Mechanic — Concrete operations

  1. Solve the quadratic in u to obtain candidate roots u = u₁, u₂, ...
  2. For each root, identify the underlying sine/cosine value (invert reciprocal if needed: cosec → 1/sin, sec → 1/cos).
  3. Compare against the range: reject if |sin x| > 1 or |cos x| > 1; equivalently reject |cosec x| < 1 or |sec x| < 1.
  4. Write an explicit one-line rejection statement, e.g. 'Reject cos x = 2 since −1 ≤ cos x ≤ 1.'
  5. Proceed with the remaining valid root(s), finding all angles in the required interval.

Worked Micro-Example

Solve $3\,\text{cosec}^2 2\theta - 13\,\text{cosec}\,2\theta - 10 = 0$ for $\theta \in [0°,180°]$. Let $u = \text{cosec}\,2\theta$: $(3u+2)(u-5)=0$, so $u=-\tfrac{2}{3}$ or $u=5$. Check: $u=-\tfrac{2}{3}\Rightarrow \sin 2\theta=-\tfrac{3}{2}$, impossible since $|\sin 2\theta|\le 1$ — reject. $u=5\Rightarrow \sin 2\theta=\tfrac{1}{5}$, valid; proceed to find $2\theta$.

Variants — Same move, different surface

Quadratic in cos x or sin x yields a root with |value| > 1
Reject directly using −1 ≤ sin x, cos x ≤ 1; state the bound explicitly.
Quadratic in sec x or cosec x yields a root with |value| < 1
Convert to the reciprocal sine/cosine and apply the standard bound, OR cite |sec x|, |cosec x| ≥ 1 directly.
Polynomial in tan x
No range restriction — tan x is unbounded — so no roots get rejected on these grounds; check instead for excluded values where tan is undefined (odd multiples of 90°).
Substitution involving e^x in a mixed equation
Reject any root with e^x ≤ 0, since e^x > 0 for all real x — same filtering logic, different function.

Common Misreadings

  • Trying to evaluate cos⁻¹(2) or sin⁻¹(−3/2) on the calculator and panicking at the error message.
  • Assuming both quadratic roots must yield angles and inventing solutions by 'fixing' the sign or magnitude.
  • Confusing the range of cosec/sec (|·| ≥ 1) with that of sin/cos (|·| ≤ 1) and rejecting the wrong root.

Pitfalls

  • Silently dropping the invalid root without writing the rejection — this loses the mark allocated for explicit range justification.
  • Forgetting to convert the reciprocal first: declaring cosec 2θ = −2/3 'fine' because |−2/3| ≤ 1, missing that it forces |sin 2θ| > 1.
  • Applying the rejection but then failing to find ALL valid angles from the surviving root within the given interval.

A-level Scope Notes

Fully within Edexcel IAL P3/P4 trigonometry. No formula-booklet entry is needed — the ranges of sin, cos, sec, cosec are assumed knowledge. Avoid invoking complex-valued inverse trig (e.g. arccos of values outside [−1,1] via complex logarithms), which is out of scope.
