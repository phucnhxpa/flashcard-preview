Integration

Split sin^n x as sin^(n-1)x · sin x to derive a reduction formula

Family: Ibp Partitioning Frequency: 1 questions

Trigger — When to use

Fires when the integrand is sin^n x (or cos^n x) with n a general positive integer parameter, and the question asks for a reduction formula relating I_n to I_{n-2}. Typical phrasings: 'show that nI_n = ...', 'prove the reduction formula', or 'hence find I_5 / I_6'. The presence of a symbolic exponent (rather than a numeric one) is the decisive cue that ordinary direct integration is unavailable.

Motivation — Why this works

Integration by parts needs one factor we can differentiate to lower the power and another we can integrate cleanly. Peeling off a single sin x as dv gives v = −cos x, while u = sin^{n-1}x differentiates to (n−1)sin^{n-2}x cos x; the resulting cos²x factor can be rewritten via 1 − sin²x to re-express the integral as a combination of I_{n-2} and I_n. This self-referential structure is exactly what produces a recurrence, and no other partition achieves it cleanly.

Thought Process — The step-by-step reasoning

  1. Notice that n is a symbolic parameter, so no antiderivative table applies — a reduction formula is required.
  2. Rule out u-substitution (no inner-function/derivative pairing) and rule out splitting as sin^{n-2}x · sin²x, which collapses to the same I_n via 1 − cos²x without giving an index drop.
  3. Commit to IBP with the asymmetric split sin^{n-1}x · sin x because it is the unique partition that lowers the exponent on differentiation and survives integration of the lone sin x.
  4. Set u = sin^{n-1}x, dv = sin x dx so that du contains sin^{n-2}x and v = −cos x — this is the bridge to writing out the IBP identity.
  5. After IBP, deploy cos²x = 1 − sin²x to expose I_{n-2} and I_n on the right, then collect like terms to solve algebraically for I_n.

Mechanic — Concrete operations

  1. Write sin^n x = sin^{n-1}x · sin x and set u = sin^{n-1}x, dv/dx = sin x.
  2. Differentiate: du/dx = (n−1)sin^{n-2}x cos x; integrate: v = −cos x.
  3. Apply IBP: I_n = [−sin^{n-1}x cos x] + (n−1)∫ sin^{n-2}x cos²x dx.
  4. Replace cos²x by 1 − sin²x: I_n = [boundary] + (n−1)I_{n-2} − (n−1)I_n.
  5. Collect: nI_n = [boundary] + (n−1)I_{n-2}, then divide by n to obtain the reduction formula.
  6. If definite on [0, π/2], note the boundary term vanishes since sin 0 = 0 and cos(π/2) = 0.

Worked Micro-Example

For $I_n=\int_0^{\pi/2}\sin^n x\,dx$, take $u=\sin^{n-1}x$, $dv=\sin x\,dx$. Then $I_n=\left[-\sin^{n-1}x\cos x\right]_0^{\pi/2}+(n-1)\int_0^{\pi/2}\sin^{n-2}x\cos^2 x\,dx$. The bracket is $0$, and $\cos^2 x=1-\sin^2 x$ gives $I_n=(n-1)I_{n-2}-(n-1)I_n$, so $nI_n=(n-1)I_{n-2}$, i.e. $I_n=\tfrac{n-1}{n}I_{n-2}$.

Variants — Same move, different surface

I_n = ∫cos^n x dx
Split as cos^{n-1}x · cos x with u = cos^{n-1}x, dv = cos x dx so v = sin x; same algebra yields nI_n = [boundary] + (n−1)I_{n-2}.
I_n = ∫sin^n x dx over [0, π] or other limits
Method is identical, but the boundary term must be evaluated explicitly — it is not automatically zero.
I_{m,n} = ∫sin^m x cos^n x dx (Further Maths)
Peel off one sin x (or one cos x) as dv; the recurrence now drops m (or n) by 2, with the other index fixed.
I_n = ∫tan^n x dx
Do NOT use IBP; instead write tan^n x = tan^{n-2}x(sec²x − 1) and integrate the sec²x piece directly.

Common Misreadings

  • Setting u = sin^n x and dv = dx, which raises rather than lowers complexity.
  • Splitting symmetrically as sin^{n/2}x · sin^{n/2}x — produces no useful index shift.
  • Choosing dv = sin^{n-1}x dx (un-integrable in closed form) instead of dv = sin x dx.

Pitfalls

  • Sign error from v = −cos x flipping the sign of the (n−1) integral.
  • Forgetting the (n−1) factor from du when transcribing the post-IBP integral.
  • Failing to use cos²x = 1 − sin²x and so not exposing I_n on the right-hand side.
  • Forgetting to collect the I_n terms algebraically, leaving the formula unsolved.

A-level Scope Notes

Reduction formulas for ∫sin^n x dx and ∫cos^n x dx are standard Edexcel IAL Further Pure content; the technique relies only on IBP and the Pythagorean identity, both within scope.
