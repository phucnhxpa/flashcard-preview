Matrices

Assemble A = PDP⁻¹ with matched eigenvector–eigenvalue column ordering

Family: Diagonalisation Frequency: 2 questions

Trigger — When to use

Fire this move when the question explicitly asks to 'diagonalise A', to 'express A in the form PDP⁻¹' (or equivalently find P such that P⁻¹AP = D), or to compute a matrix power Aⁿ / a matrix function that would be intractable by direct multiplication. The structural prerequisite is that A has n linearly independent eigenvectors (so for an n×n matrix you have already found a full set of eigenvalues and eigenvectors). Phrasings include 'find an invertible P and a diagonal D such that…', 'hence find A^k', or 'use your answer to part (a) to evaluate Aⁿ'.

Motivation — Why this works

If A has n linearly independent eigenvectors, then placing them as columns of P makes P invertible and yields A = PDP⁻¹, where D = diag(λ₁,…,λₙ). This works because AP = PD column-by-column is exactly the eigenvalue equation Ae_i = λ_i e_i. The decomposition decouples A into independent scalar actions along each eigen-direction, so any hard operation on A (powers, exponentials) becomes a trivial operation on the diagonal D: Aⁿ = PDⁿP⁻¹ with Dⁿ = diag(λ₁ⁿ,…,λₙⁿ).

Thought Process — The step-by-step reasoning

  1. Notice the question asks for diagonalisation or for a high power Aⁿ — direct repeated multiplication is combinatorially hopeless, so a structural decomposition is needed.
  2. Check the prerequisite: do I have n linearly independent eigenvectors? For a repeated eigenvalue, I must verify the eigenspace has full geometric multiplicity; otherwise A is not diagonalisable and this move fails.
  3. Rule out ad-hoc alternatives: direct expansion of Aⁿ, Cayley–Hamilton reduction (only useful when diagonalisation fails or is not requested), or guessing a pattern from A, A², A³.
  4. Commit to A = PDP⁻¹ because Dⁿ is immediate and the only non-trivial labour is computing P⁻¹ once.
  5. Tactical decision: choose a column order for P (e.g. eigenvalues ascending, or in the order given). The order is free but D must match P column-for-column.
  6. Plan the inversion route for P⁻¹: row-reduce [P | I] → [I | P⁻¹] for a 3×3, or use the cofactor/adjugate formula. If A is symmetric, eigenvectors for distinct eigenvalues are orthogonal — normalise them and use P⁻¹ = Pᵀ.

Mechanic — Concrete operations

  1. Form P = [e₁ | e₂ | … | eₙ] with eigenvectors as columns, in a chosen order.
  2. Form D = diag(λ₁, λ₂, …, λₙ) with eigenvalues on the diagonal in the SAME order as the columns of P.
  3. Verify consistency by checking AP = PD: the i-th column of AP should equal λ_i times the i-th column of P.
  4. Compute P⁻¹ by row-reducing [P | I] → [I | P⁻¹], or via P⁻¹ = (1/det P)·adj(P) for small matrices.
  5. Write A = PDP⁻¹ (or P⁻¹AP = D, depending on the form requested).
  6. If a power is required: compute Dⁿ = diag(λ₁ⁿ,…,λₙⁿ), then Aⁿ = P · Dⁿ · P⁻¹ — keep this multiplication order strictly.

Worked Micro-Example

Let A have eigenvalues λ₁=1, λ₂=4 with eigenvectors e₁=(1,1)ᵀ, e₂=(1,-1)ᵀ. Take P=\begin{pmatrix}1&1\\1&-1\end{pmatrix}, D=\begin{pmatrix}1&0\\0&4\end{pmatrix}. Then P⁻¹=\tfrac12\begin{pmatrix}1&1\\1&-1\end{pmatrix}, and A^n = P D^n P^{-1} = \tfrac12\begin{pmatrix}1+4^n & 1-4^n \\ 1-4^n & 1+4^n\end{pmatrix}.

Variants — Same move, different surface

A is symmetric (real)
Eigenvectors for distinct eigenvalues are orthogonal; normalise them so P is orthogonal and use P⁻¹ = Pᵀ — no inversion calculation needed.
Repeated eigenvalue in a 3×3 (algebraic multiplicity 2)
Find two linearly independent eigenvectors spanning that 2-dimensional eigenspace; if only one independent eigenvector exists, A is not diagonalisable and this move fails.
Question asks for D = P⁻¹AP rather than A = PDP⁻¹
Same P and D — just rearrange the equation; no recomputation needed.

Common Misreadings

  • Listing eigenvalues in D in a different order from the corresponding eigenvector columns of P — the pairing must be exact.
  • Assuming every square matrix is diagonalisable; failing to check that the eigenvectors are linearly independent.
  • Reading 'diagonalise' as merely 'find eigenvalues' and stopping before assembling P and D.

Pitfalls

  • Arithmetic errors when computing P⁻¹ for a 3×3 matrix — this is where most marks are lost; always verify by checking PP⁻¹ = I on at least one column.
  • Writing A = PDP instead of A = PDP⁻¹, or confusing P⁻¹AP = D with PAP⁻¹.
  • When computing Aⁿ = PDⁿP⁻¹, accidentally writing P⁻¹DⁿP — matrix multiplication is not commutative, so the order is fixed by the derivation.
  • Forgetting to raise EACH diagonal entry to the n-th power (writing Dⁿ as diag(λ₁,…,λₙ) instead of diag(λ₁ⁿ,…,λₙⁿ)).

A-level Scope Notes

This is core Edexcel IAL Further Pure (FP2/FP3) material on diagonalisation of 2×2 and 3×3 matrices. The eigenvalue/eigenvector machinery and the identity A = PDP⁻¹ are in scope; Jordan normal form and the matrix exponential are NOT — if a matrix is not diagonalisable, the question will route via Cayley–Hamilton instead.
