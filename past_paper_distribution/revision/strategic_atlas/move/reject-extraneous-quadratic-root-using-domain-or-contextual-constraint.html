Algebra & Functions

Reject extraneous quadratic root using domain or contextual constraint

Family: Root Selection Frequency: 3 questions

Trigger — When to use

Fires whenever solving a quadratic (via the formula, completing the square, or taking a square root) produces a ± and hence two candidate values, but the problem context allows only one. Typical cues: the question states a variable is 'positive', the point lies 'in the first quadrant', the function has a restricted domain (e.g. x > 4), or you are constructing an inverse function whose range must equal the original domain. Equivalent phrasings: 'pick the correct branch', 'discard the spurious root', 'apply the sign condition'. Recognise it as soon as you write '±' next to a surd and the question has imposed any inequality, sign, or geometric region condition.

Motivation — Why this works

A quadratic equation is symmetric in its two roots, but the original problem usually carries an asymmetric constraint (positivity, a domain, a quadrant, a length being non-negative). The squaring or quadratic-formula step introduces this symmetry artificially, so one of the two algebraic answers fails the original condition and must be discarded. Selecting correctly is what converts a purely algebraic answer into the contextually valid one — failing to do so is a logical error, not a computational one. The underlying principle: solutions of a derived equation are necessary but not sufficient; they must be checked against the original constraints.

Thought Process — The step-by-step reasoning

  1. Notice that the algebra has produced two candidate values (± a surd, or two roots from the quadratic formula).
  2. Re-read the question for any restriction: is the variable declared positive? Is the point in a specific quadrant? Does the original function have a restricted domain that forces the inverse's range?
  3. Rule out the lazy default ('always take +'): the correct branch depends on the constraint, not on a universal sign convention.
  4. Commit to testing each candidate against the constraint — either by numerical estimate (e.g. √165 ≈ 12.8), asymptotic behaviour (large-x limit), or substitution back into a second equation/sketch.
  5. Tactically choose the cheapest test: a surd estimate if the constraint is positivity; an asymptotic argument if the domain is x > k for some k; substitution into the other curve if the constraint is geometric (y > 0, first quadrant).
  6. Discard the failing root explicitly (state the reason) and present the surviving root as the final answer.

Mechanic — Concrete operations

  1. Write the two candidates explicitly: x = a + √D and x = a − √D (or analogous ±-expression).
  2. Identify the active constraint from the question: e.g. p > 0, x > 4, P in first quadrant, range of f⁻¹ ⊆ domain of f.
  3. Estimate or analyse each candidate: numerical bound (√D between consecutive integer squares), asymptotic limit as x → ∞, or substitute into the secondary equation.
  4. Compare each candidate against the constraint; mark one ✓ and one ✗ with a brief reason.
  5. State the surviving root as the final answer, citing the constraint that eliminated the other.

Worked Micro-Example

Solve $(p-3)^2 = 165$ with $p>0$. Take square roots: $p = 3 \pm \sqrt{165}$. Estimate $\sqrt{165} \approx 12.8$ (since $12^2=144$, $13^2=169$). Then $3 - \sqrt{165} \approx -9.8 < 0$ — reject. So $p = 3 + \sqrt{165}$.

Variants — Same move, different surface

Inverse functions where f has a restricted domain
Choose the ± branch so that the range of f⁻¹ equals the domain of f; test by letting x → ∞ and seeing which branch lands in the required region.
Geometric problem with two intersection points
Use a sketch or substitute each x-candidate into the other curve to check the y-sign; keep the root in the specified quadrant or arc.
Positive-length / positive-parameter constraint
Estimate the surd numerically; reject the root with negative sign.
Trigonometric equation on a restricted interval
Same logic — list all algebraic solutions, then keep only those inside the stated interval (e.g. 0 ≤ θ < 2π).

Common Misreadings

  • Defaulting to '+' because it 'looks tidier' rather than checking the constraint.
  • Interpreting 'first quadrant' as only x > 0, forgetting to verify y > 0.
  • Presenting both roots as joint answers when the question requires a single value.
  • Misreading the domain restriction as applying to the new variable rather than to the original function.

Pitfalls

  • Dropping the ± when taking a square root and so missing the valid root entirely.
  • Numerical sign error when the constant and the surd are close in magnitude (e.g. 3 vs √8).
  • Using a single test value rather than the relevant asymptotic/limit when the constraint is global (e.g. 'for all x > 4').
  • Substituting into the wrong equation when checking a geometric constraint.
