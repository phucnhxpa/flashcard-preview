Integration

Set up trapezium rule by reading h and labelling ordinates correctly

Family: Trapezium Rule Setup Frequency: 2 questions

Trigger — When to use

The question asks for a numerical estimate of a definite integral ∫ₐᵇ y dx and supplies a table of x and y values at equally spaced x. Common phrasings include 'use the trapezium rule with all the values in the table', 'estimate the integral to … d.p.', or 'using n strips'. Visually you see a fence-post layout: a row of x's from a to b in equal steps, and a matching row of y's. Before any arithmetic the student must (i) extract h, (ii) count strips vs ordinates, and (iii) decide which y-values are endpoints and which are interior.

Motivation — Why this works

The trapezium rule (h/2)[y₀ + y_n + 2(y₁ + … + y_{n−1})] weights the two endpoint ordinates by 1 and every interior ordinate by 2 — this is just the geometric statement that adjacent trapezia share an internal edge that is therefore counted twice. The whole calculation collapses if h is wrong or if endpoint/interior roles are swapped, so the setup is the load-bearing step. The fence-post principle (n ordinates ⇒ n−1 strips) and the relation h = (b−a)/(number of strips) are the two identities that make the setup robust.

Thought Process — The step-by-step reasoning

  1. Notice the table of equally spaced x-values with matching y-values — this is the signature of a trapezium-rule question, not a substitution or by-parts question (which would give an exact integrand, not a table).
  2. Rule out exact methods: I have no formula for y, only sampled values, so analytic integration is impossible; Simpson's rule is also off-syllabus at this point unless explicitly requested.
  3. Commit to the trapezium rule and immediately separate two questions: 'how many ordinates do I have?' and 'how many strips does that give?' — they differ by 1 (fence-post logic).
  4. Compute h either by reading the spacing directly from the x-row (e.g. 0.5, 1.0, 1.5 → h = 0.5) or by h = (b − a)/(number of strips); cross-check the two against each other.
  5. Label the ordinates y₀, y₁, …, y_n with y₀ and y_n the endpoints (weight 1) and y₁, …, y_{n−1} the interior ordinates (weight 2). Writing this labelling out before substituting prevents the most common slip.
  6. Form the interior sum S = y₁ + y₂ + … + y_{n−1} as a single number, then assemble (h/2)[y₀ + y_n + 2S] — keeping the prefactor h/2 (not h) and the doubling on the correct group.

Mechanic — Concrete operations

  1. Count the ordinates n_ord from the table; set number of strips n = n_ord − 1.
  2. Compute h = (b − a)/n, and verify against the visible spacing of x-values.
  3. Label y₀ = y(a), y_n = y(b) as endpoints; label the remaining n − 1 values as interior ordinates.
  4. Sum the interior ordinates: S = y₁ + y₂ + … + y_{n−1}.
  5. Sum the endpoints: E = y₀ + y_n.
  6. Apply ∫ₐᵇ y dx ≈ (h/2)[E + 2S] and round to the requested precision.

Worked Micro-Example

Given x = 0.5, 1.0, 1.5, 2.0, 2.5, 3.0 with y = 1.319, 1.075, 1.001, 1.041, 1.223, 1.850. Then n_ord = 6, n = 5 strips, h = 0.5. Endpoints E = 1.319 + 1.850 = 3.169; interior S = 1.075 + 1.001 + 1.041 + 1.223 = 4.340. Estimate = (0.5/2)[3.169 + 2(4.340)] = 0.25 × 11.849 = 2.962.

Variants — Same move, different surface

Question states 'use n strips' explicitly (e.g. 'use 6 strips on [2.5, 4]')
Take n directly, compute h = (b − a)/n, and the number of ordinates needed is n + 1; build the x-row yourself if not tabulated.
Spacing is non-uniform
The shortcut formula fails; apply the trapezium area ½(x_{i+1} − x_i)(y_i + y_{i+1}) strip-by-strip and sum.
Integrand involves a transformation, e.g. ∫ ln(x) dx with y already tabulated as ln(x)
Read the y-column as the ordinates directly — do not re-evaluate the integrand; the table has done the work.

Common Misreadings

  • Treating the number of ordinates as the number of strips (e.g. seeing 7 y-values and using n = 7), giving h = (b − a)/7 instead of (b − a)/6.
  • Using h = b − a (the whole interval) instead of h = (b − a)/n.
  • Mistaking a tabulated function value f(x_i) for an x-value when the columns are unfamiliarly laid out.

Pitfalls

  • Doubling the endpoint ordinates instead of the interior ones — swapping the two weight groups in the formula.
  • Forgetting the ½ in the prefactor and writing h[…] instead of (h/2)[…].
  • Arithmetic slip when summing the interior ordinates — every subsequent mark depends on this single sum, so it is worth recomputing.
  • Rounding intermediate ordinates before summing, which can throw the final answer outside the accepted accuracy band.

A-level Scope Notes

Fully within Edexcel IAL Pure Maths (P2/P3) and the formula booklet entry for the trapezium rule: ∫ₐᵇ y dx ≈ (h/2)[y₀ + y_n + 2(y₁ + … + y_{n−1})], where h = (b − a)/n. Do not invoke Simpson's rule or error-bound formulae unless a Further Pure paper explicitly requests them.
