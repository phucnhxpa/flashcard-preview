Algebra & Functions

Select the algebraic form that minimises work for the specific substitution

Family: Form Selection Frequency: 3 questions

Trigger — When to use

Fires when a function or model is available in MULTIPLE algebraic representations (e.g. expanded vs factorised polynomial, original quadratic vs completed-square, exponential model vs its linearised log form) AND a specific substitution or readout is required. Cue phrases: 'find the y-intercept', 'state the maximum', 'find t when N = ...', 'where the curve crosses the y-axis'. The key signal is that one form makes the required quantity readable almost by inspection, while the other form would require expansion, logs, or extra arithmetic. Also fires when a rounded form coexists with an exact form — prefer the exact form to avoid error propagation.

Motivation — Why this works

Different algebraic forms encode different information cheaply: the constant term of an expanded polynomial IS f(0); the completed-square form (x−h)²+k displays the vertex (h,k); the linearised log equation log N = mt+c isolates t with one rearrangement. Choosing the form aligned with the question saves computation AND avoids rounding error from intermediate values. The underlying principle is that algebraic identities preserve the function but rearrange which features are 'on the surface' — picking the right surface is a deliberate efficiency and accuracy decision, not a stylistic one.

Thought Process — The step-by-step reasoning

  1. I notice the question asks for one specific feature (y-intercept / vertex / a particular t-value) and I have two or more equivalent forms of the same function on the page.
  2. I ask: at the substitution point, which terms vanish or simplify? At x=0, every x-term in an expanded polynomial dies, leaving the constant. At the vertex, the squared bracket in completed-square form is zero.
  3. I rule out the alternative forms: using the factorised form at x=0 still works but requires multiplying out (5)(−3)² = 45; using the exponential model with rounded a,b would require logs AND propagate rounding error.
  4. I commit to the form that gives the answer by inspection or in one rearrangement, because it minimises both arithmetic steps and error sources.
  5. Tactical check: am I using the EXACT form (e.g. the linear equation derived from the graph) rather than a ROUNDED downstream form (e.g. N = 1200·1.43^t)? Always prefer exact.
  6. Execute the substitution in the chosen form and read off / solve.

Mechanic — Concrete operations

  1. Identify all available forms of f(x) (or the model) from earlier parts of the question.
  2. Identify the target: f(0)? vertex? f⁻¹(value)? specific t?
  3. Match: constant-term readout → expanded form; vertex/max/min → completed-square; solve for exponent → linearised log form.
  4. Substitute into the chosen form; the calculation should collapse to one or two operations.
  5. State the answer with correct coordinates / units, and double-check any 'shift' the question adds (e.g. y = f(x) − 7).

Worked Micro-Example

Given $f(x) = 3 + 12x - 2x^2 = 21 - 2(x-3)^2$, find the y-intercept of $y = f(x) - 7$. Use the original (expanded) form: $f(0) = 3$, so $y = 3 - 7 = -4$, giving $(0,-4)$. The completed-square form would require $21 - 2(-3)^2 - 7 = 21 - 18 - 7 = -4$ — same answer, three extra operations.

Variants — Same move, different surface

Finding the vertex / maximum / minimum of a quadratic
Switch to completed-square form $a(x-h)^2 + k$ and read $(h,k)$ directly; do NOT use the expanded form.
Solving for t in an exponential model when a linearised log equation is also available
Substitute log₁₀(N) into the linear equation log N = mt + c (exact gradient/intercept) rather than using rounded a,b in N = ab^t.
Finding x-intercepts of a polynomial
Use the factorised form to read roots directly; expanded form would require solving from scratch.

Common Misreadings

  • Treating all forms as equivalent in effort and defaulting to whichever was 'given' in the question stem.
  • Confusing x-intercepts (use factorised) with y-intercepts (use expanded) and applying the wrong form.
  • Using the rounded model from a previous part instead of the exact linear equation, accepting avoidable error.

Pitfalls

  • Forgetting an outer transformation: computing f(0) correctly but ignoring a '−7' or '+c' shift outside.
  • Misreading scale/units in context (e.g. log₁₀(500) instead of log₁₀(500000) when N is in thousands).
  • Arithmetic slips in the 'easy' step because the student rushes, e.g. dropping a sign on the constant term.

A-level Scope Notes

Fully within Edexcel IAL P1–P4 scope: completed-square form, polynomial expansion/factorisation, and linearisation of exponential models via log₁₀ (P2/P3). No formula-booklet entry is strictly invoked; this is a representational choice. Avoid generalising to Taylor series — at IAL only the link 'constant term = f(0)' is used.
