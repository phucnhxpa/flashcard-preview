Integration

Apply trapezium rule to tabulated ordinates to estimate a definite integral

Family: Numerical Integration Frequency: 4 questions

Trigger — When to use

The question provides a table of (x, y) values at equally spaced x-values and asks for a numerical estimate of a definite integral, often with the explicit instruction 'use the trapezium rule with all the values of y in the table'. The integrand may be analytically intractable (e.g. log₂(2x), e^(x²)) or simply untabulated as a function. Equivalent phrasings include 'estimate the area under the curve', 'estimate ∫ₐᵇ f(x) dx using n strips', or providing only discrete data with no formula. The visual cue is a horizontal table of x and y rows with uniform x-spacing.

Motivation — Why this works

The trapezium rule replaces the curve between successive ordinates by straight-line segments, so the area under the curve is approximated by the sum of trapezoid areas. With equal spacing h, this collapses to the compact formula (h/2)[y₀ + yₙ + 2(y₁ + y₂ + … + yₙ₋₁)], where the boundary ordinates appear once and the interior ordinates appear doubled (because each interior ordinate is shared by two trapezia). It converts an analytically hard or impossible integral into a routine arithmetic computation, and is the standard numerical method on the Edexcel IAL P-papers.

Thought Process — The step-by-step reasoning

  1. Notice the question supplies a table of equally spaced x-values with corresponding y-values, and either asks for an estimate or names the trapezium rule explicitly — this is the canonical trigger.
  2. Rule out analytical integration: either no closed-form antiderivative exists in the IAL toolkit (e.g. log₂(2x), e^(x²)), or the question forbids it by demanding a numerical estimate using the tabulated data.
  3. Commit to the trapezium rule because the data is discrete, equally spaced, and the question wants a number — Simpson's rule is not on the IAL specification, so trapezium is the only relevant numerical tool.
  4. Bridge to operation: count the number of ordinates m, deduce the number of strips n = m − 1, then compute h = (b − a)/n. Check the spacing is genuinely uniform.
  5. Tactically partition the ordinates into 'ends' (y₀ and yₙ, weight 1) and 'middles' (y₁,…,yₙ₋₁, weight 2). Sum each group separately to localise arithmetic errors.
  6. Substitute into A ≈ (h/2)[ (y₀ + yₙ) + 2(y₁ + y₂ + … + yₙ₋₁) ], keeping full precision until the final rounding step so subsequent parts of the question can reuse the unrounded value.

Mechanic — Concrete operations

  1. Count ordinates m and strips n = m − 1; compute h = (b − a)/n.
  2. Identify the boundary ordinates y₀ and yₙ; sum them: S_end = y₀ + yₙ.
  3. Identify the interior ordinates y₁, …, yₙ₋₁; sum them: S_mid = y₁ + y₂ + ⋯ + yₙ₋₁.
  4. Apply the formula: A ≈ (h/2)[ S_end + 2·S_mid ].
  5. Evaluate the bracket first, then multiply by h/2; keep full precision.
  6. Round only at the end to the requested number of significant figures or decimal places.

Worked Micro-Example

Estimate $\int_{2}^{4} f(x)\,dx$ from the table $x=2,2.5,3,3.5,4$ with $y=4.718, 5.617, 6.723, 8.083, 9.756$. Here $n=4$ strips and $h=0.5$. Ends: $4.718+9.756=14.474$. Middles: $5.617+6.723+8.083=20.423$. Then $A\approx \tfrac{0.5}{2}[14.474+2(20.423)] = 0.25\times 55.320 = 13.83$ (3 s.f. $\approx 13.8$).

Variants — Same move, different surface

Function given algebraically but a numerical estimate is requested (e.g. estimate ∫₀² e^(x²) dx with 4 strips)
Build the table yourself by evaluating f at the equally spaced x-values, then apply the same formula.
Only 3 ordinates (n = 2 strips)
Formula reduces to (h/2)[y₀ + 2y₁ + y₂] — same weighting principle with one middle term.
y-values given in surd or exact form
Either keep exact form through the algebra and convert at the end, or convert to decimals at the start; be consistent to avoid rounding error.
Unequally spaced x-values
The compact formula no longer applies — sum each trapezium ½(xᵢ₊₁ − xᵢ)(yᵢ + yᵢ₊₁) individually.
Follow-up part asks for an improved estimate
Halving the strip width (doubling the number of strips) typically improves accuracy; reuse already-computed ordinates.

Common Misreadings

  • Confusing the number of ordinates with the number of strips, e.g. taking n = 5 instead of n = 4 from a 5-row table and so using the wrong h.
  • Using h = b − a (the total interval width) instead of the strip width (b − a)/n.
  • Doubling the endpoint ordinates instead of (or as well as) the interior ordinates.
  • Reading the question as asking for an exact value when it specifies the trapezium rule, then attempting analytical integration.

Pitfalls

  • Forgetting the factor of 2 on the interior ordinates inside the bracket.
  • Including y₀ or yₙ inside the '2(...)' bracket by mistake.
  • Omitting the ½ in h/2, effectively doubling the final answer.
  • Premature rounding of the interior or endpoint sum, causing the final answer to fall outside the accepted tolerance.
  • Arithmetic slip when summing several decimal ordinates — keep a running total and double-check.

A-level Scope Notes

Within Edexcel IAL Pure Mathematics (P2/P3), the trapezium rule formula A ≈ (h/2)[y₀ + yₙ + 2(y₁ + … + yₙ₋₁)] is given in the formula booklet. Simpson's rule and error-bound analysis are NOT on the IAL specification and should not be invoked.
