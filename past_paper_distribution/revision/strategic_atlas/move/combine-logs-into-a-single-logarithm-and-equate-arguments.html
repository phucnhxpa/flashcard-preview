Combine logs into a single logarithm and equate arguments

Family: Log Law Simplification Frequency: 2 questions

Trigger — When to use

The equation has logarithms on BOTH sides (or can be rearranged so it does), all sharing the SAME base. Visually you see a sum/difference of log terms like log_b(A) ± log_b(B) ± log_b(C) = log_b(D), or one side is a single log and the other is a string of log terms waiting to be merged. Alternative cues: 'express y in the form px^q' after a log-linear regression line has been derived; or a transcendental equation where every term is a logarithm with no stray constants on the wrong side.

Motivation — Why this works

The product rule log A + log B = log(AB) and quotient rule log A − log B = log(A/B) collapse multiple log terms into ONE log. Once both sides reduce to log_b(·) = log_b(·), the one-to-one (strictly monotonic) property of the logarithm allows you to equate arguments directly, replacing the transcendental equation with a purely algebraic one. This is the standard 'de-logarithming' bridge from log-form to power-law or polynomial form.

Thought Process — The step-by-step reasoning

  1. I notice every term in the equation is a logarithm with the same base — there are no loose constants stuck outside a log on the wrong side.
  2. I rule out exponentiating both sides as b^(LHS) = b^(RHS): it works in principle but invites the classic error b^(A+B) = b^A + b^B; combining logs first is cleaner.
  3. I rule out 'cancelling logs term by term' — that is not a valid operation; logs only cancel once each side is a SINGLE log.
  4. I commit to the combine-then-equate route: first use product/quotient laws to merge each side into one log, then drop the logs.
  5. Tactical decision: if any term has a coefficient (e.g. 3 log x), I first absorb it as a power (log x^3) before combining; if a constant like 4 sits on the RHS, I rewrite it as log_10(10^4) so the entire side is a log.
  6. After equating arguments I read off the algebraic equation and solve by standard methods (factorising, identifying p and q, etc.), remembering to check any solutions against the original log domain.

Mechanic — Concrete operations

  1. Rewrite any constant terms on a log side as a log: e.g. 4 = log_10(10^4); rewrite any coefficients as powers: k log A = log A^k.
  2. Apply the product rule on sums: log_b A + log_b B = log_b(AB), and the quotient rule on differences: log_b A − log_b B = log_b(A/B).
  3. Reduce each side to a single logarithm of the same base: log_b(LHS-arg) = log_b(RHS-arg).
  4. Equate arguments using one-to-one property: LHS-arg = RHS-arg.
  5. Solve the resulting algebraic equation (linear, quadratic, or power form).
  6. Check solutions are admissible — every argument inside an original log must be > 0.

Worked Micro-Example

Solve $\log_2 2 + \log_2 y + \log_2(y+4) = \log_2(5-y)$.\\Combine LHS: $\log_2[2y(y+4)] = \log_2(5-y)$.\\Equate arguments: $2y(y+4) = 5-y \Rightarrow 2y^2 + 9y - 5 = 0 \Rightarrow (2y-1)(y+5)=0$.\\So $y = \tfrac{1}{2}$ (reject $y=-5$ since it makes $\log_2 y$ undefined).

Variants — Same move, different surface

Power-law extraction: $\log_{10} y = -\tfrac{2}{3}\log_{10} x + 4$
Rewrite $4 = \log_{10} 10^4$ and $-\tfrac{2}{3}\log_{10} x = \log_{10} x^{-2/3}$, combine to $\log_{10} y = \log_{10}(10^4 x^{-2/3})$, then $y = 10000\, x^{-2/3}$.
Subtraction on one side: $\log_a f(x) - \log_a g(x) = \log_a h(x)$
Use the quotient rule: $f(x)/g(x) = h(x)$.
Mixed bases (e.g. one $\log_{10}$ and one $\ln$)
Convert to a common base via change-of-base $\log_a x = \ln x / \ln a$ BEFORE attempting to equate arguments.

Common Misreadings

  • Treating $\log A + \log B$ as $\log(A+B)$ instead of $\log(AB)$.
  • Trying to 'cancel logs' term by term across the equation before merging each side into a single log.
  • Exponentiating prematurely and then writing $b^{A+B} = b^A + b^B$ — confusing exponent rules with log rules.

Pitfalls

  • Forgetting to multiply ALL arguments when three or more log terms appear on one side (e.g. dropping the leading $\log_2 2$).
  • Sign errors when moving log terms across the '=' before combining — particularly flipping a subtraction into a division the wrong way.
  • Failing to check the domain: solutions that make any original log argument $\le 0$ must be rejected.
  • Equating arguments when bases differ — the one-to-one step requires identical bases on both sides.

A-level Scope Notes

Fully within Edexcel IAL P2 (Logarithms) and reused in P3/P4 modelling contexts. Uses the laws of logarithms listed in the formula booklet ($\log_a xy$, $\log_a (x/y)$, $\log_a x^k$). Avoid invoking complex-valued logs or branch issues — at IAL level all log arguments must be positive reals.
