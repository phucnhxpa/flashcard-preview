Algebra & Functions

Apply zero-product property to extract roots from factored form

Family: Zero Product Property Frequency: 4 questions

Trigger — When to use

The equation has been fully factorised and now reads (linear)(linear) = 0, e.g. (mx + p)(nx + q) = 0, with zero on the right-hand side. This is the moment immediately after a successful factorisation of a quadratic (or higher-degree polynomial) when no further algebraic simplification is possible. Equivalent phrasings: 'a product of brackets equals zero', 'two factors multiply to give zero', or 'fully factorised form with RHS = 0'. The cue is structural: brackets multiplied together, an equals sign, and a 0.

Motivation — Why this works

The zero-product property states that if AB = 0 then A = 0 or B = 0, because the real numbers contain no zero divisors — a non-zero number times a non-zero number cannot be zero. Invoking this property is the logical bridge from a factorised equation to its individual solutions: it converts one quadratic equation into two trivial linear equations, each solved in a single step. This is the entire reason factorisation was performed in the first place. Because each linear factor contributes exactly one root, the property also guarantees that all roots have been found (a degree-2 polynomial has at most 2 real roots).

Thought Process — The step-by-step reasoning

  1. Notice that the equation is now in the form (factor₁)(factor₂) = 0, with zero isolated on the right-hand side — no further rearrangement is needed or useful.
  2. Rule out alternative manoeuvres: dividing both sides by one factor would lose a root, expanding back out would undo the factorisation, and the quadratic formula is unnecessary now that factors are explicit.
  3. Commit to the zero-product property because it is the only valid logical step — over the reals, a product equals zero iff at least one factor equals zero.
  4. Bridge to the operation: split the single equation into two independent linear equations, one per factor, joined by 'or' (not 'and') because x takes one value or the other.
  5. Tactical care: for monic factors (x + p) = 0 read off x = −p directly; for non-monic (mx + p) = 0 isolate x = −p/m, watching signs carefully.
  6. Optionally check each candidate root in the original equation, especially if a sign slip is plausible.

Mechanic — Concrete operations

  1. From (mx + p)(nx + q) = 0, set the first factor to zero: mx + p = 0 ⟹ x = −p/m.
  2. Set the second factor to zero: nx + q = 0 ⟹ x = −q/n.
  3. Connect the results with 'or': x = −p/m or x = −q/n.
  4. (Optional) Substitute each root back into the original equation to verify.

Worked Micro-Example

Solve $(3x+2)(2x-5)=0$. By the zero-product property, $3x+2=0$ or $2x-5=0$. Solving each: $x=-\tfrac{2}{3}$ or $x=\tfrac{5}{2}$.

Variants — Same move, different surface

Repeated factor, e.g. $(2x-1)^2 = 0$.
Setting the (single) repeated factor to zero gives one repeated root, here $x = \tfrac{1}{2}$ (a double root).
One factor is just $x$, e.g. $x(2x-5)=0$.
Read off $x=0$ immediately from the first factor; do not divide through by $x$, as that would discard this root.
Cubic or higher polynomial fully factorised, e.g. $x(x-1)(x+3)=0$.
Apply the same property to each linear factor independently, giving one root per factor: $x=0,\,1,\,-3$.
RHS is non-zero, e.g. $(x+1)(x+2) = 6$.
The property does NOT apply directly. First expand, move all terms to one side to obtain RHS = 0, then re-factorise before invoking the property.

Common Misreadings

  • Treating the equation as if it can be solved by dividing both sides by one factor (e.g. dividing by (x+3)), which silently discards a root.
  • Attempting to apply the property when the right-hand side is non-zero, e.g. concluding from $(x-1)(x-2)=6$ that $x-1=6$ or $x-2=6$.
  • Confusing factorised form with a difference, e.g. reading $(x+a)(x+b)$ as $(x+a)-(x+b)$ when extracting roots.

Pitfalls

  • Sign errors when isolating x: e.g. writing $x = 2/3$ instead of $x = -2/3$ from $3x+2=0$, especially with double negatives.
  • Writing 'and' instead of 'or' between solutions — x cannot equal both values simultaneously.
  • Forgetting to state both roots, particularly when one is 0 or a fraction.
  • Stopping after the first solution and not setting the second factor to zero.

A-level Scope Notes

Fully within Edexcel IAL Pure Mathematics (P1 onward) and Further Pure. No formula booklet entry is required — the zero-product property is assumed knowledge. Avoid extending this reasoning to matrices or modular arithmetic with composite moduli, where zero divisors exist and the property fails; this is out of scope for IAL.
