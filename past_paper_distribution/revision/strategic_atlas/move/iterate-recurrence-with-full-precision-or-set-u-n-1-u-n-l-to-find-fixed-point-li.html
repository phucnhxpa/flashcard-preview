Trigger — When to use

Fires when the question gives a recurrence x_{n+1} = g(x_n) (or T_{n+1} = g(T_n), u_{n+1} = f(u_n)) together with a starting value, and asks either (i) for specific iterates to a stated decimal precision, (ii) to iterate until convergence to k d.p., or (iii) for the limit as n → ∞ assuming the sequence converges. Visual cues: a 'show that' part producing T = g(T) followed by 'use the iteration with T_1 = ...', or wording like 'the sequence converges to L, find L'. Equivalent phrasings include 'apply the iterative formula', 'find the limit', or 'continue the iteration until two successive values agree to 4 d.p.'

Motivation — Why this works

A fixed-point iteration x_{n+1} = g(x_n) converges to a root of x = g(x) when |g'(x)| < 1 near the root, so repeated substitution drives the sequence toward the fixed point. If the sequence converges, then both x_n and x_{n+1} tend to the same limit L, so L must satisfy L = g(L) — converting an infinite process into a single algebraic equation. For numerical iteration, feeding back full-precision values prevents rounding-error amplification through nonlinear functions like arccos or exp, ensuring the displayed answer is correct to the requested precision.

Thought Process — Step-by-step reasoning

  1. Notice the recurrence form x_{n+1} = g(x_n) and check what is asked: numerical iterates to k d.p., or the limit L as n → ∞.
  2. Rule out solving for a closed form (e.g. u_n = A·r^n + c) — this is longer, may diverge, and is unnecessary when only iterates or the limit are wanted; rule out Newton-Raphson unless the question explicitly sets it up.
  3. Commit to the fixed-point method: for limits, substitute u_{n+1} = u_n = L and solve L = g(L); for iterates, substitute the previous value and compute, retaining full calculator precision (use ANS).
  4. Bridge to first operation: write down x_1 (given) and compute x_2 = g(x_1) on the calculator, storing the unrounded value rather than the displayed rounded one.
  5. Tactical decision on stopping: continue iterating until two consecutive iterates agree to the required number of decimal places, then quote that common value as the limit; only round at the final reporting step.
  6. Tactical decision on mode/branch: ensure radians for trig/arccos, and for non-linear fixed-point equations (e.g. quadratic L = L² − 2) pick the root the sequence actually approaches by checking the starting value.

Mechanic — Concrete operations

  1. Identify g: rewrite the recurrence as x_{n+1} = g(x_n) and confirm the starting value x_1.
  2. For a numerical iteration: compute x_2 = g(x_1) on the calculator, press = and use ANS for x_3 = g(ANS), continuing without intermediate rounding.
  3. Tabulate iterates carrying ≥ 2 extra decimals beyond the requested precision; stop when x_{n+1} = x_n to the required d.p.
  4. Round only the reported iterates (e.g. x_2 and x_5) to the stated precision at the end.
  5. For a limit L: replace BOTH u_{n+1} and u_n by L in the recurrence to obtain L = g(L).
  6. Solve L = g(L) algebraically (linear → one root; quadratic/non-linear → select the root consistent with the starting value or stated convergence).

Worked Micro-Example

Given $x_{n+1} = \tfrac{1}{3}\arccos(1.5x_n - 2)$ with $x_1 = 0.8$ (radians): $x_2 = \tfrac{1}{3}\arccos(-0.8) = 0.83269\ldots$ (store full value as ANS); $x_3 = \tfrac{1}{3}\arccos(1.5\cdot\text{ANS}-2) = 0.81786\ldots$; continuing, $x_4 = 0.81316\ldots$, $x_5 = 0.81169\ldots$. Report $x_2 = 0.8327$, $x_5 = 0.8117$ to 4 d.p. The limit $L$ satisfies $L = \tfrac{1}{3}\arccos(1.5L-2)$.

Variants — Same move, different surface

Limit of a linear recurrence u_{n+1} = au_n + b (question states convergence)
Set u_{n+1} = u_n = L to get L = aL + b ⟹ L = b/(1−a); ignore that |a|>1 would diverge — the question grants convergence.
Quadratic recurrence u_{n+1} = u_n² + c
Solve L = L² + c (a quadratic with two candidate limits); pick the root the iteration actually approaches based on the starting value.
Second-order recurrence u_{n+2} = au_{n+1} + bu_n
Set u_{n+2} = u_{n+1} = u_n = L to get L(1 − a − b) = 0.
Newton-Raphson formulation x_{n+1} = x_n − f(x_n)/f'(x_n)
Same full-precision iteration discipline; the fixed-point limit equation reduces to f(L) = 0.
Recurrence involving exp or trig with awkward arguments (e.g. T_{n+1} = (1/12)(116 − 16e^{−0.75T_n} + e^{T_n−10} − e^{−10}))
Iterate with ANS until T_{n+1} = T_n to 4 d.p.; convergence is the explicit stopping criterion.

Common Misreadings

  • Treating 'find the limit' as requiring a closed-form solution u_n = A·r^n + c when the fixed-point equation is sufficient.
  • Assuming convergence must be verified by |g'(L)| < 1 even when the question states the sequence converges.
  • Reading the iteration as a one-off substitution (only computing x_2) when the question asks for repeated iteration to convergence.

Pitfalls

  • Rounding each iterate to 4 d.p. before substituting it back — the single most common error; can shift later iterates in the last reported decimal place.
  • Calculator in degrees instead of radians for arccos/cos/sin-based iterations.
  • Replacing only one of u_{n+1}, u_n with L when forming the fixed-point equation.
  • For non-linear fixed-point equations, accepting an extraneous root without checking which branch the iteration actually approaches.
  • Stopping iteration too early (after only 2–3 steps) so that 'convergence to 4 d.p.' is not visibly demonstrated.

A-level Scope Notes

Within Edexcel IAL P3/P4 scope: numerical methods (iteration to find roots) and convergence of sequences. No formula booklet entry — students are expected to use the recurrence as given. Avoid invoking Banach fixed-point theorem or formal contraction arguments; |g'(L)| < 1 is mentioned only intuitively if at all.
