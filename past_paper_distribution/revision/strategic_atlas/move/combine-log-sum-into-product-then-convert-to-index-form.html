Combine log sum into product then convert to index form

Family: Constant To Log Conversion Frequency: 1 questions

Trigger — When to use

The equation contains a sum (or difference) of two logarithms with the same base set equal to a constant, e.g. $\log_4 a + \log_4 b = 3$. The logs obstruct algebraic manipulation, and the right-hand side is a clean number ready for exponentiation. Typical phrasings: 'Given that $\log_c x + \log_c y = k$...', or appearing as one half of a simultaneous system alongside a linear/polynomial equation in $a$ and $b$.

Motivation — Why this works

The addition law $\log_c x + \log_c y = \log_c(xy)$ collapses two log terms into a single $\log_c(\text{expression}) = \text{constant}$, which is the canonical form for the definition $\log_c N = P \iff N = c^P$. Applying this definition exponentiates away the logarithm, replacing a transcendental relation with a polynomial product equation. This product equation then pairs naturally with any other algebraic equation (often linear) to form a standard simultaneous system.

Thought Process — The step-by-step reasoning

  1. Notice two log terms with the same base added (or subtracted) and equated to a constant — the addition/subtraction law applies directly.
  2. Rule out converting each log separately (e.g. setting $\log_4 a = p$, $\log_4 b = q$), since that introduces two new unknowns without simplifying.
  3. Commit to combining first because the result $\log_c(\text{single expression}) = \text{constant}$ is the exact precondition for clean exponentiation.
  4. Bridge: write $\log_4(ab) = 3$, then read off $ab = 4^3$ using the definition of a logarithm.
  5. Tactically, evaluate $4^3 = 64$ immediately so the product equation is ready to substitute into the partner equation.

Mechanic — Concrete operations

  1. Apply the addition law: $\log_c a + \log_c b = \log_c(ab)$.
  2. Rewrite the equation as $\log_c(ab) = k$.
  3. Convert to index form using $\log_c N = P \iff N = c^P$: $ab = c^k$.
  4. Evaluate $c^k$ to a numerical value.
  5. Pair the resulting product equation with the other given equation to solve simultaneously.

Worked Micro-Example

Given $\log_4 a + \log_4 b = 3$ and $a - b = 8$. Combine: $\log_4(ab) = 3$, so $ab = 4^3 = 64$. From the linear equation $a = b + 8$, substitute: $(b+8)b = 64 \Rightarrow b^2 + 8b - 64 = 0$, giving $b = -4 + 4\sqrt{5}$ (taking $b>0$), and $a = 4 + 4\sqrt{5}$.

Variants — Same move, different surface

Difference of logs: $\log_c a - \log_c b = k$
Use the subtraction law to get $\log_c(a/b) = k$, then $a/b = c^k$.
Coefficients present: $2\log_c a + \log_c b = k$
First apply the power law $2\log_c a = \log_c(a^2)$, then combine to $\log_c(a^2 b) = k$ before exponentiating.
Constant absorbed into log: $\log_c a + \log_c b = \log_c m$
Combine LHS to $\log_c(ab) = \log_c m$ and equate arguments: $ab = m$ (no exponentiation needed).
Mixed bases
First rewrite all logs in a common base via change-of-base before applying the addition law.

Common Misreadings

  • Treating $\log_c a + \log_c b$ as $\log_c(a+b)$ — the classic addition-inside-vs-outside error.
  • Reading $ab = 4^3$ as $ab = 4 \times 3 = 12$, confusing exponentiation with multiplication.
  • Combining logs of different bases as if the addition law applied across bases.

Pitfalls

  • Exponentiating before reducing to a single $\log_c(\cdot) = \text{constant}$ form, which corrupts the conversion.
  • Losing the domain constraint $a, b > 0$ required for the original logs, leading to acceptance of negative roots from the resulting quadratic.
  • Forgetting to apply the power law first when coefficients sit in front of the log terms.

A-level Scope Notes

Standard Edexcel IAL Pure Mathematics (P2) logarithm laws: addition, subtraction, power, and the definition $\log_c N = P \iff N = c^P$. Solving the resulting linear-and-product system uses only GCSE/AS-level quadratic techniques.
