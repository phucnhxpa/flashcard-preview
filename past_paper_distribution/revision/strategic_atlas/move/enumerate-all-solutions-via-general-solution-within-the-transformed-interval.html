Trigonometry

Enumerate all solutions via general solution within the transformed interval

Family: Solution Enumeration Frequency: 2 questions

Trigger — When to use

After reducing an equation to a single trig function tan u = k, cos u = k, or sin u = k (where u is a linear function of x like 2θ, 2x+π/4, etc.), and the original variable's domain has been transformed into a u-interval that spans more than one period. Visual cues: the question gives an interval like −2π < x < 2π or asks for a high number of marks (4+); after substitution u = ax+b the u-range is wide (often 2π, 4π, or 8π). Equivalent phrasing: 'find all solutions in the given range', 'state how many solutions', or a worked equation with multiple intersections on a sketch.

Motivation — Why this works

Trigonometric equations have infinitely many solutions, but a bounded domain selects finitely many. The general solution formulas (u = arctan k + nπ for tan; u = ±arccos k + 2nπ for cos; u = arcsin k + 2nπ or π − arcsin k + 2nπ for sin) parametrise every solution by an integer n. Enumerating valid n against the transformed bounds is systematic and auditable — it guarantees completeness (no missed solution) and correctness (no spurious extras). This is the safest way to avoid the classic mark-losing error of forgetting the symmetric branch.

Thought Process — The step-by-step reasoning

  1. Notice that after the algebraic reduction, only one trig function remains equal to a constant, but the variable u = ax+b lives on a wider interval than one fundamental period — so multiple solutions are guaranteed.
  2. Rule out simply quoting the principal value (loses the symmetric and periodic copies), and rule out a graphical sketch alone (imprecise for exact answers).
  3. Commit to the general-solution + enumerate-n method because it is the only approach that demonstrably finds every solution and lets the marker check completeness.
  4. Bridge: write down the transformed interval for u explicitly (e.g. u ∈ (−15π/4, 17π/4)), then write the general solution u = f(n).
  5. Tactical decision: solve the inequality on n (e.g. lower bound < f(n) < upper bound) to get the integer range for n, rather than guessing values one by one.
  6. Back-substitute each valid u into x = (u−b)/a, simplify to a common denominator, and verify each x satisfies the original (strict) inequality.

Mechanic — Concrete operations

  1. State the transformed u-interval explicitly from the original x-interval and substitution u = ax+b.
  2. Write the general solution: tan u = k ⇒ u = arctan(k) + nπ; cos u = k ⇒ u = ±arccos(k) + 2nπ; sin u = k ⇒ u = arcsin(k) + 2nπ or π − arcsin(k) + 2nπ.
  3. For each branch, solve the double inequality on n to find the integer range (e.g. n = −4, −3, …, 3).
  4. List each u-value, then convert to x via x = (u − b)/a.
  5. Check each x lies strictly inside the original domain (exclude boundary if inequality is strict).
  6. Reject any x that violates a side-condition (e.g. makes a denominator zero in the original equation).
  7. Present answers exactly (in terms of π) or to the required accuracy (e.g. 3 s.f.).

Variants — Same move, different surface

For cos u = k on a symmetric interval like u ∈ (−π, π)
Use u = ±arccos(k); both ± branches lie in one period, so two solutions per k without needing n ≠ 0.
For sin u = k
Use u = arcsin(k) + 2nπ AND u = π − arcsin(k) + 2nπ; enumerate n on each branch separately.
For tan u = k (period π not 2π)
Single family u = arctan(k) + nπ — typically twice as many n-values fit a given interval as for sin/cos.
Quadratic in trig giving two roots k₁, k₂
Run the enumeration independently for each root, then union the solution sets; check no coincidences.

Common Misreadings

  • Treating the equation as having only the principal-value solution and stopping there.
  • Forgetting to transform the domain when substituting u = ax+b — solving on the original x-range instead of the u-range.
  • Confusing the period of tan (π) with that of sin/cos (2π) and getting the count wrong.

Pitfalls

  • Off-by-one error on n — including an n that puts u just outside the range, or excluding a valid boundary case (especially with strict vs. non-strict inequalities).
  • Forgetting the symmetric ± branch from the cosine general solution, or the π − arcsin(k) branch for sine — this halves the answer set.
  • Including boundary values (e.g. x = ±π/2) when the original inequality is strict.
  • Rounding arctan/arccos/arcsin too early, propagating error into the final 3 s.f. answers.
  • Double-counting where two families happen to coincide (rare but check by comparing principal values mod π or 2π).

A-level Scope Notes

Within Edexcel IAL P3/P4 trigonometry. Uses standard general solutions (not in formula booklet — must be memorised). Stays clear of complex-valued solutions and inverse trig identities beyond principal range. Do not invoke Euler's formula or sum-to-product reformulations of the general solution.
