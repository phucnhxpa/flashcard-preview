Proof

Bound integer variable using positivity to enumerate finite cases

Family: Domain Restriction Frequency: 1 questions

Trigger — When to use

Fires when a problem declares variables to be positive integers (or natural numbers) and gives linear/affine relations between them, e.g. a = 8 − 2b with a, b ∈ ℤ⁺. Also triggers when a 'prove that...' statement quantifies over a set whose size is not yet known. Cue words: 'positive integers', 'natural numbers', 'where a, b, c are...'. The presence of a strict positivity constraint on a dependent expression is the key signal that the parameter set is actually finite.

Motivation — Why this works

A linear constraint of the form a = p − qb with a > 0 and b ≥ 1 forces b into a bounded interval, and integrality then collapses this interval to a finite list. This converts a potentially open-ended universal claim into a manageable enumeration, legitimising proof by exhaustion. It also disciplines any algebraic proof by clarifying exactly which cases must be covered, preventing over-general arguments.

Thought Process — The step-by-step reasoning

  1. Notice that the variables are declared positive integers — this is a working constraint, not flavour text.
  2. Rule out treating b as a free real variable; the algebraic relation a = 8 − 2b plus a > 0 forces b < 4.
  3. Commit to translating each positivity statement into an inequality on b, then intersect with b ≥ 1.
  4. Bridge: solve 8 − 2b > 0 to get b < 4, and combine with b ∈ ℤ⁺ to obtain b ∈ {1, 2, 3}.
  5. Decide between two proof routes now visible: a clean algebraic argument valid on this range, or a three-case exhaustion.

Mechanic — Concrete operations

  1. Write down all positivity constraints: a > 0, b > 0, c > 0.
  2. Substitute the given relations to express each constraint in terms of one variable, e.g. 8 − 2b > 0.
  3. Solve each inequality: b < 4 from a > 0; b ≥ 1 from b ∈ ℤ⁺.
  4. Intersect the inequalities and impose integrality to list the finite case set b ∈ {1, 2, 3}.
  5. Confirm the remaining variables (here c = b + 2) are automatically positive on this set, so no extra cases are lost.

Worked Micro-Example

Given $a, b, c \in \mathbb{Z}^+$ with $a = 8 - 2b$ and $c = b + 2$. From $a > 0$: $8 - 2b > 0 \Rightarrow b < 4$. With $b \geq 1$: $b \in \{1,2,3\}$. Then $(a,b,c) \in \{(6,1,3),(4,2,4),(2,3,5)\}$, giving exactly three cases to verify.

Variants — Same move, different surface

Prove a divisibility/parity claim about $a + b + c$ where $a = 10 - 3b$, $a,b \in \mathbb{Z}^+$.
Solve $10 - 3b > 0$ to get $b \in \{1,2,3\}$ and then check the claim case-by-case.
Geometric sequence with positive terms and given sum/relation between terms.
Positivity of all terms bounds the common ratio $r$ (e.g. $r > 0$, sometimes $r < 1$ from convergence), restricting the parameter set.
Diophantine-style word problems (coins, tickets) with linear constraint and non-negativity.
Replace 'positive integer' with 'non-negative integer'; include the boundary case $b = 0$ in enumeration.
Quadratic relation such as $a = 12 - b^2$ with $a, b \in \mathbb{Z}^+$.
Solve $12 - b^2 > 0$ to get $b \in \{1,2,3\}$; the bounding inequality is non-linear but the enumeration step is identical.

Common Misreadings

  • Ignoring the positive-integer constraint entirely and attempting a proof for all real b.
  • Treating 'positive integer' as 'non-negative integer' and erroneously including b = 0.
  • Checking only one of a, b, c for positivity and missing a hidden case restriction.
  • Assuming the case set must be infinite and jumping straight to a general algebraic proof without first bounding b.

Pitfalls

  • Using b ≥ 0 instead of b ≥ 1 (positive means strictly greater than zero).
  • Writing b ≤ 4 instead of b < 4 (failing to use strict inequality from a > 0).
  • Forgetting to test b = 4 explicitly: $a = 8 - 8 = 0$ is not positive, so b = 4 is excluded.
  • Listing the finite case set but then proving the claim only for one representative value.

A-level Scope Notes

Sits squarely within Edexcel IAL Pure proof techniques (P1/P3): proof by exhaustion and direct proof on a finite domain, using only linear inequalities and integer reasoning.
