Trigonometry

Resolve SSA ambiguous case by selecting 180°−arcsin when obtuse angle required

Family: Ambiguous Case Resolution Frequency: 4 questions

Trigger — When to use

Fires whenever the Sine Rule has been used to find an unknown angle and the equation reduces to sin θ = k with 0 < k < 1. Concrete cues: (i) an SSA configuration where the known angle is opposite the shorter of two given sides, (ii) a diagram (even 'not to scale') showing the target angle as clearly obtuse, or (iii) the question text explicitly stating the angle is obtuse/greater than 90° (or > π/2). Alternative phrasing: 'arcsin only returned an acute value but the geometry demands the second-quadrant solution'.

Motivation — Why this works

The calculator's arcsin returns only the principal value in [0°, 90°] (or [0, π/2]), but the identity sin θ = sin(180° − θ) means every value 0 < k < 1 has two solutions in (0°, 180°): an acute α and an obtuse 180° − α. In the SSA (ambiguous) case both can yield valid triangles, so the student must consciously generate the supplementary candidate and use the diagram or stated constraint to choose. Skipping this step silently substitutes the wrong angle and cascades errors through every subsequent length, area, or angle calculation.

Thought Process — The step-by-step reasoning

  1. I notice I have used the Sine Rule and obtained sin θ = k with 0 < k < 1, in an SSA setup — this is the classic ambiguous case.
  2. I rule out blindly accepting arcsin(k): I recall sin is positive in both Q1 and Q2, so on (0°, 180°) there are TWO candidates, α = arcsin(k) and 180° − α.
  3. I rule out using cosine rule rearrangement (which would give a unique angle) because the sine rule is what fits the given data — so I must handle the ambiguity manually.
  4. I commit to listing both candidates, then disambiguating using either the diagram (angle visibly obtuse), an explicit statement in the question, or a feasibility check on angle sum < 180°.
  5. Tactical choice: if the problem says the angle is obtuse, I select 180° − α (or π − α in radians); if not stated, I check that 180° − α plus the other known angle is still < 180° before keeping it.
  6. Final guard: I keep the unrounded acute value α in memory and subtract from 180° at the end, to avoid rounding error in the obtuse value.

Mechanic — Concrete operations

  1. From Sine Rule, isolate sin θ = k (e.g. sin θ = 0.7731).
  2. Compute the acute principal value: α = arcsin(k) (in degrees or radians as required).
  3. Form the obtuse candidate: θ_obtuse = 180° − α (or π − α in radians).
  4. Disambiguate: use the diagram, the stated 'obtuse' constraint, or check the third angle 180° − (given angle) − θ_obtuse > 0°.
  5. Select the appropriate root and round only at the final step to the requested precision.

Worked Micro-Example

Sine Rule gives $\sin(\angle ACB) = 0.7934$ in a triangle with $\angle BAC = 35°$, and the diagram shows $\angle ACB$ obtuse. Acute root: $\alpha = \arcsin(0.7934) \approx 52.5°$. Obtuse root: $\angle ACB = 180° - 52.5° = 127.5°$. Feasibility check: $35° + 127.5° = 162.5° < 180°$, so the third angle is $17.5° > 0°$ ✓. Answer: $\angle ACB \approx 127.5°$.

Variants — Same move, different surface

Problem stated in radians (e.g. circle/sector geometry with π)
Use θ = π − arcsin(k) instead of 180° − arcsin(k); guard against accidentally writing 2π − α (that would be a reflex angle).
Both acute and obtuse roots produce valid triangles (no diagram, no constraint)
Report BOTH solutions and the two corresponding triangles — do not arbitrarily discard one.
Cosine Rule available as alternative
Switching to the cosine rule for the unknown angle eliminates ambiguity, since arccos returns a unique value in (0°, 180°).
Equation sin θ = k solved over [0°, 360°] (not a triangle)
Two solutions in the first cycle are α and 180° − α; further solutions add 360°. Do not use 360° − α (that is the cosine pattern).

Common Misreadings

  • Accepting the calculator's arcsin output uncritically and missing the supplementary solution entirely.
  • Reflexively discarding the obtuse candidate because of a half-remembered 'angles in a triangle are acute' rule (confusing with right triangles).
  • Computing 360° − α instead of 180° − α, conflating sine symmetry with cosine/full-rotation patterns.
  • Treating the diagram as decorative when it is labelled 'not to scale', and so ignoring the visual cue that the angle is obtuse.

Pitfalls

  • Forgetting to verify the obtuse choice still yields a positive third angle (i.e. given angle + obtuse candidate < 180°).
  • Rounding the intermediate acute value α too early so that 180° − α inherits a magnified error — keep full precision until the last step.
  • In radian problems, writing 2π − α (a reflex angle) instead of π − α (the second-quadrant angle).
  • Failing to check feasibility of sin θ = k itself: if k > 1 the configuration is impossible and no triangle exists.

A-level Scope Notes

Fully within Edexcel IAL Pure Maths (P2 Sine Rule, ambiguous case) and reused in P3/P4 trigonometric equations and circle geometry. The Sine Rule is given in the formula booklet; the identity sin(180° − θ) = sin θ is expected knowledge. Do not generalise to complex-valued or hyperbolic analogues — those are outside IAL scope.
