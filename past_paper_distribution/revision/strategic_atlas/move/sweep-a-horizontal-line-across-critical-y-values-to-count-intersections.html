Strategic Atlas / Trigonometry / Graphical Solution Counting
Trigonometry

Sweep a horizontal line across critical y-values to count intersections

Family: Graphical Solution Counting Frequency: 1 questions

Trigger — When to use

Fires when a question asks for values of a parameter $k$ (or $p$) such that an equation $f(x) = k$ has a specified number of solutions on a given domain, especially with $f$ a trig function like $\cos(2x)$, $\sin(x)$, or $a\sin(x)+b$. Common phrasings: 'find the values of $k$ for which the equation has exactly two solutions', 'state the set of values of $p$ such that...'. The cue is a parameter on the right-hand side combined with a fixed-shape curve on the left.

Motivation — Why this works

Solutions of $f(x) = k$ correspond bijectively to intersections of $y = f(x)$ with the horizontal line $y = k$. The intersection count only changes at y-values where the line is tangent to a turning point or passes through an endpoint of the curve — these are the 'critical y-values'. Between critical values the count is locally constant, so testing one representative per interval (plus each boundary) is exhaustive. This converts an algebraic counting problem into a finite case-check.

Thought Process — The step-by-step reasoning

  1. Notice the equation has the form $f(x) = k$ with a parameter on one side and a fixed trig curve on the other — a graphical sweep is natural.
  2. Rule out brute algebraic solving: the parameter prevents a clean closed-form, and the question asks about a count, not specific roots.
  3. Commit because the curve's shape on the stated domain is sketchable and the range is bounded — the critical y-values (max, min, any local extrema, endpoint heights) are finite and explicit.
  4. Sketch $y = f(x)$ on the domain, mark turning points and endpoint y-values, and identify the critical y-values that partition the range.
  5. Test one representative $k$ in each open strip and each critical boundary, counting intersections by inspection; assemble the answer as a union of intervals and isolated points.

Mechanic — Concrete operations

  1. Sketch $y = f(x)$ on the given domain, marking maxima, minima, zeros, and endpoint heights.
  2. List the critical y-values where intersection count can change: maxima, minima, and endpoint y-values.
  3. Partition the y-axis into open strips between consecutive critical values, plus the critical values themselves.
  4. For each strip, draw a representative horizontal line and count crossings with the sketch.
  5. For each critical y-value, check tangency/endpoint behaviour separately (these often give a different count from the adjacent strips).
  6. Collect the y-values whose count matches the required number into a union of intervals and isolated points.
  7. State the final answer with correct strict/non-strict inequalities matching the boundary tests.

Worked Micro-Example

Solve: for what $p$ does $\cos(2x) = p$ have exactly two solutions on $0 \le x \le 225^\circ$? The curve has range $[-1,1]$, with maxima at $x=0,180$ (height $1$), minimum at $x=90$ (height $-1$), and endpoint $(225, 0)$. Sweep: $p=1$ gives 2 touching points ✓; $p\in(0,1)$ gives 3 crossings ✗; $p=0$ gives 3 ($x=45,135,225$) ✗; $p\in(-1,0)$ gives 2 (both inside the trough) ✓; $p=-1$ gives 1 ✗. Answer: $-1 < p < 0$ or $p = 1$.

Variants — Same move, different surface

$a\sin(x) + b = k$ on a finite interval
Critical y-values become $b \pm a$ and the endpoint heights $a\sin(x_{\text{end}}) + b$; otherwise the sweep is identical.
Quadratic-in-trig like $\cos^2 x = k$
Sketch the squared curve (range $[0,1]$, period halved); critical values are $0$ and $1$.
Modulus graph $y = |f(x)| = k$
Reflect negative parts upward first; critical y-values include $0$ (cusps) and original $|$max$|$.
Piecewise or transformed graph from earlier part
Use the already-sketched transformed curve; critical values come from its turning points and endpoints, not the parent.
'How many solutions of $f(x) = g(x)$' with $g$ a fixed line $y = mx+c$
Sweep is no longer horizontal; instead translate the line or use intersection counts via algebra/discriminant.

Common Misreadings

  • Treating the curve as fully symmetric over the domain when the interval cuts off mid-period, missing an extra (or missing) crossing.
  • Counting tangent contact at a maximum/minimum as one solution when it is a genuine (double-root) solution still counted once for solution-counting.
  • Assuming $p \in (0,1)$ behaves like $p \in (-1,0)$ by symmetry, ignoring that the domain endpoint breaks the symmetry.

Pitfalls

  • Forgetting that the domain endpoints generate their own critical y-values (the curve's height at $x_{\min}$ and $x_{\max}$).
  • Writing $-1 \le p \le 0$ instead of $-1 < p < 0$, wrongly including boundaries that fail the count.
  • Omitting isolated valid points like $p = 1$ from the final answer because they look like boundaries.
  • Sketching the wrong period: e.g. $\cos(2x)$ has period $180^\circ$, not $360^\circ$.

A-level Scope Notes

Standard Edexcel IAL P2/P3 technique for solution-counting of trig (and other) equations on bounded domains; uses only sketching, range, and turning-point identification — no calculus required beyond what students already know.
