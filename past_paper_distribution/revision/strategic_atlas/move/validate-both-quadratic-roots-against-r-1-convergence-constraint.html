Sequences & Series

Validate both quadratic roots against |r| < 1 convergence constraint

Family: Domain Validation Frequency: 1 questions

Trigger — When to use

Fires whenever a quadratic (or higher) equation in the common ratio r has been produced via manipulation of S∞ = a/(1−r). Cue phrases: 'sum to infinity', 'convergent geometric series', 'find the possible values of r'. Any time more than one candidate value of r emerges, each must be tested against |r| < 1 before being reported as a solution.

Motivation — Why this works

The derivation of the quadratic implicitly assumed S∞ exists, which requires |r| < 1. A root violating this would make the very equation used to derive it meaningless, so such a root is extraneous rather than a genuine solution. Checking both roots safeguards against reporting a spurious value that satisfies the algebra but contradicts the convergence hypothesis underpinning the problem.

Thought Process — The step-by-step reasoning

  1. I notice the working used S∞ = a/(1−r), so any r I find is only meaningful if |r| < 1.
  2. I rule out simply quoting both roots: a quadratic from an S∞ equation can produce a root with |r| ≥ 1 that must be discarded.
  3. I commit to evaluating each root numerically (using a surd approximation if needed) and comparing |r| with 1.
  4. First concrete check: substitute the surd values and bound them — e.g. √3 ≈ 1.732 gives quick decimal estimates.
  5. Tactically, I keep the exact surd form for the final answer and use the decimal only as a validity test, and I check |r|, not r, so negative ratios are handled correctly.

Mechanic — Concrete operations

  1. List the candidate roots in exact form, e.g. r = (2 ± √3)/4.
  2. Approximate each using √3 ≈ 1.732 (or better) to obtain decimal estimates.
  3. Compute |r| for each root and compare with 1.
  4. Reject any root with |r| ≥ 1, citing failure of the S∞ convergence condition.
  5. State explicitly which root(s) are accepted and retain exact surd form in the final answer.

Worked Micro-Example

Suppose solving an S∞ problem yields $4r^2 - 4r + \tfrac14 = 0$, giving $r = \dfrac{2 \pm \sqrt{3}}{4}$. Then $r_1 \approx 0.067$ and $r_2 \approx 0.933$. Since $|r_1|<1$ and $|r_2|<1$, both are valid common ratios for a convergent GP, so both are accepted.

Variants — Same move, different surface

Only one root satisfies |r| < 1 (e.g. roots r = 1/2 and r = 2)
Explicitly reject the root with |r| ≥ 1 stating that S∞ does not exist there, and report only the valid root.
Negative root candidate (e.g. r = −3/4)
Test |r| < 1, not 0 < r < 1; a negative ratio with |r| < 1 still gives a convergent GP with alternating-sign terms.
Logarithmic equation producing two solutions
Check each candidate against argument > 0 in every original log term; discard any that makes a log undefined.
Squaring step in a surd equation
Substitute each candidate back into the original (pre-squared) equation and reject extraneous roots introduced by squaring.

Common Misreadings

  • Treating both roots as automatically valid because they came from a 'correct' quadratic.
  • Rejecting a perfectly valid root because its decimal value is close to 1, without checking the strict inequality.
  • Using 0 < r < 1 instead of |r| < 1 and so wrongly discarding negative ratios.

Pitfalls

  • Forgetting that the condition is |r| < 1, not r < 1, so negative ratios are mishandled.
  • Using a coarse approximation such as √3 ≈ 1.7 that distorts borderline comparisons with 1.
  • Failing to state the rejection reason ('|r| ≥ 1, so S∞ does not exist') in the written solution, losing method marks.

A-level Scope Notes

Sits within Edexcel IAL P2/P3 geometric series: S∞ = a/(1−r) requires |r| < 1, and validating roots against this is the standard expected check.
