Algebra & Functions

Read +c as a vertical translation of the graph

Family: Transformation Recognition Frequency: 1 questions

Trigger — When to use

Fires when a new function in a later part differs from an earlier function only by an added constant, e.g. y = f(x) + c versus y = f(x). Typical cues: 'sketch y = ... + k', 'state the new asymptote', or a follow-on 'hence' part referring to a previous sketch. Also fires when comparing y = a/x and y = a/x + c, or y = e^x and y = e^x + c, where a re-sketch is requested.

Motivation — Why this works

Adding a constant c outside the function adds c to every output, so the entire graph rises (or falls) by c units while x-coordinates are preserved. This means horizontal asymptotes shift from y = L to y = L + c, intercepts on the y-axis shift by c, but vertical asymptotes and the domain are untouched. Recognising the shift saves time and links the parts of the question as the examiner intends.

Thought Process — The step-by-step reasoning

  1. Notices that the new equation is structurally identical to a previous one except for an added constant outside the function.
  2. Rules out re-deriving the curve from scratch (table of values, full asymptote analysis) since the transformation rule does the work.
  3. Commits to the rule y = f(x) + c ⇒ translation by vector (0, c), confident this preserves shape and vertical asymptotes.
  4. Translates the previous sketch upward by c: every key feature (branches, intercepts, horizontal asymptote) moves up by c.
  5. Checks edge details: vertical asymptote position unchanged; sign of c determines up vs down; new y-intercept is old y-intercept + c.

Mechanic — Concrete operations

  1. Identify the base function f(x) from the earlier part or standard form.
  2. Spot the additive constant c outside f(x), giving y = f(x) + c.
  3. Apply translation by vector (0, c) to every feature of the base graph.
  4. Update horizontal asymptote: y = L becomes y = L + c.
  5. Keep vertical asymptotes and domain unchanged.
  6. Recompute the y-intercept (or note that it shifts by c) and re-sketch.

Worked Micro-Example

Given y = -k/x has asymptotes x = 0 and y = 0, sketch y = -k/x + k. The +k shifts the graph up by k, so the horizontal asymptote becomes y = k while x = 0 stays. The curve crosses the x-axis where -k/x + k = 0 ⇒ x = 1, giving a clean reference point for the translated sketch.

Variants — Same move, different surface

y = f(x - a) + b
Compose a horizontal shift by a with the vertical shift by b; asymptotes move accordingly in both directions.
y = e^x + c or y = ln x + c
Same vertical shift logic; horizontal asymptote of e^x moves from y = 0 to y = c, while ln x has no horizontal asymptote but its y-values all shift by c.
y = A f(x)
This is a vertical stretch by factor A, not a translation; asymptote y = L becomes y = AL, and the y-intercept scales rather than shifts.
y = f(x) + c with a modulus, e.g. y = |f(x)| + c
Apply the modulus first, then translate the resulting graph up by c.

Common Misreadings

  • Confusing y = f(x) + k with y = f(x + k) and applying a horizontal shift instead.
  • Treating +k as scaling the function (vertical stretch) rather than translating it.
  • Thinking the vertical asymptote also moves up by k.

Pitfalls

  • Sketching the new horizontal asymptote at y = -k instead of y = +k due to sign slip.
  • Forgetting to update the y-intercept or the points where the curve crosses the x-axis after the shift.
  • Leaving the old asymptote y = 0 on the sketch alongside the new one.

A-level Scope Notes

Standard Edexcel IAL P2/P3 transformation of graphs: y = f(x) + a, y = f(x + a), y = af(x), y = f(ax). Used routinely in 'sketch and state asymptote' questions involving reciprocal, exponential, and logarithmic curves.
