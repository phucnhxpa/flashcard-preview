Algebra & Functions

Carry full precision through Newton-Raphson iterations until rounded values agree

Family: Convergence Criterion Frequency: 1 questions

Trigger — When to use

Fires when a numerical-methods question (Newton-Raphson, fixed-point iteration, or interval bisection) asks for a root 'correct to n decimal places' or 'correct to n significant figures'. The cue is the explicit accuracy demand combined with an iterative formula already established in earlier parts. Phrasings include 'show that α = ... is correct to 3 d.p.', 'find the root to 4 d.p.', or 'continue the iteration until...'. Distinguishes itself from a one-shot iteration by requiring justification of accuracy.

Motivation — Why this works

Newton-Raphson converges quadratically, so unrounded inputs let each iterate roughly double its correct digits; rounding mid-stream pollutes this. Edexcel's accepted justification for 'correct to n d.p.' is that two successive iterates round to the same n-d.p. value — a practical surrogate for a formal error bound. Storing full calculator precision and only rounding at the comparison stage guarantees this criterion is met in the fewest iterations and avoids spurious disagreement caused by accumulated rounding.

Thought Process — The step-by-step reasoning

  1. I notice the question demands a specific decimal-place accuracy, not just 'one more iteration'.
  2. I rule out using the rounded answer from the previous part as a fresh input — that would discard precision I already paid for.
  3. I commit to the standard Edexcel convergence test: iterate until two consecutive x-values round to the same n d.p.
  4. Bridge: store the previous iterate at full calculator precision (Ans or memory), then apply the Newton-Raphson formula once more.
  5. Tactically, I round only at the final comparison step and explicitly write 'both round to ... so α = ... to n d.p.' to secure the justification mark.

Mechanic — Concrete operations

  1. Retain x_n at full calculator precision (use Ans / memory, never the rounded value)
  2. Compute f(x_n) and f'(x_n) at full precision
  3. Apply x_{n+1} = x_n − f(x_n)/f'(x_n)
  4. Round both x_n and x_{n+1} to the required n d.p.
  5. If they agree, state the agreement and conclude α = (rounded value) to n d.p.; if not, iterate again with x_{n+1}

Worked Micro-Example

Given x_1 = 0.346520146… for f(x) = \cos x - 2x. Compute f(x_1) ≈ 0.002135, f'(x_1) = -\sin x_1 - 2 ≈ -2.3398. Then x_2 = 0.346520146 - 0.002135/(-2.3398) ≈ 0.347432. Rounding: x_1 ≈ 0.347 and x_2 ≈ 0.347 (3 d.p.), so α = 0.347 correct to 3 d.p.

Variants — Same move, different surface

Accuracy stated as significant figures (e.g. '5 s.f.')
Round both successive iterates to that number of s.f. and check agreement; one extra iteration is often required versus the d.p. version.
Fixed-point iteration x_{n+1} = g(x_n)
Convergence is linear, so expect to need more iterations; same agreement test applies but compare x_n and x_{n+1} at the demanded accuracy.
Sign-change / interval bisection justification
Replace the agreement test by showing f(α − 0.5×10^{-n}) and f(α + 0.5×10^{-n}) have opposite signs, with f continuous on that interval.
Question asks to 'show α lies in [a,b]' before refining
First establish the sign change to locate the root, then carry full precision through the iteration to the required accuracy.

Common Misreadings

  • Treating 'correct to 3 d.p.' as a single iteration with rounding, rather than a convergence demonstration.
  • Feeding the rounded answer from part (b) into part (c) as the new starting value, losing accuracy.
  • Believing more iterations always improve accuracy, even if rounded intermediates have already corrupted the digits.

Pitfalls

  • Rounding x_n before substituting into f and f', which can shift x_{n+1} enough to delay agreement.
  • Failing to write the explicit comparison sentence ('x_1 and x_2 both round to 0.347') — examiners require this for the accuracy mark.
  • Truncating instead of rounding (e.g. 0.3474 → 0.347 vs 0.3475 → 0.348).
  • Using degree mode when the function involves trig in radians (or vice versa), invalidating every iterate.

A-level Scope Notes

Fully within Edexcel IAL P3/P4 numerical-methods scope: Newton-Raphson, fixed-point iteration, and sign-change arguments. No formal error analysis or convergence proofs are expected — only the 'two successive iterates agree' justification.
