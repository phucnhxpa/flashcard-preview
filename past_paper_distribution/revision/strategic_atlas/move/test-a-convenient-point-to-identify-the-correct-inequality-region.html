Complex Numbers

Test a convenient point to identify the correct inequality region

Family: Point Testing Frequency: 1 questions

Trigger — When to use

Fires when a question asks to shade or describe a region of the Argand diagram defined by one or more complex inequalities such as $|z - z_1| \le r$, $|z - z_1| \le |z - z_2|$, or $\arg(z - z_1) \le \theta$. The boundary loci have been drawn (circles, perpendicular bisectors, half-lines) but it is unclear which side satisfies the inequality. Also triggered when combining multiple inequalities and you must locate the common region. Phrases like 'shade the region', 'indicate on the Argand diagram', or 'find the set of points' are typical cues.

Motivation — Why this works

Each locus equation $|z-z_1|=r$ etc. is a boundary that splits the plane into two open regions; the inequality picks exactly one of them (plus the boundary if non-strict). Because the defining function (e.g. $|z-z_1|$) is continuous, its sign relative to the boundary is constant on each side, so a single interior test point determines the entire region. Substituting a concrete value like $z=0$ converts a geometric question into a one-line arithmetic check, bypassing error-prone reasoning about 'which side is closer'.

Thought Process — The step-by-step reasoning

  1. I notice the boundary curves are drawn but I haven't yet decided which side to shade.
  2. I rule out reasoning purely geometrically ('inside means smaller modulus') because for $\ge$ inequalities or perpendicular bisectors the rule flips and I might mis-sign it.
  3. I commit to point-testing because it gives an unambiguous true/false verdict in one substitution.
  4. I look for the simplest point clearly on one side of every boundary — $z=0$ is the default, otherwise I pick a point on a coordinate axis well away from any locus.
  5. I substitute and read off: if true, shade the side containing my test point; if false, shade the other side. For intersections, the test point must satisfy every inequality simultaneously to lie in the required region.

Mechanic — Concrete operations

  1. Sketch all boundary loci first (circles, perpendicular bisectors, half-lines).
  2. Pick a test point $z_0$ that is clearly off every boundary — usually $z_0 = 0$, or another simple value if $0$ lies on a boundary.
  3. Substitute $z_0$ into each given inequality and evaluate.
  4. For each inequality: if it holds at $z_0$, the required region is the side containing $z_0$; if not, it is the opposite side.
  5. For combined regions, shade only the intersection where every inequality is satisfied at the test point's side.
  6. Mark the boundary as solid (included) for $\le$, $\ge$ and dashed (excluded) for $<$, $>$.

Worked Micro-Example

Shade the region defined by $|z-3| \le |z-i|$. Boundary is the perpendicular bisector of $3$ and $i$. Test $z=0$: LHS $=|{-3}|=3$, RHS $=|{-i}|=1$. Since $3 \le 1$ is false, the origin is in the wrong region, so shade the side of the bisector not containing $0$ (i.e. the side containing $3$).

Variants — Same move, different surface

Region $|z - z_1| \ge r$ (exterior of a circle)
Test $z=0$: if outside the circle and the inequality holds, shade the exterior; otherwise shade the side not containing $0$.
Half-line inequality $\alpha \le \arg(z - z_1) \le \beta$
Pick a test point with an obviously computable argument relative to $z_1$ (e.g. $z = z_1 + 1$ gives $\arg = 0$) and check the bound.
Intersection of a disc and a half-plane
Test point must satisfy both inequalities; if it fails one, that single failure already excludes its side for that inequality.
Cartesian curve sketching of $y > f(x)$
Same idea: test a point not on the curve, e.g. $(0,0)$, and shade the side where the inequality holds.

Common Misreadings

  • Assuming the interior of a circle always satisfies $|z-a| \le r$ without recording the direction — fine here, but the same student then assumes interior for $|z-a| \ge r$.
  • Treating $|z - z_1| \le |z - z_2|$ as 'shade near $z_2$' instead of 'shade near $z_1$'.
  • Forgetting that for strict inequalities the boundary itself is excluded (dashed line).

Pitfalls

  • Picking a test point that lies on a boundary, producing equality and no decision.
  • Using $z=0$ when the origin is one of the foci or sits on the perpendicular bisector — choose another point.
  • Testing only one inequality in a combined-region question and shading too large a set.
  • Algebra slip when computing $|z_0 - z_1|$ — forgetting to take the modulus of a complex number, e.g. writing $|{-i}|=-1$.

A-level Scope Notes

Standard Edexcel IAL Further Pure technique for Argand-diagram region questions involving $|z-z_1|$, perpendicular bisectors $|z-z_1|=|z-z_2|$, and half-lines $\arg(z-z_1)=\theta$, including their intersections.
