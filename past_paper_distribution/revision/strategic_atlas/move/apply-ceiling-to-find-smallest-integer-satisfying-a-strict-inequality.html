Trigger — When to use

Fires whenever a sequence/series problem ends with a continuous inequality like N > 21.96... (or n > c after taking logs) and the question asks for the first/smallest integer n for which a quantity 'exceeds', 'first exceeds', 'is greater than', or 'first reaches more than' a threshold. Common phrasings: 'after how many years does the population first exceed...', 'find the least value of n such that u_n > k', 'determine the smallest n for which S_n exceeds...'. The signal is a strict-inequality bound producing a non-integer cutoff together with a context (years, terms, payments) forcing n to be a positive integer.

Motivation — Why this works

Because n is integer-valued and the inequality is strict, the smallest valid n is ⌈c⌉ when c is not itself an integer. Any integer below c fails the inequality, while ⌈c⌉ is the first integer above c and therefore the first to satisfy the strict bound. This is the discrete counterpart of taking infimum on the continuous inequality and rounding upward. Rounding the wrong way (down) gives a value that the inequality just fails to hold for, which is the most common error.

Thought Process — Step-by-step reasoning

  1. I notice my algebra/logs have produced a non-integer bound such as n > 21.96..., and n must be a whole number.
  2. I rule out simply rounding to the nearest integer or truncating, since 21 fails the strict inequality.
  3. I commit to the ceiling because the question asks for the first n that exceeds the threshold — I need the smallest integer strictly greater than 21.96.
  4. I check the boundary: test n = 21 (fails) and n = 22 (passes) by substituting back into the original inequality, not just the log form.
  5. I state n = 22 with a brief justification ('smallest integer satisfying n > 21.96...') so the discrete reasoning is explicit.

Mechanic — Concrete operations

  1. Reduce the problem to a clean inequality in n, e.g. n > c (or n < c after a sign flip from log of a number < 1).
  2. Identify c numerically to enough decimal places to be unambiguous.
  3. Take ⌈c⌉ if c is not an integer; if c happens to be an integer and the inequality is strict, take c + 1.
  4. Verify by substituting n = ⌈c⌉ and n = ⌈c⌉ − 1 back into the original condition.
  5. State the answer as 'the least value of n is ...' to match the question's wording.

Worked Micro-Example

A population follows $P_n = 30000(1.045)^n$. Find the least $n$ with $P_n > 75000$. Solve $1.045^n > 2.5$, so $n > \dfrac{\ln 2.5}{\ln 1.045} = 20.82\ldots$. Since $n \in \mathbb{Z}^+$ and the inequality is strict, take $n = \lceil 20.82 \rceil = 21$. Check: $1.045^{20} \approx 2.41 < 2.5$ and $1.045^{21} \approx 2.52 > 2.5$. ✓

Variants — Same move, different surface

Decaying geometric sequence asking when a value 'falls below' a threshold
Taking logs of a number less than 1 flips the inequality to n > c; ceiling logic still applies.
Sum of a geometric series first exceeding a value (e.g. cumulative savings)
Solve S_n > T for n via the GP sum formula, then apply ceiling to the resulting bound.
Arithmetic sequence first exceeding a value
Linear inequality a + (n−1)d > T gives n > c directly; take ⌈c⌉.
Question phrased with 'at least' (≥) rather than 'exceeds' (>)
If c is not an integer, answer is identical (⌈c⌉); if c is an integer, the answer is c itself rather than c+1.
Question asks for greatest n such that u_n < T (below-threshold)
Use floor instead: take ⌊c⌋, with the analogous boundary check at integer c.

Common Misreadings

  • Treating the bound as if n could be any real number and quoting n = 21.96 as the answer.
  • Reading 'exceeds' as 'reaches' and taking the floor instead of the ceiling.
  • Forgetting that taking logs of a base less than 1 reverses the inequality, then rounding the wrong way.

Pitfalls

  • Rounding 21.96 down to 21 — this integer fails the strict inequality.
  • Not checking the boundary case when c comes out exactly an integer (then strict '>' requires c+1, not c).
  • Carrying too few decimals so that c appears integer when it isn't (e.g. quoting 22.00 instead of 21.998).
  • Verifying only in the log-transformed form rather than substituting back into the original sequence/sum.

A-level Scope Notes

Standard Edexcel IAL P1/P2 material on geometric and arithmetic sequences, where logs are used to solve exponential inequalities and answers are required as positive integers (years, terms, payments).
