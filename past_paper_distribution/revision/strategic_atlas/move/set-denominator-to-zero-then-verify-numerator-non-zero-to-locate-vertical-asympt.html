Strategic Atlas / Algebra & Functions / Asymptote Detection
Algebra & Functions

Set denominator to zero, then verify numerator non-zero, to locate vertical asymptotes

Family: Asymptote Detection Frequency: 1 questions

Trigger — When to use

A rational function f(x) = N(x)/D(x) is given and the question asks for vertical asymptotes, equations of asymptotes, or the domain. Cues include phrases like 'state the equations of the asymptotes', 'find the vertical asymptote', or sketching a curve with division-by-zero behaviour. The move also fires when analysing discontinuities of a rational expression to distinguish poles from removable holes.

Motivation — Why this works

A rational function blows up to ±∞ exactly at x-values where the denominator vanishes but the numerator does not, because near such a point the magnitude of 1/D(x) becomes unbounded while N(x) stays bounded and non-zero. If both numerator and denominator vanish, there may instead be a removable discontinuity (a hole) after cancellation, so the non-zero numerator check is essential. This principle underlies all vertical-asymptote detection in IAL rational-function work.

Thought Process — The step-by-step reasoning

  1. Notice the function is a quotient of polynomials, so the only candidates for vertical asymptotes are zeros of the denominator.
  2. Rule out the possibility of a removable hole by checking whether the numerator also vanishes at each candidate — if it does, factor and cancel before deciding.
  3. Commit to solving D(x) = 0 because vertical asymptotes are entirely controlled by the denominator's zeros.
  4. Bridge to operation: solve the denominator equation, then substitute each root into the numerator to confirm it is non-zero.
  5. Tactical decision: state the asymptote as an equation x = a, not just a value, and explicitly mention the numerator check to secure full marks.

Mechanic — Concrete operations

  1. Identify the denominator D(x) of the rational function.
  2. Solve D(x) = 0 for all real roots; these are the candidate x-values.
  3. For each candidate x = a, evaluate N(a); confirm N(a) ≠ 0.
  4. If N(a) ≠ 0, declare a vertical asymptote at x = a; if N(a) = 0, factor and cancel to test for a hole.
  5. Write the final answer as an equation of the form x = a.

Worked Micro-Example

For $g(x) = \dfrac{x^2 - 5}{2x - 8}$: solve $2x - 8 = 0 \Rightarrow x = 4$. Check the numerator: $4^2 - 5 = 11 \neq 0$. Hence $x = 4$ is a vertical asymptote (no cancellation possible).

Variants — Same move, different surface

Quadratic denominator, e.g. $f(x) = \dfrac{x+1}{x^2 - 5x + 6}$
Factor the denominator as $(x-2)(x-3)$ and test each root in the numerator; both yield vertical asymptotes $x = 2$ and $x = 3$.
Common factor present, e.g. $f(x) = \dfrac{x^2 - 4}{x - 2}$
Cancel the $(x-2)$ factor first; $x = 2$ is a removable hole, not an asymptote, and the simplified function has no vertical asymptote.
Irreducible quadratic denominator, e.g. $f(x) = \dfrac{x}{x^2 + 1}$
The denominator has no real roots, so there are no vertical asymptotes — only a horizontal one to consider.
Repeated root in denominator, e.g. $f(x) = \dfrac{1}{(x-3)^2}$
The single asymptote $x = 3$ still applies, but both one-sided limits go to $+\infty$ — useful when sketching.

Common Misreadings

  • Treating every zero of the denominator as an asymptote without checking for cancellation, missing removable holes.
  • Confusing vertical with horizontal asymptotes and applying limits at infinity instead of locating poles.
  • Reporting the asymptote as a number (e.g. '4') rather than an equation ('x = 4').

Pitfalls

  • Forgetting to state explicitly that the numerator is non-zero at the candidate, which examiners may require for full marks.
  • Failing to factor the denominator fully when it is quadratic or higher, missing one of multiple asymptotes.
  • Cancelling a common factor silently without noting the resulting hole in the domain.

A-level Scope Notes

Fully within Edexcel IAL Pure (rational functions, curve sketching) and Further Pure (partial fractions, sketching with asymptotes); no limit notation beyond informal 'tends to ±∞' is required.
