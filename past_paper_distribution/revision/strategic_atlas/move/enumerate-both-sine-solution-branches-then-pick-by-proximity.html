Trigonometry

Enumerate both sine solution branches then pick by proximity

Family: General Solution Selection Frequency: 1 questions

Trigger — When to use

Fires when an equation of the form sin(ax+b) = k has been partially solved (a principal solution α is given or found) and the question asks for a different specific solution — e.g. the largest negative root, the smallest root above some bound, or the next solution after α. Common phrasings: 'in terms of α, find...', 'the negative solution closest to zero', 'the next positive solution'. The presence of an inner argument like x/12 or 2x means the period is rescaled and both families must be tracked carefully.

Motivation — Why this works

The equation sin(θ) = k has exactly two solution families per period: θ = α + 2nπ and θ = π − α + 2nπ, because sine takes each value in (−1,1) twice per cycle. Restricting attention to only one family loses half the solutions. Once both families are written in terms of x (after undoing the inner argument), substituting integer n produces every candidate, and the required one is selected by the constraint (sign, magnitude, or interval).

Thought Process — The step-by-step reasoning

  1. Notice the question references α (the smallest positive solution) but asks for a different specific solution, signalling that further roots must be generated.
  2. Rule out simply negating α or adding/subtracting one period to α, since sine's symmetry means the nearest solution may come from the reflected branch π − α, not the same family.
  3. Commit to writing the full general solution with both families, because only systematic enumeration guarantees the closest root is found.
  4. Bridge: undo the inner argument (here, multiply by 12) so both families are expressed directly in x, with the correct rescaled period 24π.
  5. Tactically test small integer n (typically n = −1 or n = 0) in each family, then compare magnitudes against the constraint to select the answer.

Mechanic — Concrete operations

  1. Write both general solutions for the inner argument: x/12 = α/12 + 2nπ or x/12 = π − α/12 + 2nπ.
  2. Multiply through by 12 to obtain x in each family: x = α + 24nπ or x = 12π − α + 24nπ.
  3. Substitute n = −1 in each family: Set 1 gives x = α − 24π; Set 2 gives x = −12π − α.
  4. Confirm both are negative and compare magnitudes: |α − 24π| ≈ 24π, |−12π − α| ≈ 12π.
  5. Select the smaller magnitude: x = −12π − α is the negative solution closest to zero.

Worked Micro-Example

Suppose sin(x/12) = k with smallest positive root α, so 0 < α < 6π. General solution: x = α + 24nπ or x = 12π − α + 24nπ. Taking n = −1: candidates are α − 24π and −12π − α. Since α > 0, |−12π − α| = 12π + α < 24π − α = |α − 24π|. Hence the negative solution nearest zero is x = −(12π + α).

Variants — Same move, different surface

cos(ax) = k with given principal root α
Use the two families x = α + (2π/a)n and x = −α + (2π/a)n; both come from ±cos⁻¹(k).
tan(ax) = k with given principal root α
Only one family x = α + (π/a)n; no reflected branch, so simply step n by ±1 to find adjacent roots.
sin(ax + b) = k asking for the next positive root after α
Compute 12π − α (Set 2 with n = 0) and α + 24π (Set 1 with n = 1); the smaller positive value is the answer.
Equation given in degrees
Replace 2nπ by 360n° and π − α by 180° − α throughout; logic is identical.

Common Misreadings

  • Considering only the family containing α (x = α + 24nπ) and missing the reflected π − α branch.
  • Assuming the negative solution closest to zero is simply −α.
  • Treating the period as 2π instead of 24π, forgetting the inner argument x/12 rescales the period.

Pitfalls

  • Forgetting to multiply 2nπ by 12 when clearing the x/12 denominator, giving wrong period spacing.
  • Comparing the two candidates by value instead of by magnitude when both are negative.
  • Sign errors when expanding 12(π − α/12) — must give 12π − α, not 12π − 12α.

A-level Scope Notes

This uses only standard general solutions of sin θ = k taught in Edexcel IAL Pure (P2/P3) and stays within the syllabus; no advanced identities or complex roots required.
