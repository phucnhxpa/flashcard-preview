Matrices

Diagonalise A = PDP⁻¹ to reduce matrix powers to scalar powers

Family: Diagonalisation Frequency: 3 questions

Trigger — When to use

The question asks you to express a square matrix A in the form PDP⁻¹, or to compute A^n (often for general n, or for a large specific n like A^10), or to evaluate an expression involving powers/inverses of A. A strong cue is that the previous parts of the question have walked you through finding eigenvalues and eigenvectors. The method requires a full set of n linearly independent eigenvectors (guaranteed if the n×n matrix has n distinct eigenvalues). Phrasings include 'find P and D such that A = PDP⁻¹', 'hence find A^n', or 'use your answer to evaluate A^k'.

Motivation — Why this works

If A = PDP⁻¹ then A^n = (PDP⁻¹)(PDP⁻¹)…(PDP⁻¹) = PD^nP⁻¹ because every interior P⁻¹P collapses to I. Since D is diagonal, D^n = diag(λ₁^n, λ₂^n, λ₃^n) is trivial — scalar exponentiation along each diagonal entry. This converts a hopeless repeated 3×3 multiplication into one matrix multiplication sandwich. Geometrically, diagonalisation decouples A's action into independent stretches along each eigenvector direction.

Thought Process — The step-by-step reasoning

  1. Notice that the question demands A^n (or a specific high power, or A^{-1}), so direct multiplication is impractical and a structural method is needed.
  2. Rule out alternatives: brute force multiplication is too long; Cayley–Hamilton works for low powers but is awkward for general n; Jordan form is out of scope. Diagonalisation is the standard A-level/Further Maths route.
  3. Confirm diagonalisability: the n×n matrix has n linearly independent eigenvectors (often guaranteed by n distinct eigenvalues already found in earlier parts).
  4. Commit to A = PDP⁻¹ and recall the key identity A^n = PD^nP⁻¹ — this is the whole point of the move.
  5. Tactical decision: fix an ordering of eigenvalues for the diagonal of D, then place the corresponding eigenvectors as the columns of P in the SAME order — consistency between P and D is non-negotiable.
  6. Plan the arithmetic: compute D^n entry-wise, compute P⁻¹ via the adjugate/determinant or by row-reducing [P | I], then multiply PD^nP⁻¹ — this last multiplication is where errors usually creep in.

Mechanic — Concrete operations

  1. Form P = [v₁ | v₂ | v₃] with the eigenvectors as columns.
  2. Form D = diag(λ₁, λ₂, λ₃) with eigenvalues in the order matching the columns of P.
  3. (Optional check) Verify AP = PD, equivalently A = PDP⁻¹.
  4. Compute D^n = diag(λ₁^n, λ₂^n, λ₃^n).
  5. Compute P⁻¹ using P⁻¹ = (1/det P)·adj(P), or by row-reducing [P | I] → [I | P⁻¹].
  6. Form A^n = P D^n P⁻¹ and multiply out (typically PD^n first, then ×P⁻¹).

Worked Micro-Example

Suppose A has eigenvalues λ₁=1, λ₂=2, λ₃=3 with eigenvectors v₁,v₂,v₃. Set P=[v₁|v₂|v₃], D=diag(1,2,3). Then A^n = P·diag(1^n, 2^n, 3^n)·P⁻¹. For e.g. n=5: D^5=diag(1,32,243), and A^5 = P·diag(1,32,243)·P⁻¹ — a single matrix sandwich rather than five multiplications of A.

Variants — Same move, different surface

Computing A⁻¹ instead of A^n
Use A⁻¹ = PD⁻¹P⁻¹ where D⁻¹ = diag(1/λ₁, 1/λ₂, 1/λ₃) — valid only if no eigenvalue is zero.
A is symmetric
Eigenvectors can be chosen orthonormal, so P is orthogonal and P⁻¹ = Pᵀ — saves the inverse computation entirely.
Only A² or A³ requested
Cayley–Hamilton may be quicker: write A^k as a low-degree polynomial in A, avoiding the P⁻¹ computation.
2×2 matrices
Same recipe with 2 eigenvectors and a 2×2 diagonal D; P⁻¹ is much easier (use the 2×2 inverse formula).

Common Misreadings

  • Writing A = P⁻¹DP or A = PDP instead of A = PDP⁻¹ — the order and the inverse both matter.
  • Treating diagonalisation as always available, even when an eigenvalue is repeated and the eigenspace is too small (matrix is defective).
  • Reading 'find A^n' as a request for a numerical pattern guess rather than a structural derivation.

Pitfalls

  • Mismatching the column order of P with the diagonal order of D — gives a completely wrong A^n even when each piece is individually correct.
  • Placing eigenvectors as rows instead of columns of P.
  • Arithmetic slips when computing P⁻¹ for a 3×3 matrix — this is the single most error-prone step; double-check via PP⁻¹ = I.
  • Forcing the method when there are insufficient independent eigenvectors — diagonalisation simply fails, and you must say so.
  • Forgetting to apply P⁻¹ on the right at the end and stopping at PD^n.

A-level Scope Notes

This is core Edexcel IAL Further Pure (FP3) material on eigenvalues, eigenvectors and diagonalisation of 2×2 and 3×3 matrices. The identity A^n = PD^nP⁻¹ is expected to be derived/used directly; Jordan normal form for defective matrices is OUT of scope and should not be invoked.
