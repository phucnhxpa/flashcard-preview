Algebra & Functions

Verify candidate roots against original-equation domain to discard extraneous solutions

Family: Domain Validation Frequency: 3 questions

Trigger — When to use

Fires whenever you have solved a transformed equation (logarithmic equation reduced to a polynomial by equating arguments, an equation cleared of square roots by squaring, or an equation simplified by a substitution like u = e^x or x = √(...)) and now have one or more candidate roots. The cue is that the original equation has a restricted domain — log arguments must be > 0, radicands must be ≥ 0, principal square roots must be ≥ 0, exponentials must be > 0 — but the algebraic equation you actually solved is valid on a larger set. Equivalent phrasings: 'check log arguments are positive', 'reject negative root from √', 'check for extraneous roots after squaring', 'back-substitute to eliminate spurious solutions'.

Motivation — Why this works

The algebraic manipulation (equating arguments of logs, squaring, substituting) is a one-way implication: every valid solution of the original equation satisfies the transformed equation, but not vice versa. The transformation drops the domain constraints, so candidate roots may correspond to log(negative), √(negative), or a negative value being equated to a principal square root. Validating each candidate against the ORIGINAL domain restores the missing implication and isolates the genuine solutions. Edexcel mark schemes explicitly award marks for stating and justifying the rejection.

Thought Process — The step-by-step reasoning

  1. I notice the original equation involves logs / a square-root substitution / a squaring step, so its domain is strictly smaller than ℝ.
  2. I considered just presenting all algebraic roots, but I rule that out — the transformation I used (e.g. log A = log B ⇒ A = B, or squaring) is not reversible without domain conditions.
  3. I commit to a domain-validation pass: I will list every constraint imposed by the ORIGINAL equation, intersect them, and test each candidate.
  4. Concretely, I write down each constraint (arg of every log > 0; every radicand ≥ 0; substituted variable in its natural range like u > 0 for u = e^x or x ≥ 0 for x = √(...)) and form the intersection.
  5. I test each candidate against the intersection. Any failure, even on a single constraint, kills the candidate — I do not need to check the others, but I must state the explicit reason (e.g. 'log₂(−1) is undefined' or '√ cannot equal a negative number').
  6. Surviving candidates are the final solutions; I write them out and, where the question is 'hence', back-substitute to recover the original variable.

Mechanic — Concrete operations

  1. Identify all domain constraints from the ORIGINAL equation: for each log_b(f(x)), require f(x) > 0; for each √f(x), require f(x) ≥ 0; for substitutions, require the substituted variable to lie in its natural range (e.g. x = √(y+3) ⇒ x ≥ 0; u = e^y ⇒ u > 0).
  2. Form the intersection of these constraints to get the master domain D.
  3. For each candidate root r from the algebraic equation, test r ∈ D. If any single constraint fails, reject r and STATE the failing constraint as justification.
  4. Collect the surviving roots; if the problem used a substitution, back-substitute to express the answer in the original variable.
  5. Write the final solution set, e.g. 'x = 3/2 only (rejecting x = −2 since 16x must be positive)'.

Worked Micro-Example

Solve $\log_2 y + \log_2(y+4) = \log_2(5-y)$. Combining and equating: $y(y+4) = 5-y \Rightarrow y^2+5y-5=0$… (after correct manipulation) candidates $y = 1/2$ and $y = -5$. Domain: $y>0$, $y+4>0$, $5-y>0$ ⇒ $00$ (and $\log_2(-1)$ is undefined) — reject. $y=1/2$ lies in $(0,5)$ — accept. Final: $y=\tfrac12$.

Variants — Same move, different surface

Logarithmic equation reduced by equating arguments
Require every log argument > 0 in the ORIGINAL equation; intersect all such conditions before testing candidates.
Equation solved by squaring both sides
Substitute each candidate back into the unsquared form to detect sign-flip extraneous roots (squaring can turn A = −B into A² = B²).
Substitution u = e^x or u = a^x
Require u > 0; reject any non-positive root of the polynomial in u before taking ln to recover x.
Substitution x = √(expression), e.g. x = √(y+3)
Require x ≥ 0 (principal square root); reject negative candidates before back-substituting y = x² − 3.

Common Misreadings

  • Treating the algebraic equation as fully equivalent to the original and presenting every root as a solution.
  • Checking only one of several log arguments, missing the constraint that fails on a different argument.
  • Forgetting that a substitution variable carries its own range restriction (e.g. assuming x = √(...) can be negative).

Pitfalls

  • Stating 'reject x = −2' without giving the explicit reason (e.g. 'because log₂(16x) requires x > 0') — the mark scheme requires justification.
  • Checking only the most obvious constraint (e.g. y > 0) and missing a tighter one (e.g. y < 5 from log₂(5−y)).
  • Producing a spurious extra answer by back-substituting a rejected root (e.g. computing y = (−1/2)² − 3 = −11/4 from a discarded x = −1/2).

A-level Scope Notes

Entirely within Edexcel IAL P2/P3/P4 scope: log laws, exponential/log domains, square-root principal-value convention, and standard substitutions. No formula-booklet entry is invoked — this is a method/communication step. Avoid generalising to complex logarithms or multi-valued square roots, which are out of scope.
