Trigger — When to use

A quantity changes by a fixed percentage each fixed time period (e.g. 'increases by 2% per year', 'grows by 5% each month', 'depreciates by 8% annually'). The problem gives an initial value and asks for a value after n periods, a total accumulated, or when a threshold is crossed. Wording such as 'each year', 'per annum', 'compound', 'rate of r%' all signal repeated multiplication rather than repeated addition.

Motivation — Why this works

If a quantity is scaled by the same factor every period, then after n periods it has been multiplied by that factor n times — this is exactly the definition of a geometric sequence with common ratio r = 1 ± (percentage/100). Recasting the worded scenario in this form unlocks the standard formulae u_n = ar^{n-1} and S_n = a(r^n − 1)/(r − 1), and converts threshold questions into clean inequalities solvable by logarithms. The model is exact for fixed-percentage compound change, in contrast with arithmetic growth which corresponds to fixed absolute change.

Thought Process — Step-by-step reasoning

  1. Notice the phrase 'increases/decreases by k% each [period]' — this means the new value depends multiplicatively on the previous value.
  2. Rule out an arithmetic model: a fixed percentage of a changing base does not give a constant absolute increment, so common difference fails.
  3. Commit to a geometric model because the same scaling factor is applied each period, matching u_{n+1} = r·u_n.
  4. Bridge to operations by setting a = (initial value) and r = 1 + k/100 (or 1 − k/100 for decrease).
  5. Decide whether the question is asking for u_n (a single year's value), S_n (a cumulative total), or an n satisfying an inequality (then take logs).

Mechanic — Concrete operations

  1. Identify the initial amount and call it a = u_1 (be careful about whether 'start of year 1' or 'end of year 1' is the first term).
  2. Convert the percentage to a multiplier: r = 1 + (percentage/100) for growth, r = 1 − (percentage/100) for decay.
  3. Write the general term u_n = a · r^{n−1} and, if needed, the sum S_n = a(r^n − 1)/(r − 1).
  4. Substitute the required n (aligning the year/period index carefully with n=1) or solve a · r^{n−1} ⋛ target using logarithms.

Worked Micro-Example

Population is 50000 at the start of 2018 and grows by 2% per year. Take a = 50000 and r = 1 + 2/100 = 1.02. Then the population at the start of year n (with 2018 as n=1) is u_n = 50000 \times 1.02^{\,n-1}. For example, at the start of 2025 (n = 8): u_8 = 50000 \times 1.02^{7} \approx 57434.

Variants — Same move, different surface

Depreciation / fixed-percentage decrease (e.g. car loses 15% of value each year)
Use r = 1 − (percentage/100), giving 0 < r < 1; same formulae, and S_∞ = a/(1−r) becomes available.
Compound interest on a lump sum
Take a = principal P and r = 1 + (interest rate per period); value after n periods is P·r^n (note the exponent n, not n−1, if 'after n periods' is meant).
Savings plan / annuity with a fixed deposit each period earning compound interest
Sum a geometric series: total = D·(r^n − 1)/(r − 1), where D is the regular deposit and r = 1 + rate.
'When does the amount first exceed/fall below a target?'
Set a·r^{n−1} > T (or <), take logs, and round n up (growth) or up to next integer satisfying inequality (decay).

Common Misreadings

  • Treating 'increases by 2% each year' as arithmetic with d = 0.02 × 50000 = 1000 added each year, ignoring that the base updates.
  • Confusing 'value at the start of year n' with 'value after n years', causing an off-by-one error in the exponent.
  • Reading a decrease of k% as r = −k/100 rather than r = 1 − k/100.

Pitfalls

  • Writing r = 0.02 (the growth rate alone) instead of r = 1.02 (the full multiplier).
  • Using u_n = a·r^n instead of a·r^{n−1} when the convention u_1 = a has been set.
  • Mixing up units of time (e.g. annual rate applied per month) without converting the rate.
  • Forgetting to round n up when solving an inequality for the first period a threshold is exceeded.

A-level Scope Notes

Fully within Edexcel IAL P1/P2 sequences and series: geometric sequences, sums S_n and S_∞ for |r|<1, and use of logarithms to solve r^n inequalities. No continuous exponential models (e^{kt}) or differential equations are needed at this stage.
