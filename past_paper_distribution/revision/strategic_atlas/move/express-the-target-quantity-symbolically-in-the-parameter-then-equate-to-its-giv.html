Algebra & Functions

Express the target quantity symbolically in the parameter then equate to its given value

Family: Equation Selection Frequency: 1 questions

Trigger — When to use

A geometric or numerical quantity (area, length, sum, integral) is given a specific value, and the figure or expression depends on an unknown parameter. Rather than guessing the parameter or testing values, you derive the quantity as a clean function of the parameter and equate. Cues include phrases like 'given that the area is...', 'the total is equal to...', or 'find the value of k such that...'. Fires whenever decomposition or integration has already produced a tidy closed form in the unknown.

Motivation — Why this works

Carrying the parameter symbolically through the geometric/algebraic work converts the problem into a single equation in one unknown, which is almost always linear or low-degree. This is faster and more reliable than trial substitution, and it exposes the structural dependence (e.g. linear decrease) so you can predict the number of solutions. It also lets you check feasibility constraints (positivity, range) once an explicit equation is in hand.

Thought Process — The step-by-step reasoning

  1. I notice the problem fixes a numerical value for the quantity but the figure/expression still contains an unknown parameter.
  2. I rule out plugging in trial values of the parameter — that is slow and won't generalise if the answer is a fraction.
  3. I commit to keeping the parameter symbolic through the decomposition/integration so the final expression is a clean function of it.
  4. Once I have Quantity(a) in closed form, I set Quantity(a) = given value to produce one equation in one unknown.
  5. I solve the equation, then back-substitute through any intermediate relation (e.g. a = 2/k) and check the parameter satisfies any geometric range constraints.

Mechanic — Concrete operations

  1. Carry the parameter through the geometric/algebraic derivation to obtain Quantity = f(parameter).
  2. Set f(parameter) = given numerical value.
  3. Solve the resulting equation (usually linear; possibly quadratic).
  4. If multiple solutions, reject those violating domain constraints (positivity, bounds).
  5. Back-substitute through any defining relation to recover the originally requested unknown.

Worked Micro-Example

Suppose decomposition gives total area $A = 9 - a$ and the problem states $A = \tfrac{27}{4}$. Set $9 - a = \tfrac{27}{4}$, so $a = \tfrac{36}{4} - \tfrac{27}{4} = \tfrac{9}{4}$. Given $a = \tfrac{2}{k}$, invert: $k = \tfrac{2}{a} = \tfrac{2}{9/4} = \tfrac{8}{9}$.

Variants — Same move, different surface

Area expression is quadratic in the parameter
Solve the quadratic and reject roots that violate geometric constraints (e.g. parameter must be positive or within a fixed interval).
Definite integral $\int_0^a f(x)\,dx = C$
Compute the integral as a function of $a$, equate to $C$, and solve for $a$.
Sum of an arithmetic/geometric series equals a given value
Write $S_n$ in terms of the unknown (first term, ratio, or $n$), equate to the given total, solve.
Length/distance condition involving a parameterised point
Express the squared length in the parameter, equate to the given value squared, solve.
Probability or expectation depending on an unknown $p$
Form $P(\text{event}) = f(p)$, equate to the stated probability, solve for $p$ within $[0,1]$.

Common Misreadings

  • Treating the parameter as already known and substituting a guessed value before equating.
  • Confusing the requested unknown with the intermediate parameter (e.g. solving for $a$ but forgetting to convert to $k$).
  • Assuming the equation must be quadratic and introducing spurious squaring.

Pitfalls

  • Arithmetic slips when combining fractions, e.g. converting $9$ to $\tfrac{36}{4}$.
  • Inverting a defining relation incorrectly, e.g. writing $k = a/2$ instead of $k = 2/a$.
  • Forgetting to check the solution lies in the admissible range for the geometric configuration.
  • Dropping the symbolic parameter mid-derivation and reverting to numerical guessing.

A-level Scope Notes

Fully within Edexcel IAL Pure Maths: applies to area/integration problems (P2–P4), series sums, and coordinate geometry where a parameter is implicitly defined and a target quantity is fixed.
