Complex Numbers

Compute argument from Cartesian form using sign-based quadrant check

Family: Quadrant Aware Arctan Frequency: 1 questions

Trigger — When to use

Fires when a question asks for arg(z) (or arg(z₁/z₂), arg(z₁z₂)) and the complex number is already in — or can cheaply be put into — Cartesian form a + bi. Cue phrases include 'find the argument', 'giving your answer in radians to 2 d.p.', and 'hence find arg(...)'. Especially relevant when the number lies in Q2, Q3, or Q4, where a naïve arctan(b/a) would return the wrong angle. Also fires when a previous part has produced the Cartesian form, making the quadrant inspection a single-step task.

Motivation — Why this works

The principal argument lives in (−π, π], but arctan only returns values in (−π/2, π/2), covering only quadrants I and IV. To recover the correct principal argument, one must first locate the point by inspecting sign(Re) and sign(Im), then either accept arctan(b/a) directly (Q1, Q4) or add/subtract π (Q2, Q3). This procedure is faster and less error-prone than computing arg(z₁) − arg(z₂) separately, since it requires only one quadrant analysis on the final Cartesian form.

Thought Process — The step-by-step reasoning

  1. I already have the number in the form a + bi from a previous part, so I can read off Re and Im directly rather than work with arg(z₁) − arg(z₂).
  2. I rule out the 'subtract arguments' route: it works but needs two quadrant checks (one for z₁, one for z₂), doubling the chance of a sign slip.
  3. I commit to the Cartesian-form route because one quadrant check on a + bi is the minimum-work path.
  4. Bridge: classify the quadrant by checking sign(a) and sign(b), then decide whether arctan(b/a) needs a ±π correction.
  5. Tactically: simplify b/a as a single fraction first (invert-and-multiply for fractions), evaluate arctan in radians, then double-check the sign matches the quadrant before rounding.

Mechanic — Concrete operations

  1. Read off Re(z) = a and Im(z) = b from the Cartesian form.
  2. Determine the quadrant from signs: (+,+) Q1, (−,+) Q2, (−,−) Q3, (+,−) Q4.
  3. Compute the ratio b/a, simplifying any compound fractions (invert the divisor when dividing fractions).
  4. Evaluate θ₀ = arctan(b/a) on a calculator in radian mode.
  5. Apply quadrant correction: Q1 or Q4 → arg = θ₀; Q2 → arg = θ₀ + π; Q3 → arg = θ₀ − π.
  6. Confirm arg ∈ (−π, π] and round to required precision (commonly 2 d.p.).

Worked Micro-Example

Given $\dfrac{z_1}{z_2} = \dfrac{3}{5} - \dfrac{7}{10}i$: Re $= \tfrac{3}{5} > 0$, Im $= -\tfrac{7}{10} < 0$, so Q4. Ratio: $\dfrac{-7/10}{3/5} = -\dfrac{7}{10}\cdot\dfrac{5}{3} = -\dfrac{7}{6}$. Then $\arg = \arctan(-7/6) \approx -0.8621$ rad. Q4, so no correction: $\arg(z_1/z_2) \approx -0.86$ rad.

Variants — Same move, different surface

Number in Q2, e.g. z = −2 + 3i
arctan(3/(−2)) ≈ −0.98 rad, but Q2 requires +π, giving arg ≈ 2.16 rad.
Number in Q3, e.g. z = −1 − i
arctan(1) = π/4, but Q3 requires −π correction, giving arg = π/4 − π = −3π/4.
Pure imaginary or pure real
Skip arctan: arg(bi) = π/2 if b > 0, −π/2 if b < 0; arg(a) = 0 if a > 0, π if a < 0.
arg(z₁z₂) or arg(z₁/z₂) without Cartesian form available
Use arg(z₁) ± arg(z₂), then reduce result into (−π, π] by adding or subtracting 2π if it overshoots.

Common Misreadings

  • Skipping the quadrant check and reporting arctan(|b|/|a|) as a positive acute angle regardless of signs.
  • Inverting the ratio and computing arctan(a/b) instead of arctan(b/a).
  • Treating arg(z₁/z₂) as arg(z₁)/arg(z₂) rather than arg(z₁) − arg(z₂).

Pitfalls

  • Mishandling compound fractions: (−7/10)/(3/5) must invert the divisor to give −7/6, not −7/50 or −21/50.
  • Calculator left in degree mode, returning e.g. −49.4° instead of −0.86 rad.
  • Dropping the negative sign and reporting +0.86 rad (the Q1 reflection of the correct Q4 answer).
  • Adding π when the point is actually in Q4 — the correction is only for Q2/Q3.

A-level Scope Notes

Fully within Edexcel IAL Further Pure (FP1/FP2) treatment of complex numbers, where the principal argument is defined on (−π, π] and quadrant-aware arctan is the standard technique. No use of atan2 or branch-cut machinery beyond this convention is required.
