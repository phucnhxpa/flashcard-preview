Algebra & Functions

Rewrite bare constant as same-base logarithm to enable merging

Family: Constant To Log Conversion Frequency: 1 questions

Trigger — When to use

Fires when a logarithmic equation contains a bare numerical constant alongside one or more logarithms of the same base, e.g. $\log_b A + \log_b B = k$ or $\log_b A = \log_b B + k$. The constant blocks direct application of the product/quotient laws, which require every term to be a logarithm of the same base. Cue phrases: 'constant on one side', 'no log on the RHS but logs on the LHS', or any equation of the form $\log_b(\cdot) = c$ where $c$ is rational.

Motivation — Why this works

The identity $k = \log_b(b^k)$ allows any real constant to be rewritten as a logarithm of the same base, since $\log_b$ and $b^{(\cdot)}$ are inverse functions. Once every term is a $\log_b$, the addition/subtraction laws reduce the equation to the form $\log_b X = \log_b Y$, which is solved by equating arguments. This bridge is essential because log laws operate only on log expressions, not on stray constants.

Thought Process — The step-by-step reasoning

  1. Student notices the equation mixes $\log_b$ terms with a bare constant — laws can't be applied directly.
  2. Rules out exponentiating the whole equation immediately, since that produces a messy expression with $b^{\text{sum of logs}}$ that still needs simplifying.
  3. Commits to the cleaner route: convert the constant into a $\log_b$ so every term is homogeneous in base.
  4. Recalls $k = \log_b(b^k)$ and computes $b^k$ explicitly to get a single numerical argument.
  5. Substitutes back and proceeds to merge logs via product/quotient laws, then equates arguments.

Mechanic — Concrete operations

  1. Identify the bare constant $k$ and the log base $b$ in the equation.
  2. Apply the identity $k = \log_b(b^k)$, computing $b^k$ as a single number.
  3. Substitute the new log expression back into the equation so every term is $\log_b(\cdot)$.
  4. Combine logs using product/quotient laws to reach $\log_b X = \log_b Y$.
  5. Equate arguments $X = Y$ and solve, then check domain validity (arguments positive).

Worked Micro-Example

Solve $\log_3(1-x)^2 = 3 - \log_3(\text{something})$. Convert the constant: $3 = \log_3(3^3) = \log_3 27$. The RHS becomes $\log_3 27 - \log_3(\cdot) = \log_3\!\left(\tfrac{27}{\cdot}\right)$, so the equation reduces to $\log_3(1-x)^2 = \log_3\!\left(\tfrac{27}{\cdot}\right)$, giving $(1-x)^2 = \tfrac{27}{\cdot}$.

Variants — Same move, different surface

Base-10 logarithm equation with a constant $c$
Rewrite $c = \log_{10}(10^c)$; e.g. $2 = \log_{10} 100$.
Natural logarithm equation with a constant $c$
Rewrite $c = \ln(e^c)$; the argument stays in exponential form unless $c$ is small/integer.
Constant on the same side as the logs (e.g. $\log_b A + k = \log_b B$)
Same conversion; after rewriting, isolate logs on each side or merge into a single log equation.
Negative or fractional constant
$-k = \log_b(b^{-k}) = \log_b(1/b^k)$; e.g. $-2 = \log_3(1/9)$.

Common Misreadings

  • Treating the constant as if it could be 'added inside' a logarithm, e.g. writing $\log_b A + k = \log_b(A+k)$.
  • Exponentiating the whole equation prematurely, producing $b^{\text{LHS}} = b^{\text{RHS}}$ with sums of logs in the exponent that still need the laws to simplify.
  • Using the wrong base when converting, e.g. writing $3 = \log_{10} 1000$ in an equation with $\log_3$ terms.

Pitfalls

  • Confusing $b^k$ with $b\cdot k$, e.g. writing $3 = \log_3 9$ instead of $\log_3 27$.
  • Dropping the conversion on only one occurrence of the constant when it appears multiple times.
  • Forgetting to verify that solutions keep all log arguments positive after equating arguments.

A-level Scope Notes

Fully within Edexcel IAL P2/P3 logarithm content: uses only the identity $k=\log_b(b^k)$ and the standard product/quotient/power laws of logarithms.
