Split the middle term using the ac product-sum search to factorise a non-monic quadratic

Family: Factorisation By Grouping Frequency: 6 questions

Trigger — When to use

A quadratic ax² + bx + c = 0 (or expression) appears with integer coefficients and leading coefficient a ≠ 1, e.g. 4p² − 33p + 8, 9p² + 26p − 3, 2x² − 7x + 5, 3p² − 10p − 8, 2k² + 3k − 35. The product ac is a manageable integer with few factor pairs, and contextual cues (a 'show/hence' instruction, a non-calculator paper, or a perfect-square discriminant) signal that rational roots are intended. Equivalent phrasings: 'factorise this quadratic exactly', 'solve without the formula', 'the discriminant b² − 4ac is a perfect square so it must factor nicely'.

Motivation — Why this works

For a non-monic quadratic the factorisation (px + q)(rx + s) is not obvious by inspection because pr = a and qs = c can be split many ways. The ac-method bypasses guesswork: any valid factorisation must produce a middle term whose two parts multiply to ac and sum to b, so finding that integer pair lets us rewrite bx as the sum of two terms and factor by grouping. This delivers exact rational roots faster and more reliably than the quadratic formula, and keeps the answer in bracket form — useful for sign analysis in inequalities or substitution into later parts of a question.

Thought Process — The step-by-step reasoning

  1. Notice the quadratic has integer coefficients and a ≠ 1; check whether it is likely to factorise by computing the discriminant or by spotting that ac has few factor pairs.
  2. Rule out the quadratic formula (works but is slower and risks arithmetic slips when clean factors exist) and completing the square (overkill for finding roots).
  3. Commit to the ac-method: compute ac and search for an integer pair (m, n) with mn = ac and m + n = b. Use sign logic — if ac > 0 both factors share b's sign; if ac < 0 the factors have opposite signs and the larger-magnitude one carries b's sign.
  4. Bridge to operation: rewrite bx as mx + nx, splitting the middle term so the four-term polynomial is set up for grouping.
  5. Inside the move, choose the order of the split so the first pair shares an obvious common factor — this guarantees the second pair produces the same binomial bracket and avoids sign-handling errors when the leading term of the second group is negative.
  6. Extract the common binomial, set each factor to zero, and read off the rational roots.

Mechanic — Concrete operations

  1. Identify a, b, c from ax² + bx + c and compute the product P = ac.
  2. Find integers m, n with mn = P and m + n = b by enumerating factor pairs of |P| and assigning signs.
  3. Rewrite the quadratic: ax² + mx + nx + c.
  4. Group as (ax² + mx) + (nx + c) and factor each pair: x(a x + m_reduced) + k(a x + m_reduced), so both groups share the same binomial.
  5. Extract the common bracket to obtain (linear)(linear) = 0.
  6. Apply the zero-product principle and solve each linear factor for x.

Worked Micro-Example

Solve $3p^2 - 10p - 8 = 0$. Compute $ac = 3(-8) = -24$; find $-12$ and $+2$ (product $-24$, sum $-10$). Split: $3p^2 - 12p + 2p - 8 = 0$. Group: $3p(p-4) + 2(p-4) = 0$, so $(3p+2)(p-4) = 0$, giving $p = -\tfrac{2}{3}$ or $p = 4$.

Variants — Same move, different surface

Monic quadratic (a = 1)
Skip the splitting step — directly find factor pair of c summing to b and write (x + m)(x + n).
Discriminant is not a perfect square / ac has no nice factor pair
Abandon factorisation and use the quadratic formula or completing the square instead.
Disguised quadratic, e.g. in e^x, sin x, or a substitution u = x²
Apply the same ac-method to the variable substitute, then back-substitute and solve.
Quadratic inequality
Factorise identically; the bracket form gives the critical values directly for a sign table or sketch.

Common Misreadings

  • Searching for factor pairs of c (the constant) instead of ac when a ≠ 1.
  • Reaching for the quadratic formula reflexively without first checking whether the coefficients factorise cleanly.

Pitfalls

  • Sign errors when ac < 0 or when extracting a factor from a group whose leading term is negative — always verify the second group reproduces the same bracket as the first.
  • Selecting a pair that satisfies the product but not the sum (or vice versa) — verify both conditions before splitting.
  • Choosing an unfortunate split order so the two groups don't share a common binomial; if this happens, swap the order of m and n.

A-level Scope Notes

Standard technique within Edexcel IAL P1/P2 (and reused throughout Pure and Further Pure when solving quadratic equations or factorising quadratic expressions). No formula-booklet entry is required; the quadratic formula in the booklet is the explicit fallback. Do not generalise to cubics by 'splitting' — for cubics use the factor theorem instead.
