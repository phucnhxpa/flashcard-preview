Complex Numbers

Apply de Moivre's theorem with 2kπ offsets to extract all n distinct nth roots

Family: De Moivre Root Extraction Frequency: 3 questions

Trigger — When to use

Fires whenever the equation has the form z^n = w where w is given (or has been put) into modulus–argument form r(cos θ + i sin θ) or r·cis(θ) or re^{iθ}, and the question asks for all n roots (not just one). Visual cues: an explicit power z^n on the left, a complex constant on the right, and wording such as 'find all solutions', 'find the n roots', 'solve for z', or 'express in the form a + ib'. Equivalently, any 'roots of unity' or 'nth roots of w' problem. If only one root is requested the move still applies but only k = 0 is needed.

Motivation — Why this works

De Moivre's theorem says [r(cos θ + i sin θ)]^n = r^n(cos nθ + i sin nθ). Reversing it for the 1/n power gives the candidate root r^{1/n} cis(θ/n), but this is only ONE root — and the Fundamental Theorem of Algebra guarantees exactly n. The fix exploits the periodicity of cos and sin: θ and θ + 2kπ describe the same complex number, so writing w = r·cis(θ + 2kπ) before taking the nth root produces n different arguments (θ + 2kπ)/n for k = 0, 1, …, n−1, equally spaced by 2π/n around the circle of radius r^{1/n}. This is the only systematic method that guarantees no root is missed.

Thought Process — The step-by-step reasoning

  1. I notice z^n = (something complex) and the question wants all n solutions — by the Fundamental Theorem of Algebra there must be exactly n.
  2. I rule out factorising (z^n − w) directly (no nice real factorisation in general) and rule out Cartesian z = x + iy with simultaneous equations (works for n = 2 but explodes for larger n).
  3. I commit to de Moivre because the RHS is, or can easily be, written in modulus–argument form, and de Moivre converts an nth-power equation into a single linear equation in the argument.
  4. Bridge: I rewrite w as r·cis(θ + 2kπ) — adding 2kπ is legitimate because it doesn't change the value of w but it WILL change (θ + 2kπ)/n once I divide by n.
  5. Tactical decision: I let k run over n consecutive integers (usually 0, 1, …, n−1, but sometimes − ⌊n/2⌋, …, ⌊n/2⌋ if the question wants principal arguments in (−π, π]).
  6. Finally I take the nth root of the modulus (r^{1/n}, the unique positive real root) AND divide each shifted argument by n, then convert to a + ib if required.

Mechanic — Concrete operations

  1. Write w in modulus–argument form: w = r(cos θ + i sin θ), with r > 0 and θ a chosen argument.
  2. Insert the periodicity: w = r[cos(θ + 2kπ) + i sin(θ + 2kπ)] for integer k.
  3. Apply de Moivre in reverse: z_k = r^{1/n} [cos((θ + 2kπ)/n) + i sin((θ + 2kπ)/n)].
  4. Substitute k = 0, 1, 2, …, n−1 to list the n distinct roots.
  5. If required, adjust each argument into the principal range (−π, π], or convert each root to Cartesian form a + ib using exact surd values.

Worked Micro-Example

Solve $z^3 = 8i$. Write $8i = 8\,\mathrm{cis}(\tfrac{\pi}{2} + 2k\pi)$. Then $z_k = 8^{1/3}\,\mathrm{cis}\!\left(\tfrac{\pi/2 + 2k\pi}{3}\right) = 2\,\mathrm{cis}\!\left(\tfrac{\pi}{6} + \tfrac{2k\pi}{3}\right)$ for $k=0,1,2$, giving arguments $\tfrac{\pi}{6}, \tfrac{5\pi}{6}, -\tfrac{\pi}{2}$, i.e. $z = \sqrt 3 + i,\ -\sqrt 3 + i,\ -2i$.

Variants — Same move, different surface

Roots of unity: z^n = 1
Set r = 1, θ = 0; the n roots are simply cis(2kπ/n) for k = 0, 1, …, n−1.
Equation reads z^n = −r (negative real)
Use θ = π, so arguments become (π + 2kπ)/n = (2k+1)π/n — the often-seen 'odd-multiples-of-π/n' form.
Question gives or expects exponential form z = ρe^{iφ}
Same logic: z_k = r^{1/n} e^{i(θ + 2kπ)/n}. Cosmetic difference only.
Quadratic/quartic where you first solve for w, then z^2 = w (e.g. (z^2)^2 + bz^2 + c = 0)
Apply the move twice: solve for each w-value, then take square roots of each, giving 2 × 2 = 4 roots of z.

Common Misreadings

  • Treating z^n = w as if it had only one solution — taking just the principal nth root and stopping.
  • Starting k at 1 (or running k = 1, …, n) and so missing the k = 0 root or duplicating the k = n root.
  • Confusing this with binomial expansion of (a+ib)^n — that is the FORWARD direction of de Moivre, not the root-extraction direction.

Pitfalls

  • Forgetting to take the nth root of the modulus: writing r·cis(...) instead of r^{1/n}·cis(...).
  • Dividing θ by n but forgetting to also divide the 2kπ term, so roots end up in the wrong place.
  • Arithmetic slips when simplifying (θ + 2kπ)/n, especially when θ is a fraction of π.
  • Leaving final arguments outside the principal range (−π, π] when the question explicitly asks for principal arguments.
  • Failing to convert to exact Cartesian form a + ib using surd values of cos and sin when the question requires it.

A-level Scope Notes

Fully within Edexcel IAL Further Pure (FP2/FP3) — de Moivre's theorem and nth roots of a complex number are listed in the formula booklet. Avoid invoking general Galois-theoretic or logarithm-of-complex-number arguments; stay with the (cos θ + i sin θ) / cis / e^{iθ} formulations the syllabus uses.
