Trigger — When to use

Fires when a question first establishes a model y = f(x) with all parameters determined (often in part (a)), then in a later part supplies a value of the output y and asks for the corresponding input x. Cues include phrases like 'use your model to find the time when…', 'find the value of t for which…', or being given the dependent variable and asked for the independent one. The model is fully specified, so the task collapses to solving a single equation in one unknown.

Motivation — Why this works

Once parameters are pinned down, the model is just an equation; substituting the given output reduces it to one equation in one unknown. Algebraic inversion (isolate, root-extract, log, etc.) typically yields multiple mathematical solutions, but the modelling context (time, length, population) constrains the variable to a physical domain. Selecting the admissible root is a modelling decision that must be justified, not a silent algebraic choice. Carrying unrounded parameter values through the inversion preserves accuracy in the final reported answer.

Thought Process — Step-by-step reasoning

  1. I notice the model is fully determined and I'm being asked to go backwards: output is given, input is unknown.
  2. I rule out re-fitting or re-deriving the model — parts (a) values stand and should be reused unrounded.
  3. I commit to substituting the given output and isolating the unknown algebraically as a single equation.
  4. I bridge to the first operation: substitute, then peel off operations in reverse order (subtract constant, divide coefficient, take root/log).
  5. After root extraction I check the physical meaning of the variable (e.g. t ≥ 0) and explicitly reject any root outside that domain.

Mechanic — Concrete operations

  1. Substitute the given output value into the fitted model, being careful with any transformation (e.g. H³ when H = 5 gives 125, not 5).
  2. Isolate the term containing the unknown by subtracting the constant and dividing by the coefficient.
  3. Apply the inverse operation (square root, cube root, ln, etc.) to free the unknown.
  4. List all mathematical roots, then reject those incompatible with the physical context, citing the reason.
  5. Round only at the end, using unrounded parameter values throughout.

Worked Micro-Example

Given the model $H^3 = 0.364t^2 + 4.724$, find $t$ when $H = 5$.\\ Substitute: $125 = 0.364t^2 + 4.724 \Rightarrow t^2 = \dfrac{120.276}{0.364} \approx 330.43$.\\ So $t = \pm\sqrt{330.43}$; reject the negative root since $t$ is years after planting.\\ Therefore $t \approx 18.2$ years (1 d.p.).

Variants — Same move, different surface

Kinematics: given displacement $s$, find $t$ from $s = ut + \tfrac{1}{2}at^2$
Solve a quadratic in $t$; reject negative or pre-launch roots based on the motion's start time.
Exponential growth/decay model $P = A e^{kt}$, given $P$ find $t$
Divide by $A$ and take $\ln$; only one real root, but check it lies in the modelled time interval.
Logarithmic model $y = a + b\ln x$, given $y$ find $x$
Isolate $\ln x$ then exponentiate; the unique root is automatically positive, but check domain.
Power-law model $y = ax^n$ from log-linear regression, given $y$ find $x$
Rearrange to $x = (y/a)^{1/n}$; for even $n$, reject the negative root by context.

Common Misreadings

  • Substituting the given value of $H$ directly where $H^n$ appears (e.g. using $5$ instead of $5^3 = 125$).
  • Treating the question as a fresh fitting problem rather than a reuse of the part (a) parameters.
  • Reading 'find $t$' as asking for $t^2$ and stopping at the squared quantity.

Pitfalls

  • Using pre-rounded parameter values (e.g. $q = 4.72$ instead of $4.724$), which can push the final answer outside the accepted range.
  • Forgetting to take the square/cube root after isolating $t^2$ or $t^3$.
  • Including the negative root in the final answer without justifying its rejection on physical grounds.
  • Rounding intermediate quantities and then rounding again at the end, compounding error.

A-level Scope Notes

Standard Edexcel IAL territory: appears in modelling questions across P2/P3/P4 (kinematics, exponential models, log-linear regression contexts) where a fitted model is run in reverse and a physical-domain argument selects the valid root.
