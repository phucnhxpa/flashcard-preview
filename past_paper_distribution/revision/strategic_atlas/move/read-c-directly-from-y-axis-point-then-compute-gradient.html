Coordinate Geometry

Read c directly from y-axis point, then compute gradient

Family: Intercept Recognition Frequency: 1 questions

Trigger — When to use

Fires when finding the equation of a straight line through two given points and one point has x-coordinate 0, so it lies on the y-axis. Common phrasings include 'the line passes through (0, k) and (a, b)', or in log-linearised contexts 'when t = 0, log N = ...'. Also fires in experimental/graph-reading questions where a y-intercept is read off the axis directly. The cue is: an x = 0 entry in the data.

Motivation — Why this works

In y = mx + c, the constant c is by definition the y-value when x = 0, so any point of the form (0, c) hands the intercept to you for free. This collapses the line-fitting problem to a single gradient calculation, bypassing point-slope rearrangement or solving simultaneous equations for m and c. It is the fastest correct route and minimises algebraic error. The same logic transfers to log-linear equations like log y = mx + c, where the y-axis intercept is the value of log y at x = 0.

Thought Process — The step-by-step reasoning

  1. I notice that one of the two given points has x = 0, so its y-coordinate is the y-intercept of the line.
  2. I rule out the point-slope form y − y₁ = m(x − x₁) and full simultaneous equations for m and c, since c is already visible.
  3. I commit to y = mx + c because c is read off and only m remains unknown.
  4. I set up the gradient calculation m = (y₂ − y₁)/(x₂ − x₁) using the two points as given.
  5. I keep track of which variable sits on the LHS — if the axes are log₁₀ N versus t, the final equation must read log₁₀ N = mt + c, not N = mt + c.

Mechanic — Concrete operations

  1. Identify the point with x = 0 and read off c as its y-coordinate.
  2. Compute the gradient m = (y₂ − y₁)/(x₂ − x₁) using both points.
  3. Substitute m and c into y = mx + c.
  4. Re-label the LHS/RHS variables to match the axes if the context is log-linear (e.g. log₁₀ N, ln y, etc.).

Worked Micro-Example

Line through $(0,\,3.08)$ and $(5,\,3.85)$. Intercept: $c = 3.08$. Gradient: $m = \dfrac{3.85 - 3.08}{5 - 0} = \dfrac{0.77}{5} = 0.154$. Equation: $\log_{10} N = 0.154\,t + 3.08$.

Variants — Same move, different surface

Neither point has x = 0 (e.g. (2, 5) and (6, 13))
Use point-slope form y − y₁ = m(x − x₁), then rearrange into y = mx + c if c is required.
Log-linear model log₁₀ y = mx + c with a tabulated value at x = 0
Read c off the (0, log₁₀ y) entry; gradient is computed in the log-coordinates, not in y itself.
Graph with a clearly marked y-intercept and one other readable point
Take c straight from the axis crossing and compute m from the second point and (0, c).
Exponential model y = Ab^x linearised as log y = (log b)x + log A
The y-intercept of the linearised graph gives log A, so A = 10^c (or e^c for ln).

Common Misreadings

  • Treating (0, k) as just another generic point and launching into simultaneous equations unnecessarily.
  • Confusing the y-intercept of a log-linear graph with the actual y-intercept of the original (non-linearised) curve.
  • Reading the intercept from a point that has y = 0 (the x-intercept) instead of x = 0.

Pitfalls

  • Swapping y-values in the gradient formula, producing a sign error in m.
  • Writing N = mt + c instead of log₁₀ N = mt + c when the vertical axis is log₁₀ N.
  • Rounding c or m too early and propagating error into later parts of the question.
  • Mixing the order of points so that (x₂ − x₁) and (y₂ − y₁) are taken inconsistently.

A-level Scope Notes

Fully within Edexcel IAL Pure Maths (coordinate geometry of straight lines) and routinely used in S/M contexts where exponential or power models are linearised via logarithms.
