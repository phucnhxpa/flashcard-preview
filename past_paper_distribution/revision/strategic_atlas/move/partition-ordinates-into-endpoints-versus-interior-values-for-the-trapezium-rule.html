Integration

Partition ordinates into endpoints versus interior values for the trapezium rule

Family: Formula Selection Frequency: 1 questions

Trigger — When to use

Fires whenever the trapezium rule is being applied to a table of y-values (or values computed from a given function at equally spaced x-values). Cues include phrases like 'use the trapezium rule with n strips' or 'estimate the integral using the values in the table'. Also fires when a student writes out the formula (h/2)[y₀ + yₙ + 2(y₁ + … + yₙ₋₁)] and must decide which tabulated values go where. The move is structural, not numerical — it must occur before any arithmetic.

Motivation — Why this works

The trapezium rule sums the areas of n adjacent trapezia, each of width h. When these are added, every interior ordinate is shared by exactly two trapezia, while the two endpoint ordinates each appear in only one. Algebraically collecting like terms therefore produces coefficient 1 on y₀ and yₙ and coefficient 2 on every y₁,…,yₙ₋₁. Partitioning the table cleanly into {endpoints} and {interior} is the structural core of applying the formula without error.

Thought Process — The step-by-step reasoning

  1. I notice a table of equally spaced x-values with corresponding y-values, and the question asks for a trapezium-rule estimate — so the formula (h/2)[y₀ + yₙ + 2(y₁ + … + yₙ₋₁)] is the right tool.
  2. I rule out treating all ordinates symmetrically (which would be the rectangle/midpoint idea) and rule out Simpson's-rule weights (1,4,2,4,…,1) — those belong to a different rule and are out of scope here.
  3. I commit to splitting the y-values into two disjoint groups before any arithmetic: the two endpoint ordinates (weight 1) and every interior ordinate (weight 2).
  4. Concretely, I label the leftmost tabulated y as y₀ and the rightmost as yₙ; everything strictly between them is interior.
  5. Tactically, I compute S_ends = y₀ + yₙ and S_mid = y₁ + y₂ + … + yₙ₋₁ as two separate sub-totals, then assemble (h/2)(S_ends + 2·S_mid) — this isolates the most common slip (forgetting the factor 2) into a single visible step.

Mechanic — Concrete operations

  1. Identify h, the common x-spacing, and confirm n = (number of ordinates) − 1.
  2. Read y₀ (first table value) and yₙ (last table value); compute S_ends = y₀ + yₙ.
  3. Read all interior values y₁, y₂, …, yₙ₋₁; compute S_mid = y₁ + y₂ + … + yₙ₋₁.
  4. Form the bracket B = S_ends + 2 · S_mid.
  5. Area estimate ≈ (h/2) · B; round to the precision requested.

Worked Micro-Example

Estimate $\int_{1}^{3} f(x)\,dx$ from the table $x=1,1.5,2,2.5,3$ with $y=2.0,\,2.7,\,3.4,\,3.9,\,4.2$. Here $h=0.5$, $n=4$. Endpoints: $S_{\text{ends}} = 2.0 + 4.2 = 6.2$. Interior: $S_{\text{mid}} = 2.7 + 3.4 + 3.9 = 10.0$. Bracket $B = 6.2 + 2(10.0) = 26.2$. Estimate $\approx \tfrac{0.5}{2}(26.2) = 6.55$.

Variants — Same move, different surface

Function given rather than a table
First tabulate y-values at x = a, a+h, …, b yourself, then partition exactly as above.
Number of strips n specified instead of number of ordinates
Compute h = (b − a)/n and note there will be n+1 ordinates, of which n−1 are interior.
Table contains an extra column (e.g. x and y² for a volume of revolution)
Apply the partition to the column actually being integrated (e.g. y² values), not to the y values themselves.
Two separate trapezium-rule estimates compared (e.g. n = 4 vs n = 8)
Re-partition for each value of n; an ordinate that was an endpoint in one calculation may become interior in the other.

Common Misreadings

  • Doubling all y-values including the endpoints, producing an overcount.
  • Doubling the endpoints and leaving the interior values single — the inverse weighting error.
  • Using Simpson's-rule weights (1, 4, 2, 4, …, 1) instead of trapezium weights (1, 2, 2, …, 2, 1).
  • Treating the number of table entries as n rather than n+1, leading to a wrong h.

Pitfalls

  • Misreading a y-value from the table (e.g., rounding or transposing digits), especially with several decimal places.
  • Forgetting to multiply the interior sum by 2 and only noticing at the final answer stage.
  • Applying the (h/2) factor before doubling the interior sum, then doubling the entire bracket — a sign of muddled order of operations.
  • Using h = b − a instead of (b − a)/n when h is not stated explicitly.

A-level Scope Notes

Fully within Edexcel IAL Pure Mathematics: the trapezium rule with equally spaced ordinates is the only numerical-integration rule examined, and questions stay at modest n (typically 4–6 strips).
