Differentiation

Sketch curve via end-behaviour and turning points to count solutions of f(x)=k

Family: Curve Sketching For Solutions Frequency: 2 questions

Trigger — When to use

Fires when a question asks for the values (or range) of a constant k for which an equation f(x) = k has a specified number of distinct roots — e.g. 'find the values of k such that f(x) = k has exactly two solutions' or 'state the range of k for which f(x) = k has at least two roots'. The function f is typically transcendental (involving e^{ax}, ln x, or a polynomial × exponential), so f(x) = k cannot be solved algebraically. The cue is: turning points have already been found in an earlier part, and now the student must convert that information into a count of intersections.

Motivation — Why this works

Because f(x) = k is transcendental, no algebraic solution exists; instead, the number of roots equals the number of intersections of the horizontal line y = k with the graph of y = f(x). The global shape of the curve is fully determined by (i) its end behaviour at ±∞, (ii) its stationary points and their values, and (iii) any zeros or asymptotes. Once the curve is sketched, reading off the k-range for n intersections is immediate. This is the standard 'number-of-roots-from-a-graph' principle and is the only viable route at A-level for such questions.

Thought Process — The step-by-step reasoning

  1. Notice the question asks about the *number* of solutions of f(x) = k, not the solutions themselves — so this is a counting/graphical problem, not an algebraic one.
  2. Rule out solving f(x) = k algebraically: the equation is transcendental (mixes polynomial and exponential parts), so no closed-form root exists.
  3. Commit to a graphical argument: the count of roots = count of intersections of y = k with y = f(x), which depends only on the curve's shape.
  4. Bridge to operations: gather the data needed to determine the shape — limits as x → ±∞, the stationary points (already found in the previous part), and the values of f at those points, plus any sign information.
  5. Trace the curve mentally from x = −∞ to x = +∞, recording the sequence of heights it passes through, including any horizontal asymptotes that are *approached but not attained*.
  6. Tactical decision at boundaries: at a value of k equal to a local max/min, the line is *tangent*, so that intersection counts once (not twice); at a value of k equal to an asymptotic limit that is never attained, the line does *not* intersect there. Decide carefully whether each boundary k should be included (≤) or excluded (<).

Mechanic — Concrete operations

  1. Compute the end behaviour: evaluate lim_{x→−∞} f(x) and lim_{x→+∞} f(x), being careful about whether limits are approached from above (0⁺) or below (0⁻).
  2. List the stationary points (x_i, f(x_i)) found in the previous part, classifying each as local max or local min.
  3. Identify any zeros or sign restrictions of f (e.g. f(x) ≥ 0 if f contains a square factor).
  4. Sketch (or describe) the curve: a monotone segment between each pair of consecutive critical features, with the correct heights at each turning point and asymptote.
  5. For each candidate range of k (separated by the heights of the turning points and asymptotic limits), count the intersections of y = k with the sketch.
  6. Read off the k-values matching the required number of roots, deciding inclusion/exclusion of each boundary by checking tangency vs. asymptotic non-attainment.

Worked Micro-Example

Let $f(x) = (x-2)^2 e^{3x}$ with stationary points at $(4/3,\, 4e^4/9)$ (max) and $(2, 0)$ (min). End behaviour: $f(x) \to 0^+$ as $x \to -\infty$, $f(x) \to +\infty$ as $x \to +\infty$, and $f(x) \geq 0$ everywhere. Tracing: rises from $0^+$ to peak $4e^4/9$, falls to $0$, rises to $\infty$. Hence $f(x)=k$ has $\geq 2$ roots iff $0 < k \leq 4e^4/9$ (k=0 gives only x=2; k=4e^4/9 gives a tangent at the max plus one root on the right).

Variants — Same move, different surface

Curve given parametrically as x = g(y) (e.g. x = y e^{2y}) with horizontal line x = k
Same logic applied with axes swapped: analyse g(y) as y → ±∞, locate its turning points in y, and count intersections of the vertical-in-(x,y) line x = k with the curve.
Question asks for 'exactly n' rather than 'at least n' roots
Same sketch, but each boundary k (turning-point height or asymptotic limit) must be tested individually to decide which exact count it produces.
Function has a vertical asymptote (e.g. involves ln x or 1/x)
Add the asymptote behaviour to the sketch data; the curve is now in pieces, and intersections are counted piece by piece.

Common Misreadings

  • Including a boundary k where the line is tangent to a turning point as if it gave two distinct roots (it gives one repeated root, often counted as one intersection point).
  • Including k equal to an asymptotic limit (e.g. k = 0 when f → 0⁺ but never reaches 0), mistaking the asymptote for an actual intersection.
  • Confusing 'horizontal line y = k' with 'vertical line x = k' (or vice versa in parametric setups), leading to an entirely wrong counting picture.

Pitfalls

  • Getting the sign of the limit wrong (e.g. claiming f(x) → 0⁻ when actually f(x) → 0⁺), which flips the inclusion of k = 0.
  • Forgetting that exponential decay dominates polynomial growth, so x^n e^{ax} → 0 (not ±∞) as x → −∞ when a > 0.
  • Writing strict inequality where weak is needed, or vice versa, at tangent boundaries — always re-check whether the tangent k yields the required number of roots.

A-level Scope Notes

Fully within Edexcel IAL P3/P4 and FP curve-sketching expectations: stationary points via f'(x)=0, standard limits of x^n e^{ax}, and graphical counting of roots. No formula booklet entry is required. Avoid invoking L'Hôpital's rule or formal asymptotic analysis — qualitative dominance arguments ('exponential beats polynomial') are sufficient and expected.
