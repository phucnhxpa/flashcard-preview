Strategic Atlas / Proof / Induction Conclusion
Proof

Close induction with a complete formal conclusion to secure the final A-mark

Family: Induction Conclusion Frequency: 3 questions

Trigger — When to use

The algebraic work of an induction proof is complete: the base case has been verified and the inductive step P(k) ⟹ P(k+1) has been algebraically established. The question is an Edexcel-style 'Prove by induction that...' problem (typically worth 6+ marks in FP1/FP2/CP1/CP2), where the mark scheme reserves a dedicated final A-mark for the concluding sentence. This trigger fires regardless of the specific result being proved (sums, divisibility, matrix powers, derivatives, recurrence formulae) — whenever you have finished the induction machinery, you must explicitly tie it together.

Motivation — Why this works

Mathematical induction is a logical argument, not just an algebraic manipulation: showing P(1) is true and P(k) ⟹ P(k+1) only proves the result for all n in the stated domain when the principle of induction is explicitly invoked. The conclusion sentence is what converts two finite checks into a universal statement. Edexcel mark schemes consistently allocate a dedicated mark for a properly worded conclusion, and this mark is frequently dropped — even by students whose algebra is flawless — because they treat the conclusion as decorative rather than as the logical capstone of the proof.

Thought Process — The step-by-step reasoning

  1. I notice I have just finished verifying the inductive step (e.g. shown the target expression for n = k+1 in closed form). The proof feels 'done' algebraically — but I remind myself that on Edexcel papers an induction conclusion is a separately marked component.
  2. I rule out simply writing 'hence proved' or 'true for all n' — these vague closings are exactly what the mark scheme penalises. I also rule out splitting the conclusion across earlier lines; examiners want one clean closing statement.
  3. I commit to writing a single sentence (or short paragraph) that explicitly names all four required elements: (i) the base case is true, (ii) the implication P(k) ⟹ P(k+1) has been shown, (iii) the words 'by (the principle of) mathematical induction', and (iv) the domain, stated precisely as 'for all positive integers n' or 'for all n ∈ ℤ⁺'.
  4. I check the question wording for the correct domain — usually 'all positive integers n' (n ≥ 1), but occasionally 'n ≥ 0' or 'n ≥ 2'. My conclusion must match this exactly.
  5. I write the conclusion in a templated form so I cannot accidentally omit a component under exam pressure.

Mechanic — Concrete operations

  1. After the inductive step, write: 'The statement is true for n = 1 (base case).'
  2. Write: 'If the statement is true for n = k, then it has been shown to be true for n = k + 1.'
  3. Write: 'Therefore, by the principle of mathematical induction, the statement is true for all positive integers n.' (Adjust the domain to match the question.)
  4. Optionally compress into one sentence: 'Since the result is true for n = 1, and true for n = k implies true for n = k + 1, by the principle of mathematical induction the result holds for all n ∈ ℤ⁺.'

Worked Micro-Example

Having shown $\sum_{r=1}^{1} r = 1 = \tfrac{1\cdot 2}{2}$ (base case) and $\sum_{r=1}^{k+1} r = \tfrac{k(k+1)}{2} + (k+1) = \tfrac{(k+1)(k+2)}{2}$ (inductive step), close with: 'The result is true for $n = 1$. If the result is true for $n = k$, it has been shown to be true for $n = k + 1$. Therefore, by the principle of mathematical induction, $\sum_{r=1}^{n} r = \tfrac{n(n+1)}{2}$ for all positive integers $n$.'

Variants — Same move, different surface

Base case is n = 0 (e.g. divisibility statements starting at n = 0)
Replace 'true for n = 1' with 'true for n = 0' and conclude 'for all non-negative integers n' or 'for all n ∈ ℤ, n ≥ 0'.
Base case is n = 2 (e.g. some inequality proofs)
State the base case at n = 2 and conclude 'for all integers n ≥ 2'.
Strong induction (Further Maths level, e.g. recurrence relations needing P(k) and P(k−1))
The implication clause becomes 'if the result is true for all integers up to and including k, it has been shown to be true for n = k + 1', and base cases for both n = 1 and n = 2 must be referenced.
Matrix induction (proving Aⁿ formula)
Same template; ensure the domain 'for all positive integers n' is explicit, since matrix powers are only defined for n ≥ 1 (or n ≥ 0 if A is invertible and stated).

Common Misreadings

  • Treating the conclusion as a formality and writing 'hence proved' or 'QED' — this loses the mark because no mechanism (induction) is named.
  • Believing that once the algebra of the inductive step is complete, the proof is finished — the logical closing is a separate, marked component.
  • Confusing 'for all n' with 'for all positive integers n' — the latter (or n ∈ ℤ⁺) is required for precision.

Pitfalls

  • Omitting 'by (the principle of) mathematical induction' — examiners want to see the proof method named explicitly.
  • Referencing only the implication ('true for k so true for k+1') without also referencing the base case — both must be cited.
  • Stating the wrong domain (e.g. 'for all real n' or 'for all n') — must match the question's stated set, typically positive integers.
  • Running out of time and skipping the conclusion entirely — write it as soon as the inductive step finishes, not as an afterthought.

A-level Scope Notes

This applies to all Edexcel IAL Further Maths induction proofs (FP1 sums and divisibility; FP2/CP2 matrix powers, derivatives, recurrence). No formula-booklet entry is invoked — induction is a proof method, not a formula. Avoid invoking 'transfinite' or 'structural' induction, which are out of scope.
