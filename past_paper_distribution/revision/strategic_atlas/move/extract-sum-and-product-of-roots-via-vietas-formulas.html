Extract sum and product of roots via Vieta's formulas

Family: Vieta'S Formulas Frequency: 3 questions

Trigger — When to use

The question presents a quadratic equation ax² + bx + c = 0 and labels its roots α and β (or similar), then asks for expressions involving those roots — typically α+β, αβ, α²+β², 1/α + 1/β, or new equations whose roots are functions of α, β. The phrasing 'roots are α and β' or 'write down α+β and αβ' is the giveaway. Crucially, the question never demands you solve for α and β individually, and often the discriminant is negative or yields ugly surds, signalling that explicit roots are not the intended path.

Motivation — Why this works

Vieta's formulas link the coefficients of a polynomial directly to the elementary symmetric polynomials in its roots: for ax² + bx + c = 0, α+β = −b/a and αβ = c/a. This bypasses the quadratic formula entirely — useful when roots are complex, irrational, or simply unnecessary. Every symmetric function of α and β (e.g. α²+β² = (α+β)² − 2αβ) can be rebuilt from these two quantities, so extracting them is the universal first move for any roots-of-polynomial problem.

Thought Process — The step-by-step reasoning

  1. Notice the question names the roots α, β but never asks for their numerical values — only for symmetric expressions in them.
  2. Consider solving via the quadratic formula, but rule it out: either the discriminant is negative (complex roots), or it produces surds that would make later arithmetic ugly.
  3. Commit to Vieta's: it gives exact, clean values for α+β and αβ directly from the coefficients, and these are the only ingredients needed because every subsequent expression is symmetric in α, β.
  4. Rewrite the equation in standard form ax² + bx + c = 0 if it is not already, and read off a, b, c carefully — paying attention to signs.
  5. Apply α+β = −b/a and αβ = c/a, watching the double-negative trap when b is itself negative.

Mechanic — Concrete operations

  1. Rearrange the equation to standard form ax² + bx + c = 0 if needed.
  2. Identify the coefficients a, b, c (with their signs).
  3. Compute α + β = −b/a.
  4. Compute αβ = c/a.
  5. Record both values; these become the building blocks for all later parts.

Worked Micro-Example

Given $3x^2 - 6x + 5 = 0$ with roots $\alpha, \beta$: identify $a=3,\ b=-6,\ c=5$. Then $\alpha+\beta = -\frac{-6}{3} = 2$ and $\alpha\beta = \frac{5}{3}$. These two values now feed every subsequent symmetric expression, e.g. $\alpha^2+\beta^2 = (\alpha+\beta)^2 - 2\alpha\beta = 4 - \tfrac{10}{3} = \tfrac{2}{3}$.

Variants — Same move, different surface

Cubic equation ax³ + bx² + cx + d = 0 with roots α, β, γ
Use Σα = −b/a, Σαβ = c/a, αβγ = −d/a — same coefficient-to-symmetric-function principle, extended to three elementary symmetric polynomials.
Equation given in non-standard form, e.g. 5 = 6x − 3x²
Rearrange to 3x² − 6x + 5 = 0 first, then read off a, b, c. Never read coefficients off an unrearranged equation.
Quartic ax⁴ + bx³ + cx² + dx + e = 0
The pattern continues: signs alternate, giving four elementary symmetric polynomials in the four roots (within Further Maths scope when relevant).

Common Misreadings

  • Reading the question as 'find the roots' and launching into the quadratic formula, producing complex or surd roots that derail the rest of the problem.
  • Confusing which coefficient is b and which is c when the equation is written in non-standard order (e.g. with terms rearranged).
  • Writing α+β = b/a instead of −b/a, forgetting the sign convention.

Pitfalls

  • Sign error from the double negative: when b is itself negative, α+β = −b/a is positive — e.g. for 3x² − 6x + 5, α+β = −(−6)/3 = 2, not −2.
  • Forgetting to divide by the leading coefficient a when a ≠ 1 — e.g. writing α+β = 5 instead of 5/2 for 2x² − 5x + 7.
  • Applying Vieta's to an equation not yet in standard form, picking up wrong coefficients.

A-level Scope Notes

Fully within Edexcel IAL Further Pure scope (FP1: Roots of polynomial equations). The quadratic case is also implicit in P1/P2 work on quadratics. Cubic and quartic extensions belong to FP1/FP2. Avoid invoking Newton's identities or general symmetric function theory — stay with elementary symmetric polynomials only.
