Coordinate Geometry

Read implicit boundary (axis) from the figure to close the region

Family: Boundary Completeness Check Frequency: 1 questions

Trigger — When to use

Fires when a question asks you to define a shaded region R by inequalities, but the named curves/lines in the stem do not by themselves enclose a finite area. Typical cue: the figure shows R hugging an axis (or a previously drawn axis line) that is not relabelled in the stem. Also fires when two of the named boundaries meet at the origin and a third is a vertical/horizontal line, leaving one side open. Phrases like 'the region R bounded by l₁, C and l₂' with a diagram are the classic setup.

Motivation — Why this works

A region described by a system of inequalities must be fully enclosed; otherwise the inequalities admit points outside the shaded figure. Examiners deliberately let the diagram carry one boundary (commonly an axis) to test whether students extract information from figures, not just text. Adding the missing inequality x ≥ 0 (or y ≥ 0) closes the region and matches the mark scheme's full description. Without it the description is incomplete and a definition mark is lost.

Thought Process — The step-by-step reasoning

  1. Notice that the stem names only three boundaries but the diagram clearly shows R is finite.
  2. Rule out the assumption that the named boundaries alone suffice by mentally testing a point just to the left of the y-axis — it would still satisfy the three given inequalities, so the description is open.
  3. Commit to inspecting the figure for an unlabelled boundary; identify the y-axis (or x-axis) as the missing edge.
  4. Translate that geometric edge into an inequality: the y-axis bounding on the left gives x ≥ 0.
  5. Decide whether the boundary is included (≥/≤) or excluded (>/<); a solid line on the diagram and a closed region indicate ≥, then combine with the existing x ≤ 16 into 0 ≤ x ≤ 16.

Mechanic — Concrete operations

  1. Inspect the figure and list every line/curve that touches R, including unlabelled axes.
  2. Compare with the inequalities you have so far; identify which side of R is not yet bounded.
  3. Write the missing boundary as an inequality (e.g. x ≥ 0 for the y-axis, y ≥ 0 for the x-axis).
  4. Use ≥/≤ since boundary points belong to R (solid line in the figure).
  5. Where possible, combine with an existing opposite-side inequality into a compact double inequality such as 0 ≤ x ≤ 16.

Worked Micro-Example

Figure shows R bounded by l₁: y = 2x, the curve C: y = \sqrt{x}, and l₂: x = 16, with R sitting in x ≥ 0. The named pieces give y ≤ 2x, y ≥ \sqrt{x}, x ≤ 16, but a point like (-1, -1) satisfies all three, so R is not yet closed. Reading the y-axis as the left boundary, we add x ≥ 0 and write the full description as 0 ≤ x ≤ 16, \sqrt{x} ≤ y ≤ 2x.

Variants — Same move, different surface

Region sits above the x-axis with curves meeting it
Add y ≥ 0 as the implicit lower boundary.
Region lies in the first quadrant bounded only by a curve and a slanted line
Add both x ≥ 0 and y ≥ 0 to close the region.
Polar/parametric region whose Cartesian sketch hugs an axis
Convert the axis to its Cartesian inequality (x ≥ 0 or y ≥ 0) before integrating.
Strict inequality from a dashed line in the figure
Use > or < instead of ≥/≤ for that boundary.

Common Misreadings

  • Treating only the boundaries named in the stem as the full set and ignoring the axis shown in the diagram.
  • Assuming x ≥ 0 is automatic from context and not stating it, when the question asks for a complete set of inequalities defining R.
  • Confusing the y-axis (x = 0) with y = 0 when writing the inequality.

Pitfalls

  • Writing x > 0 instead of x ≥ 0 when the boundary point (origin or a point on the y-axis) lies in R.
  • Writing the inequality the wrong way round (x ≤ 0) by mis-identifying which side of the axis R lies on.
  • Forgetting to merge x ≥ 0 with x ≤ 16 into 0 ≤ x ≤ 16 when the question asks for a tidy description.

A-level Scope Notes

Standard for Edexcel IAL Pure (C2/C34) area-under-curve and 'define the region' questions, and for Further Pure region-of-integration setups; uses only Cartesian inequalities and direct figure-reading, no advanced topology.
