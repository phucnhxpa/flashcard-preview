Algebra & Functions

Validate candidate solutions against the domain constraint that produced them

Family: Domain Validity Check Frequency: 2 questions

Trigger — When to use

Fires whenever solving has been done under a local assumption — a piecewise/case branch valid only on an interval (e.g. |x+a| split into x ≥ −a and x < −a), or a substitution that hides a domain restriction (y = e^x forces y > 0; y = sin x forces |y| ≤ 1; y = √(...) forces y ≥ 0). Once raw candidates emerge from each branch's algebra, each one must be tested against the very assumption that made the algebra legal. Recognise the cue: 'I solved a linear/quadratic equation that was only true under a side condition' — that side condition has not yet been enforced.

Motivation — Why this works

Each branch's equation is only equivalent to the original problem on its assumed sub-domain; outside that sub-domain it is a different equation entirely. A root that lies outside its branch's domain is therefore extraneous — it satisfies the surrogate equation but not the original. Filtering these out prevents phantom intersection points (in modulus problems) and undefined operations such as ln of a non-positive number (in exponential substitutions). The underlying principle is simply 'check that every implication used was a genuine equivalence on the relevant set'.

Thought Process — The step-by-step reasoning

  1. Notice that the candidate solutions were not produced by the original equation directly, but by a simplified version that depended on an assumption (an interval, or a positivity constraint from a substitution).
  2. Rule out the temptation to just list every algebraic root — recall that squaring, case-splitting, and substituting can each introduce extraneous roots.
  3. Commit to testing each candidate against its own branch's hypothesis: for a piecewise case, plug into the interval inequality; for a substitution y = f(x), check the natural range of f.
  4. Translate the check into a concrete inequality: e.g. 'is 5a/2 < −5a given a > 0?' or 'is y = 4 > 0?'. Use the sign of any parameter (here a > 0) to evaluate signs cleanly.
  5. Discard any candidate that fails its own hypothesis, briefly stating why; retain the survivors and continue (e.g. back-substitute via x = ln y).
  6. Optionally sanity-check the surviving count against geometric intuition (two upward V-shapes meet at most twice, etc.).

Mechanic — Concrete operations

  1. List each candidate alongside the assumption under which it was derived (interval bound, or y > 0, or |y| ≤ 1, etc.).
  2. Substitute the candidate into the assumption and simplify to a numeric/sign inequality.
  3. Use known parameter signs (e.g. a > 0) to determine the truth value without ambiguity.
  4. Mark each candidate ✓ (retain) or ✗ (reject as extraneous), with one-line justification.
  5. Carry only the ✓ candidates into the next step (back-substitution, final answer list, or graph-intersection count).

Worked Micro-Example

Solve $|x-1| = 2x$ by cases. Case A ($x \ge 1$): $x-1 = 2x \Rightarrow x = -1$; check $-1 \ge 1$? No — reject. Case B ($x < 1$): $1-x = 2x \Rightarrow x = 1/3$; check $1/3 < 1$? Yes — accept. Final solution: $x = 1/3$.

Variants — Same move, different surface

Modulus equation split into linear cases on intervals (e.g. |x+a| = |x−2a|)
After solving each linear equation, substitute the candidate into the interval inequality that defined the case; reject if it violates either bound.
Exponential substitution y = e^x in a quadratic
Reject any root with y ≤ 0 since e^x > 0; only positive roots can be inverted via x = ln y.
Trig substitution y = sin x or y = cos x
Reject quadratic roots outside [−1, 1] before applying arcsin/arccos.
Surd substitution y = √(expression)
Reject negative roots since √ denotes the principal (non-negative) value.
Squaring both sides of an equation
All algebraic roots must be tested in the original (unsquared) equation, since squaring can introduce sign-flipped extraneous solutions.

Common Misreadings

  • Treating every algebraic root as automatically valid and reporting all of them as solutions/intersection points.
  • Confusing the domain of the substituted variable with the domain of the original variable, and checking the wrong constraint.

Pitfalls

  • Reversing an inequality direction when a parameter's sign is misjudged — always pin down whether a > 0 or a < 0 before evaluating expressions like −11a/12.
  • Forgetting that in some problems all roots fail (e.g. 2e^{2x} + 9e^x + 4 = 0 yields two negative y-roots, so the original equation has no real solution) and forcing ln of a negative number.
  • Checking only one of two bounds in a case interval (e.g. verifying x < −a/3 but not x ≥ −5a).

A-level Scope Notes

Standard Edexcel IAL Pure technique used in P2/P3 modulus equations and in P3 exponential/logarithmic equations via substitution. No formula-booklet entry is invoked; the move is a logical hygiene check. Avoid generalising to complex roots or to multi-valued inverse functions, which are out of scope.
