Sequences & Series

Iterate the recurrence forward (or downward) to generate required terms

Family: Recurrence Iteration Frequency: 4 questions

Trigger — When to use

A first-order recurrence relation u_{n+1} = f(u_n) (or a reduction formula I_n = g(n)·I_{n−2}) is given together with a known initial/base value, and the question asks for one or more specific later terms — either numerical (e.g. u_3, u_4, I_6) or symbolic in a parameter (e.g. a_2, a_3 in terms of p, or u_3 in terms of k). The cue is that the recurrence only links consecutive (or near-consecutive) terms, so the requested term cannot be reached without computing the intermediates. Equivalent phrasings: 'Find u_3 given u_1 and u_{n+1}=…', 'Express a_3 in terms of p', 'Use the reduction formula to evaluate I_6'.

Motivation — Why this works

A recurrence defines each term implicitly through its predecessor(s), so the only direct route to a later term is the chain u_1 → u_2 → u_3 → … . Sequential substitution converts the implicit definition into explicit values or symbolic expressions, and — crucially — keeps everything linear/polynomial in any unknown parameter so a later constraint (sum, equality) can be solved. For reduction formulae, iterating drops the index by a fixed step until a base case (I_0 = π/2 or I_1 = 1, etc.) evaluable by direct integration is reached. Deriving a closed form via the auxiliary equation is overkill when only a handful of terms are required.

Thought Process — The step-by-step reasoning

  1. Notice the recurrence only links consecutive terms (or terms two apart for a reduction formula), so the requested index cannot be jumped to directly.
  2. Rule out closed-form/auxiliary-equation methods: for just 2–3 specific terms these are slower and more error-prone, and the question doesn't ask for u_n in general.
  3. Commit to forward (or downward) iteration: identify the smallest index given, then plan the chain of substitutions ending at the requested index.
  4. Bridge to action by substituting n = 1 into u_{n+1} = f(u_n) to obtain u_2, then n = 2 to obtain u_3, etc. — keeping everything symbolic if a parameter (p, k) is present so the expression remains usable in the next part of the question.
  5. At each step, simplify carefully (expand brackets, collect like terms) before substituting into the next iteration, since errors compound; for reduction formulae, leave fractions unmultiplied until the chain is complete and only then evaluate the product.

Mechanic — Concrete operations

  1. Identify the initial/base value (u_1, a_1, or I_0/I_1) and the recurrence u_{n+1} = f(u_n).
  2. Substitute n = 1 to obtain u_2 = f(u_1); simplify fully.
  3. Substitute n = 2 into the same recurrence using the previous result to obtain u_3 = f(u_2); simplify fully.
  4. Repeat for further indices as required, taking care with signs, the constant term, and powers of any parameter.
  5. For reduction formulae, terminate the chain at the base case (I_0 = π/2 for even n, I_1 = 1 for odd n) and multiply the accumulated coefficients.

Worked Micro-Example

Given $u_1 = 6$ and $u_{n+1} = ku_n + 3$, find $u_3$ in terms of $k$. Set $n=1$: $u_2 = k(6)+3 = 6k+3$. Set $n=2$: $u_3 = k(6k+3)+3 = 6k^2+3k+3$.

Variants — Same move, different surface

Linear recurrence with constant difference, e.g. a_{n+1} = a_n − 4 with a_1 = p − 3
Each iteration subtracts the constant; terms remain linear in the parameter p, so a_k = (p−3) − 4(k−1).
Geometric recurrence a_{n+1} = r·a_n
Each iteration multiplies by r, giving a_k = r^{k−1}·a_1; same chaining logic.
Second-order recurrence u_{n+2} = au_{n+1} + bu_n
Two initial values are required; iteration still proceeds term-by-term but each step references the two previous terms.
Reduction formula I_n = ((n−1)/n)·I_{n−2} for ∫₀^{π/2} sin^n x dx
Iterate downward in steps of 2 until reaching I_0 = π/2 (n even) or I_1 = 1 (n odd); multiply the chain of fractions.
Request for a far-out term such as u_50
Iteration becomes impractical; switch to solving the recurrence (auxiliary equation / closed form).

Common Misreadings

  • Reversing the recurrence direction, e.g. reading u_{n+1} = ku_n + 3 as u_n = ku_{n+1} + 3.
  • Using n = 3 to try to obtain u_3 directly, when n = 2 in u_{n+1} = … is what produces u_3.
  • Using the wrong base case for a reduction formula (I_1 when n is even, or I_0 when n is odd).

Pitfalls

  • Sign errors when chaining subtractions or substituting negative intermediates, e.g. 2(−3) − 5 written as −1 instead of −11; or (p−3) − 4 written as p−1.
  • Dropping the constant term at each iteration (e.g. writing u_2 = 6k instead of 6k + 3).
  • Failing to expand brackets correctly when the recurrence is applied symbolically, e.g. k(6k+3) becoming 6k² + 3 instead of 6k² + 3k.
  • Forgetting to compute the unrequested intermediate term (e.g. u_2) before reaching the requested one (u_3).
  • Arithmetic slips when multiplying the chain of fractions in a reduction-formula problem.

A-level Scope Notes

Fully within Edexcel IAL P1/P2 (sequences defined by a recurrence) and FP3 / Further Pure (reduction formulae for integrals). No formula booklet entry is required for the iteration itself; standard integrals (sin^n x, cos^n x, x^n e^{−x}) come from prior chapters. Avoid invoking general closed-form solutions of linear recurrences via auxiliary equations beyond the FP syllabus.
