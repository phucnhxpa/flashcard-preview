Algebra & Functions

Exponentiate both sides to eliminate the logarithm

Family: Inverse Power Operation Frequency: 1 questions

Trigger — When to use

Fires when an equation has the form $\log_b N = (\text{expression})$ but the target/required form has $N$ isolated without any logarithm — typically when converting a linearised log-relationship back to an exponential model $N = ab^t$. Cues include: 'express N in the form...', 'find N when...', or a regression line in log-space that must be turned into a multiplicative model. Also fires whenever a single $\log$ term sits alone on one side after using log laws.

Motivation — Why this works

By definition, $\log_b N = Z \iff N = b^Z$, so applying $b^{(\cdot)}$ to both sides is the exact inverse of $\log_b$. This converts an additive (linear-in-$t$) relationship in log-space into a multiplicative (exponential-in-$t$) relationship in the original variable. The base used to exponentiate must match the base of the logarithm, otherwise the inverse property fails. This is the standard bridge between linearised data analysis and exponential models.

Thought Process — The step-by-step reasoning

  1. I notice the equation has $\log_{10} N$ on one side, but the required form $N = ab^t$ contains no logarithm at all.
  2. I rule out rearranging algebraically (no way to 'cancel' a log without its inverse) and rule out using $\ln$ because the log here is base 10, not base $e$.
  3. I commit to applying $10^{(\cdot)}$ to both sides, since $10^{\log_{10} N} = N$ exactly undoes the log.
  4. First concrete operation: write $N = 10^{(\text{whole RHS})}$, keeping the RHS bracketed so the entire expression sits in the exponent.
  5. Then I plan to use index laws ($10^{A+B} = 10^A \cdot 10^B$) to split the constant term off and recover the $a \cdot b^t$ shape.

Mechanic — Concrete operations

  1. Identify the base of the logarithm (here $\log_{10}$).
  2. Write each side as a power of that base: $10^{\log_{10} N} = 10^{(\text{RHS})}$.
  3. Simplify the LHS to $N$ using $10^{\log_{10} N} = N$.
  4. If needed, apply $10^{A+B} = 10^A \cdot 10^B$ to separate the constant term from the $t$-term.
  5. Identify $a = 10^{c}$ and $b = 10^{m}$ to match the target form $N = ab^t$.

Worked Micro-Example

Given $\log_{10} N = 0.0646t + 1.478$, exponentiate both sides with base 10: $N = 10^{0.0646t + 1.478}$. Split using index laws: $N = 10^{1.478} \cdot 10^{0.0646t} = 30.06 \cdot (10^{0.0646})^{t} \approx 30.06 \cdot (1.160)^t$. So $a \approx 30.06$ and $b \approx 1.160$.

Variants — Same move, different surface

Equation uses $\ln N = mt + c$
Exponentiate with base $e$: $N = e^{mt+c} = e^c \cdot (e^m)^t$.
Equation uses $\log_2 N = \text{expression}$
Exponentiate with base 2: $N = 2^{\text{expression}}$.
Multiple log terms, e.g. $\log_{10}(y) - \log_{10}(x) = k$
Combine logs first using $\log A - \log B = \log(A/B)$, then exponentiate: $y/x = 10^k$.
Solving $\log_{10}(2x+1) = 1.5$
Exponentiate to get $2x+1 = 10^{1.5}$, then solve linearly for $x$.

Common Misreadings

  • Treating $\log_{10}$ as a multiplicative coefficient and 'dividing' by it: writing $N = (0.0646t + 1.478)/\log_{10}$.
  • Reading $\log_{10} N$ as $\log_{10} \times N$ and trying to factor it out.
  • Believing the equation is already in exponential form and skipping the inversion step entirely.

Pitfalls

  • Using the wrong base: writing $N = e^{0.0646t+1.478}$ when the original log was base 10.
  • Forgetting to bracket the entire RHS in the exponent: writing $N = 10^{0.0646t} + 1.478$ instead of $N = 10^{0.0646t + 1.478}$.
  • Failing to split $10^{c+mt}$ into $10^c \cdot 10^{mt}$, so $a$ and $b$ are not identified.
  • Computing $b = 10^{0.0646}$ by writing $10 \times 0.0646$ instead of evaluating the power.

A-level Scope Notes

Standard Edexcel IAL P2/P3 content: inverse relationship between logarithms and exponentials, log laws, and exponential models $N = ab^t$ obtained from linearised log-data. No content beyond syllabus is required.
