Matrices

Assemble P from eigenvectors and D from eigenvalues to write A = PDP⁻¹

Family: Diagonalisation Frequency: 2 questions

Trigger — When to use

Fires when the question asks to 'diagonalise A', 'express A in the form PDP⁻¹', 'find a matrix P such that P⁻¹AP is diagonal', or sets up a downstream computation like Aⁿ, eigen-based recurrences, or matrix exponentials. Structurally, you have already obtained n eigenvalues with n linearly independent eigenvectors for an n×n matrix (typically 3×3 at IAL Further Maths). The move is also signalled when a follow-up part says 'hence find A⁵' or 'hence solve the system' after eigenvalue work.

Motivation — Why this works

The defining relations Av_i = λ_i v_i (i = 1,…,n) bundle into the single matrix equation AP = PD, where P has the eigenvectors as columns and D is diagonal with the corresponding eigenvalues. If the eigenvectors are linearly independent then P is invertible, giving the canonical decomposition A = PDP⁻¹. This decouples A's action into independent scalings along the eigenvector directions, which is exactly what makes powers (Aⁿ = PDⁿP⁻¹) and functions of A computable, since Dⁿ just raises each diagonal entry to the nth power.

Thought Process — The step-by-step reasoning

  1. I notice I have a full set of n linearly independent eigenvectors for an n×n matrix, so A is diagonalisable — no need for Jordan form or other workarounds.
  2. I rule out trying to compute Aⁿ by direct multiplication or by induction on entries: those are far more error-prone than diagonalisation when eigen-data is already available.
  3. I commit to A = PDP⁻¹ because the relation Av_i = λ_i v_i packages neatly as AP = PD, and invertibility of P (from independence) lets me isolate A.
  4. Bridging to computation: I form P by stacking eigenvectors as columns and D as the diagonal matrix of eigenvalues, being careful that the i-th column of P matches the i-th diagonal entry of D.
  5. Tactical choice: if I have freedom in ordering, I pick an order that makes any subsequent inverse or arithmetic cleanest (e.g. placing the simplest eigenvector first), and I sanity-check by verifying AP = PD column by column before inverting P.

Mechanic — Concrete operations

  1. Write P = [v₁ | v₂ | v₃] with eigenvectors as columns.
  2. Write D = diag(λ₁, λ₂, λ₃) with eigenvalues in the matching column positions.
  3. State the decomposition A = PDP⁻¹ (equivalently D = P⁻¹AP).
  4. Compute P⁻¹ when required, using either the adjugate/cofactor formula P⁻¹ = (1/det P)·adj(P) or row-reduction [P | I] → [I | P⁻¹].
  5. (Optional verification) Compute AP and PD separately and check equality column by column.
  6. (If asked for Aⁿ) Use Aⁿ = PDⁿP⁻¹ where Dⁿ = diag(λ₁ⁿ, λ₂ⁿ, λ₃ⁿ).

Worked Micro-Example

Suppose A has eigenpairs (λ₁,v₁)=(2,(1,0,0)ᵀ), (λ₂,v₂)=(3,(1,1,0)ᵀ), (λ₃,v₃)=(−1,(1,1,1)ᵀ). Then P = [[1,1,1],[0,1,1],[0,0,1]], D = diag(2,3,−1). A short row-reduction gives P⁻¹ = [[1,−1,0],[0,1,−1],[0,0,1]], so A = PDP⁻¹ and A⁵ = P·diag(32,243,−1)·P⁻¹.

Variants — Same move, different surface

A is symmetric (Aᵀ = A)
Eigenvectors for distinct eigenvalues are orthogonal; normalise them to obtain an orthogonal P with P⁻¹ = Pᵀ, so A = PDPᵀ — no inverse computation needed.
A repeated eigenvalue still yields enough independent eigenvectors
Diagonalisation still works; just ensure the chosen basis of the eigenspace gives linearly independent columns of P.
Question asks for Aⁿ or for solving u_{k+1} = Au_k
Use Aⁿ = PDⁿP⁻¹; the diagonalisation is the engine, not the destination.

Common Misreadings

  • Treating any three eigenvectors as automatically independent without checking — repeated eigenvalues require explicit verification.
  • Mismatching column order: putting eigenvalues in D in a different order from the eigenvectors in P breaks AP = PD.
  • Confusing A = PDP⁻¹ with A = P⁻¹DP — the order matters since matrices do not commute.

Pitfalls

  • Arithmetic slips in computing P⁻¹ for a 3×3 P (sign errors in cofactors, mis-tracked row operations).
  • Forgetting that Dⁿ = diag(λ₁ⁿ,…) only works because D is diagonal; trying to raise a non-diagonal matrix entry-wise to a power is wrong.
  • Writing D = P⁻¹AP but then using A = P⁻¹DP — the rearrangement gives A = PDP⁻¹.
  • Failing to scale eigenvectors consistently — any nonzero scalar multiple is fine, but the same vector must be used in P throughout.

A-level Scope Notes

Sits within Edexcel IAL Further Pure (FP1/FP3) diagonalisation of 2×2 and 3×3 matrices, including the symmetric case where P is orthogonal. Jordan normal form and defective (non-diagonalisable) matrices are out of scope and should not be invoked.
