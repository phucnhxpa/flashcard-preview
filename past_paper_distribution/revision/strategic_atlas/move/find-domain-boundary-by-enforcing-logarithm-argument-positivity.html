Algebra & Functions

Find domain boundary by enforcing logarithm argument positivity

Family: Domain Restriction Frequency: 1 questions

Trigger — When to use

Fires whenever a function contains ln(...) (or log) and the question asks for the domain, the largest set of valid x, the vertical asymptote, or a constant k in a domain inequality x > k. Cues include phrasings like 'state the domain', 'find the value of k such that f is defined for x > k', or 'write down the equation of the vertical asymptote' for a log-containing curve. Outer transformations (modulus, linear scaling, addition of constants) do not change where the logarithm fails, so the log argument is the sole restriction source.

Motivation — Why this works

ln(u) is only defined for u > 0, so any x making the argument non-positive is excluded from the domain. The boundary u = 0 corresponds to a vertical asymptote of ln(u), which transfers to the original variable through the inner function. Because polynomials, moduli and linear wrappers are defined on all of ℝ, the log argument supplies the unique domain restriction. Solving argument > 0 therefore gives both the domain and the asymptote location in one step.

Thought Process — The step-by-step reasoning

  1. Notice the function contains ln(...) and the question references a domain bound x > k or a vertical asymptote.
  2. Rule out other potential restriction sources: modulus is defined everywhere, linear/affine wrappers don't restrict, no division by zero or even root present.
  3. Commit to the principle that ln requires its argument strictly positive, so the boundary comes from argument = 0.
  4. Translate this into the inequality (inner expression) > 0 in x.
  5. Solve the linear inequality and read off k as the boundary value, matching the form x > k.

Mechanic — Concrete operations

  1. Identify the argument u(x) of the logarithm.
  2. Set u(x) > 0.
  3. Solve the resulting inequality for x.
  4. Read the lower bound as k (or state the domain / asymptote x = k accordingly).

Worked Micro-Example

Given $f(x) = 2 - 4\ln(x+1)$, the only restriction is from the log: require $x+1 > 0$, hence $x > -1$. So the domain is $x > -1$, giving $k = -1$, and the vertical asymptote is $x = -1$.

Variants — Same move, different surface

Argument is a general linear expression, e.g. $\ln(2x-3)$
Solve $2x-3>0$ to get $x>3/2$; the boundary scales with the coefficient of x.
Argument involves a constant shift only, e.g. $\ln(x-a)$
Domain is $x>a$; vertical asymptote at $x=a$.
Negated inner expression, e.g. $\ln(3-x)$
Require $3-x>0$, giving $x<3$; the inequality direction flips.
Composite like $f(x) = a + b\ln(cx+d)$ with $c>0$
Outer constants $a,b$ are irrelevant to domain; solve $cx+d>0$ to get $x> -d/c$.
Modulus wrapping the log, e.g. $|\ln(x+1)|$
Domain is unchanged ($x>-1$); modulus only affects range, not where ln is defined.

Common Misreadings

  • Treating the x-intercept (where $\ln(\cdot)=0$) as the domain boundary instead of where the argument equals 0.
  • Assuming the domain is $x>0$ by default for any log function, ignoring the inner shift.
  • Confusing the constant $k$ in $x>k$ with a y-value such as the horizontal level of an asymptote (logs have vertical asymptotes, not horizontal).

Pitfalls

  • Writing $x > 0$ instead of $x > -1$ by overlooking the $+1$ inside the logarithm.
  • Using $\geq$ instead of $>$; the inequality must be strict because $\ln 0$ is undefined.
  • Forgetting to flip the inequality when the argument has a negative coefficient of $x$.
  • Reporting $k$ as the asymptote of the whole curve in $y$ rather than the $x$-boundary.

A-level Scope Notes

Fully within Edexcel IAL Pure Mathematics (P2/P3) treatment of exponential and logarithmic functions, including domain, range and asymptotes of curves of the form $y = a + b\ln(cx+d)$.
