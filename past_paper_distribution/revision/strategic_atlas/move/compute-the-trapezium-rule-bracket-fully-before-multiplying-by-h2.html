Integration

Compute the trapezium-rule bracket fully before multiplying by h/2

Family: Order Of Operations Frequency: 1 questions

Trigger — When to use

Fires whenever a numerical integration formula presents a multiplicative pre-factor (typically h/2 for the trapezium rule) acting on an additive bracket of tabulated y-values. Cues include: the question quoting the formula $\int_a^b y\,dx \approx \tfrac{h}{2}\big[(y_0+y_n)+2(y_1+\dots+y_{n-1})\big]$, a table of ordinates, and an instruction like 'use the trapezium rule with n strips'. Also fires when Simpson's rule is requested with prefactor h/3.

Motivation — Why this works

Evaluating the bracket B in full before scaling once by h/2 mirrors the printed formula exactly, so each mark-scheme mark (value of h, structure of bracket, correct substitution, final answer) is visibly earned. It collapses n potential multiplications into a single one, which reduces rounding and arithmetic slips. Keeping intermediate work as exact sums of tabulated values preserves accuracy until the last step. Crucially, even if the final multiplication is mis-keyed, the structural marks for the bracket are still awarded.

Thought Process — The step-by-step reasoning

  1. I notice the formula has the shape (prefactor) × [bracket of y-values], so the natural unit of work is the bracket itself.
  2. I rule out the strip-by-strip approach (computing each $\tfrac{h}{2}(y_i+y_{i+1})$ separately and summing) because it multiplies n times and invites rounding drift.
  3. I commit to: first sum the endpoints, then sum the interior ordinates, then assemble B = S_ends + 2·S_mid, then multiply by h/2 exactly once.
  4. Bridge: I write the formula line $A \approx \tfrac{h}{2}\big[(y_0+y_n)+2(y_1+\dots+y_{n-1})\big]$ and substitute symbolic sums before any decimal arithmetic.
  5. Tactically, I keep ordinates to the precision given in the table, only round at the very end to the accuracy specified, and I write outer square brackets clearly so the h/2 visibly multiplies the entire bracket.

Mechanic — Concrete operations

  1. Identify h = (b−a)/n from the strip count and interval.
  2. Compute S_ends = y_0 + y_n using exact table values.
  3. Compute S_mid = y_1 + y_2 + … + y_{n−1} using exact table values.
  4. Form the bracket B = S_ends + 2·S_mid (write this line explicitly).
  5. Multiply once: Area ≈ (h/2) × B.
  6. Round only at the final step to the accuracy requested (e.g. 3 s.f. or 4 d.p.).

Worked Micro-Example

With $h=0.5$ and ordinates $y_0=1,\ y_1=1.284,\ y_2=1.649,\ y_3=2.117,\ y_4=2.718$: $S_{\text{ends}} = 1 + 2.718 = 3.718$; $S_{\text{mid}} = 1.284+1.649+2.117 = 5.050$; $B = 3.718 + 2(5.050) = 13.818$; Area $\approx \tfrac{0.5}{2}(13.818) = 0.25 \times 13.818 = 3.4545 \approx 3.45$ (3 s.f.).

Variants — Same move, different surface

Simpson's rule (Further Maths)
Same 'bracket-first, scale-last' strategy, but the prefactor is h/3 and the bracket is $(y_0+y_n) + 4(y_{\text{odd}}) + 2(y_{\text{even}})$; compute the three weighted sums, assemble, then multiply by h/3.
Trapezium rule applied to a transformed integrand (e.g. $\int \ln x\,dx$ via tabulated $\ln x_i$)
Tabulate the transformed ordinates first, then apply identical bracket-first scaling — do not mix the transformation into each strip individually.
Estimating area between a curve and a chord/line via two trapezium-rule applications
Compute each estimate's bracket separately, scale each by its own h/2, then subtract — keep the two scalings independent to avoid mixing prefactors.
Question gives unequal x-values but states ordinates are evenly spaced
Re-derive h from any two consecutive x-values before forming the bracket; the bracket-first strategy is unchanged.

Common Misreadings

  • Treating the formula as a sum of strip areas and multiplying h/2 into each strip — wastes time and risks rounding error.
  • Reading '2(y_1+…+y_{n−1})' as '2y_1 + y_2 + … + y_{n−1}', i.e. doubling only the first interior ordinate.
  • Forgetting the 1/2 in h/2 and using h as the prefactor — doubles the final answer.
  • Including y_0 or y_n inside the doubled interior sum.

Pitfalls

  • Premature rounding of S_ends or S_mid when ordinates are given to 4+ decimal places, distorting the final answer beyond the required accuracy.
  • Writing 'h/2 × y_0 + y_n + 2(…)' without the outer bracket, so order of operations multiplies only y_0 by h/2.
  • Using the wrong h when n counts ordinates (n+1 values) rather than strips (n values) — always compute h = (b−a)/(number of strips).
  • Mis-keying the final multiplication on the calculator; mitigated by writing the bracket value down before scaling.

A-level Scope Notes

Fully within Edexcel IAL Pure Mathematics (trapezium rule, P-unit numerical methods) and IAL Further Pure (Simpson's rule). The formula is provided in the formula booklet, so reproducing its exact bracketed structure aligns directly with the published mark scheme.
