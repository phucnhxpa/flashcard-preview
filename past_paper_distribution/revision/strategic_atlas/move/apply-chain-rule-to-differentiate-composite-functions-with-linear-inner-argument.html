Differentiation

Apply chain rule to differentiate composite functions with linear inner argument

Family: Chain Rule Frequency: 2 questions

Trigger — When to use

The expression to differentiate has the form g(f(x)) where the outer function g is one of the standard library functions (a^u, e^u, ln u, sin u, cos u, etc.) and the inner function f is a simple linear expression like ax+b, −t, or kx. Visual cues: a constant coefficient multiplying a non-trivial argument inside an exponential, logarithm, or trig function (e.g. 1.4^(−t), ln(2x+1), e^(3x), sin(5x)). Equivalent phrasings: 'differentiate a^(linear)', 'derivative of ln(linear)', 'composite with affine inner function'. Fires whenever the inner expression is NOT just x or t alone, so a naive application of the base derivative rule would miss a multiplicative factor.

Motivation — Why this works

The chain rule d/dx[g(f(x))] = g'(f(x)) · f'(x) is the universal mechanism for handling composition. With a linear inner function f(x) = ax+b, the factor f'(x) = a is just a constant, so the chain rule contributes a single clean multiplier — but forgetting it is the dominant source of error. Knowing the standard outer derivatives (d/du[a^u] = a^u ln a, d/du[ln u] = 1/u, d/du[e^u] = e^u) and combining them with the inner derivative produces the answer in one step, avoiding the need to rewrite via e^(u ln a) or other detours.

Thought Process — The step-by-step reasoning

  1. I notice the function has a composite structure: an outer 'library' function (exponential, logarithm, trig) wrapped around a non-trivial inner expression like −t or 2x+1.
  2. I rule out applying the base rule directly (e.g. d/dt[a^t] = a^t ln a) because the exponent is not just t — that would miss a factor. I also rule out rewriting via natural log/exp conversion as unnecessarily long.
  3. I rule out the power rule: the variable is in the exponent (or inside ln), not the base, so this is an exponential/log situation, not a polynomial one.
  4. I commit to the chain rule: identify u = inner function, compute du/dx, then multiply g'(u) by du/dx.
  5. Tactically, since the inner function is linear (au+b), du/dx is a constant — I just attach it as a multiplier and tidy any leading coefficients.
  6. Finally I sanity-check the sign and magnitude against context (e.g. a decay model should give a negative derivative).

Mechanic — Concrete operations

  1. Identify the outer function g and inner function u = f(x); pull out any constant multipliers.
  2. Write down g'(u) using the standard derivative (a^u ln a, 1/u, e^u, cos u, etc.).
  3. Compute du/dx (for a linear inner function this is just the coefficient of x).
  4. Multiply: dy/dx = (constant) · g'(u) · du/dx, keeping signs explicit.
  5. Substitute the specific value of x or t if a numerical derivative is required, deferring rounding to the final step.

Worked Micro-Example

Differentiate $y = 2\ln(2x+1)$. Outer: $\ln u$ with $u = 2x+1$. Then $g'(u) = 1/u$ and $du/dx = 2$, so $\frac{d}{dx}\ln(2x+1) = \frac{2}{2x+1}$. Multiplying by the leading 2 gives $\frac{dy}{dx} = \frac{4}{2x+1}$.

Variants — Same move, different surface

Constant-base exponential with negative linear exponent, e.g. x = 400·1.4^(−t)
Use d/dt[a^(−t)] = −a^(−t) ln a; the chain rule contributes the −1, ensuring dx/dt is negative as required for decay.
Natural exponential e^(kx)
d/dx[e^(kx)] = k·e^(kx); ln(e) = 1 so no log factor appears, only the inner derivative k.
Logarithm of a linear function, e.g. ln(ax+b)
d/dx[ln(ax+b)] = a/(ax+b); the inner derivative a sits on top.
Non-linear inner function, e.g. ln(x²+1) or a^(t²)
Replace the constant inner derivative with the actual f'(x) (e.g. 2x), preserving the same chain-rule structure.

Common Misreadings

  • Treating a^t as a power and applying the power rule t·a^(t−1) — wrong because the variable is the exponent, not the base.
  • Reading 'use calculus' as 'substitute into the original function' rather than 'differentiate first'.
  • Mistaking ln(2x+1) for ln(2x) + ln(1) or similar invalid log split, then differentiating the wrong expression.

Pitfalls

  • Forgetting the inner derivative du/dx — writing 1/(2x+1) instead of 2/(2x+1), or a^(−t) ln a instead of −a^(−t) ln a.
  • Dropping the ln a factor for non-natural-base exponentials (using d/dt[a^t] = a^t).
  • Sign errors when the inner function has a negative coefficient (e.g. exponent −t).
  • Premature rounding of ln(a) or a^(−n) before the final multiplication, producing inaccurate numerical answers.

A-level Scope Notes

Within Edexcel IAL P3/P4 scope: derivatives of e^x, ln x, a^x, sin x, cos x, tan x are all in the formula booklet, and the chain rule is a core technique. Avoid generalising to implicit functions or to chain rule with multivariable inner functions, which are out of scope at this level.
