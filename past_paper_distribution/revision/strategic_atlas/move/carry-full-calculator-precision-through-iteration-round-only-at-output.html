Sequences & Series

Carry full calculator precision through iteration, round only at output

Family: Iterative Precision Management Frequency: 1 questions

Trigger — When to use

Fires whenever a recurrence $x_{n+1} = f(x_n)$ must be iterated multiple times and a later iterate (e.g. $x_5$, $x_6$) is requested to a fixed number of decimal places. Cue phrases include 'find $x_2$, $x_3$, $\ldots$, $x_n$ giving your answers to 3 decimal places' or 'use the iteration formula ... starting with $x_1 = a$'. Particularly critical when the function involves transcendentals (arctan, exp, log, roots) where small input changes amplify. Also applies to Newton–Raphson iteration and fixed-point iteration questions.

Motivation — Why this works

Each iterate is fed back as the input to the next, so any rounding error in $x_n$ propagates into $x_{n+1}$ and can compound. Edexcel mark schemes typically allow only $\pm 1$ in the last quoted digit, so a single premature rounding can shift the final iterate out of the accepted range. Storing the full calculator value (via the ANS key or a memory register) and rounding only when writing the answer guarantees that the displayed 3 d.p. value is the correctly rounded image of the true iterate. This is the standard numerical-methods discipline examiners deliberately probe by asking for a deep iterate.

Thought Process — The step-by-step reasoning

  1. Notice the recurrence is to be applied several times and a late iterate is requested to a specified accuracy.
  2. Rule out hand-substituting rounded intermediates — the cumulative error can flip the last digit of the final answer.
  3. Commit to keeping every $x_n$ in calculator memory at full precision, displaying rounded values only on the answer line.
  4. Set the calculator to radians (if a trig function appears), enter $x_1$, then build the iteration as $f(\mathrm{ANS})$ and press = repeatedly.
  5. Record each iterate to the required d.p. as you go, but never re-type the rounded number back into the calculator.

Mechanic — Concrete operations

  1. Check angle mode (radians for arctan/sin/cos/tan in pure-maths iterations).
  2. Enter $x_1$ on the calculator and press =.
  3. Type the iteration formula using ANS in place of $x_n$, e.g. $2\arctan(4/\mathrm{ANS})$.
  4. Press = repeatedly; after each press write down the displayed value rounded to the required d.p. as $x_2, x_3, \ldots$.
  5. Do NOT clear ANS or retype rounded values; the calculator silently retains full precision.
  6. Quote the final requested iterate to the stated number of decimal places.

Worked Micro-Example

Given $x_{n+1} = 2\arctan(4/x_n)$ with $x_1 = 2$ (radians): pressing = produces $x_2 = 2.21429\ldots \to 2.214$, $x_3 = 2.13169\ldots \to 2.132$, $x_4 = 2.16545\ldots \to 2.165$, $x_5 = 2.15169\ldots \to 2.152$, $x_6 = 2.15730\ldots \to 2.157$ (3 d.p.). Re-entering 2.214 by hand at step 3 gives $x_6 \approx 2.158$, losing the mark.

Variants — Same move, different surface

Newton–Raphson iteration $x_{n+1} = x_n - f(x_n)/f'(x_n)$ to find a root to 4 d.p.
Define $f$ and $f'$ separately on the calculator or store the whole NR formula in ANS form; convergence is faster, but precision discipline is identical.
Fixed-point iteration with a logarithm or surd, e.g. $x_{n+1} = \sqrt{5 - x_n}$.
Same ANS technique; watch for domain issues (negative argument under the root) when $x_1$ is a poor starting value.
Recurrence given by $u_{n+1} = au_n + b$ where an explicit closed form exists.
Either iterate with ANS or use the closed-form $u_n = A\cdot a^{n-1} + b/(1-a)$; closed form avoids rounding entirely.
Trapezium-rule or Simpson's-rule numerical integration with several ordinates.
Compute each ordinate $y_i$ to full precision in a list/table; round only the final $\int$ estimate.

Common Misreadings

  • Treating 'give to 3 d.p.' as an instruction to use 3 d.p. throughout the calculation, rather than only in the final display.
  • Believing the iteration formula expects degrees because it 'looks like' a trig expression, when pure-maths iterations always use radians.
  • Thinking $x_1$ is the first iterate produced, rather than the given starting value.

Pitfalls

  • Rounding $x_2$ to 2.214 and retyping it as the input for $x_3$ — this can shift $x_6$ to 2.158 instead of the correct 2.157.
  • Calculator left in degree mode, giving $\arctan(2) \approx 63.43$ and a completely wrong $x_2$.
  • Miscounting iterations: producing $x_5$ when $x_6$ was asked for (the starting value $x_1$ is not produced by the formula).
  • Using a bracket error such as $2\arctan 4/x_n$ interpreted as $(2\arctan 4)/x_n$ — always bracket the argument.

A-level Scope Notes

Standard Edexcel IAL P3/P4 numerical-methods technique for iterative root-finding and Newton–Raphson; no content beyond the syllabus is invoked.
