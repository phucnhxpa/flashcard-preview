Trigonometry

Specialize a compound angle formula by setting B = A to obtain a double angle identity

Family: Identity Selection Frequency: 1 questions

Trigger — When to use

The question explicitly instructs 'use cos(A+B) = cosAcosB − sinAsinB to show that cos 2A = ...' (or the analogous sin/tan version), and the target involves a doubled angle 2A in a single variable. The cue is the mismatch between a two-variable compound angle identity and a single-variable double angle target. Equivalent phrasings include 'hence show', 'deduce', or 'starting from sin(A+B) prove sin 2A = ...'. Whenever 2A appears as the argument and a compound angle identity is on the table, this specialization fires.

Motivation — Why this works

Because 2A is literally A + A, the compound angle formula applies directly with both inputs equal, collapsing it into a single-variable identity. This is the canonical derivation route for the double angle formulas in the IAL syllabus. The intermediate form (cos²A − sin²A, 2 sinA cosA, or 2tanA/(1−tan²A)) is the raw double angle identity, from which Pythagorean substitution then yields the alternative forms.

Thought Process — The step-by-step reasoning

  1. I notice the target has argument 2A while the given identity has A+B; since 2A = A+A, setting B = A is the bridge.
  2. I rule out other substitutions (B = 0, B = π/2, etc.) because none of them produce a 2A argument.
  3. I commit to B = A because the 'use ... to show' instruction forces a forward derivation from the compound formula.
  4. Concretely, I write cos(A+A) and expand it using the given identity term by term.
  5. I anticipate that the raw output (cos²A − sin²A) won't yet match the requested form, so I plan a follow-up move using sin²A + cos²A = 1 to convert it.

Mechanic — Concrete operations

  1. Write the LHS of the target as cos(A+A).
  2. Apply the given identity with B replaced by A: cos(A+A) ≡ cosA·cosA − sinA·sinA.
  3. Simplify products to powers: cos 2A ≡ cos²A − sin²A.
  4. If the target is a different form (e.g. 2cos²A − 1), apply sin²A = 1 − cos²A as a follow-up.

Worked Micro-Example

Show cos 2A ≡ 2cos²A − 1 using cos(A+B) ≡ cosAcosB − sinAsinB.\nSet B = A: cos(A+A) ≡ cosA·cosA − sinA·sinA, so cos 2A ≡ cos²A − sin²A.\nUsing sin²A ≡ 1 − cos²A: cos 2A ≡ cos²A − (1 − cos²A) = 2cos²A − 1. ∎

Variants — Same move, different surface

Deriving sin 2A from sin(A+B) ≡ sinAcosB + cosAsinB
Set B = A to get sin 2A ≡ 2 sinA cosA directly; no Pythagorean follow-up needed.
Deriving tan 2A from tan(A+B) = (tanA + tanB)/(1 − tanA tanB)
Set B = A to get tan 2A = 2tanA/(1 − tan²A); note the domain restriction tan²A ≠ 1.
Showing cos 2A ≡ 1 − 2sin²A
After cos 2A ≡ cos²A − sin²A, substitute cos²A = 1 − sin²A instead.
Deriving cos 3A or sin 3A
Write 3A = 2A + A and use the compound angle formula with B = A applied iteratively, combined with the double angle results.

Common Misreadings

  • Treating the instruction 'use cos(A+B) ...' as optional and quoting cos 2A = 2cos²A − 1 directly, which fails to demonstrate the derivation requested.
  • Working backwards from the target identity instead of forwards from the given compound formula, breaking the logical 'show that' structure.
  • Assuming a different specialization (e.g. B = −A) is required, which produces cos 0 = 1 rather than cos 2A.

Pitfalls

  • Writing cos(2A) = cos²(2A) − sin²(2A) by confusing 'double the argument' with applying the identity to 2A itself.
  • Dropping the identity symbol ≡ in a 'show that' proof, or skipping the explicit B = A substitution step the marker is looking for.
  • Forgetting to invoke sin²A + cos²A ≡ 1 when the target form requires only cos²A or only sin²A.
  • Misexpanding signs: writing cosAcosA + sinAsinA (the cos(A−B) expansion) instead of the minus form.

A-level Scope Notes

This is the standard Edexcel IAL P3/C3-level derivation of the double angle formulas from the compound angle identities, and is regularly examined as a 'use ... to show that' opener before a follow-up part.
