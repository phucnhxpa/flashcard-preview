Integration

Use concavity and strip-width to judge and improve trapezium-rule accuracy

Family: Trapezium Rule Error Analysis Frequency: 2 questions

Trigger — When to use

Fires whenever a question asks (a) whether a trapezium-rule estimate is an overestimate or underestimate, or (b) how the estimate could be made more accurate. Visual cues include data presented in a table of ordinates, a context like 'cross-section of a river/tunnel/ground profile' that swells then returns to zero (a dome shape), or wording such as 'state, with a reason, whether your answer is an over/underestimate' and 'suggest one way to improve the accuracy'. Equivalent phrasings: 'comment on the accuracy', 'how could the estimate be refined', 'is the true area greater or smaller than your estimate'.

Motivation — Why this works

The trapezium rule replaces the true curve on each strip by the straight chord joining its endpoints. By concavity: on a concave-down (∩) section the chord lies BELOW the curve, so trapezia underestimate; on a concave-up (∪) section the chord lies ABOVE the curve, so trapezia overestimate. The error on each strip scales with the strip width h (formally with h² times the curvature), so halving h roughly quarters the error — hence 'use more strips / smaller h' is the standard accuracy-improvement answer. This combines the geometric meaning of concavity with the structural source of approximation error in the rule.

Thought Process — The step-by-step reasoning

  1. Notice which sub-question is being asked: a sign-of-error question (over/underestimate) or an accuracy-improvement question — the two need different reasonings but share the same geometric picture.
  2. For the over/underestimate part: visualise the data points and sketch the smooth curve through them; identify the overall concavity (e.g. a river cross-section bulges outward, giving a ∩-shape, hence concave down).
  3. Rule out irrelevant features: monotonicity (increasing/decreasing) does NOT determine over/underestimate; only concavity does. Also rule out 'rounding' explanations — these address arithmetic precision, not method error.
  4. Commit to the chord-vs-curve argument: state explicitly whether the chord lies above or below the curve, and translate that into trapezia area > or < true area.
  5. For the accuracy part: recognise that the approximation error stems from chord ≠ curve, and this error shrinks as h shrinks. Commit to the standard mark-scheme phrasing: 'use more strips' / 'decrease h' / 'increase the number of ordinates'.
  6. Finally, propagate the conclusion: if the question asks about a derived quantity (volume = area × length, etc.), an underestimated area gives an underestimated volume.

Mechanic — Concrete operations

  1. Sketch (mentally or on paper) the data points and the implied smooth curve.
  2. Classify the curve between consecutive ordinates as concave down (∩) or concave up (∪).
  3. State the chord position: concave down ⇒ chord below curve; concave up ⇒ chord above curve.
  4. Conclude over- or underestimate: chord below ⇒ trapezia area < true area ⇒ underestimate (and vice versa).
  5. If asked about a derived quantity (e.g. volume V = A × L), carry the inequality through: underestimate of A ⇒ underestimate of V.
  6. For the accuracy part, write: 'Use more strips (decrease h / increase the number of ordinates)' — one short sentence is sufficient for the mark.

Worked Micro-Example

Data for a river cross-section give depths $0, 1.2, 1.8, 1.5, 0$ at equally spaced points. The profile is dome-shaped, hence concave down. Each chord between consecutive points lies below the curve, so $A_{\text{trap}} < A_{\text{true}}$: the estimate is an underestimate. Since $V = A \times L$, the volume is also underestimated. To improve accuracy: use more strips (smaller $h$).

Variants — Same move, different surface

Concave-up curve such as $y = x^2$ or $y = e^x$ on a positive interval
Chords lie above the curve, so trapezia OVERestimate the area. Same logic, opposite inequality.
Curve with an inflection point inside the integration interval
Some strips overestimate, others underestimate; the net sign cannot be determined without further analysis. Acknowledge this rather than guessing.
Question phrased 'state one way the estimate could be made more accurate'
Identical answer: more strips / smaller strip width / more ordinates. Avoid 'use more decimal places' (rounding error, not method error).
Further Maths context comparing methods
Mention Simpson's rule as a higher-order alternative, but for IAL P-paper marks stick to 'more strips'.

Common Misreadings

  • Confusing concavity with monotonicity: arguing 'the function is increasing, so trapezia overestimate' — increasing/decreasing is irrelevant.
  • Reversing the chord-vs-curve inequality: thinking 'chord below curve' means MORE trapezia area instead of less.
  • Reading the dome-shaped data as concave up because the values rise initially — concavity is about the bend, not the slope direction.

Pitfalls

  • Writing 'underestimate' or 'overestimate' with no geometric justification — the explanation mark is lost without a chord/concavity reason.
  • Vague phrasing like 'the curve curves' or 'it's not straight' — must explicitly say chord lies above/below the curve.
  • Answering the accuracy question with 'use more decimal places' or 'use a calculator' — these address arithmetic precision, not the inherent method error.
  • Forgetting to extend an area conclusion to volume (or vice versa) when the question asks about the derived quantity.

A-level Scope Notes

Fully within Edexcel IAL Pure Maths (P2/P3) trapezium rule, formula booklet entry $\int_a^b y\,dx \approx \tfrac{h}{2}\{y_0 + y_n + 2(y_1 + \cdots + y_{n-1})\}$. The error formula $-\tfrac{(b-a)h^2}{12}f''(\xi)$ is OUT of scope at IAL — do not quote it; argue geometrically via concavity instead.
