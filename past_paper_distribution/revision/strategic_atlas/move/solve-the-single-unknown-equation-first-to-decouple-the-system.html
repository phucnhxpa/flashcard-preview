Algebra & Functions

Solve the single-unknown equation first to decouple the system

Family: Strategic Equation Ordering Frequency: 1 questions

Trigger — When to use

Fires when a linear system (often from equating components of vector/parametric equations, or from simultaneous equations) contains one equation in which all but one unknown have zero coefficients. Common cue: a direction vector has a zero component, so the corresponding scalar parameter drops out of that coordinate equation. Also triggers when a row of a matrix system, or one equation after partial elimination, reduces to a single variable. Phrasings like 'one equation only involves \(\lambda\)' or 'the \(z\)-component gives directly...' signal this move.

Motivation — Why this works

A linear system is fastest to solve along its weakest link: an equation with only one unknown gives that unknown immediately, with no substitution or elimination overhead. Once that variable is fixed, the remaining equations become single-unknown equations themselves, so the entire system collapses sequentially. This exploits the triangular structure already hidden in the system and minimises algebraic manipulation, cutting both time and error risk. It is the manual analogue of back-substitution in row-echelon form.

Thought Process — The step-by-step reasoning

  1. I scan all equations and count how many unknowns appear in each before doing any algebra.
  2. I rule out starting with equations that contain two or more unknowns, since those would require elimination or substitution.
  3. I commit to the equation with a single unknown because it yields a value in one step with no dependencies.
  4. I bridge to action by solving that equation directly, then feeding the result into the next-simplest equation.
  5. Inside the move I keep track of which equations remain unused, so I can verify consistency with the final equation as a check.

Mechanic — Concrete operations

  1. Inspect each equation and identify any that involves only one unknown (often because a coefficient is structurally zero).
  2. Solve that equation directly for its single unknown.
  3. Substitute the found value into the next equation that now becomes single-unknown.
  4. Repeat back-substitution until all unknowns are determined.
  5. Substitute into the unused equation to verify consistency (especially important for intersection/skew checks).

Worked Micro-Example

Equating \(\mathbf{r}_1 = \mathbf{r}_2\) gives the system: (1) \(2+\lambda = 3+\mu\), (2) \(\lambda = -\mu\), (3) \(-1+3\lambda = 2\). Equation (3) involves only \(\lambda\): \(3\lambda = 3 \Rightarrow \lambda = 1\). Substituting into (2): \(1 = -\mu \Rightarrow \mu = -1\). Check in (1): \(2+1 = 3+(-1)\)? \(3 \ne 2\), so the lines do not intersect.

Variants — Same move, different surface

3D line intersection where a direction vector has a zero component
The coordinate equation along that axis becomes single-parameter; solve it first, then substitute into the remaining two.
Simultaneous equations after partial elimination has produced a single-variable row
Treat the reduced row as the entry point and back-substitute upward.
Plane-line intersection \(\mathbf{r} = \mathbf{a} + \lambda\mathbf{d}\) into \(\mathbf{r}\cdot\mathbf{n} = k\)
The substitution yields one equation in \(\lambda\) alone — solve it directly, no system needed.
Matrix equation \(M\mathbf{x}=\mathbf{b}\) where \(M\) is upper/lower triangular
Use back- or forward-substitution row by row, each row being single-unknown given previous results.
Partial fractions with a cover-up opportunity at a simple root
Substituting the root kills all but one unknown coefficient, isolating it immediately.

Common Misreadings

  • Treating the system as fully coupled and launching into elimination without scanning for a decoupled equation.
  • Assuming the third equation is redundant rather than the easiest, and discarding it.
  • Confusing 'zero component of direction vector' with 'parallel lines', missing the algebraic simplification it offers.

Pitfalls

  • Forgetting to substitute into the unused equation to check consistency, leading to a false 'intersection' for skew lines.
  • Sign errors when the single-variable equation has a negative constant (e.g. \(-1+3\lambda=2\) misread as \(3\lambda=1\)).
  • Solving the easy equation correctly but then substituting into the wrong remaining equation, reintroducing both unknowns.

A-level Scope Notes

Fully within Edexcel IAL scope: applies to 3D vector line intersection (FP3), simultaneous linear equations (P1), matrix back-substitution (FP1), and partial-fraction cover-up (P2/P3).
