Trigger — When to use

Fires immediately after clearing denominators in a partial fraction problem, leaving an identity of the form N(x) ≡ A·(factor₁) + B·(factor₂) + ... that must hold for ALL x. The visual cue is a sum of products on the RHS where each term contains a different linear factor of the original denominator. Equivalently: whenever you have just written 'multiply both sides by the common denominator' and obtained a polynomial identity, look for the roots of the linear factors. Especially powerful when the denominator factorises as a product of distinct linear terms (x−a)(x−b)... .

Motivation — Why this works

Because the cleared identity holds for every value of x, you may substitute any convenient value — and choosing x equal to a root of a linear denominator factor zeroes out every term containing that factor, leaving a single unknown isolated in one arithmetic step. This exploits the fact that f(x)·(x−a) vanishes at x=a, so all 'other' partial fraction contributions disappear. The result is the Heaviside cover-up principle: cover the factor, evaluate the rest at its root, and read off the coefficient. This avoids expanding the RHS and solving a full simultaneous system.

Thought Process — Step-by-step reasoning

  1. Notice the structure: after clearing denominators, I have a polynomial identity N(x) ≡ A(x−b)(x−c)... + B(x−a)(x−c)... + ... that holds for ALL x.
  2. Rule out the brute-force route: I could expand the entire RHS, collect powers of x, and equate coefficients to solve a simultaneous system — but that's slow and error-prone.
  3. Commit to substitution: since the identity holds for any x, I should pick values that kill as many terms as possible. The roots of the linear denominator factors do exactly that.
  4. Bridge to operation: list the linear factors (x−a), (x−b), ... and their roots a, b, ... . Each substitution x = (root) zeroes every term except one, isolating one unknown instantly.
  5. Tactical decision: if ALL factors are distinct and linear, substitution alone finds every constant. If there is an irreducible quadratic factor (Bx+C numerator) or a repeated factor, use substitution for the linear roots first, then finish off the remaining unknowns by comparing coefficients of the highest powers of x — a hybrid that's faster than either method alone.
  6. Verify by checking one untouched coefficient (e.g. the constant term or x¹ coefficient) to catch arithmetic slips.

Mechanic — Concrete operations

  1. Multiply both sides of the partial fraction setup by the common denominator to obtain a polynomial identity N(x) ≡ Σ (numerator)·(product of other factors).
  2. Identify the roots of each linear denominator factor: for (x−a), the root is x = a; for (x+k), the root is x = −k.
  3. Substitute x = (root of factor i): all terms containing (x − root) vanish, leaving N(root) = (numerator i)·(product of other factors evaluated at the root). Solve for that single unknown.
  4. Repeat for each linear factor's root to peel off one constant per substitution.
  5. If an irreducible quadratic (or repeated factor) leaves unknowns unresolved, expand the RHS using the constants already found and equate coefficients of the highest powers of x to solve for the remaining unknowns.
  6. Verify by checking one further coefficient or by substituting a simple value like x = 0.

Worked Micro-Example

Decompose $\frac{26x+35}{(x+1)(x+2)} = \frac{C}{x+1} + \frac{D}{x+2}$. Clearing denominators: $26x+35 \equiv C(x+2) + D(x+1)$. Set $x=-1$: $26(-1)+35 = C(1) \Rightarrow C = 9$. Set $x=-2$: $26(-2)+35 = D(-1) \Rightarrow D = 17$. Therefore $\frac{26x+35}{(x+1)(x+2)} = \frac{9}{x+1} + \frac{17}{x+2}$.

Variants — Same move, different surface

Denominator contains an irreducible quadratic factor, e.g. (x−1)(x²+1) with numerator (Bx+C) over the quadratic
Substitute the real root (x = 1) to find A, then expand and equate coefficients of x² and x¹ to solve for B and C — substitution alone cannot finish the job because x²+1 has no real root.
Repeated linear factor like (x+1)²
Substitute x = −1 to find the coefficient of the highest-power term 1/(x+1)², then use a second convenient value (e.g. x = 0) or coefficient comparison to find the 1/(x+1) coefficient.
After a substitution u = ... the new identity has roots like u = 0, u = 2
Same logic in the new variable: pick u-values that are roots of the u-denominator factors. Combine substitution (for clean roots) with coefficient comparison for any leftover unknowns.
Heaviside cover-up shorthand
Mentally 'cover' the factor (x−a) in the original fraction and evaluate the rest at x = a — this is the same logic in compact notation, useful for mental computation.

Common Misreadings

  • Treating the identity as only valid for 'most' x and hesitating to substitute roots that make a denominator factor zero — the cleared identity is a polynomial identity valid for ALL x.
  • Trying to use cover-up on an irreducible quadratic factor by substituting a real value — there is no real root, so this term cannot be isolated by real substitution alone.
  • Confusing the cleared polynomial identity with the original fraction (which is undefined at the roots) and refusing to substitute those roots.

Pitfalls

  • Sign errors when substituting negative roots: e.g. evaluating D(x+1) at x = −2 gives D(−1) = −D, not D.
  • Substituting x-values that are NOT roots of denominator factors — this fails to zero out terms and produces a messy simultaneous system.
  • Forgetting that for an irreducible quadratic factor, cover-up only handles the linear factors; the (Bx+C) numerator still requires coefficient comparison.
  • Arithmetic slips when evaluating the surviving bracket (e.g. computing (root − other root) with wrong sign).

A-level Scope Notes

Fully within Edexcel IAL P3/P4 partial fractions scope (denominators with distinct linear factors, repeated linear factors, or one irreducible quadratic). The technique is sometimes called the 'cover-up rule' or attributed to Heaviside; either name is acceptable. Avoid substituting complex roots (e.g. x = i for x²+1) — that is a Further Maths/complex analysis extension and not expected at this level.
