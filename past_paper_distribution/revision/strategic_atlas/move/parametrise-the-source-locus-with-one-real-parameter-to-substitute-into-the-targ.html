Complex Numbers

Parametrise the source locus with one real parameter to substitute into the target equation

Family: Parametric Substitution Frequency: 2 questions

Trigger — When to use

Fires when a problem couples a complex locus (half-line arg(z − z₁) = θ, real/imaginary axis, vertical/horizontal line, or circle) with a second condition: another locus to intersect, or a Möbius/rational map whose image is required. Visual cues: 'find the points of intersection of arg(z − z₁) = θ and |z − z₂| = r', or 'find the image of the imaginary axis under w = f(z)'. Equivalent phrasings include 'where does this half-line meet that circle' and 'sketch the image of Re(z) = 0 in the w-plane'. The key signal is that one of the loci already admits a clean one-parameter description, so brute-force Cartesian work is avoidable.

Motivation — Why this works

A locus defined by a single linear/angular constraint is intrinsically one-dimensional, so it can be encoded by exactly one real parameter (t for a half-line, y for the imaginary axis, θ for a circle). Substituting this parametric form into the second equation collapses two simultaneous constraints into a single equation in one real unknown, which is dramatically easier to solve. This works because the parametrisation automatically enforces the source locus condition (e.g. the arg or Re(z) = 0 condition is built in), so it never has to be checked separately. The underlying principle is dimensional matching: a 1-D object embedded in 2-D is best described by 1 parameter, not 2 coordinates plus 1 constraint.

Thought Process — The step-by-step reasoning

  1. Notice that one locus (the half-line, the axis, or the unit circle) is one-dimensional and has an obvious single-parameter description: z = z₁ + t(cos θ + i sin θ) with t > 0, or z = iy, or z = e^{iφ}.
  2. Consider the alternative: write z = x + iy and impose both conditions in Cartesian form. Reject this — it introduces two unknowns plus a constraint and typically forces messy simultaneous equations or quadratics in two variables.
  3. Consider also inverting a Möbius map to express z in w and imposing the locus condition afterwards. Reject this — partial-fraction algebra and sign tracking are error-prone.
  4. Commit to direct substitution: feed the parametric z into the second equation (the circle |z − z₂| = r, or the map w = f(z)). The arg/Re/Im condition is enforced automatically, so only one real equation remains.
  5. Decide on the parameter range carefully: t > 0 strictly for a half-line (excluding the endpoint z₁), y ∈ ℝ for an axis, and exclude any parameter value that makes a denominator vanish.
  6. After solving, reconstruct z (or w) from the parameter value(s) and reject any solution that violates the parameter's allowed range.

Mechanic — Concrete operations

  1. Identify the one-parameter form of the source locus: half-line → z = z₁ + t(cos θ + i sin θ), t > 0; imaginary axis → z = iy, y ∈ ℝ; real axis → z = x, x ∈ ℝ; circle → z = z₀ + r e^{iφ}.
  2. Substitute this expression for z into the target equation (|z − z₂| = r, or w = f(z), etc.).
  3. For an intersection: square the modulus to get (Re part)² + (Im part)² = r², expand, and obtain a quadratic in the parameter.
  4. For an image curve: simplify w as a function of the single real parameter, separating Re(w) and Im(w).
  5. Solve for the parameter; discard values violating range (t ≤ 0, or values making any denominator zero such as y = −1 when (z + i) appears).
  6. Substitute valid parameter values back to recover the required z (intersection point) or eliminate the parameter to obtain a Cartesian relation in (u, v) where w = u + iv.

Worked Micro-Example

Find where arg(z − 1) = π/4 meets |z| = √5. Parametrise: z = 1 + t(cos(π/4) + i sin(π/4)) = 1 + (t/√2)(1+i), t > 0. Then |z|² = (1 + t/√2)² + (t/√2)² = 5, giving t² + √2 t − 4 = 0, so t = (−√2 + √18)/2 = √2. Hence z = 1 + (√2/√2)(1+i) = 2 + i.

Variants — Same move, different surface

Both loci are circles
Parametrise one as z = z₀ + r e^{iφ} and substitute into the other's modulus equation; solve trigonometric equation in φ.
One locus is a perpendicular bisector or general line
Write z = x + i(mx + c) and substitute into |z − z₂| = r, yielding a quadratic in x.
Image of the unit circle under a Möbius map
Set z = cos φ + i sin φ, substitute into w = f(z), and eliminate φ via cos²φ + sin²φ = 1.
Image of a vertical line Re(z) = c
Write z = c + iy, substitute into w = f(z), then eliminate y between Re(w) and Im(w).

Common Misreadings

  • Trying to equate arguments of two complex expressions instead of parametrising the half-line — rarely productive.
  • Writing z = x + iy and then imposing the locus constraint after substitution, doubling the algebraic workload.
  • Treating the half-line as a full line and getting spurious intersection points on the wrong side of z₁.

Pitfalls

  • Forgetting the strict t > 0 condition for a half-line; t = 0 gives z₁ (excluded) and t < 0 gives the opposite ray.
  • Algebraic slips when expanding |(p + t cos θ) + i(q + t sin θ)|² — each bracket must be expanded carefully and the cos²θ + sin²θ = 1 simplification used.
  • Solving the quadratic in t and forgetting to test both roots for positivity.
  • Forgetting to exclude parameter values that make a denominator vanish under a Möbius map (e.g. y = −1 when (z + i) appears in the denominator).
  • Failing to eliminate the parameter at the end when a Cartesian image equation is required.

A-level Scope Notes

Entirely within Edexcel IAL Further Pure scope (FP1/FP2 loci of complex numbers and transformations). Uses only standard parametrisation z = a + ib, modulus identity |z|² = (Re z)² + (Im z)², and cos²θ + sin²θ = 1 from the formula booklet. Avoid generalisations to inversive geometry or cross-ratio invariants — these are out of scope.
