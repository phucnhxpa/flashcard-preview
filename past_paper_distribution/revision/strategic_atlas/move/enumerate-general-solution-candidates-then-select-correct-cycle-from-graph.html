Trigger — When to use

The question asks for the coordinates of a specific labelled point (often Q or P) on a sinusoidal graph that is either a maximum, minimum, or x-intercept. The underlying trig equation (e.g. sin(x)=0, cos(x−a)=−1, cos(x)=1) has infinitely many solutions, so a restricted-domain algebraic answer is impossible. The phrase 'with reference to Figure …' or a clearly-drawn graph extending over multiple periods is the visual cue. Equivalently: 'find the x-coordinate of Q' where Q sits several cycles along the positive x-axis.

Motivation — Why this works

Standard trig equations like sin θ = 0 or cos θ = ±1 produce infinite arithmetic progressions of solutions (spacing equal to the period or half-period). When the question pins down one specific solution via a diagram rather than a domain restriction, the only reliable strategy is to write the general solution, list the candidate x-values in order, and match against the figure. This combines the algebraic technique (general solution of a trig equation) with graphical interpretation, and prevents the common error of guessing the principal value.

Thought Process — Step-by-step reasoning

  1. Notice that Q is a special point on a trig curve (max, min, or x-intercept), so I need to solve a standard trig equation like sin x = 0 or cos(x − a) = −1.
  2. Rule out a restricted-domain approach — the graph extends over several cycles, so there is no single algebraic answer; I must use the figure to pick the right cycle.
  3. Commit to the general-solution-then-enumerate strategy: write θ = principal value + n·(period), substitute back any phase shift, and list the candidates in order.
  4. Bridge to operation: solve the underlying equation algebraically, then re-add the phase shift (x = θ + shift) so the candidates are in terms of the original variable.
  5. Tactical decision: enumerate enough candidates (n = 0, 1, 2, …) to comfortably bracket where Q is drawn, then count maxima/minima/intercepts on the figure to match.
  6. Final check: confirm the chosen x makes geometric sense — e.g. a maximum should sit at the top of a peak in the figure, and the y-coordinate should match the amplitude.

Mechanic — Concrete operations

  1. Identify the equation Q satisfies (e.g. sin x = 0, cos(x − 30)° = −1).
  2. Write the general solution: θ = θ₀ + n·T where θ₀ is the principal value and T is the period (or half-period for zeros of sin/cos).
  3. Re-introduce any phase shift / scaling so the candidates are values of x, not θ.
  4. Enumerate candidates for n = 0, 1, 2, … as an arithmetic sequence.
  5. Count the corresponding feature (peaks/troughs/zeros) on the figure to determine which n applies to Q.
  6. State Q as an ordered pair (x, y), where y is read directly from the curve's amplitude/axis.

Worked Micro-Example

For $y = 3\cos(x - 30)°$ with Q a minimum shown as the third trough on the graph: solve $\cos(x-30)° = -1 \Rightarrow x - 30 = 180 + 360n \Rightarrow x = 210 + 360n$. Enumerate: $210°, 570°, 930°, \dots$. The third trough corresponds to $n = 2$, so $Q = (930°, -3)$.

Variants — Same move, different surface

For x-intercepts of y = A sin(x), where sin x = 0
Use x = 180n° (spacing is the half-period 180°, not the full period 360°); count intercepts from the origin, treating the origin as the n = 0 intercept.
For x-intercepts of y = A cos(x), where cos x = 0
General solution x = 90° + 180°n; same enumerate-and-match logic.
For transformed curves like y = sin(Bx) or y = cos(Bx − C)
The period becomes 360°/B, so candidate spacing shrinks accordingly; remember to add back the phase shift C/B when listing x-values.
Restricted-domain version (e.g. 0 ≤ x ≤ 360°)
Selection becomes algebraic — keep only candidates inside the domain, no graph needed.

Common Misreadings

  • Assuming Q is the principal solution (n = 0) without consulting the graph, e.g. giving x = 210° when the figure shows x = 930°.
  • Treating the origin as the 'first' intercept rather than the zeroth, producing an off-by-one error.
  • Misreading the x-axis scale on the figure, especially when it extends past 720°.

Pitfalls

  • Forgetting to add the phase shift back: solving cos(x − 30)° = −1 gives x − 30 = 180 + 360n, so x = 210 + 360n, not x = 180 + 360n.
  • Confusing the period (360°) with the zero-spacing (180° for sin x or cos x), leading to half as many or twice as many candidates.
  • Using the wrong period for y = A cos(Bx − C): the period is 360°/B, not 360°.
  • Reading the y-coordinate incorrectly — at a maximum y = +A, at a minimum y = −A, at a zero y = 0.

A-level Scope Notes

Sits within Edexcel IAL P2/P3 trigonometry: general solutions of sin θ = k, cos θ = k. No formula booklet entry is needed beyond knowing the periods (360° for sin/cos, 180° for tan). Avoid introducing complex-number or radian general-solution forms unless the question is posed in radians.
