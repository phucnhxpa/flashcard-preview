Integration

Set up strip width and ordinate table for trapezium rule

Family: Numerical Integration Frequency: 1 questions

Trigger — When to use

Fires whenever a question instructs you to apply the trapezium rule with a stated number of strips (or stated number of ordinates) over a definite interval [a,b]. Typical phrasings: 'use the trapezium rule with n strips', 'with n+1 ordinates', or 'complete the table and hence estimate'. Often paired with a 'show that' target value, signalling the need for accurate intermediate values. Also fires when the integrand is awkward to integrate exactly (e.g. log, exponential, or root expressions).

Motivation — Why this works

The trapezium rule formula \(\tfrac{h}{2}[y_0+y_n+2(y_1+\dots+y_{n-1})]\) presupposes equally spaced sample points, so h and the x-grid must be fixed before any y is computed. Setting h=(b-a)/n first guarantees the correct spacing and the correct count of ordinates (n+1). Tabulating y-values systematically eliminates miscounts of which ordinates are 'ends' versus 'middles'. Carrying sufficient decimal places preserves the accuracy needed to match a 'show that' target.

Thought Process — The step-by-step reasoning

  1. Notice the cue 'trapezium rule with n strips' and the explicit interval, so the formula and grid are fully determined.
  2. Rule out exact integration (often the integrand has no elementary antiderivative at A-level) and rule out using n+1 as the divisor when computing h.
  3. Commit to the standard pipeline: compute h, list x-values, evaluate y-values, then substitute — in that order — to avoid bookkeeping errors.
  4. Bridge: write h=(b-a)/n explicitly, then build the x-grid x_i=a+ih for i=0,…,n.
  5. Tactically, keep at least 4–5 decimal places in each y_i, identify y_0 and y_n as endpoint ordinates, and group the interior ordinates for the ×2 factor.

Mechanic — Concrete operations

  1. Read n (number of strips) and the interval [a,b] from the question.
  2. Compute h = (b-a)/n.
  3. List the x-values x_0=a, x_1=a+h, …, x_n=b (giving n+1 ordinates).
  4. Evaluate each y_i = f(x_i) to at least 4–5 decimal places.
  5. Substitute into Area ≈ (h/2)[y_0 + y_n + 2(y_1 + y_2 + … + y_{n-1})].
  6. Simplify and round to the precision requested (or to match the 'show that' target).

Worked Micro-Example

Estimate \(\int_2^{14}\log_{10}x\,dx\) with 4 strips. Then \(h=(14-2)/4=3\), giving \(x=2,5,8,11,14\) and \(y\approx 0.30103,\,0.69897,\,0.90309,\,1.04139,\,1.14613\). Area \(\approx \tfrac{3}{2}[0.30103+1.14613+2(0.69897+0.90309+1.04139)] = 1.5\times 6.73406 \approx 10.10.\)

Variants — Same move, different surface

Question gives 'n+1 ordinates' instead of 'n strips'
Convert: number of strips = (number of ordinates) − 1, then compute h = (b-a)/n as usual.
x-values supplied in a pre-printed table
Read h directly as the common difference of consecutive x-values; just compute the missing y-values.
Integrand involves a parameter or is given as y=f(x) tabulated from an earlier part
Reuse the earlier y-values; do not recompute, and check the spacing is uniform before applying the rule.
Interval includes a point where f is undefined or improper
Outside A-level scope — at IAL the integrand is always defined and finite at every x_i.
Comparison with exact value to assess over/under-estimate
After computing the estimate, examine concavity of f on [a,b]: concave-up ⇒ overestimate, concave-down ⇒ underestimate.

Common Misreadings

  • Confusing 'n strips' with 'n ordinates', leading to h=(b-a)/(n+1) or h=(b-a)/(n-1).
  • Treating 'four ordinates' as four strips, producing the wrong h.
  • Assuming the interval endpoints are excluded (they are y_0 and y_n and must be included once).

Pitfalls

  • Rounding y-values too aggressively (e.g. 2 d.p.), so the final estimate fails to match the 'show that' target.
  • Multiplying y_0 or y_n by 2, or forgetting the ×2 on the interior ordinates.
  • Arithmetic slip in computing h, propagating through every x_i.
  • Using degrees instead of radians when f involves trigonometric functions.
  • Forgetting the outer factor h/2 (writing h instead).

A-level Scope Notes

Fully within Edexcel IAL Pure Mathematics (P2/P3): the trapezium rule is applied to continuous integrands on a closed interval with a stated number of strips, and accuracy/over-under-estimation arguments use concavity only.
