Complex Numbers

Insert +2kπ before taking the nth root to capture all roots

Family: Multi Valued Argument Frequency: 1 questions

Trigger — When to use

Fires whenever an equation of the form z^n = w (with n ≥ 2) must be solved completely. Cues include phrases like 'find all roots', 'solve z^n = ...', or being told the equation has n solutions while only one has been produced. Also fires when asked to find nth roots of unity, or to mark all roots on an Argand diagram. The signature is: a single modulus–argument form has been written, but n distinct answers are required.

Motivation — Why this works

The argument of a non-zero complex number is defined only modulo 2π, so w = re^{iθ} = re^{i(θ + 2kπ)} for every integer k. When the nth root is taken, the exponent becomes (θ + 2kπ)/n, and now consecutive integers k differ by 2π/n in argument — genuinely different points on the circle. Exactly n consecutive values of k produce all distinct roots before the pattern repeats. This is the standard mechanism producing n equally spaced roots on a circle of radius r^{1/n}.

Thought Process — The step-by-step reasoning

  1. Notices that z^n = w is a degree-n polynomial equation, so there must be exactly n roots, yet writing w with a single argument has only delivered one.
  2. Rules out adding 2kπ after dividing — that just shifts the argument by full turns and reproduces the same root.
  3. Commits to the principle that argument is multi-valued: w = re^{i(θ + 2kπ)} must be used before extracting the root.
  4. Bridges to the operation: take the nth root of both sides to get z = r^{1/n} e^{i(θ + 2kπ)/n}, then list k = 0,1,…,n−1.
  5. Tactically chooses the k-range to give simplest arguments (often centred on 0, e.g. k = −2,−1,0,1,2 for n = 5) if a principal-value form is desired.

Mechanic — Concrete operations

  1. Write w in modulus–argument form with general argument: w = re^{i(θ + 2kπ)}, k ∈ ℤ.
  2. Take nth root of both sides: z = r^{1/n} e^{i(θ + 2kπ)/n}.
  3. Substitute n consecutive integer values of k (typically k = 0,1,…,n−1) to list the n distinct roots.
  4. Simplify each argument to lie in the desired range (e.g. (−π, π]) if required.
  5. Optionally verify roots are equally spaced by 2π/n on a circle of radius r^{1/n}.

Worked Micro-Example

Solve $z^5 = 32i$. Write $32i = 32e^{i(\pi/2 + 2k\pi)}$. Then $z = 2e^{i(\pi/2 + 2k\pi)/5} = 2e^{i(\pi + 4k\pi)/10}$. Taking $k = 0,1,2,3,4$ gives the five roots with arguments $\tfrac{\pi}{10}, \tfrac{\pi}{2}, \tfrac{9\pi}{10}, \tfrac{13\pi}{10}, \tfrac{17\pi}{10}$, equally spaced by $\tfrac{2\pi}{5}$ on the circle $|z| = 2$.

Variants — Same move, different surface

Roots of unity: z^n = 1
Use 1 = e^{i·2kπ}, giving z = e^{2kπ i/n} for k = 0,…,n−1.
z^n = w with w in Cartesian form
First convert w to modulus–argument form (compute |w| and arg w), then apply the standard procedure.
Find one root, then rotate
Find a single root z₀, then the others are z₀ · e^{2kπ i/n} for k = 1,…,n−1 — equivalent to the +2kπ method.
Equation (z − a)^n = w
Apply the move to the bracket to get z − a = r^{1/n} e^{i(θ+2kπ)/n}, then add a.

Common Misreadings

  • Believing only one root exists because only one argument was initially identified.
  • Adding +2kπ after dividing by n, which merely shifts by full revolutions and reproduces the same single root.
  • Confusing arg(w) (multi-valued) with the principal value Arg(w) and forgetting the multi-valued nature is what generates the extra roots.

Pitfalls

  • Omitting the +2kπ entirely and reporting only one of the n required roots.
  • Using k = 1,2,…,n instead of k = 0,1,…,n−1 — works but risks miscounting and double-listing.
  • Failing to reduce arguments into the requested range (e.g. (−π, π]) when the question demands principal arguments.
  • Forgetting to take the nth root of the modulus as well, leaving |z| = r instead of r^{1/n}.

A-level Scope Notes

Standard Edexcel IAL Further Pure content (FP2): nth roots of complex numbers, de Moivre's theorem, and Argand diagram representation of equally spaced roots.
