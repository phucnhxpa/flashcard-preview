Split the middle term via the ac-method to factorise a non-monic quadratic

Family: Quadratic Factorisation Frequency: 2 questions

Trigger — When to use

Fires when you face a quadratic ax² + bx + c with a ≠ 1 (a non-monic / non-unit leading coefficient) that you need to factorise into two linear brackets — typically to solve ax² + bx + c = 0, to match a stated factorised form like (5x − 1)(x + 2), or to find roots of f'(x) for stationary points. Equivalent cues: 'factorise 5x² + 9x − 2', 'show that f'(x) = (px + q)(x + r)', or 'solve' a non-monic quadratic where the discriminant looks like a perfect square. Use this when ac is small enough that an integer factor pair summing to b can be spotted quickly. If ac is awkward or no integer pair exists, switch to the quadratic formula instead.

Motivation — Why this works

The ac-method works because if ax² + bx + c = (px + q)(rx + s) with integer coefficients, then pr = a, qs = c and ps + qr = b — so the two cross-product numbers ps and qr multiply to (ps)(qr) = (pr)(qs) = ac and add to b. Finding the pair (m, n) with mn = ac and m + n = b lets you split bx = mx + nx, after which factoring by grouping pulls out a common binomial. This is systematic, gives exact rational roots immediately, and avoids the sign-juggling guesswork of trying (px ± ?)(rx ± ?) by inspection.

Thought Process — The step-by-step reasoning

  1. I notice the quadratic is non-monic (a ≠ 1), e.g. 5x² + 9x − 2, so I cannot just look for two numbers multiplying to c and summing to b — that shortcut is only valid when a = 1.
  2. I consider alternatives: (i) guess-and-check binomial factors — risky on signs; (ii) the quadratic formula — always works but slower and gives surd-looking output even when roots are rational; (iii) the ac-method — systematic and fast if ac has a nice integer factor pair.
  3. I commit to the ac-method because ac = 5·(−2) = −10 is small, so I can mentally enumerate factor pairs of −10 and check which sums to b = 9.
  4. Bridge to operation: I list pairs of −10 — (1,−10), (−1,10), (2,−5), (−2,5) — and pick (10, −1) since 10 + (−1) = 9 = b. ✓
  5. Tactical choice: when I split bx = 10x + (−1)x, the order of the two split terms doesn't matter for the answer, but I order them so the first pair shares an obvious GCF with ax² — here writing 5x² + 10x − x − 2 lets me factor 5x from the first pair cleanly.
  6. I then group, extract the common binomial (x + 2), and read off the factorisation (5x − 1)(x + 2).

Mechanic — Concrete operations

  1. Identify a, b, c from ax² + bx + c. Compute the product ac.
  2. Find an integer pair (m, n) with mn = ac and m + n = b. (Enumerate factor pairs of ac systematically; keep track of signs.)
  3. Rewrite the middle term: ax² + bx + c = ax² + mx + nx + c.
  4. Group into two pairs and extract the GCF of each: e.g. 5x² + 10x − x − 2 = 5x(x + 2) − 1(x + 2).
  5. Factor out the common binomial: (5x − 1)(x + 2).
  6. If solving = 0, set each factor to zero: 5x − 1 = 0 ⇒ x = 1/5, x + 2 = 0 ⇒ x = −2.

Worked Micro-Example

Factorise $5x^2 + 9x - 2$. Compute $ac = 5\cdot(-2) = -10$; need pair summing to $9$: $(10,-1)$. Split: $5x^2 + 10x - x - 2$. Group: $5x(x+2) - 1(x+2) = (5x-1)(x+2)$. Hence roots $x = \tfrac{1}{5},\ x = -2$.

Variants — Same move, different surface

Monic quadratic (a = 1), e.g. x² + 5x + 6
ac reduces to c, so just find two numbers multiplying to c and summing to b; no splitting/grouping needed — write (x + m)(x + n) directly.
ac is large or no integer pair exists (irrational/complex roots)
Abandon the ac-method; use the quadratic formula x = (−b ± √(b² − 4ac))/(2a), or complete the square.
Quadratic appears as a numerator/derivative to be matched against a stated form like (px + q)(x + r)
Same procedure; after factorising, compare coefficients with the given template to identify any unknown constant (e.g. 'find k' such that the split is correct).
Hidden quadratic in a substitution, e.g. 5u² + 9u − 2 = 0 where u = e^x or u = √x
Apply ac-method to the u-quadratic, then back-substitute to recover x.

Common Misreadings

  • Treating a non-monic quadratic as if a = 1 — looking for numbers multiplying to c (not ac) and summing to b, which gives wrong factors.
  • Jumping to the quadratic formula when integer factorisation would be quicker and cleaner, then mishandling the surd simplification.

Pitfalls

  • Sign errors when c < 0: confusing which of the pair is positive and which negative (e.g. picking (−10, 1) instead of (10, −1)).
  • Grouping incorrectly so the two pairs do not share the same binomial — e.g. writing 5x(x + 2) + 1(−x − 2) and failing to factor out −1 to expose the common (x + 2).
  • Forgetting that the pair must MULTIPLY to ac (not just c) and ADD to b — swapping these conditions.
  • After obtaining (5x − 1)(x + 2) = 0, mis-solving 5x − 1 = 0 as x = −1/5 instead of x = 1/5.

A-level Scope Notes

Fully within Edexcel IAL Pure Maths (P1 onward). No formula-booklet entry is needed; the quadratic formula is provided in the booklet as a fallback. Avoid generalising to cubics/quartics via 'ac-method' — that does not extend; for higher degree use the factor theorem instead.
