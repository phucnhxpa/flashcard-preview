Triggersestimate the medianlinear interpolationcumulative frequencylower quartileupper quartileQ1Q3
Assumptions
  • Data is uniformly distributed within each class (justifies linear interpolation)
  • Use class boundaries (not class limits) as the endpoints for interpolation
  • Median position = n/2 (not (n+1)/2) for grouped continuous data in S1
Approaches
A: Direct linear interpolation formula
When: Standard approach for all grouped data median/quartile questions
  1. 1. Construct a cumulative frequency column
  2. 2. Find the target position: median at n/2, Q1 at n/4, Q3 at 3n/4
  3. 3. Identify the class that contains this position (the class where cumulative frequency first reaches/exceeds the target)
  4. 4. Apply the formula: estimate = L + ((target position − CF_before) / f_class) × w
  5. where L = lower class boundary, CF_before = cumulative frequency up to preceding class, f_class = frequency of the median class, w = class width
  6. 5. Check the answer lies within the identified class
+ Pros: Direct and formulaic; quick once the correct class is identified
– Cons: Easy to use wrong class boundaries or miscalculate CF_before
B: Graphical approach using cumulative frequency curve (ogive)
When: When a cumulative frequency diagram is already drawn or requested; good for checking answers
  1. 1. Plot cumulative frequency against upper class boundaries
  2. 2. Join points with a smooth curve (ogive)
  3. 3. Read across from n/2 (or n/4, 3n/4) on the y-axis to the curve
  4. 4. Read down to the x-axis to obtain the estimate
  5. 5. State values to appropriate accuracy
+ Pros: Visual and intuitive; useful for checking interpolation answers
– Cons: Less precise than calculation; drawing errors affect accuracy
Formulas
$Q_2 = L + \left(\frac{n/2 - CF_{\text{before}}}{f_{\text{class}}}\right) \times w$ (MEMORIZE)
$Q_1$: use $n/4$ in place of $n/2$; $Q_3$: use $3n/4$ (MEMORIZE)
$\text{IQR} = Q_3 - Q_1$ (MEMORIZE)
Pitfalls
  • Using class limits (e.g. 20, 30) instead of class boundaries (e.g. 19.5, 29.5) as L and for class width w
  • Getting the wrong class by miscounting cumulative frequency — always write out the CF column fully
  • Using (n+1)/2 for grouped data instead of n/2 (the (n+1)/2 rule is for listed/discrete data)
  • Forgetting to subtract CF_before (using total CF of the class instead of the frequency within it)

Triggershistogramfrequency densityunequal class widthsarea represents frequencyscale factor
Assumptions
  • Frequency density = frequency / class width
  • Area of each bar is proportional to (or equal to) the frequency of that class
  • Class boundaries (not limits) define the edges of the bars
Approaches
A: Calculate frequency density for each class
When: When you need to draw a histogram from a frequency table with unequal class widths
  1. 1. Determine class boundaries for each class
  2. 2. Calculate class width w = upper boundary − lower boundary
  3. 3. Calculate frequency density = frequency / class width for each class
  4. 4. Draw bars with heights equal to frequency density on a continuous x-axis
  5. 5. Label axes: x-axis = variable, y-axis = frequency density
+ Pros: Straightforward once class boundaries are identified
– Cons: Requires careful identification of class boundaries first
B: Use a reference bar to find the scale factor, then read off frequencies
When: When a histogram is given and you must extract frequencies from it
  1. 1. Identify one class where both the frequency and bar height are known
  2. 2. Calculate scale factor k = frequency / (bar height × class width), or simply use frequency density = height
  3. 3. For each other bar: frequency = bar height × class width (if height = frequency density)
  4. 4. Alternatively: frequency = k × area of bar
  5. 5. Verify by summing all frequencies to match given total if provided
+ Pros: Works even if the y-axis scale is not explicitly labelled
– Cons: Misreading bar heights from the diagram can introduce errors
Formulas
$\text{Frequency density} = \frac{\text{frequency}}{\text{class width}}$ (MEMORIZE)
$\text{Frequency} = \text{frequency density} \times \text{class width}$ (MEMORIZE)
Area of bar $\propto$ frequency (MEMORIZE)
Pitfalls
  • Drawing bar heights proportional to frequency instead of frequency density when class widths are unequal
  • Forgetting to use class boundaries (e.g. using 10 instead of 9.5) for bar edges and class widths
  • Misreading the scale on the given histogram axis
  • Not using the provided reference bar to correctly calibrate the scale

Triggersskewnesspositive skewnegative skewsymmetricalshape of distributioncomment on the skewness
Assumptions
  • Positive skew: mean > median, or Q3 − Q2 > Q2 − Q1, or long upper whisker
  • Negative skew: mean < median, or Q3 − Q2 < Q2 − Q1, or long lower whisker
  • Symmetric: mean ≈ median, Q3 − Q2 ≈ Q2 − Q1
Approaches
A: Quartile-based skewness measure
When: When quartiles are already calculated — common in box plot questions
  1. 1. Compute Q3 − Q2 and Q2 − Q1
  2. 2. If Q3 − Q2 > Q2 − Q1 → positive skew
  3. 3. If Q3 − Q2 < Q2 − Q1 → negative skew
  4. 4. Optionally compute the Bowley skewness coefficient: (Q3 − 2Q2 + Q1)/(Q3 − Q1)
  5. 5. State the skewness with a contextual interpretation
+ Pros: Quick and uses already-computed values
– Cons: Only gives a rough indication; must interpret, not just state the number
B: Mean vs median comparison
When: When both mean and median are available
  1. 1. Compare mean and median
  2. 2. If mean > median → positive skew
  3. 3. If mean < median → negative skew
  4. 4. State conclusion in context
+ Pros: Very quick; often just one line of working
– Cons: Needs both mean and median to be computed/given
Formulas
Bowley skewness: $\frac{Q_3 - 2Q_2 + Q_1}{Q_3 - Q_1}$ (MEMORIZE)
Pearson skewness: $\frac{3(\text{mean} - \text{median})}{\sigma}$ (formula booklet)
Pitfalls
  • Stating 'positive skew' without justification — must show the comparison (e.g. Q3 − Q2 > Q2 − Q1)
  • Confusing the direction: long right tail = positive skew, not negative
  • Forgetting to interpret in context (e.g. 'most students scored at the lower end')

Triggersstem-and-leafback-to-backkeyorderedcompare distributions
Assumptions
  • Leaves must be in ascending order from the stem
  • A key must be provided (e.g. 3|5 means 35)
  • For back-to-back diagrams, leaves on the left are read outward from the stem (so in descending order left to right toward the stem)
  • Data is exact (not grouped) so use (n+1)/2 for median position
Approaches
A: Read data, find summary statistics, compare
When: Standard approach for all stem-and-leaf questions
  1. 1. Read all data values from the diagram, confirming with the key
  2. 2. Count n to verify no values are missed
  3. 3. Find median at position (n+1)/2, Q1 at (n+1)/4, Q3 at 3(n+1)/4
  4. 4. Calculate IQR = Q3 − Q1
  5. 5. If asked, calculate mean = Σx/n and s.d. = √(Σx²/n − x̄²)
  6. 6. For back-to-back: repeat for both sides and compare using named statistics (median, IQR) in context
+ Pros: Straightforward; uses exact ordered data so no interpolation needed
– Cons: Miscounting leaves is the biggest risk — must be meticulous
Formulas
Median position for listed data: $\frac{n+1}{2}$ (MEMORIZE)
$Q_1$ at $\frac{n+1}{4}$, $Q_3$ at $\frac{3(n+1)}{4}$ (MEMORIZE)
Pitfalls
  • Miscounting leaves — always recount n and cross-check with any stated total
  • Forgetting to include the key on a stem-and-leaf diagram you construct
  • Not ordering the leaves (especially on the left side of a back-to-back diagram)
  • Using n/2 instead of (n+1)/2 for the median position with listed data

Triggerscombinedtwo groupsaltogetherpooledtotal meantotal standard deviationrecover Σx and Σx²
Assumptions
  • Can recover Σx from n × x̄ and Σx² from n(σ² + x̄²)
  • Combined mean uses total of all sums divided by total n
  • Combined variance uses total Σx² / total n − (combined mean)²
Approaches
A: Recover sums and combine
When: Standard and only reliable method for combining groups
  1. 1. For each group, recover Σx_i = n_i × x̄_i
  2. 2. For each group, recover Σx_i² = n_i(σ_i² + x̄_i²)
  3. 3. Combined Σx = Σx_1 + Σx_2; combined n = n_1 + n_2
  4. 4. Combined mean: x̄_c = (Σx_1 + Σx_2) / (n_1 + n_2)
  5. 5. Combined Σx² = Σx_1² + Σx_2²
  6. 6. Combined variance: σ_c² = (Σx_1² + Σx_2²)/(n_1 + n_2) − x̄_c²
  7. 7. Combined s.d.: σ_c = √(σ_c²)
+ Pros: Works for any number of groups; systematic
– Cons: Requires careful recovery of Σx² — easy to forget the x̄² term
Formulas
$\Sigma x = n\bar{x}$ (MEMORIZE)
$\Sigma x^2 = n(\sigma^2 + \bar{x}^2)$ — rearranged from $\sigma^2 = \frac{\Sigma x^2}{n} - \bar{x}^2$ (MEMORIZE)
$\bar{x}_c = \frac{\Sigma x_1 + \Sigma x_2}{n_1 + n_2}$ (MEMORIZE)
$\sigma_c = \sqrt{\frac{\Sigma x_1^2 + \Sigma x_2^2}{n_1 + n_2} - \bar{x}_c^2}$ (MEMORIZE)
Pitfalls
  • Averaging the two means or the two s.d.s directly — this is WRONG unless n₁ = n₂ (and even then wrong for s.d.)
  • Forgetting to add x̄² when recovering Σx²: writing Σx² = nσ² instead of n(σ² + x̄²)
  • Using sample variance (n−1) from one formula and population variance (n) from another — stay consistent with S1 convention (n divisor)

Triggersbox plotbox-and-whiskeroutliersIQRinterquartile range1.5 × IQRstem and leaf
Assumptions
  • For listed/ordered data of size n: use (n+1)/4 positions for quartiles
  • Outlier boundaries: below Q1 − 1.5×IQR or above Q3 + 1.5×IQR
  • Whiskers extend to the most extreme non-outlier values (not to the outliers or the fences)
  • Outliers are marked individually with crosses (×) or asterisks (*)
Approaches
A: Systematic five-number summary + outlier check
When: Always — this is the standard method for box plot questions
  1. 1. Ensure data is in ascending order (read stem-and-leaf carefully, checking the key)
  2. 2. Count n, the total number of data values
  3. 3. Find Q2 (median) at position (n+1)/2; if halfway between two values, average them
  4. 4. Find Q1 at position (n+1)/4; find Q3 at position 3(n+1)/4
  5. 5. Calculate IQR = Q3 − Q1
  6. 6. Calculate lower fence = Q1 − 1.5×IQR and upper fence = Q3 + 1.5×IQR
  7. 7. Identify any data values beyond these fences as outliers
  8. 8. Draw box from Q1 to Q3 with a line at Q2
  9. 9. Draw lower whisker from Q1 to smallest non-outlier; upper whisker from Q3 to largest non-outlier
  10. 10. Mark each outlier with a cross (×) at its value
+ Pros: Covers every mark in a structured way; hard to miss steps
– Cons: Must count positions very carefully — easy to be off by one
B: Compare two box plots for interpretation
When: When asked to compare two data sets (often the final part of the question)
  1. 1. Compare medians — state which distribution has higher/lower central tendency and what that means in context
  2. 2. Compare IQRs (or box widths) — state which has more/less spread and interpret in context
  3. 3. Compare skewness — look at relative positions of Q2 within the box and whisker lengths
  4. 4. Mention outliers if present and interpret in context
  5. 5. Use data context (e.g. 'On average, group A waited longer because their median is higher')
+ Pros: Structured comparison earns all interpretation marks
– Cons: Students often forget to use context — generic statements score poorly
Formulas
Median position: $\frac{n+1}{2}$ for listed data (MEMORIZE)
$Q_1$ position: $\frac{n+1}{4}$, $Q_3$ position: $\frac{3(n+1)}{4}$ for listed data (MEMORIZE)
$\text{IQR} = Q_3 - Q_1$ (MEMORIZE)
Outlier if $x < Q_1 - 1.5 \times \text{IQR}$ or $x > Q_3 + 1.5 \times \text{IQR}$ (MEMORIZE)
Pitfalls
  • Miscounting the position of quartiles — always write out (n+1)/4 explicitly and verify n
  • Extending whiskers to outlier values instead of to the nearest non-outlier data point
  • Only checking the upper fence for outliers and forgetting the lower fence
  • In stem-and-leaf diagrams: misreading the key (e.g. 3|5 means 35 not 3.5)
  • Forgetting to include a scale/axis on the box plot

Triggerscodingcoded datay = (x − a)/boriginal meanoriginal standard deviationΣyΣy²
Assumptions
  • Linear coding y = (x − a)/b means x = by + a
  • Coding affects the mean: x̄ = bȳ + a
  • Coding affects standard deviation by scale factor only: σ_x = |b| × σ_y (shift a has no effect on spread)
  • Use population s.d. (divide by n) in S1 unless stated otherwise
Approaches
A: Compute coded statistics then reverse the coding
When: Standard approach whenever coded summary statistics are given
  1. 1. From given Σy and n, compute coded mean: ȳ = Σy / n
  2. 2. From given Σy² and ȳ, compute coded variance: σ_y² = Σy²/n − ȳ²
  3. 3. Take square root for coded s.d.: σ_y = √(σ_y²)
  4. 4. Reverse coding for mean: x̄ = bȳ + a
  5. 5. Reverse coding for s.d.: σ_x = |b| × σ_y
  6. 6. Check: the mean should be sensible in context; s.d. must be positive
+ Pros: Clean and direct; each step is worth marks
– Cons: Must identify a and b correctly from the coding formula
B: Recover original sums then compute directly
When: When you need Σx and Σx² for combining groups (e.g. in combined mean/variance questions)
  1. 1. From coding x = by + a, recover Σx = bΣy + na
  2. 2. Recover Σx² from Σx² = b²Σy² + 2abΣy + na² (expand (by + a)² and sum)
  3. 3. Compute x̄ = Σx / n
  4. 4. Compute σ_x = √(Σx²/n − x̄²)
  5. 5. Use Σx and Σx² directly if combining with another group
+ Pros: Gives you raw sums needed for combining groups
– Cons: More algebraic manipulation; risk of errors in expanding squares
Formulas
If $y = \frac{x - a}{b}$ then $\bar{x} = b\bar{y} + a$ (MEMORIZE)
If $y = \frac{x - a}{b}$ then $\sigma_x = |b| \sigma_y$ (MEMORIZE — shift does NOT affect s.d.)
$\sigma_y = \sqrt{\frac{\Sigma y^2}{n} - \bar{y}^2}$ (formula booklet)
Pitfalls
  • Forgetting that the additive constant a does NOT affect the standard deviation — only the multiplier b does
  • Computing Σy²/n without subtracting ȳ² (giving Σy²/n instead of variance)
  • Getting the coding direction wrong: if y = (x−50)/10 then x = 10y + 50, not y = 10x + 50
  • Not multiplying σ_y by |b| when reversing the coding for standard deviation

Triggersgrouped frequency tableestimate the meanestimate the standard deviationmidpointsΣfxΣfx²
Assumptions
  • Data within each class is uniformly distributed (justifies using midpoints and linear interpolation)
  • Class midpoint = (lower boundary + upper boundary) / 2 — use class BOUNDARIES not class limits
  • Use population standard deviation (divide by n, not n−1) unless told otherwise in S1
Approaches
A: Direct calculation using Σfx and Σfx²
When: Standard approach when given a grouped frequency table — works every time
  1. 1. Identify class boundaries (not class limits) for each class
  2. 2. Calculate midpoint x for each class: x = (lower boundary + upper boundary) / 2
  3. 3. Build columns: f, x, fx, fx² (or x², fx²)
  4. 4. Sum to get Σf = n, Σfx, Σfx²
  5. 5. Mean: x̄ = Σfx / Σf
  6. 6. Variance: σ² = Σfx² / Σf − x̄²
  7. 7. Standard deviation: σ = √(Σfx² / Σf − x̄²)
  8. 8. Present answer to appropriate decimal places (usually 3 s.f. or as specified)
+ Pros: Systematic and reliable; easy to set up in a table
– Cons: Arithmetic-heavy; easy to make calculation errors with large numbers
B: Coded data approach to simplify arithmetic
When: When midpoints are large or awkward numbers — coding reduces calculation effort
  1. 1. Choose a coding y = (x − a)/b where a is near the centre of the data and b is the class width
  2. 2. Compute coded midpoints y for each class
  3. 3. Build columns: f, y, fy, fy²
  4. 4. Compute coded mean: ȳ = Σfy / Σf
  5. 5. Compute coded variance: σ_y² = Σfy² / Σf − ȳ²
  6. 6. Reverse coding for mean: x̄ = bȳ + a
  7. 7. Reverse coding for s.d.: σ_x = |b| × σ_y
  8. 8. Check answer is sensible (mean should lie within the data range)
+ Pros: Much simpler arithmetic with smaller numbers; fewer calculation mistakes
– Cons: Must remember to reverse the coding correctly; an extra conceptual step
Formulas
$\bar{x} = \frac{\Sigma fx}{\Sigma f}$ (MEMORIZE)
$\sigma = \sqrt{\frac{\Sigma fx^2}{\Sigma f} - \bar{x}^2}$ (formula booklet)
Coding: if $y = \frac{x - a}{b}$ then $\bar{x} = b\bar{y} + a$ and $\sigma_x = |b|\sigma_y$ (MEMORIZE)
Pitfalls
  • Using class limits (e.g. 10-19) instead of class boundaries (9.5-19.5) for midpoints
  • Forgetting to square the mean when computing variance: writing Σfx²/Σf instead of Σfx²/Σf − x̄²
  • Squaring fx instead of squaring x then multiplying by f: computing (fx)² rather than f(x²)
  • Using sample s.d. (n−1 divisor) when the formula booklet and S1 convention uses population s.d. (n divisor)
  • Rounding midpoints or intermediate sums prematurely

Must memorize

  • $\bar{x} = \frac{\Sigma fx}{\Sigma f}$ — mean from grouped data
  • $\sigma = \sqrt{\frac{\Sigma fx^2}{\Sigma f} - \bar{x}^2}$ — standard deviation (in formula booklet but must know how to apply with f)
  • Linear interpolation: $Q = L + \frac{(\text{target position} - CF_{\text{before}})}{f_{\text{class}}} \times w$
  • Outlier fences: $Q_1 - 1.5 \times IQR$ and $Q_3 + 1.5 \times IQR$
  • Coding: $y = \frac{x-a}{b} \Rightarrow \bar{x} = b\bar{y}+a$ and $\sigma_x = |b|\sigma_y$
  • Recovering sums: $\Sigma x = n\bar{x}$ and $\Sigma x^2 = n(\sigma^2 + \bar{x}^2)$
  • Frequency density $= \frac{\text{frequency}}{\text{class width}}$ (for histograms with unequal class widths)
  • Listed data quartile positions: $Q_1$ at $\frac{n+1}{4}$, $Q_2$ at $\frac{n+1}{2}$, $Q_3$ at $\frac{3(n+1)}{4}$
  • Grouped data quartile positions: $Q_1$ at $\frac{n}{4}$, $Q_2$ at $\frac{n}{2}$, $Q_3$ at $\frac{3n}{4}$

Global strategy

  • 1. ALWAYS clarify whether data is listed (use (n+1)/2 for median) or grouped (use n/2 for median) — this distinction drives correct quartile positions throughout.
  • 2. Build a full working table (f, x, fx, fx², cumulative f) for every grouped data question — this earns method marks even if arithmetic slips occur.
  • 3. Class BOUNDARIES vs class LIMITS: before any calculation, convert to class boundaries (e.g. 10-19 becomes 9.5-19.5 for continuous data). This affects midpoints, interpolation, and histogram bars.
  • 4. For box plot questions, follow a strict checklist: Q1, Q2, Q3, IQR → fences → identify outliers → whiskers to nearest non-outlier → mark outliers with ×.
  • 5. Know the variance formula inside-out: σ² = Σfx²/Σf − x̄². The most common single error in this chapter is forgetting to subtract x̄².
  • 6. When coding is involved, write the forward AND reverse coding equations before computing. Remember: shift doesn't affect s.d., only the multiplier does.
  • 7. In comparison questions (box plots, back-to-back stem-and-leaf), always compare using NAMED STATISTICS (median, IQR) with CONTEXT — vague statements score 0.
  • 8. Time management: grouped frequency mean/s.d. + interpolation questions can take 10-15 minutes of arithmetic. Set up tables neatly and double-check row totals.

Exam tips

  1. Show ALL intermediate columns (f, x, fx, x², fx²) in a table — examiners award method marks for these even if the final answer is wrong.
  2. When drawing a box plot, always include a clearly labelled scale on the axis and mark outliers with × — unlabelled diagrams lose marks.
  3. After computing mean and s.d., do a sanity check: the mean should lie within the data range, and s.d. should be positive and smaller than the range.
  4. In interpolation, explicitly state which class contains the median/quartile and write L, CF_before, f_class, and w before substituting — this earns the method mark.
  5. For 'interpret in context' marks, do NOT just say 'the median is higher'. Say something like 'On average, patients in hospital A waited longer, since the median waiting time (42 min) is greater than that for hospital B (31 min).'
  6. If a question says 'use the coding y = ...' you MUST use it — don't compute directly from x or you may lose method marks.
  7. Double-check your n by summing all frequencies before computing anything — a wrong n propagates errors through every calculation.
  8. Write down outlier fence values explicitly (e.g. 'Q3 + 1.5×IQR = 58 + 1.5×12 = 76') and then compare each suspect data value against them — this earns the mark even if your Q3 was slightly off.
  9. Keep at least 4 decimal places in intermediate working and round only at the final step to avoid accumulation of rounding errors.
