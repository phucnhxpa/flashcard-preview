Triggerscumulative distribution functionF(x)P(X ≤P(X >find F(x)
Assumptions
  • F(x) = P(X ≤ x) is the cumulative distribution function
  • F is a step function for discrete random variables: F(x) increases only at the values X can take
  • F(largest value) = 1 and F(value below smallest) = 0
Approaches
A: Build CDF from probability table
When: When you have P(X=x) for all x and need to construct F(x)
  1. List all values of X in increasing order: x₁ < x₂ < ... < xₙ
  2. F(x₁) = P(X = x₁)
  3. F(x₂) = P(X = x₁) + P(X = x₂)
  4. Continue: F(xᵢ) = F(xᵢ₋₁) + P(X = xᵢ)
  5. Verify F(xₙ) = 1
+ Pros: Straightforward cumulative addition
– Cons: Must be careful with running totals; check final value is 1
B: Extract P(X=x) from given CDF
When: When F(x) is given in a table and you need individual probabilities
  1. For the smallest value x₁: P(X = x₁) = F(x₁)
  2. For subsequent values: P(X = xᵢ) = F(xᵢ) − F(xᵢ₋₁)
  3. Verify all extracted probabilities sum to 1
  4. Use these to compute E(X), Var(X) etc.
+ Pros: Clean extraction of probabilities; avoids mistakes in probability tables
– Cons: Must correctly identify the 'previous' CDF value — easy to subtract wrong pair
Formulas
$F(x) = P(X \le x) = \sum_{t \le x} P(X = t)$ (MEMORIZE)
$P(X = x) = F(x) - F(x^{-})$ where $x^{-}$ is the previous value in the support (MEMORIZE)
$P(a < X \le b) = F(b) - F(a)$ (MEMORIZE)
Pitfalls
  • Forgetting that P(X = x₁) = F(x₁) for the smallest value (not F(x₁) − F(x₀) since F(x₀)=0)
  • Subtracting CDF values in the wrong order: P(X=x) = F(x) − F(previous), not F(previous) − F(x)
  • Confusing P(X ≤ c) with P(X < c): for discrete variables, P(X < c) = F(c) − P(X=c)
  • Forgetting that F(0) includes P(X=0) plus all probabilities for values below 0

Triggerstwo independent observationsX₁ and X₂P(X₁ < X₂)productsum of two observations
Assumptions
  • X₁ and X₂ are independent and identically distributed
  • Independence means P(X₁=a, X₂=b) = P(X₁=a)·P(X₂=b)
  • Enumerate all possible (X₁, X₂) pairs systematically
Approaches
A: Full enumeration via sample space grid
When: When the number of values is small (e.g. X takes 3-5 values)
  1. List all possible pairs (x₁, x₂) in a grid/table
  2. For each pair, compute the joint probability P(X₁=x₁)·P(X₂=x₂)
  3. Identify which pairs satisfy the required condition (e.g. X₁ < X₂)
  4. Sum the relevant joint probabilities
+ Pros: Thorough and transparent — hard to miss cases
– Cons: Time-consuming for large support; must be systematic to avoid missing pairs
B: Symmetry argument for P(X₁ < X₂)
When: When X₁ and X₂ are i.i.d. and you need P(X₁ < X₂)
  1. By symmetry: P(X₁ < X₂) = P(X₁ > X₂)
  2. Compute P(X₁ = X₂) = Σ[P(X=x)]²
  3. Then P(X₁ < X₂) = [1 − P(X₁ = X₂)] / 2
+ Pros: Very fast; avoids full enumeration
– Cons: Only works for the specific comparison X₁ < X₂ with i.i.d. variables
Formulas
$P(X_1 = a, X_2 = b) = P(X_1 = a) \cdot P(X_2 = b)$ for independent $X_1, X_2$ (MEMORIZE)
$P(X_1 < X_2) = \frac{1 - P(X_1 = X_2)}{2}$ for i.i.d. variables (MEMORIZE — useful shortcut)
$P(X_1 = X_2) = \sum [P(X = x)]^2$ (MEMORIZE)
Pitfalls
  • Forgetting to halve P(X₁≠X₂) when using symmetry to find P(X₁ < X₂)
  • Missing pairs in the enumeration — use a systematic grid
  • Confusing P(X₁ ≤ X₂) with P(X₁ < X₂) — the former includes P(X₁ = X₂)

Triggersprobability distributionrandom variable Xgiven in the following tablefind the value ofP(X=x)
Assumptions
  • X is a discrete random variable taking only the listed values
  • All probabilities are non-negative: P(X=x) ≥ 0 for all x
  • The sum of all probabilities equals exactly 1: ΣP(X=x) = 1
  • Each value of X is taken with the probability given — no other values are possible
Approaches
A: Sum-to-1 then direct computation
When: Standard approach when the table has one or two unknown parameters
  1. Write out ΣP(X=x) = 1 and substitute all given probabilities to form an equation in the unknown(s)
  2. If a second equation is needed, use a given condition such as E(X) = value, or P(X ≤ c) = value
  3. Solve the simultaneous equations for the unknown parameter(s)
  4. Check all resulting probabilities are between 0 and 1 inclusive
  5. Compute E(X) = Σx·P(X=x) by multiplying each x by its probability and summing
  6. Compute E(X²) = Σx²·P(X=x)
  7. Compute Var(X) = E(X²) − [E(X)]²
+ Pros: Systematic and always works; easy to check at each stage
– Cons: Can be arithmetic-heavy with many table entries; easy to make sign/fraction errors
B: Use symmetry (when distribution is symmetric)
When: When the probability distribution is symmetric about a central value
  1. Identify the axis of symmetry of the distribution
  2. Conclude E(X) = central value by symmetry
  3. Still compute E(X²) = Σx²·P(X=x) explicitly
  4. Compute Var(X) = E(X²) − [E(X)]²
+ Pros: Saves time on E(X) calculation and provides a quick check
– Cons: Only applicable when symmetry is present; still need full computation for variance
Formulas
$\sum_{\text{all } x} P(X=x) = 1$ (MEMORIZE)
$E(X) = \sum x \cdot P(X=x)$ (formula booklet)
$E(X^2) = \sum x^2 \cdot P(X=x)$ (MEMORIZE — derived from definition)
$\text{Var}(X) = E(X^2) - [E(X)]^2$ (formula booklet)
Pitfalls
  • Forgetting that all probabilities must sum to exactly 1, leading to incorrect parameter values
  • Computing Var(X) as E(X²) − E(X) instead of E(X²) − [E(X)]² — must square the mean
  • Arithmetic errors when working with fractions — keep exact fractions throughout
  • Not checking that all computed probabilities are non-negative after solving

Triggersdiscrete uniform distributionequally likelyeach value has the same probabilityP(X=x) = 1/n
Assumptions
  • Each of the n outcomes is equally likely with probability 1/n
  • The values are consecutive integers or equally spaced
  • If X ~ DU(1,n) then E(X) = (n+1)/2 and Var(X) = (n²−1)/12
Approaches
A: Use standard discrete uniform formulas
When: When X takes values 1, 2, ..., n each with probability 1/n
  1. Identify that X ~ Discrete Uniform on {1, 2, ..., n}
  2. Apply E(X) = (n+1)/2
  3. Apply Var(X) = (n²−1)/12
  4. For linear transformations Y = aX+b, apply E(Y) = aE(X)+b and Var(Y) = a²Var(X)
+ Pros: Very fast if you recognise the distribution and know the formulas
– Cons: Formulas may not be in the booklet — must memorise; doesn't apply if values aren't 1 to n
B: First principles calculation
When: When the uniform distribution is on non-standard values or to verify your answer
  1. List all values and their equal probabilities
  2. Compute E(X) = (1/n)·Σxᵢ directly
  3. Compute E(X²) = (1/n)·Σxᵢ²
  4. Compute Var(X) = E(X²) − [E(X)]²
+ Pros: Always works regardless of the values; provides a check
– Cons: Slower for large n; more room for arithmetic error
Formulas
$E(X) = \frac{n+1}{2}$ for $X \sim DU(1, n)$ (MEMORIZE — not always in booklet)
$\text{Var}(X) = \frac{n^2 - 1}{12}$ for $X \sim DU(1, n)$ (MEMORIZE)
$\sum_{r=1}^{n} r = \frac{n(n+1)}{2}$ (formula booklet)
$\sum_{r=1}^{n} r^2 = \frac{n(n+1)(2n+1)}{6}$ (formula booklet)
Pitfalls
  • Not recognising the discrete uniform distribution when all outcomes are equally likely
  • Using the wrong formula: confusing (n+1)/2 with n/2 for the mean
  • Applying the standard DU formulas when the values aren't 1 to n — must adjust or use first principles

TriggersinequalityP(X >find the values of Xsolvegreater than
Assumptions
  • X is discrete — solutions must be integers or discrete values from the support
  • After solving a continuous inequality, restrict to valid discrete values
  • The inequality may involve a linear transformation of X
Approaches
A: Solve continuous inequality then restrict to discrete support
When: When asked which values of X satisfy a given inequality
  1. Solve the inequality algebraically as if X were continuous
  2. Identify the critical boundary value
  3. Restrict to only those discrete values of X in the support that satisfy the inequality
  4. State the answer clearly listing the valid discrete values
+ Pros: Systematic — uses algebra then applies the discrete constraint
– Cons: Must remember X is discrete; the algebraic solution alone is not the final answer
Formulas

Pitfalls
  • Forgetting that X is discrete: after solving X > 0.5, the valid values start from X = 1 (or the next discrete value), not X = 0.5
  • Not listing all discrete values that satisfy the inequality

TriggersY = 1/XY = X²new random variablefind the distribution of Yfind E(Y)
Assumptions
  • Y = g(X) is a well-defined function for all values X can take
  • If g is non-linear, E(g(X)) ≠ g(E(X)) in general
  • Different values of X may map to the same value of Y — combine their probabilities
Approaches
A: Construct the new probability distribution table
When: Always for non-linear transformations
  1. For each value xᵢ in the support of X, compute yᵢ = g(xᵢ)
  2. If two or more xᵢ give the same yᵢ, add their probabilities: P(Y=yᵢ) = ΣP(X=xⱼ) for all xⱼ mapping to yᵢ
  3. Write out the probability distribution table for Y
  4. Verify ΣP(Y=y) = 1
  5. Compute E(Y) = Σy·P(Y=y) from this new table
+ Pros: Clear and systematic; impossible to go wrong if table is correctly constructed
– Cons: Can be time-consuming if X has many values
B: Use E(g(X)) = Σg(x)·P(X=x) directly
When: When you only need E(Y) and not the full distribution of Y
  1. Apply E(g(X)) = Σg(xᵢ)·P(X=xᵢ) using the original X table
  2. No need to construct the Y distribution table
+ Pros: Faster when only E(Y) is needed
– Cons: Cannot find Var(Y) or P(Y=y) without the full distribution; easy to make the E(1/X) = 1/E(X) error
Formulas
$E(g(X)) = \sum g(x) \cdot P(X=x)$ (MEMORIZE)
$E(g(X)) \neq g(E(X))$ in general for non-linear $g$ (MEMORIZE — critical concept)
Pitfalls
  • Using E(1/X) = 1/E(X) — this is WRONG for non-linear functions; must compute E(1/X) from the definition
  • Forgetting to combine probabilities when multiple X values map to the same Y value
  • Using E(X²) = [E(X)]² — these are different quantities

TriggersY = aX + blinear transformationfind E(Y)find Var(Y)given that a > 0
Assumptions
  • The transformation is linear: Y = aX + b where a, b are constants
  • a and b are real constants; if a > 0 is stated, do not consider negative roots
  • The underlying distribution of X is already known or E(X) and Var(X) are given
Approaches
A: Direct formula application
When: When E(X) and Var(X) are known and you need E(Y) and Var(Y)
  1. Write Y = aX + b
  2. Apply E(Y) = aE(X) + b
  3. Apply Var(Y) = a²Var(X) — note the constant b vanishes and a is squared
  4. If required, SD(Y) = |a| · SD(X)
+ Pros: Very fast — just plug in values
– Cons: Must remember to square the coefficient for variance; easy to forget b drops out
B: Reverse-engineer a and b from E(Y) and Var(Y)
When: When Y = aX + b and you are given E(Y) and Var(Y) and must find a and b
  1. From Var(Y) = a²Var(X), solve for a² then a (use a > 0 if specified)
  2. From E(Y) = aE(X) + b, substitute the value of a and solve for b
  3. State Y = aX + b with the found values
+ Pros: Systematic approach to a common exam sub-question
– Cons: Must be careful about sign of a; always check any constraint given in the question
Formulas
$E(aX + b) = aE(X) + b$ (formula booklet)
$\text{Var}(aX + b) = a^2 \text{Var}(X)$ (formula booklet)
Pitfalls
  • Using Var(aX+b) = a·Var(X) instead of a²·Var(X) — must square the coefficient
  • Including the constant b in the variance formula — Var(aX+b) = a²Var(X), not a²Var(X)+b
  • Taking the negative square root for a when the question specifies a > 0
  • For Var(4−3X): writing Var = 3·Var(X) instead of 9·Var(X) — must square the −3

Must memorize

  • $\sum P(X=x) = 1$ — sum of all probabilities equals 1
  • $E(X) = \sum x \cdot P(X=x)$ (formula booklet)
  • $\text{Var}(X) = E(X^2) - [E(X)]^2$ (formula booklet)
  • $E(aX + b) = aE(X) + b$ (formula booklet)
  • $\text{Var}(aX + b) = a^2 \text{Var}(X)$ (formula booklet)
  • $E(g(X)) = \sum g(x) \cdot P(X = x)$ — definition of expectation of a function (MEMORIZE)
  • $F(x) = P(X \le x)$; $P(X = x) = F(x) - F(x^{-})$ (MEMORIZE)
  • For $X \sim DU(1, n)$: $E(X) = \frac{n+1}{2}$, $\text{Var}(X) = \frac{n^2 - 1}{12}$ (MEMORIZE — may not be in booklet)

Global strategy

  • Always start by writing ΣP(X=x) = 1 — this gives you the key equation for finding unknown parameters
  • For every Var(X) calculation, compute E(X) and E(X²) separately, then use Var(X) = E(X²) − [E(X)]² — never skip squaring the mean
  • Set up a systematic table with columns: x, P(X=x), x·P(X=x), x²·P(X=x) — this organises your work and reduces arithmetic errors
  • For linear transformations Y = aX + b, remember: constants shift the mean but only the coefficient squared affects the variance
  • For non-linear functions Y = g(X), never assume E(g(X)) = g(E(X)) — always go back to the definition E(g(X)) = Σg(x)·P(X=x)
  • When building a CDF, add probabilities cumulatively from the smallest value and verify the final entry equals 1
  • When two independent observations are involved, consider using the symmetry shortcut: P(X₁ < X₂) = [1 − ΣP(X=x)²] / 2
  • Keep all working in exact fractions — avoid decimals until the final answer if the question requires a decimal

Exam tips

  1. Draw a clear table with columns x | P(X=x) | x·P | x²·P — this prevents errors and earns method marks even if arithmetic slips
  2. Always verify ΣP = 1 after finding unknown parameters — this is a quick sanity check that catches algebraic mistakes
  3. In Var(X) = E(X²) − [E(X)]², circle or underline the square on [E(X)]² to remind yourself — this is the #1 error across all sessions
  4. For Var(aX+b) = a²Var(X): write out the formula explicitly before substituting — examiners award marks for quoting the correct formula
  5. If a question says 'a > 0', only take the positive square root when solving a² = k — state this reasoning explicitly
  6. When working with CDFs, clearly state F(x) for each x including values below and above the support range
  7. Show full working for E(X) and E(X²) calculations — partial credit is awarded for correct method even with arithmetic errors
  8. In 'two observation' problems, draw a grid of all (x₁, x₂) outcomes to ensure no pairs are missed
