Triggersproduct moment correlation coefficientΣx²Σxysummary statisticscalculate r
Assumptions
  • Data are bivariate and come from a sample of size n
  • The PMCC measures linear association only — not causation
  • r lies in the interval [-1, 1]; values near ±1 indicate strong linear correlation
Approaches
A: Direct formula using summary statistics
When: Always — this is the standard and expected method when summary statistics are provided
  1. Compute Sxx = Σx² − (Σx)²/n
  2. Compute Syy = Σy² − (Σy)²/n
  3. Compute Sxy = Σxy − (Σx)(Σy)/n
  4. Calculate r = Sxy / √(Sxx × Syy)
  5. Check that −1 ≤ r ≤ 1 as a sanity check
+ Pros: Systematic, works every time, directly from formula booklet
– Cons: Prone to arithmetic errors with large numbers — must be careful with calculator work
B: Recover Sxy from given regression coefficient b
When: When the regression line is already given and you need to find r (e.g. Jun 2021)
  1. Note that b = Sxy/Sxx, so Sxy = b × Sxx
  2. Compute Sxx and Syy from summary statistics
  3. Calculate r = Sxy / √(Sxx × Syy) = (b × Sxx) / √(Sxx × Syy)
+ Pros: Saves time when b is already provided
– Cons: Only applicable when the regression gradient is given or already calculated
Formulas
$S_{xx} = \sum x^2 - \frac{(\sum x)^2}{n}$ (formula booklet)
$S_{yy} = \sum y^2 - \frac{(\sum y)^2}{n}$ (formula booklet)
$S_{xy} = \sum xy - \frac{(\sum x)(\sum y)}{n}$ (formula booklet)
$r = \frac{S_{xy}}{\sqrt{S_{xx} \cdot S_{yy}}}$ (formula booklet)
Pitfalls
  • Forgetting to divide (Σx)² by n — writing Sxx = Σx² − (Σx)² instead of Σx² − (Σx)²/n
  • Mixing up Sxx and Syy or Sxy in the formula
  • Arithmetic errors with large squared sums — always double-check on the calculator
  • Giving r to insufficient decimal places (typically 3 s.f. or 4 d.p. required)

Triggersestimatepredictreliableextrapolationinterpolationwithin the rangeoutside the rangePriya claims
Assumptions
  • Interpolation (predicting within the observed range of x) is generally reliable if r is strong
  • Extrapolation (predicting outside the observed range of x) is unreliable because the linear model may not hold
  • The regression line of y on x should only be used to predict y from x, not x from y
Approaches
A: Substitute and comment
When: Always when asked to predict and comment on reliability
  1. Substitute the given x-value into the regression equation y = a + bx
  2. State the predicted value with appropriate units and precision
  3. Determine whether the x-value lies within the range of the original data
  4. If within range → interpolation → prediction is reliable (assuming strong correlation)
  5. If outside range → extrapolation → prediction is unreliable as the linear model may not hold outside the observed range
  6. If commenting on another person's claim, explicitly state why it is or is not valid
+ Pros: Straightforward and systematic — easy marks if you remember to comment
– Cons: Students often forget the comment part and lose marks
Formulas
Interpolation: prediction at $x$ within $[x_{\min}, x_{\max}]$ of observed data (MEMORIZE definition)
Extrapolation: prediction at $x$ outside $[x_{\min}, x_{\max}]$ of observed data (MEMORIZE definition)
Pitfalls
  • Using the regression line of y on x to predict x from y — this is invalid; you need x on y for that
  • Not recognising that x = 0 is often extrapolation (e.g. interpreting the intercept as meaningful when 0 is far from the data range)
  • Forgetting to comment on reliability after making the prediction — this is almost always worth a mark
  • Saying extrapolation is 'wrong' — it's not wrong, it's unreliable/less reliable

Triggersinterpretin contextwhat does this value meandescribe the correlationcomment on
Assumptions
  • Correlation does not imply causation
  • The PMCC only measures the strength of linear association
  • Interpretations must refer to the actual variables in the question, not just 'x and y'
Approaches
A: Structured interpretation using sign, strength, and context
When: Always when asked to interpret r
  1. State the sign: positive r means as x increases, y tends to increase; negative means as x increases, y tends to decrease
  2. State the strength: |r| close to 1 = strong, close to 0.5 = moderate, close to 0 = weak
  3. Put it in context: replace 'x' and 'y' with the actual variable names from the question
  4. Example: 'r = 0.92 indicates a strong positive linear correlation between floor area and house price'
+ Pros: Covers all marking points systematically
– Cons: Students sometimes forget one element (e.g. 'linear' or the context)
B: Interpreting gradient b in context
When: When asked what b represents
  1. State that b represents the change in y for each unit increase in x
  2. Use the actual variable names and units from the question
  3. Be careful with scaled variables (e.g. if y is in £1000s, b = 2.5 means £2500 increase per unit increase in x)
  4. Example: 'For each additional square metre of floor area, the house price increases by approximately £2500'
+ Pros: Direct and concise
– Cons: Easy to forget unit scaling
Formulas
$-1 \le r \le 1$ (MEMORIZE)
$r = \pm 1$ means perfect linear correlation (MEMORIZE)
$r = 0$ means no linear correlation (MEMORIZE)
Pitfalls
  • Saying 'strong correlation' without specifying 'linear' — examiners want the word 'linear'
  • Failing to use the context of the question — writing 'x and y are correlated' instead of naming the actual variables
  • Interpreting r close to 0 as 'no relationship' — it only means no LINEAR relationship
  • Forgetting unit conversions: if y is in £1000s then b = 3 means £3000 not £3 per unit increase in x

Triggersregression liney on xequation of the lineleast squarespredict
Assumptions
  • The regression line of y on x is used to predict y from x (y is the response/dependent variable)
  • The regression line always passes through the point (x̄, ȳ)
  • The model assumes a linear relationship between x and y
Approaches
A: Standard calculation from summary statistics
When: When you have Σx, Σy, Σx², Σxy and n
  1. Calculate x̄ = Σx/n and ȳ = Σy/n
  2. Calculate Sxx = Σx² − (Σx)²/n
  3. Calculate Sxy = Σxy − (Σx)(Σy)/n
  4. Compute gradient b = Sxy / Sxx
  5. Compute intercept a = ȳ − b × x̄
  6. Write the equation as y = a + bx (or ŷ = a + bx)
  7. If asked, verify the line passes through (x̄, ȳ)
+ Pros: Completely standard and expected — always gets full marks
– Cons: Multiple steps increase risk of arithmetic error
B: Using (x̄, ȳ) and one other point to draw/check line
When: When asked to plot the regression line on a scatter diagram
  1. Calculate (x̄, ȳ) and plot this point
  2. Substitute another convenient x-value into y = a + bx to get a second point
  3. Draw a straight line through both points
  4. Ensure the line extends only within the data range (avoid extrapolation on the plot)
+ Pros: Quick visual check and easy to plot accurately
– Cons: Only useful for graphical parts, not for calculation marks
Formulas
$b = \frac{S_{xy}}{S_{xx}}$ (formula booklet)
$a = \bar{y} - b\bar{x}$ (formula booklet)
Regression line of $y$ on $x$: $y = a + bx$ (formula booklet)
Pitfalls
  • Using b = Sxy/Syy instead of b = Sxy/Sxx — the denominator is ALWAYS the S-value for the explanatory variable
  • Computing the regression line of x on y when asked for y on x — always check which variable is being predicted
  • Rounding b and a too early — keep intermediate values to at least 4 s.f.
  • Forgetting to state the equation clearly in the form y = a + bx with numerical values

Triggersregression line passes throughfind Σxgiven that the equation isshow that
Assumptions
  • The regression line of y on x always passes through the mean point (x̄, ȳ)
  • b = Sxy/Sxx and a = ȳ − bx̄ can be used in reverse to recover unknowns
Approaches
A: Use the mean point property
When: When you know the regression equation and ȳ (or Σy/n) but need x̄ (or Σx)
  1. Write ȳ = a + b × x̄
  2. Substitute the known values of a, b, and ȳ
  3. Solve for x̄
  4. Recover Σx = n × x̄ if needed
+ Pros: Very quick — just one equation to solve
– Cons: Only works when the regression equation is fully given
B: Use b = Sxy/Sxx to recover Sxy
When: When you know b and Sxx and need Sxy (e.g. to find r)
  1. Calculate Sxy = b × Sxx
  2. Then use r = Sxy / √(Sxx × Syy) as needed
+ Pros: Direct algebraic manipulation
– Cons: Requires Sxx to be known or calculable
Formulas
The regression line passes through $(\bar{x}, \bar{y})$ (MEMORIZE)
$\bar{x} = \frac{\sum x}{n}$, $\bar{y} = \frac{\sum y}{n}$ (MEMORIZE)
Pitfalls
  • Forgetting that the regression line passes through (x̄, ȳ) — this is the key property to exploit
  • Confusing x̄ with Σx — remember to multiply/divide by n appropriately

Triggersoverestimateunderestimateexceedsinequalityregression vs actual
Assumptions
  • The regression line predicts ŷ = a + bx
  • The identity line y = x (or f = p etc.) represents 'prediction equals actual'
  • The inequality reverses direction depending on the sign of (b − 1)
Approaches
A: Solve the inequality algebraically
When: When asked to find the range of x for which the regression prediction exceeds the actual
  1. Set up the inequality: a + bx > x (or a + bx < x depending on the question)
  2. Rearrange: a > x(1 − b) or a < x(1 − b)
  3. Solve for x, being careful about the direction of the inequality when dividing by (1 − b)
  4. State the range clearly and check it makes sense in context
+ Pros: Quick and elegant
– Cons: Must be careful with inequality direction when dividing by a negative number
Formulas
Regression prediction: $\hat{y} = a + bx$ (formula booklet)
Identity line: $y = x$ (MEMORIZE for this type of problem)
Pitfalls
  • Reversing the inequality when dividing by a negative quantity (1 − b) when b > 1
  • Not linking the algebraic answer back to the context of the problem

Triggerscodinglinear transformationw = 50 − peffect on rnew variable
Assumptions
  • Linear coding y = a + bx does not change |r|, only changes the sign if b < 0
  • If the coding is w = c − p (negative coefficient on p), then r(d, w) = −r(d, p)
  • Two variables related by an exact linear equation have r = ±1
Approaches
A: Apply the linear coding rule for PMCC
When: When a linear transformation is applied to one or both variables
  1. Identify the transformation: e.g. w = 50 − p means w = 50 + (−1)p
  2. The multiplier of p is −1 (negative), so the sign of r is reversed
  3. Therefore r(d, w) = −r(d, p)
  4. If asked for r between w and p directly: since w = 50 − p is exact linear, r(w, p) = −1
+ Pros: Very quick — no calculation needed once the rule is known
– Cons: Easy to forget the sign-reversal rule under exam pressure
Formulas
If $w = \alpha + \beta p$ then $r(d, w) = \text{sign}(\beta) \times r(d, p)$ (MEMORIZE)
If $w = \alpha + \beta p$ is exact (not statistical), then $r(w, p) = \text{sign}(\beta) = \pm 1$ (MEMORIZE)
Pitfalls
  • Forgetting that a negative coefficient in the coding reverses the sign of r
  • Thinking that coding changes the magnitude of r — it does NOT; only the sign can change
  • Not recognising that w = 50 − p is an exact linear relationship, so r(w, p) = −1 exactly

Triggerscorrectedshould have beenupdatedrecalculateerror in data
Assumptions
  • When only a y-value is corrected, Σx and Σx² (hence Sxx) do not change
  • When only an x-value is corrected, Σy and Σy² (hence Syy) do not change
  • Updating means: new sum = old sum − old value + new value
Approaches
A: Adjust each affected summary statistic individually
When: Whenever a single data point is corrected
  1. Identify which value changed (x or y or both) and the old and new values
  2. Update Σy: new Σy = old Σy − old_y + new_y
  3. Update Σy²: new Σy² = old Σy² − (old_y)² + (new_y)²
  4. Update Σxy: new Σxy = old Σxy − x × old_y + x × new_y (where x is the unchanged x-value of that data point)
  5. Note: if only y changed, Σx and Σx² remain the same, so Sxx is unchanged
  6. Recalculate Sxy, Syy (and Sxx if x also changed) with the new summary statistics
  7. Recompute b, a, and r as required
+ Pros: Efficient — avoids recalculating everything from scratch
– Cons: Must be careful to update every affected sum and not miss one
Formulas
Updated $\sum y = \text{old } \sum y - y_{\text{old}} + y_{\text{new}}$ (MEMORIZE)
Updated $\sum y^2 = \text{old } \sum y^2 - y_{\text{old}}^2 + y_{\text{new}}^2$ (MEMORIZE)
Updated $\sum xy = \text{old } \sum xy - x \cdot y_{\text{old}} + x \cdot y_{\text{new}}$ (MEMORIZE)
Pitfalls
  • Changing Sxx when only a y-value was corrected — Sxx does NOT change in this case
  • Forgetting to update Σxy as well as Σy and Σy²
  • Using the old n when a data point is added/removed (though correction usually keeps n the same)

Must memorize

  • $S_{xx} = \sum x^2 - \frac{(\sum x)^2}{n}$ (formula booklet)
  • $S_{yy} = \sum y^2 - \frac{(\sum y)^2}{n}$ (formula booklet)
  • $S_{xy} = \sum xy - \frac{(\sum x)(\sum y)}{n}$ (formula booklet)
  • $r = \frac{S_{xy}}{\sqrt{S_{xx} \cdot S_{yy}}}$ (formula booklet)
  • $b = \frac{S_{xy}}{S_{xx}}$, $a = \bar{y} - b\bar{x}$ (formula booklet)
  • The regression line of $y$ on $x$ always passes through $(\bar{x}, \bar{y})$ (MEMORIZE)
  • Interpolation = within observed range (reliable); Extrapolation = outside observed range (unreliable) (MEMORIZE)
  • Linear coding $w = \alpha + \beta p$ does not change $|r|$; sign of $r$ changes if $\beta < 0$ (MEMORIZE)
  • Regression of $y$ on $x$ is used to predict $y$ from $x$; regression of $x$ on $y$ is used to predict $x$ from $y$ (MEMORIZE)

Global strategy

  • Almost every question follows the same skeleton: compute Sxx, Syy, Sxy → find r → find regression line → interpret → predict and comment on reliability. Master this workflow and you can handle any variant.
  • Always start by carefully noting n, Σx, Σy, Σx², Σy², Σxy from the question — write them out clearly before computing anything.
  • Double-check Sxx, Syy, Sxy calculations: the most common source of lost marks is forgetting to divide (Σx)² by n in the formula Sxx = Σx² − (Σx)²/n.
  • When asked to interpret, always use the context of the question (name the actual variables and their units) and include the word 'linear' when describing correlation.
  • Always identify whether a prediction involves interpolation or extrapolation — this is almost always asked and is easy marks.
  • Know which regression line to use: y on x predicts y from x; x on y predicts x from y. Using the wrong one is a fatal error.
  • Keep at least 4 significant figures in intermediate calculations and round only the final answer to the precision requested.
  • For coding questions, remember: linear coding does not change |r|; a negative multiplier flips the sign of r.

Exam tips

  1. Write out all given summary statistics at the start and label them clearly — this reduces substitution errors.
  2. When computing Sxx, Syy, Sxy, use brackets on your calculator: e.g. type (Σx)² ÷ n as a single calculation to avoid premature rounding.
  3. If your value of r comes out greater than 1 or less than −1, you have made an error — go back and check your Sxx, Syy, Sxy immediately.
  4. For interpretation marks: always write a full sentence using the variable names from the question (e.g. 'temperature' and 'sales'), include the word 'linear', and describe direction (positive/negative) and strength (strong/moderate/weak).
  5. When the question says 'comment on the reliability', explicitly state whether it is interpolation or extrapolation, and why this matters.
  6. Never use the regression line of y on x to predict x — if asked to find x for a given y, either rearrange cautiously (with a comment about limitations) or state that the x on y line is needed.
  7. In data-correction questions, update each affected sum systematically: Σy, Σy², Σxy — and explicitly note which sums (like Σx, Σx², Sxx) remain unchanged.
  8. Check your regression equation by substituting x̄ and verifying you get ȳ — this catches errors in a and b quickly.
