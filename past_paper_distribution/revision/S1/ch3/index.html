Addition rule and conditional probability from given event probabilities
#1 most important

TriggersP(A ∪ B)P(A|B)independentmutually exclusivegiven thataddition ruleevents A and B
Assumptions
  • Events are NOT independent unless proven or stated
  • Events are NOT mutually exclusive unless P(A ∩ B) = 0 is shown
  • All probabilities lie in [0, 1]
Approaches
A: Algebraic formula manipulation
When: When probabilities are given numerically and you need to find unknowns step by step
  1. Write down all given information: P(A), P(B), P(A ∪ B), P(A|B), etc.
  2. If P(A|B) is given, convert to P(A ∩ B) = P(A|B) × P(B)
  3. Apply addition rule: P(A ∪ B) = P(A) + P(B) − P(A ∩ B) to find unknown
  4. Use complement: P(A') = 1 − P(A), P(A' ∩ B') = 1 − P(A ∪ B)
  5. For independence: check whether P(A ∩ B) = P(A) × P(B) — state conclusion clearly
  6. For mutual exclusivity: check whether P(A ∩ B) = 0 — state conclusion clearly
+ Pros: Direct, efficient, and works for all standard sub-parts
– Cons: Must be careful about which formula to apply first when multiple unknowns exist
B: Two-way table / Venn diagram as organiser
When: When information is complex or multiple conditional probabilities are involved
  1. Draw a 2×2 table with rows A, A' and columns B, B' plus totals
  2. Fill in known values: marginal totals from P(A), P(B), grand total = 1
  3. Use P(A ∩ B) to fill the intersection cell
  4. Complete remaining cells by subtraction
  5. Read off any required probability directly from the table
  6. Test independence/mutual exclusivity from the completed table
+ Pros: Visual and reduces algebraic errors; all probabilities visible at once
– Cons: Takes slightly longer to set up
Formulas
$P(A \cup B) = P(A) + P(B) - P(A \cap B)$ (MEMORIZE)
$P(A|B) = \frac{P(A \cap B)}{P(B)}$ (formula booklet)
$P(A' \cap B') = 1 - P(A \cup B)$ (MEMORIZE)
$P(A \cap B) = P(A) \times P(B) \iff \text{A and B independent}$ (MEMORIZE)
$P(A \cap B) = 0 \iff \text{A and B mutually exclusive}$ (MEMORIZE)
Pitfalls
  • Forgetting to subtract P(A ∩ B) in the addition rule — double-counting the intersection
  • Confusing P(A|B) with P(B|A) — the denominator must match the 'given' event
  • Dividing by P(A) instead of P(B) when computing P(A|B)
  • Stating events are independent without showing the numerical check P(A ∩ B) vs P(A)×P(B)
  • Forgetting that mutually exclusive events with non-zero probabilities CANNOT be independent

Triggersindependentmutually exclusivejustifygive a reasondetermine whether
Assumptions
  • Independence requires P(A ∩ B) = P(A) × P(B) — this is the ONLY valid test
  • Mutual exclusivity requires P(A ∩ B) = 0
  • Two events with P(A) > 0 and P(B) > 0 that are mutually exclusive CANNOT be independent
Approaches
A: Direct numerical comparison
When: Always — this is the standard and expected method
  1. State the definition: 'A and B are independent if and only if P(A ∩ B) = P(A) × P(B)'
  2. Compute P(A) × P(B) numerically
  3. Compare with the known value of P(A ∩ B)
  4. If equal: conclude independent. If not equal: conclude not independent
  5. For mutual exclusivity: state 'ME ⟺ P(A ∩ B) = 0', then check
+ Pros: Quick, definitive, and always earns full marks
– Cons: None — this is the required method
Formulas
$\text{Independent} \iff P(A \cap B) = P(A) \times P(B)$ (MEMORIZE)
$\text{Mutually exclusive} \iff P(A \cap B) = 0$ (MEMORIZE)
Pitfalls
  • Just stating 'they are independent' without showing the numerical check — loses marks
  • Checking P(A|B) = P(A) as independence test is valid but students often compute P(A|B) incorrectly
  • Confusing independence with mutual exclusivity — they are different concepts and (for non-trivial events) mutually contradictory
  • Rounding intermediate values and then claiming approximate equality proves independence

Triggerstree diagramwithout replacementgiven thatfirstsecondbagselectedtransferred
Assumptions
  • Branches at each node must have probabilities summing to 1
  • 'Without replacement' means the sample space reduces after each draw
  • Paths through the tree represent intersections (multiply along branches)
  • Mutually exclusive paths can be added to find P(event)
Approaches
A: Full tree diagram enumeration
When: When there are 2-3 stages with a small number of outcomes at each stage
  1. Draw the tree: first-stage branches from the root, second-stage from each first-stage outcome
  2. Label each branch with its conditional probability (adjust denominators for without-replacement)
  3. For P(specific outcome): multiply along the unique path
  4. For P(event spanning multiple paths): identify all paths leading to the event and sum their products
  5. For conditional probability P(A|B): compute P(A ∩ B) / P(B) where P(B) is the sum of all paths giving B
+ Pros: Extremely clear and visual; hard to miss cases
– Cons: Can become unwieldy with many stages or outcomes
B: Bayes' theorem / reverse conditional probability
When: When asked for the probability of an earlier-stage event given a later-stage outcome
  1. Identify the 'forward' conditional probabilities from the tree
  2. Compute P(later event) using the law of total probability: sum over all branches
  3. Apply Bayes: P(earlier|later) = P(earlier ∩ later) / P(later)
  4. Simplify and present the answer
+ Pros: Systematic for 'reverse' conditional probability questions
– Cons: Requires careful identification of all contributing branches for total probability
Formulas
$P(A \cap B) = P(A) \times P(B|A)$ — multiplication rule (MEMORIZE)
$P(B) = P(B|A)P(A) + P(B|A')P(A')$ — law of total probability (MEMORIZE)
$P(A|B) = \frac{P(B|A)P(A)}{P(B)}$ — Bayes' theorem (formula booklet)
$P(A|B) = \frac{P(A \cap B)}{P(B)}$ (formula booklet)
Pitfalls
  • Forgetting to reduce the denominator for 'without replacement' on the second branch
  • Forgetting a branch: e.g. missing the 'Yellow first' path when computing P(Yellow second)
  • In Bayes-type questions, forgetting to sum ALL branches for the denominator (law of total probability)
  • Confusing P(A|B) with P(B|A) — always ask: 'what is the given condition?'
  • When items are transferred between bags, forgetting the new composition of the receiving bag depends on what was transferred

TriggersVenn diagramP(A ∪ B)P(A ∩ B)complementregionexactly oneat most oneneither
Assumptions
  • All probabilities in the Venn diagram (including the outer region) must sum to 1
  • Events are NOT independent unless explicitly stated or proven
  • Events are NOT mutually exclusive unless P(A ∩ B) = 0
Approaches
A: Direct region reading + formula application
When: When the Venn diagram is pre-drawn with all regions labelled
  1. Identify each region of the Venn diagram and its probability
  2. Check all regions sum to 1 (find unknown region if needed)
  3. For P(A ∪ B): add all regions inside at least one circle
  4. For P(A'): add all regions outside circle A (including outer region)
  5. For P(A ∩ B'): read the region inside A but outside B
  6. For conditional probability P(A|B): identify P(A ∩ B) and P(B), then divide
+ Pros: Fast and visual — directly read off the diagram
– Cons: Easy to misidentify regions, especially in 3-set diagrams
B: Populate Venn diagram from given information
When: When you are given P(A), P(B), P(A ∩ B) etc. and must construct the diagram
  1. Start by placing P(A ∩ B) in the intersection region
  2. Subtract intersection from P(A) to get 'only A' region: P(A) − P(A ∩ B)
  3. Subtract intersection from P(B) to get 'only B' region: P(B) − P(A ∩ B)
  4. Compute outer region: 1 − P(A ∪ B) = 1 − [P(A) + P(B) − P(A ∩ B)]
  5. For 3-set problems, work from the centre outward: triple intersection first, then pairwise-only, then single-only, then outer
  6. Verify all regions sum to 1
+ Pros: Systematic and reliable for building up the diagram
– Cons: More time-consuming; algebraic errors if given conditional probabilities requiring simultaneous equations
Formulas
$P(A \cup B) = P(A) + P(B) - P(A \cap B)$ (MEMORIZE)
$P(A') = 1 - P(A)$ (MEMORIZE)
$P(A|B) = \frac{P(A \cap B)}{P(B)}$ (formula booklet)
$P(A' \cap B') = 1 - P(A \cup B)$ — De Morgan's complement (MEMORIZE)
Independence test: $P(A \cap B) = P(A) \times P(B)$ (MEMORIZE)
Pitfalls
  • Forgetting the outer region (complement of all sets) when regions must sum to 1
  • For P(A ∪ B ∪ C)', forgetting to apply De Morgan's law correctly
  • Confusing 'at most one defect' (includes zero defects) with 'exactly one defect'
  • In 3-set Venn diagrams, misidentifying which region corresponds to which intersection
  • Using P(A) as denominator when computing P(A|B) instead of P(B)

TriggersVenn diagramfind the value of p and qgiven that P(A|B) =show that
Assumptions
  • All region probabilities are non-negative
  • All regions must sum to 1
  • If independence is given, P(A ∩ B) = P(A) × P(B) provides an equation
Approaches
A: Set up equations from probability rules
When: When the Venn diagram has algebraic unknowns and conditions are given
  1. Express P(A), P(B), P(A ∩ B) in terms of the diagram's unknowns
  2. Use the constraint that all regions sum to 1 to get one equation
  3. Convert each given condition (e.g. P(A|B) = k) into an equation using the formula
  4. If independence is given: write P(A ∩ B) = P(A) × P(B) as an equation
  5. Solve the resulting system of simultaneous (possibly quadratic) equations
  6. Check solutions are valid (probabilities between 0 and 1)
+ Pros: Systematic and handles any combination of constraints
– Cons: Can lead to quadratic or cubic equations requiring careful algebra
Formulas
$\sum \text{all regions} = 1$ (MEMORIZE)
$P(A|B) = \frac{P(A \cap B)}{P(B)}$ (formula booklet)
$P(A \cap B) = P(A) \times P(B)$ for independent events (MEMORIZE)
Pitfalls
  • Forgetting the equation from all regions summing to 1 — this often provides the crucial extra equation
  • Making algebraic errors when expressions for P(A) involve multiple regions
  • Not checking that solutions give valid probabilities (all between 0 and 1)

Triggerscumulative distribution functionF(x)without replacementselected at randomcombination
Assumptions
  • P(X = k) = F(k) − F(k−1) for discrete CDF
  • F(x) is non-decreasing with F(max) = 1
  • Without replacement: use hypergeometric-style counting
Approaches
A: Extract probabilities from CDF then use combinatorics
When: When F(x) is given in tabular or piecewise form
  1. Compute P(X = k) = F(k) − F(k−1) for each value of k
  2. For combinatorial part: count favourable outcomes using $\binom{n}{r}$
  3. Compute total outcomes: $\binom{N}{r}$ for selecting r from N without replacement
  4. Set up equation: computed P(X = k) = favourable/total
  5. Solve for any unknowns (may be a polynomial equation — try integer solutions first)
+ Pros: Structured approach that separates the CDF work from the counting
– Cons: Polynomial equations may require trial-and-error for verification
Formulas
$P(X = k) = F(k) - F(k-1)$ (MEMORIZE)
$\binom{n}{r} = \frac{n!}{r!(n-r)!}$ (formula booklet)
$P(\text{selecting } r \text{ of type A from } N) = \frac{\binom{a}{r}\binom{N-a}{n-r}}{\binom{N}{n}}$ — hypergeometric (MEMORIZE structure)
Pitfalls
  • Confusing P(X = k) with F(k) — remember to subtract F(k−1)
  • Forgetting the combinatorial factor: e.g. P(X = 1) involves choosing WHICH one of the selected items is the special type
  • Arithmetic errors in solving cubic/quadratic equations — always verify by substitution

Must memorize

  • $P(A \cup B) = P(A) + P(B) - P(A \cap B)$ — Addition rule (MEMORIZE, not in formula booklet in this form)
  • $P(A|B) = \frac{P(A \cap B)}{P(B)}$ — Conditional probability (formula booklet)
  • $P(A') = 1 - P(A)$ — Complement rule (MEMORIZE)
  • $P(A' \cap B') = 1 - P(A \cup B)$ — De Morgan's complement (MEMORIZE)
  • $\text{Independent} \iff P(A \cap B) = P(A) \times P(B)$ (MEMORIZE)
  • $\text{Mutually exclusive} \iff P(A \cap B) = 0$ (MEMORIZE)
  • $P(B) = P(B|A)P(A) + P(B|A')P(A')$ — Law of total probability (MEMORIZE)
  • $P(A|B) = \frac{P(B|A)P(A)}{P(B)}$ — Bayes' theorem (formula booklet)

Global strategy

  • ALWAYS draw a diagram first — Venn diagram, tree diagram, or two-way table. This organises information and prevents errors.
  • Start with P(A ∩ B): this is the most fundamental quantity. Most other probabilities (union, conditional, complement intersections) flow from knowing P(A ∩ B).
  • When given conditional probabilities, IMMEDIATELY convert them to intersection probabilities: P(A ∩ B) = P(A|B) × P(B).
  • Check your work: all regions in a Venn diagram must sum to 1; all branches from a node in a tree must sum to 1.
  • For independence/mutual exclusivity, always show the numerical comparison — never just assert the conclusion.
  • Read the question carefully for 'with replacement' vs 'without replacement' — this fundamentally changes the second-stage probabilities.
  • In multi-part questions, earlier parts typically set up values needed in later parts. If stuck on a part, use a 'show that' result from an earlier part and proceed.
  • Practice De Morgan's laws: P(A' ∩ B') = 1 − P(A ∪ B) and P(A' ∪ B') = 1 − P(A ∩ B) — these appear frequently.

Exam tips

  1. Conditional probability is the most frequently examined sub-topic — practise converting P(A|B) to P(A ∩ B)/P(B) until it is automatic.
  2. Always write the formula BEFORE substituting numbers — this earns method marks even if arithmetic is wrong.
  3. For 'show that' questions, work towards the given answer and present every step clearly. Do not skip algebra.
  4. When testing independence, write: 'P(A) × P(B) = ... and P(A ∩ B) = ... Since these are [equal/not equal], A and B are [independent/not independent].' This template guarantees full marks.
  5. If a question asks 'are A and B mutually exclusive?', check P(A ∩ B): if it equals 0, they are; if not, they aren't. State the value explicitly.
  6. In Venn diagram questions, label every region — including the outer region (neither event). This prevents forgetting it.
  7. Tree diagrams: write probabilities on branches AND joint probabilities at the end of each path. This makes subsequent calculations much easier.
  8. Double-check that your final probability answer is between 0 and 1 — an impossible value signals an error.
